## 2. 共同模板规则

### 2.1 共同字段

所有模板必须登记以下参数引用：

- `max_health`、体型和空间占用。
- 移动速度或固定占用；容纳单位写`none`。
- 核心行为周期、射程、覆盖数和基础效果；无行为模板写`none`。
- 死亡输出数量、谱系与腐败规则。
- 批次合并键所需状态字段。

第一版不使用随机命中、暴击、浮动伤害或隐蔽闪避。所有攻击在合法目标池中使用确定性命中槽。

### 2.2 阵营与位置

| 类别 | 合法位置 |
| --- | --- |
| 基础幼体 | 所属虫巢的幼体容器 |
| 己方可机动单位 | 虫群大军或节点内；形成后不能同时位于两处 |
| 己方固定菌毯 | 节点内 |
| 敌方可机动单位 | 已生成节点内 |
| 敌方和中立固定单位 | 已生成节点内固定空间段 |

领地采集、自由游荡、当前战场、接触、纵深和后备都是位置之上的活动或空间投影，不建立额外库存。

### 2.3 无目标规则

- 己方战斗单位和敌方活动单位：前往各自阵营推进锚点。
- 工蜂：没有合法采集目标时待命。
- 根脉孢体：没有合法扎根位置时待命并继续随阵营推进规则移动到可搜索空间。
- 固定菌毯和固定结构：待命。

目标失效只清空目标与不兼容进度，不触发第二次效果、退款或随机重抽。

## 3. 通用行为与效果定义

### 3.1 搬运资产

`v1.effect.carry_process_asset`调用工蜂原子预留和五阶段转换：

```text
现场 -> 已预留 -> 携带中 -> 已保障 -> 已回流
```

目标顺序完全读取L3服务层级：已承诺搬运、临近永久损失、阻断生产、常规采集；同层再按等待老化、可形成载荷、服务成本、拥堵和稳定ID排序。内容不提供玩家可配置权重。

### 3.2 接触伤害

`v1.effect.contact_damage`声明伤害标签、基础伤害参数、单次覆盖、合法空间层和目标池。它只伤害可达接触目标；过量伤害不转移到后备。

### 3.3 纵深伤害

`v1.effect.spine_barrage`只覆盖已处于明确射程、视线和空间范围内的目标批次。形成草案时必须发布可见预警区域；不能随机命中未暴露后备或节点外工蜂。

### 3.4 自身变形

`v1.effect.root_transform`把根脉孢体等量变形为根脉菌毯，沿用L3耐久换算、位置继承、状态白名单和下一步骤生效规则。来源与目标数量不能同时保留。

### 3.5 生成单位

`v1.effect.spawn_shell_guard`从来源单位的有限储备或明确环境输入中形成壳卫体，并在节点右侧入口或来源声明空间提交。来源死亡后已形成草案可以在同一步提交一次，后续不再生成。

### 3.6 共振状态

`v1.effect.resonance_frenzy`使用通用`刷新持续`状态实例，修改同阵营活动单位和出兵源已有核心行为的周期。状态：

- 不增加第二行为。
- 来源、作用范围、修正值和失效步骤可见。
- 离开范围或来源死亡后不再刷新，已有实例按声明步骤失效。
- 不作用于没有周期行为的封锁壳壁和共振孵育床。

### 3.7 根网被动范围

`v1.effect.root_network_aura`是根脉菌毯的被动范围定义，分别调用公开修正接口影响：

- 友方通行容量或移动成本。
- 合法补位吞吐。
- 工蜂在范围内的交互通路。

它不直接造成治疗、资源或控制权。来源死亡后下一合法重算边界全部修正消失。

## 11. 共振孵育床

| 字段 | 定义 |
| --- | --- |
| `content_id` | `v1.unit.sample_nursery` |
| 阵营 | 中立 |
| 标签 | `fixed`、`sample_recovery_source`、`non_occupation_blocker`、`indestructible_v1` |
| 核心行为 | `v1.effect.restore_first_sample_chain` |
| 启动条件 | 首次解析未提交，且不存在存活载体、可回收残骸、已保障/已回流样本或处理中样本 |
| 输出 | 节点未占领时生成恢复载体；已占领时生成现场恢复过程资产 |
| 死亡输出 | `none`；第一版不可攻击、搬运或转化 |

行为同一时刻最多维持一条恢复任务。任一合法样本链重新出现时，未提交恢复任务停止。首次候选组形成后永久休眠。

恢复输出只恢复首次样本链，不产生普通资源、外部同化、击杀统计、候选机会或飞升成果。孵育过程、剩余进度、当前阻断和输出位置可观察。

| `v1.asset.mutagenic_tissue` | 异变样本组织 | 消化池分离异变腺结 | 突触解析腔 | 保持来源事件与普通候选谱系 |
| `v1.asset.root_remains` | 菌毯残骸 | 根脉菌毯主动战场死亡 | 腐解消化池 | 内部 |

未占节点后台战斗中的玩家固定单位死亡不形成`root_remains`。恢复载体的样本残骸带恢复谱系，后续不会形成重复外部统计。

## 13. 内容解锁

| 内容 | 资格来源 | 保底边界 |
| --- | --- | --- |
| 采质工蜂 | 开局模板可用 | 无工蜂时核心巢室允许唯一应急恢复 |
| 噬咬体 | 胚流孵化室完成有效幼体循环 | 不依赖战场掉落或候选 |
| 根脉孢体 | 观察到合法可扎根地表 | 首轮必需节点生成前可观察位置 |
| 根脉菌毯 | 不是单独孵化模板资格，由孢体等量变形 | 不建立第二份生产配方 |
| 裂壳体 | 观察并实际伤害出兵源，或保障硬壳结构残骸 | 可选；缺少时噬咬体仍能完成必要结构 |

观察到内容只更新发现或模板资格；能否当前孵化仍检查房间绑定、幼体、生物质、功率、任务和输出位置。

## 14. 第一版战斗内容组合边界

- 小规模接触至少使用壳卫体和固定共振样本体，不需要无法绕过的持续出兵源。
- 出兵源压力节点至少使用一个增殖囊巢，并通过空间段限制接触或补位。
- 综合出口至少包含壳卫体、针脊体、一个占领阻断源和合法菌毯位置。
- 封锁壳壁、增殖囊巢和边界脉门都可由噬咬体低效完成；裂壳体只改变处理关系。
- 第一版没有必须使用专用阶段的Boss。数量为1的关键样本体和固定结构可以显示独立状态条。
- 所有敌方出兵必须来自已生成实例、有限储备或可见环境输入。

## 15. 表现接口

单位表现只读取模板、批次、状态队列、真实位置、空间、行为进度和效果事件：

| 表现类别 | 用途 |
| --- | --- |
| `individual_readable` | 早期小数量、关键样本体和数量为1的结构 |
| `batch_representative` | 大规模活动单位和接触批次 |
| `fixed_cluster` | 菌毯、囊巢、壳壁和脉门 |
| `background_flow` | 后备、战时新生、撤离流和领地游荡 |
| `aggregate_remains` | 大规模尸体与过程资产批次 |

画面代表不具有权威耐久、目标或死亡输出。改变代表配额只重建投影，不播放可被结算监听的死亡或生成事件。

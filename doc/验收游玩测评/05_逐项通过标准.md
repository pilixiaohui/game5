# game5 第一版逐项通过标准

## 1. 适用范围

本标准验收 `variants/desktop_v1/` 的桌面第一版，范围包括一座初始虫巢、一个区域、一个活跃战场、样本与候选链、出口后的相邻区域信息和飞升只读预览。

正式飞升、第二地区实装、第二虫巢和多战场不作为本版完成项，但锁定内容不得暴露可执行命令、修改存档或假装已经开放。

## 2. 结论枚举

每个验收项只能使用：

- `PASS`：证据完整且达到门槛。
- `FAIL`：证据证明未达到门槛。
- `CONDITIONAL`：仅允许用于非核心 `S2` 或统计证据临界项，附批准偏差和复测日期。
- `INVALID`：测试基础设施或数据缺失使结果不可判定。
- `NOT_RUN`：尚未执行。
- `OUT_OF_SCOPE`：经范围文件明确排除，不能由测试人员临时指定。

`INVALID`、`NOT_RUN` 不能按通过计算。

## 3. 体验门槛登记

以下为第一轮推荐值，不是资源或战斗平衡数值。形成性先导轮可以调整一次，但必须在终验构建发放前冻结并记录理由。

| parameter | 推荐冻结值 | 含义 |
| --- | --- | --- |
| `U_START_MIN` | `0.90` | 能独立开始新档并完成第一项有效建设的比例 |
| `U_CORE_MIN` | `0.80` | 60 分钟内独立完成生产、首占、样本、突变和出口主链的比例 |
| `U_SAVE_MIN` | `0.95` | 独立完成保存、退出、重开、继续的比例 |
| `HELP_MAX_INDEPENDENT` | `H1` | `H2` 及以上不计独立完成 |
| `PXI_CORE_MEDIAN_MIN` | `+1.0` | 核心 PXI 维度各分层中位数下限 |
| `PXI_NEGATIVE_MAX` | `0.25` | 核心 PXI 维度各分层低于 0 的答卷比例上限 |
| `PXI_SUPPORT_MEDIAN_MIN` | `0.0` | 支持维度中位数不得为负 |
| `CONTENT_EXHAUSTION_P50_MIN` | `45 min` | 内容耗尽时间中位数下限；按生存分析处理未耗尽场次 |
| `VOLUNTARY_45M_MIN` | `0.75` | 第 45 分钟仍自愿游玩的比例 |
| `MANDATORY_IDLE_MAX` | `180 s` | 没有任何有意义替代行动的单次强制等待上限 |
| `MENTAL_MODEL_MEDIAN_MIN` | `1.5 / 2` | 每个核心因果主题的分层中位数下限 |
| `TARGET_FPS` | `60` | 最低硬件、目标分辨率下的渲染目标 |
| `SCALE_TARGET_60M` | 平衡测试后冻结 | 首小时末虫群规模目标；当前不凭猜测填写 |

最终样本量必须通过先导数据和功效计划登记。终验报告同时给出比例的 Wilson 95% 区间和量表区间；不能只报达到门槛的点估计。

## 4. 核心路径定义

核心路径是结果集合，不是规定顺序：

1. 从干净新档建立能源、资源形成和单位生产能力。
2. 形成工蜂和至少一种战斗单位。
3. 进入节点战斗，并经历占领或一次撤离/失败后的恢复。
4. 保障残骸或样本，经房间处理形成生物质或基因。
5. 生成一个候选组并同化一个候选，其他候选消失。
6. 占领区域出口，查看相邻区域信息和飞升预览。
7. 保存、完全退出、重新载入并继续模拟。

玩家可以采用不同房间布局、节点路线、战斗次数和候选。验收不得把自动化夹具的固定顺序当作唯一正确玩法。

## 12. 最终判定算法

按顺序执行：

1. 构建哈希、范围、最低硬件、样本和门槛均已冻结，否则 `FAIL` 或 `INVALID`。
2. 任一构建、稳定性、存档、规则、数据或缺陷硬门槛不是 `PASS`，整版不得 `PASS`。
3. 分别计算 `NOVICE`、`GENRE`、`RETURNING` 和目标 `ACCESS` 结果，不以总体均值覆盖分层失败。
4. `ACC-UX-*` 和 `ACC-PLY-*` 达到冻结门槛后，才可签发 `PASS`。
5. 只有非核心 `S2` 或统计证据临界项可以申请 `CONDITIONAL PASS`；必须写明偏差、负责人、到期版本和复测场景。
6. 任何层级的高分都不能抵消另一层级的硬失败。

最终结论和签字使用 [06_终验报告模板.md](06_终验报告模板.md)。

## 1. 契约职责与写入边界

### 1.1 本文负责

- 单位模板如何形成单位批次与逻辑个体状态。
- 单位总账、形成、变形、死亡、真实数量和唯一位置。
- 虫群大军活动、可参战资格、进攻转入、战时新生补充和回归。
- 单一核心行为、移动、目标选择、通用效果与最早生效步骤。
- 工蜂如何选择、预留、携带、保障和投递过程资产。
- 接触、纵深、后备、路径、空间与交互容量如何影响行动。
- 确定性伤害、状态效果、被动范围、死亡输出和批量结算。
- 固定菌毯、阻断结构、出兵点、主动战场与后台战斗的共同执行。
- 撤离与全灭时的单位和携带资产事务。

### 1.2 本文不负责

- 节点图、空间段内容、节点控制、区域出口和地图生成；由`L3_世界地图与节点契约.md`负责。
- 过程资产五阶段、残骸账目、入账和安全缓冲容量；由`L3_资源与能源契约.md`负责。
- 幼体形成配方、孵化任务、房间端口和生产队列；由`L3_虫巢与房间契约.md`负责。
- 候选生成、突变同化和修正来源；由`L3_候选与进化契约.md`负责。
- 固定步长、命令总排序、十二阶段和保存时点；由统一时间契约负责。
- 单位具体名称、属性值、动画、画面实体和镜头。

## 2. 单位定义、批次与逻辑个体

### 2.1 单位资格

满足以下任一事实的对象使用单位模型：

- 具有真实数量和存活耐久。
- 可以受伤、死亡或变形。
- 可以占据节点空间。
- 可以周期执行至多一个核心行为。
- 可以作为单位行为的目标。

房间、地形、路径、过程资产、外部来源和候选组不满足该身份。可破坏出兵点和阻断结构满足，因此属于移动形态为`固定`的单位。

### 2.2 单位模板最低字段

每个模板必须登记：

- 稳定定义标识、阵营、标签与内容版本。
- 移动形态：`容纳 / 可机动 / 固定`之一。
- 合法位置、移动配置、体型与空间占用。
- 最大耐久、伤害标签、抗性标签和目标资格。
- 零或一个核心行为定义。
- 零或多个被动范围修正定义。
- 死亡输出配方、控制资格和可见特殊目标资格。
- 无目标规则：`待命`或`前往阵营推进锚点`。
- 表现类别与允许的批次合并键；表现字段不参与结算。

模板不得携带关卡专用更新循环、专用撤离或专用死亡步骤。

### 2.3 单位批次

单位总账只拥有批次，不为百万个同质单位建立百万个运行实例。每个批次至少保存：

- 稳定批次标识、模板、阵营、真实整数数量和来源事件。
- 唯一真实位置及其虫巢、节点或空间段引用。
- 移动形态、空间层、活动、最早生效步骤和生产归因。
- 规范状态队列、核心行为进度、目标池或输出位置。
- 携带资产、交互预留、路径、容量和状态效果引用。
- 创建、拆分、合并和最后变更步骤。

只有模板、阵营、位置、空间、耐久、状态、行为、目标、携带、预留、生效步骤与必要来源份额全部兼容的批次才能合并。

### 2.4 规范状态队列

批次内部使用规范状态单元保存“完整玩法键 -> 真实数量”。完整玩法键只包含会改变后续资格、目标、结算或守恒的事实，至少覆盖精确剩余耐久、状态多重集、行为进度、目标、路径、携带/预留、位置与必要的玩法来源标签。运行批次ID、容器顺序、画面代表和纯统计生产事件不得进入玩法键。

纯统计生产归因保存在与状态单元并列的稳定稀疏份额表中，并始终满足`归因份额之和 = 状态单元真实数量`。若某个来源后来会改变玩法，该来源必须显式提升为玩法标签；不能继续藏在统计份额里供战斗读取。

任何改变玩法键的事务提交后都必须按完整玩法键排序、合并同键单元并形成唯一规范哈希。规范化必须幂等，且拆分与合并满足：

```text
拆分前真实数量 = 全部拆分后真实数量之和
拆分前总剩余耐久 = 全部拆分后总剩余耐久之和
携带资产、状态层数、行为承诺和来源份额分别守恒
```

不能通过平均耐久/持续时间、裁剪尾部状态、随机抽样伤亡或限制队列数量改变死亡结果。表现可以合并显示；运行容器可以分片；两者都不得进入权威状态或改变规范哈希。

## 3. 唯一位置、虫群大军与活动

### 3.1 单位真实位置

存活单位批次只能位于：

| 位置 | 必要引用 | 合法形态 |
| --- | --- | --- |
| `虫巢内` | 虫巢与单位容器 | `容纳` |
| `虫群大军` | 当前轮次 | 玩家`可机动` |
| `节点内` | 节点与空间段 | 敌我`可机动`或`固定` |
| `死亡` | 唯一死亡事件 | 已终结批次 |

当前战场、节点留置、接触、纵深、后备和领地活动都是筛选，不是第二份数量。

### 3.2 虫群大军活动

对每种玩家可机动模板：

```text
虫群大军
= 虫巢作业 + 领地采集 + 自由游荡 + 待命
```

- 活动保存目标虫巢或节点的`活动锚点`，但不把单位位置复制到该对象。
- 活动锚点只用于作业资格、时间、容量和表现投影，不形成玩家可配置的运输或驻军系统。
- 一个单位同一时刻只能处于一种活动。
- 页面切换和投影数量不改变活动分配。

### 3.3 工蜂活动与可参战资格

工蜂处于下列任一事实时不可参战：

- 已携带过程资产。
- 已占用不可无代价中断的取货、交接或投递容量。
- 已形成等待第10阶段提交的搬运草案。

其他待命或仍可无损释放预留的工蜂属于可参战候选。后一类只有在第2阶段进攻事务能够把采集资产恢复到原来源`现场`、释放路径与交互/投递容量并清空对应行为进度时，才进入最终可参战集合。任一释放失败时该工蜂保持原作业且不转移；系统不会为增加参战数量取消不可中断搬运，也不会让同一工蜂同时留在作业投影和节点内。

这里的“释放失败”分为两类：正常的`不可释放`只使该工蜂留在原作业，不改变任何事实；已经进入最终释放集合后出现所有者、数量或状态不一致则是跨域事务失败，整笔进攻不发布，不能只留下部分已释放工蜂。

### 3.4 战斗单位领地活动

已占节点中的战斗单位自由游荡是虫群大军活动投影：

- 第一版不改变真实位置，不建立驻军。
- 不形成敌军、战损、控制权或额外资源。
- 进攻时活动立即结束，单位仍按普通可参战资格进入目标节点。
- 表现投影总量不得超过对应活动真实数量。

## 4. 形成、离巢、变形与死亡

### 4.1 形成

形成请求必须包含模板、数量、来源、目标位置、容量、最早生效步骤和幂等标识。合法来源只有：

- 房间孵化或幼体培育输出。
- 已提交地图初始内容。
- 出兵点或恢复来源行为。
- 明确的后续跨轮初始事实。

第11阶段形成的单位最早在下一完整步骤第4阶段生效。形成数量不能读取动画实体数。

### 4.2 幼体与离巢

- 幼体是`容纳`单位，无核心行为，只能位于虫巢单位容器。
- 房间契约负责幼体形成任务和单位输入预留；本文原子写入幼体数量。
- 分化使用等量变形，不先死亡再形成新单位。
- 离巢时没有主动战场，玩家可机动单位进入虫群大军。
- 存在主动战场时，非工蜂可机动单位进入当前节点左侧后备；新工蜂先参加下一步骤作业分配，仍待命的富余部分再进入主动节点。
- 已确认撤离等待提交时，新离巢单位留在虫群大军，不加入即将结束的战场。

### 4.3 变形

第一版变形默认等量，统一继承：

```text
新剩余耐久
= clamp(floor(旧剩余耐久 * 新最大耐久 / 旧最大耐久), 1, 新最大耐久)
```

- 来源与目标模板不能同时保留同一数量。
- 默认保持节点和空间位置；幼体分化随后执行离巢位置事务。
- 只保留目标模板允许继承的状态，其他状态正常终止但不触发死亡输出。
- 清空旧目标、路径、行为进度与不兼容预留。
- 保留形成来源和生产归因，记录唯一变形事件。
- 新形态从下一完整步骤生效，不能同一步再次行动。

后续一对多变形必须登记守恒来源和输出数量，不能覆盖本节默认规则。

### 4.4 死亡

第9阶段统一执行：

```text
耐久归零或合法终结
-> 从存活位置移除
-> 释放尚未提交的目标、空间和交互预留
-> 携带中资产在死亡位置形成现场资产
-> 按模板和战斗上下文形成一次死亡输出
-> 移除空间占用、被动范围与占领阻断资格
-> 写入死亡和单位守恒账目
```

- 主动战场敌我死亡按模板形成残骸或样本过程资产。
- 未占节点后台战斗中的玩家固定单位死亡不形成可回收输出，但正常登记死亡。
- 后台敌方死亡可以形成节点现场资产，仍需工蜂回流和房间处理。
- 同一死亡幂等标识只能形成一次输出。

### 4.5 单位守恒

对每个模板：

```text
期初数量 + 累计形成 + 累计变形进入
= 虫巢内存活 + 虫群大军存活 + 全部节点内存活
 + 累计变形离开 + 累计死亡
```

每次进攻、补充、撤离、占领和全灭还必须满足L2单次战役公式。

## 5. 单一核心行为与通用效果

### 5.1 行为管线

每个模板至多一个核心行为，并复用：

```text
读取自身状态
-> 过滤合法目标或输出位置
-> 按内容定义的有序目标组稳定选择
-> 接近目标
-> 检查空间、射程、容量和输入
-> 积累行为进度
-> 形成一个事件草案
-> 在统一阶段提交或失效
```

移动、受伤、死亡、待命、被动范围和响应进攻/撤离不占第二行为。

### 5.2 行为运行事实

行为只保存：

- 当前目标池或输出位置。
- 当前路径与空间关系。
- 行为进度、最早生效步骤和状态修正。
- 已承诺交互数量、输入或容量引用。

事件草案只存在于当前未发布步骤工作区，不跨步骤保存。跨步骤等待继续由行为进度或任务事实表达。

### 5.3 六类通用效果

| 效果 | 提交性质 | 用途 |
| --- | --- | --- |
| 伤害 | 形成后必结算 | 对单位状态队列降低耐久 |
| 搬运资产 | 存活后提交 | 推进过程资产五阶段 |
| 自身变形 | 存活后提交 | 等量改变执行者模板 |
| 生成单位 | 形成后必结算 | 消耗储备或输入形成单位 |
| 施加状态 | 形成后必结算 | 建立有生效步骤的状态效果 |
| 生成过程资产 | 形成后必结算 | 形成仍需采集处理的物理资产 |

一次行为周期只能形成一个主要效果。互斥配方必须在周期开始前按公开条件选定；条件失效则清空本周期进度并重新选择，不能一次提交多种配方。

### 5.4 目标失效

- 草案形成前，目标、路径、容量或输入失效时释放尚未占用预留并清空该次进度。
- 伤害、状态和生成草案形成后不因来源同一步死亡取消。
- 搬运和自身变形在第10阶段再次检查执行者存活及全部预留。
- 内容不能通过目标失效重复形成随机、资源或单位结果。

## 6. 工蜂采集契约

### 6.1 可采集目标

工蜂把以下对象统一视为带类型的过程资产来源：

- 虫巢房间物理输出与中间产物。
- 初始母床和已占节点外部物质来源形成的现场资产。
- 主动战场和已占领地的普通残骸、样本残骸与特殊处理物。
- 安全回流缓冲中需要继续交接的已保障资产。

外部物质来源先由世界与资源跨域事务在来源位置形成`现场`过程资产。工蜂必须完成五阶段回流后，腐殖滤囊或其他房间才能处理；房间不能远程抽取节点物质。

### 6.2 合法目标过滤

目标必须同时满足：

1. 资产标签与工蜂能力兼容。
2. 存在未预留真实数量。
3. 存在合法交互关系和可计算服务成本。
4. 来源交互容量可用。
5. 至少一个兼容投递端或安全交接缓冲存在。
6. 投递容量可以原子承诺。
7. 工蜂存活、已生效且没有待提交撤离冲突。

已占领地活动不建立玩家配置的运输路线。来源节点、目标虫巢和端口由系统根据兼容性与稳定成本选择，玩家只通过生产工蜂、房间容量和布局改变结果。

### 6.3 服务层级与防饥饿

从高到低：

| 基础层级 | 条件 |
| --- | --- |
| 已承诺搬运 | 已携带或进入不可无代价中断的交接 |
| 临近永久损失 | 即将腐败、被夺回或溢出终结 |
| 阻断生产 | 持续占满必要输出、输入或交接容量 |
| 常规采集 | 其他合法目标 |

同层按等待老化、可形成载荷、服务成本、拥堵和稳定目标标识排序。等待老化可以把长期未服务目标提高到与临近永久损失相同的有效层级，但不能伪造风险事实，也不能越过已经承诺的搬运。

玩家不能设置工蜂优先级。样本只因真实风险、保障或堵塞事实获得顺序，不因系统认为它“更应该拿”而暗中加权。

### 6.4 原子预留

工蜂领取任务时同笔预留：

- 源资产数量。
- 取货交互容量。
- 投递端或安全交接容量。
- 本次可执行工蜂数量与活动份额。

任一失败则任务不成立。批次容量不足时确定性拆分，可执行部分取得预留，其余保持原活动并继续选择。

成功预留形成一个由单位行为拥有的`采集交互预留束`，保存资产、房间/世界路径、源交互端、投递端、工蜂活动份额的稳定所有者句柄，以及行为尝试、精确数量、建立步骤、可逆截止边界和状态。它不是第二份资产或运输任务。第10阶段取货、目标失效、执行者死亡、第1阶段撤离或第2阶段进攻释放都必须用同一预留束终结全部仍有效句柄。

### 6.5 阶段转换

| 单位结果 | 资源契约结果 |
| --- | --- |
| 第7阶段领取目标 | `现场 -> 已预留` |
| 第10阶段成功取货 | `已预留 -> 携带中` |
| 完成安全交接 | `携带中 -> 已保障` |
| 到达兼容输入 | `携带中/已保障 -> 已回流` |
| 到达账本接入口 | 物理资产终结并同笔正式入账 |
| 工蜂第9阶段死亡 | 携带资产在死亡位置回到`现场` |

具体数量和谱系始终由资源契约写入。已保障资产不因原工蜂后来死亡退回现场。

## 7. 进攻、战时补充与战役结束

### 7.1 进攻输入

世界契约提供目标节点合法性，时间契约提供命令顺序。本文在第2阶段计算可参战集合：

- 全部生效、存活、位于虫群大军且没有不可中断作业的玩家可机动单位。
- 不包含幼体、固定单位、已携带、待提交交接或无法完整释放作业预留的工蜂。
- 不读取生产虫巢、节点距离、玩家投放比例或画面选择。

至少存在一个可参战单位且主动战场槽空闲时，进攻按一个跨域原子事务执行：

1. 对待命、携带、不可中断交接和持有可逆预留的工蜂按稳定批次与预留束标识分类。
2. 对可逆候选分别调用资源、房间和世界接口执行无副作用释放准备；正常`不可释放`者保持原作业并从最终集合排除。
3. 由准备成功者与原待命单位形成最终可参战集合，再验证节点、主动战场槽和全部单位份额。
4. 以同一进攻事务标识原子提交资产回现场、房间/世界路径与端口释放、行为进度和活动份额清空、全部最终单位进入目标节点左侧后备、战役期初账目和主动战场引用。

准备阶段不写状态。任一已准备子结果在提交时失败，整笔进攻不发布；不能留下已释放但未进攻、已进攻仍占原作业容量或只释放资产未释放端口的半状态。

### 7.2 战时新生补充

- 非工蜂玩家可机动单位离巢后，在存在主动战场且没有待提交撤离时自动进入当前节点左侧后备。
- 新生工蜂先参与虫巢和领地合法采集分配；没有未服务目标的待命部分在下一步骤进入主动节点。
- 补充是单位位置事务，不建立队列、运输或投放比例。
- 第11阶段新生最早从下一步骤行动。

### 7.3 撤离

撤离由时间契约在第1阶段最高优先级执行：

```text
校验当前主动节点
-> 节点内玩家可机动单位回到虫群大军
-> 玩家固定单位留在原节点
-> 已预留采集资产释放回现场
-> 携带中资产转为已保障
-> 兼容容量内继续回流，超额保持已保障并堵塞
-> 释放主动战场引用
-> 形成一次撤离事件与战役期末账目
```

撤离没有时间、成功率、途中受击或取消。该节点本步骤不形成新的移动、行为、出兵和环境伤害。

### 7.4 占领回归接口

世界契约判定节点满足占领条件时，本文同笔执行：

- 玩家可机动单位回到虫群大军并重新进入活动分配。
- 玩家固定单位继续留在节点。
- 结束战役账期并提供回归、死亡、留置与期末数量。
- 释放主动战场引用。

本文不写节点控制，只返回可供世界事务原子提交的单位子结果。

### 7.5 全灭与仅剩固定单位

当前主动节点不再存在玩家可机动单位时：

- 第12阶段结束战役并释放主动战场槽。
- 玩家固定单位保持节点位置。
- 未占节点从下一步骤进入敌方恢复或后台战斗活动。
- 节点、敌军、残骸、空间和出兵源不重置。

## 8. 空间、移动、目标与战线

### 8.1 空间层

节点内单位分别位于`接触 / 纵深 / 后备`之一。空间段提供：

- 通行、接触、受击和交互容量。
- 路径边、方向、地表、视线和推进锚点。
- 可扎根位置、环境修正和固定单位占用。

容量按真实数量与体型计算，不读取画面碰撞体。

### 8.2 移动

- 有合法行为目标时沿最低合法成本路径接近。
- 无目标且模板允许推进时前往阵营推进锚点，否则待命。
- 路径成本只读取距离、拥堵、移动方式、体型、公开环境与阻断。
- 同成本路径按稳定路径标识选择。
- 移动只改变位置，不直接产生伤害、资源或控制权。

### 8.3 补位与突破

- 接触容量出现空位后，合法纵深和后备单位按抵达步骤、行为需求与稳定标识补位。
- 一方只有在阻断占用消失、路径可通行且对方未补满缺口时推进。
- 宏观兵力差通过补位持续性、空间占用、纵深暴露和出兵储备表现，不直接生成隐藏伤害倍率。
- 后备只有进入明确覆盖区域或暴露空间后才能受伤。

### 8.4 目标组

行为定义一个或多个有序目标组。通用过滤顺序：

1. 阵营与目标标签。
2. 存活、可作用与生效资格。
3. 路径、射程、方向、视线和空间层。
4. 交互、受击或输出容量。
5. 已有目标承诺与互斥事实。

系统只在当前组没有合法目标时进入下一组。同组排序为：

```text
空间暴露
-> 预计到达或作用距离
-> 剩余耐久或内容声明的稳定次序
-> 模板与完整玩法键的规范序列
```

战斗目标选择不读取最近失败、资源价值、画面实体、运行批次ID或纯统计生产归因。只有已登记为玩法事实的来源标签可以影响资格和规范顺序。

## 9. 伤害、状态与被动范围

### 9.1 攻击包

伤害行为完成时形成：

- 来源模板、真实行动次数和稳定目标池。
- 基础伤害、固定与比例修正。
- 单次覆盖数量、伤害标签、射程和合法空间层。
- 来源步骤、定义内效果槽序与幂等标识。

### 9.2 有效伤害

```text
单次有效伤害
= max(0, floor(
    (基础伤害 + 固定伤害修正)
    * 比例伤害修正
    * 目标承伤修正
    * 伤害标签与抗性标签乘区
  ))
```

第一版没有随机命中、随机暴击、浮动伤害和独立护甲减法。完全免疫使用明确规则替换或零乘区。

### 9.3 命中槽分配

1. `命中槽数 = 行动次数 * 单次覆盖数量`。
2. 目标池按空间暴露、距离、剩余耐久、模板和完整玩法键形成唯一规范逻辑序列；同键单元必须先合并。
3. 对池内`N`个逻辑个体和`K`个命中槽，计算`q = K div N`与`r = K mod N`；规范序列前`r`个个体承受`q+1`次命中，其余承受`q`次，直接用整数前缀区间拆分状态单元，不逐个遍历。
4. `N=0`时攻击包无目标失效；单个逻辑个体获得的过量伤害不转移。
5. 穿透、范围或连锁必须在攻击包中预先声明额外覆盖。

同一步敌我攻击及不同效果包都从第8阶段同一目标快照形成。每类攻击包独立计算商余边界，再对规范区间叠加完整命中/状态向量并一次提交；不能先提交一种攻击，再让其受伤结果改变同一步另一种攻击的目标。被本步击杀的来源仍完成已经形成的攻击。

### 9.4 状态效果

状态必须声明`刷新持续 / 取最强 / 独立实例 / 固定叠层`之一，并保存生效、失效步骤与来源。四种模式的权威语义为：

- `刷新持续`：同一效果键只保留一个规范实例，更新公开幅度与最终失效步骤。
- `取最强`：保留全部尚未失效候选的聚合多重集，当前最强只是派生值；较弱项不能因暂时未生效而删除。
- `独立实例`：效果、幅度、失效步骤和后续负载键相同的实例按真实数量聚合，不同实例不取平均。
- `固定叠层`：保存0至声明上限的整数层，并使用内容定义的统一或逐层失效规则。

第N步第9阶段形成的持续状态默认从N+1步第4阶段生效。不同失效步骤、幅度、来源玩法标签或后续负载不能通过平均值合并。

状态不能无限递归产生伤害、单位或资源。未声明叠加方式的内容不能进入运行状态。

### 9.5 被动范围

被动范围只在来源单位存活、位置合法且目标位于范围时派生：

- 不积累行为进度。
- 不占第二核心行为。
- 来源死亡或离开后立即从下一合法重算边界消失。
- 叠加遵循L2修正顺序。

菌毯的通行、补位和采集改善属于被动范围；其对敌伤害仍来自唯一伤害行为。

## 10. 固定单位、出兵与后台战斗

### 10.1 固定单位

- 固定单位不移动、不随撤离回归，但可以受伤、死亡、执行一个行为或提供被动范围。
- 阻断结构无行为也合法；作用来自空间占用和标签。
- 固定不等于无敌，房间也不会因外形相似而使用单位耐久。

### 10.2 出兵点

出兵点使用`生成单位`行为，并保存真实有限储备或明确输入：

- 行为完成时原子减少储备并形成单位草案。
- 储备不足时停止，不生成负储备。
- 同一步出兵点死亡不取消已经形成的生成草案。
- 新生从下一步骤行动。
- 出兵点死亡或储备耗尽不创建另一套隐藏完成状态；控制资格继续读取存活标签。

### 10.3 敌方恢复

世界契约提供恢复来源、恢复资格与目标节点。本文只执行合法单位形成和后续战斗：

- 只能由存活出兵源、右侧有限储备、入口或明确环境恢复规则形成。
- 永久摧毁的来源不能因退出、读档或摘要恢复。
- 没有玩家单位时恢复仍推进实际敌军位置，不使用隐藏控制百分比。

### 10.4 未占节点后台战斗

节点存在玩家固定单位且敌方恢复或敌军仍活动时，执行完整后台战斗：

- 使用与主动战场相同的移动、目标、伤害、状态、死亡和占领阻断事实。
- 不创建画面代表，不占主动战场槽，不接受战役命令。
- 玩家后台死亡不形成残骸或回收收益。
- 敌方后台死亡形成现场资产，不自动入账。
- 满足占领条件时向世界契约提交同一占领请求。

### 10.5 已占节点

第一版己方控制节点不执行敌方恢复、战斗伤害或控制反转。工蜂领地采集和战斗单位自由游荡使用虫群大军活动，不建立节点单位副本。

## 11. 占领事实接口

每个主动或后台步骤结束时，本文向世界契约提供：

- 存活敌方`占领阻断`单位集合。
- 未完成且可见的特殊目标单位集合。
- 存活玩家可机动与固定单位集合。
- 当前单位位置、出兵储备和已经提交的死亡事实。

本文不计算隐藏进度。世界契约只有在阻断集合与可见特殊目标集合均为空时才能提交节点控制。占领事件必须与移动单位回归、主动槽释放和来源开放组成跨域原子事务。

## 16. 保存、离线与确定性

### 16.1 必须保存

- 全部单位批次、模板、真实数量、来源与生产归因。
- 唯一位置、虫巢/节点/空间段、空间层和活动锚点。
- 规范状态单元的完整玩法键、真实数量、规范顺序与统计归因稀疏份额表。
- 状态模式所需的全部候选/实例/叠层及其幅度、生效/失效步骤、来源玩法标签和最早生效步骤。
- 行为进度、目标、路径、预留，以及用于验证规范序列的内容与参数版本。
- 携带资产引用、出兵储备、状态生效/失效步骤和被动来源。
- 未终结采集交互预留束、全部所有者句柄、精确工蜂份额、可逆边界和最近进攻释放事务标识。
- 战役账期、主动节点引用、后台战斗和累计形成、变形、死亡事实。
- 一次性死亡输出、出兵、撤离、占领回归的幂等标识。

稳定快照不保存步骤内攻击或搬运草案。

### 16.2 离线

- 无主动战场时，未占节点后台战斗、领地采集和房间单位形成按统一步骤推进。
- 有主动战场时整个世界冻结，单位、行为、状态和恢复均不推进。
- 已占节点安全活动不产生战斗。
- 离线事件跳跃必须与逐步执行得到相同目标、伤亡、单位和资产账目。

### 16.3 确定性与表现独立

- 相同快照、命令、参数和模拟时长得到相同单位状态与事件序列。
- 容器顺序、批次分片、线程、帧率、页面和画面配额不能改变目标与结果。
- 画面代表只读取批次和空间形成投影，不能成为目标、伤害、采集或死亡来源。
- 百万级真实数量必须使用状态队列和整数商余数结算，不逐个创建运行对象。

## 17. 第一版约束与排除

- 第一版全局最多一个主动战场。
- 进攻使用全部合法可参战单位，不选择比例或路线。
- 基础战役命令只有进攻和撤离。
- 已占节点安全，不发生失守或塔防。
- 玩家不设置工蜂优先级、单位目标、阵型和个体路径。
- 第一版战斗无随机命中、暴击和浮动伤害。
- 基础撤离不脱根固定菌毯。
- 第一版不提供房间受伤、虫巢战斗和复杂Boss专用阶段。

## 18. 待平衡参数身份

| 参数族 | 需要登记 | 必须验证 |
| --- | --- | --- |
| 单位模板 | 耐久、体型、移动、标签与抗性 | 可读差异、基础可完成性 |
| 行为 | 周期、射程、覆盖、目标组与效果 | 单一行为、构筑涌现与无支配答案 |
| 工蜂 | 携带、交互、服务窗口、老化和活动容量 | 防饥饿、堵塞与恢复 |
| 空间 | 接触、纵深、后备、通行与补位容量 | 宏观数量差和长战场节奏 |
| 伤害 | 基础值、乘区、状态持续和叠加上限 | 确定性、伤亡曲线与抗性价值 |
| 出兵恢复 | 储备、周期、位置和恢复速率 | 多次推进、后台战斗与无无限源 |
| 批次性能 | 规范状态单元、状态实例、效果包、归因键的夹具包络与参考硬件 | 无损精度、确定性、长时增长和候选时间/内存预算 |

批次性能登记是工程验证预算，不是玩法数量或状态队列运行上限；代表配额属于L5表现预算。参数不能改变位置枚举、阶段顺序、单一行为和原子撤离。

## 20. 不变量责任与完成条件

### 20.1 不变量责任

本文直接维护：

- INV-001至INV-003：单位、批次、行为和表现具有稳定身份、唯一所有权且无第二份真相。
- INV-004至INV-006：单位、战役、幼体与变形守恒且不免费治疗。
- INV-010：工蜂搬运只请求资源契约的合法五阶段变化。
- INV-013、INV-014：单主动战场和全合法兵力进攻。
- INV-015：撤离单位与携带资产子事务原子且不可取消。
- INV-016、INV-017：占领只提供实际阻断事实，已占节点安全。
- INV-020至INV-023：单位执行确定、表现独立、原子且幂等。

本文协同维护：

- INV-008：工蜂只把节点来源变为过程资产，不直接形成核心资源。
- INV-019：已形成单位和战局不因读档重抽。
- INV-024：工蜂恢复、全灭结束和单位生产共同保证恢复可达。

### 20.2 完成条件

- 所有第一版生物与可破坏结构都能由同一模板、批次、位置和行为管线表示。
- 形成、变形、死亡、进攻、补充、回归和撤离均满足单位与战役守恒。
- 工蜂从外部来源到房间输入不存在远程物质捷径、重复预留或永久饥饿。
- 规范状态队列经过任意合法分片、合并、保存和重放都得到同一哈希；纯统计归因不改变玩法。
- 批量商余与同时效果在百万数量下保持精确，表现配额和工程超预算处理不改变目标、伤害和死亡。
- 主动与后台战斗复用同一移动、目标、伤害、状态和死亡规则。
- 本文只提供占领事实，节点控制由世界契约唯一提交。
- UBC-001至UBC-032全部映射到L6自动测试；未通过前本文状态不能升级为已验证机制。

世界、候选和轮次调用规则已经由对应L3契约封闭；本文后续只随单位战斗接口变更同步更新。

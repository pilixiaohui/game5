## 9. 根网脉动

| 字段 | 定义 |
| --- | --- |
| `content_id` | `v1.candidate.root_pulse` |
| 目标接口 | 根脉菌毯被动范围与资源能源功率请求 |
| 收益 | 通行、补位和工蜂交互三类根网修正应用提高乘区 |
| 压力 | 每个合法存活根脉菌毯真实单位占用一个真实扎根锚点，并按真实锚点数量累计安全层级4持续功率请求；同态单位可以聚合提交，但不按批次数量计费 |
| 能源不足 | 请求未服务时只使额外强化按服务结果降级；基础菌毯行为和未修改被动不消失 |
| 生命周期 | 本轮定义级修正 |
| 重复 | 第一版不可重复 |

基础请求功率按以下关系形成：

```text
根网脉动基础请求功率
= 合法存活且已占据扎根锚点的根脉菌毯真实数量
  * mutation.root_network.power_per_anchor
```

状态队列、运行批次或容器分片只能把同态请求聚合为较少记录，不能改变真实数量之和、请求总功率或同层分配结果。受伤、状态变化、保存载入和表现配额导致的批次拆分也不能额外增加功耗。

其他参数引用：`mutation.root_network.support_multiplier`、最低服务和服务缩放。固定菌毯留置后台时仍按其真实存活数量形成同一规则的功率请求；页面或画面不可见不停止请求。

## 11. 首次组内容约束

首次共振样本解析从六项公共池形成一个受保护组：

- 组大小使用`candidate.first_group_size`参数，发布候选不得少于两个。
- 成员使用同一`candidate.first_assimilation_genes`成本档。
- 组内不能全部作用于同一对象类型或同一代价结构。
- 每项作用对象在首轮必然可重建，不能因为玩家尚未建房而永久失效。
- 组可以延后选择，不倒计时、不舍弃、不刷新、不重抽。
- 同化一个成员后，其他成员同笔终结，受保护槽关闭。

参数必须满足：

```text
首次解析基因产量
>= 首次组最高同化成本
 + max(主动诱导生成成本, 第一版普通池最高同化成本)
```

这只保证恢复可达，不承诺所有普通来源都能同时免费完成。

## 12. 普通组内容约束

- 第一版至少一个普通持有槽。
- 主动诱导、培养腔和特殊战利品使用同一六项公共池，并过滤已同化不可重复成员。
- 组内多样性继续检查作用对象与代价结构。
- 普通组可以整组舍弃；生成成本、时间、功率和处理资产不返还。
- 来源缓冲组不能直接舍弃，释放槽后按L3稳定顺序自动迁入。
- 合法池不足时拒绝来源或停止房间任务，不生成空组、重复答案或临时占位候选。
- 地图和候选生成都不读取最近战损或当前敌方防线保证克制项。

## 13. 异变腺结处理内容

W节点生成的`v1.asset.mutagenic_gland`使用以下链：

```text
异变腺结现场批次
-> 工蜂保障并回流腐解消化池
-> v1.recipe.separate_mutagenic_gland
-> v1.asset.mutagenic_tissue
-> 工蜂投递突触解析腔
-> v1.recipe.analyze_mutagenic_tissue
-> v1.source.mutagenic_gland_analysis形成一个普通候选组
```

该链可以形成样本标签，但第一版不要求额外基因输出。候选组已有普通槽时进入持有；槽满时进入处理来源缓冲。资产、组和来源事件都只能提交一次。

| 专属池预览 | 血潮回灌扩展、捕食性突触、腐殖统御 |
| 地图字段 | `none` |

参数：`leader.world_devourer.enemy_remains_multiplier`、`frontline_worker_cycle_multiplier`、`candidate_weight_multiplier`、`blood_tide_hatch_modifier`、`blood_tide_cap`、`stable_supply_multiplier`。后台我方死亡和内部回收不能触发血潮回灌。

### 16.4 第一版显示边界

三位领袖只显示基准值、修正、结果、适用标签、独有机制、专属池和弱项。第一版不能选择、解锁或生效领袖；重复打开预览不生成随机候选或修改地图。

## 17. 保留与重置预览

第一版飞升页必须明确显示：

| 保留 | 重置 |
| --- | --- |
| 图鉴与内容知识 | 本轮世界种子、区域图、节点和情报 |
| 已解锁领袖与确认领袖 | 虫巢、房间、任务和本地容器 |
| 已购买永久收益 | 生物质、基因、储能和过程资产 |
| 明确永久化资格 | 幼体、虫群大军、敌我单位、菌毯和残骸 |
| 未消费飞升点和历史记录 | 候选组、本轮突变、本轮发现和本轮随机流 |

第一版只读预览不能建立正式保留列表实例、结束当前轮次或创建新种子。

## 18. 信息与表现要求

候选卡必须显示：

- 中文名、科技族、作用对象和来源。
- 规则变化与参数化收益。
- 新增压力、条件或弱项。
- 生成成本与同化成本分离。
- 重复、互斥和目标当前合法性。

不得显示“推荐”“适合当前敌人”“胜率提升”或动态克制提示。候选动画和卡片顺序读取已提交组，不参与抽取。

飞升预览必须显示具体数字。发布参数缺失时显示配置错误并阻止发布验收，不能用“较高”“擅长”或静态示例替代。

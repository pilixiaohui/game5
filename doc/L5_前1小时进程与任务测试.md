## 1. 首小时体验目标

第一版必须让玩家通过实际状态变化形成以下认识：

1. 虫巢是有空间、输入输出、功率、容量和工蜂交互的生产生态，不是领取菜单。
2. 玩家通过房间布局、生产构成、候选选择、进攻和撤离间接统御虫群，不控制单体。
3. 单位数量从第一只工蜂连续增长到宏观洪流，真实数量与画面代表可以区分并对账。
4. 战损和撤离具有真实代价，但全灭、零储能、堵塞和偏科构筑都有可执行恢复路径。
5. 地图支路、自由房间和有限候选允许不同完成方法，不形成唯一标准路线。
6. 新玩家和熟悉玩家使用同一机制；差异来自理解与操作效率，不来自隐藏补贴或跳过因果链。

首小时不负责证明长期飞升是否平衡，也不要求完成第二虫巢、正式领袖选择、完整插件或无限世界。

## 2. 进程组织原则

### 2.1 因果门而非线性任务

第一版只固定以下不可颠倒的因果：

```text
合法蓝图和自由槽
-> 建设房间
-> 房间任务与网络贡献运行
-> 工蜂完成物理交互

幼体形成
-> 等量分化
-> 单位离巢
-> 虫群大军
-> 进攻或继续积累

固定样本体死亡
-> 样本保障与回流
-> 消化分离
-> 突触解析
-> 基因、资格与首次组
-> 同化一个突变

首次突变完成
-> 普通候选来源开放
-> 普通组形成、选择或舍弃

出口占领
-> 相邻概要与着床条件公开
-> 飞升预览继续读取真实账本
```

E、B、P和W的访问顺序，房间位置、并行程度、单位比例、首次候选结果和撤离时机都不固定。

### 2.2 允许重叠

- 房间建设、已有房间生产、敌方恢复和领地采集可以同时推进。
- 玩家可以在等待孵化时观察地图、调整布局或查看账目。
- 第一次撤离、普通残骸闭环和样本闭环可以发生在同一次或不同次战役。
- E、B、P、W支路可以穿插，出口打开后仍可回访未完成支路。
- 首次候选可以延后选择，但第一版完成里程碑要求最终同化并观察一次规则变化。

### 2.3 不允许的节奏手段

- 固定现实倒计时解锁。
- 页面访问或教程关闭直接完成玩法里程碑。
- 为新玩家暗中增加产出、降低敌军或保证候选。
- 为熟悉玩家跳过资源处理、样本链或战损。
- 用推荐兵力、胜率或标准路线代替玩家判断。

## 3. 首小时因果阶段

阶段描述体验关系，不规定固定分钟。相邻阶段可以重叠，支路可以回访。

### 3.1 P0 轮次建立与虫巢苏醒

权威起点：步骤0已经原子提交初始虫巢、腐殖盆地全部隐藏内容、少量生物质与储能、零基因、自由空槽和内容版本。

玩家可观察：

- 核心巢室与自由空槽。
- 生物质、储能/上限、功率供给/请求/净变化和基因。
- 已知H、C连接及未揭示分支边界。
- 开局可用的腐殖滤囊、热化代谢腔、胚流孵化室和工蜂模板。

玩家可决定：首批房间类型、位置、并行建设程度、是否使用时间倍率。系统不标记唯一正确槽位或建设顺序。

必须保留的恢复：核心巢室第一能源建设保障和第一工蜂应急闭环。

### 3.2 P1 建立生产生态

需要自然出现的关系：

- 建设预留生物质并形成临时功率请求。
- 并行建设与已有基础负载共同改变净功率。
- 腐殖滤囊需要外源物质、工蜂和输出端才能正式入账。
- 热化代谢腔从环境服务形成供给与储能上限，不发能源包。
- 胚流孵化室先形成幼体再分化单位。
- 相邻兼容同类模块自动形成一个房间群。

玩家至少需要观察一个真实缺料、缺能、堵塞或高负载事实，并通过任一合法操作改变该事实。系统只显示来源、位置和影响，不推荐具体修复方案。

### 3.3 P2 第一只工蜂与持续生产

第一只工蜂必须从真实数量`1`建立认知：

1. 没有工蜂时，物理输出停留在房间缓冲。
2. 受限应急链形成一只工蜂，不形成免费库存。
3. 工蜂离巢后自动选择合法物理输出或外源物质。
4. 取货、携带、保障、投递和正式入账可以从详情与动画对应。
5. 工蜂不足、路径或端口拥堵会改变入账速度。

玩家不点击领取，也不设置采集优先级。生产队列可以自动循环，玩家只控制房间、目标规模、绑定、循环与暂停。

### 3.4 P3 军队构成与地图选择

有效幼体循环使噬咬体进入内容池；观察到可扎根地表使根脉孢体进入内容池。玩家通过不同孵化房间群选择持续生产方向。

区域地图在C占领前只提供H到C的明确主方向。玩家观察C时看到已知敌方类型、空间倾向、样本迹象、可参战总量和未知边界，不看到胜率或推荐比例。

发起进攻后：

- 全部合法可参战单位进入节点左侧后备。
- 可逆采集预留先原子释放，携带或不可中断工蜂继续作业。
- 战时新生按L3资格自动补充。
- 玩家只观察和决定是否撤离，不下达个体目标或技能。

### 3.5 P4 持续战场、撤离与再进入

首轮必须让玩家看见：

- 左右入口、接触、纵深、后备和真实补位。
- 噬咬体猎杀、根脉孢体扎根、菌毯截杀与工蜂采集的不同自主行为。
- 单位死亡形成真实残骸，样本载体形成特殊残骸。
- 出兵源存在时，敌方补充来自可见对象或有限储备。
- 战线、损耗、残骸和五阶段资产持续更新。

玩家至少经历以下之一，并在同一存档看到后续状态：

- 确认撤离后再进入同一节点。
- 全灭后生产恢复并再次进攻。
- 留置菌毯与恢复敌军进行后台战斗。

撤离确认后原子提交，当前步骤不再攻击。移动单位回归、固定菌毯留置、预留释放、携带资产保障和超额安全堵塞必须同时可对账。

### 3.6 P5 残骸、样本和房间回流

普通残骸闭环：

```text
单位死亡 -> 现场残骸 -> 工蜂保障/回流
-> 腐解消化池 -> 生物质浆 -> 工蜂投递 -> 生物质入账
```

首次样本闭环：

```text
共振样本体死亡 -> 样本残骸保障/回流
-> 腐解消化池分离样本组织
-> 突触解析腔解析
-> 基因 + 样本标签 + 首次受保护候选组
```

任一步容量不足时保留资产并显示堵塞，不用结算弹窗跳过。样本保障前损失时，共振孵育床按已确认规则恢复唯一链，但不返还战损、时间或普通资源。

### 3.7 P6 第一次突变与可观察差异

玩家主动打开首次组时世界进入关键选择暂停。候选必须显示作用对象、收益关系、压力、来源和公开成本。

玩家选择一个后：

- 基因消费、突变实例、未选成员终结和受保护槽关闭原子提交。
- 返回进入前运行状态与倍率。
- 已选突变在下一合法生效边界作用于现有或未来对象。
- 玩家能在同一房间、物流链或后续战斗中观察至少一次规则变化及其代价。

系统不评价选择优劣，也不因为最近失败更换已形成候选。

### 3.8 P7 普通候选与网状支路

首次同化后，主动诱导、诱变培养腔和W特殊战利品进入实际可运行范围：

- 主动诱导由进化页发起，支付公开生成基因，选择不二次收费。
- 培养腔由房间持续形成，承担生物质、功率、时间和槽位压力。
- 异变腺结走完整保障、消化和解析链，形成普通组。

第一版完成要求玩家至少通过主动诱导或培养腔实际生成并处理一个普通组；W特殊来源必须存在且可运行，但玩家可以跳过，不阻止出口完成。

此阶段玩家可按自己判断访问E、B、P或W。地图只说明已知生态、来源、敌军、连接和历史事实，不输出最优路线。

### 3.9 P8 出口、继续游玩与远景

玩家通过任一合法P路径到达X并完成综合长战场。占领后：

- 移动单位回到虫群大军，固定菌毯留置。
- X进入己方控制并打开唯一通路。
- 世界图公开至少一个相邻区域的主题、连接、资源倾向、整体压力和待着床虫巢节点。
- 第二地区进攻和建巢明确为尚未开放，不生成完整节点网。
- 飞升页用当前真实账本显示六项贡献、准确点数、永久收益和三位领袖差异。

完成第一版体验不清除未占支路，不强迫停止积累，也不自动执行飞升。

## 6. 新玩家与熟悉玩家

### 6.1 共同规则

- 相同起始参数、地图生成、敌军、候选池、账目和恢复边界。
- 相同内容里程碑和第一版完成定义。
- 相同命令，不提供新手专用隐藏产出或熟悉玩家跳过命令。
- 首次关键结果可暂停观察，重复表现可以加速或跳过。

### 6.2 新玩家所需反馈

- 房间状态、端口、缓冲和工蜂路径能在不读公式时建立因果。
- 第一次小规模战斗的真实数量接近可逐个核对范围。
- 首次撤离确认显示准确回归、留置和五阶段资产快照。
- 首次候选有足够时间比较作用对象和代价。
- 未知信息明确标记为未知，不用推荐文本替代情报。

### 6.3 熟悉玩家的压缩方式

- 开局即可使用已开放的统一时间倍率。
- 可以并行建设、快速重排和批量确认合法结构事务。
- 可以跳过重复房间、孵化和战斗镜头，不跳过事件。
- 可以更快决定支路与候选，但候选随机和成本不变。
- 任何倍率下相同模拟时长得到同一结果。

## 7. 失败与恢复体验

| 失败状态 | 必须保留的真实代价 | 最低恢复入口 |
| --- | --- | --- |
| 工蜂全灭 | 工蜂死亡、携带资产掉回现场、时间和战损 | 核心巢室唯一第一工蜂应急链 |
| 储能归零 | 已结算能源、未服务任务零进度 | 第一能源建设保障或能源房间冷启动，玩家可暂停负载 |
| 主动战场全灭 | 单位死亡、节点敌军/结构/残骸持续 | 房间继续生产并再次进攻；固定菌毯按后台规则保留 |
| 首次样本未保障 | 现场样本损失、战损和等待 | 共振孵育床唯一恢复链 |
| 房间输出堵塞 | 已形成资产占用真实容量 | 增加/重组容量、恢复工蜂、暂停来源或清空处理链 |
| 普通候选槽满 | 已形成组和来源成本保留 | 选择或舍弃普通持有组后自动迁入旧缓冲组 |
| 构筑偏科 | 已同化成本和本轮突变保留 | 房间重排、单位生产调整、后续普通候选和可选支路 |

系统只陈述状态、来源和可执行命令，不把某一恢复入口标成标准答案。

## 8. 首小时规模体验

- 第一只工蜂固定以真实数量`1`出现。
- 真实数量在早期可接近一比一表现，超过可读范围后逐步转为批次代表和背景流量。
- 首小时末以约百万级真实虫群作为目标体验量级，不是未经测试的发布区间。
- 数量增长必须来自已形成单位，死亡、变形、进攻、撤离和留置均可对账。
- 代表实体配额变化不改变模拟；百万级不能只显示一个总数字而缺少接触、后备、补充和损耗关系。
- 具体增长曲线、阶段时点、代表阈值和发布区间全部进入L6参数扫描与试玩。

## 9. 任务测试脚本

测试主持人只给出目标，不说明最优策略、内部公式或标准路线。

### 9.1 T-01 建立可运行虫巢

目标：让至少一种资源房间和一种孵化房间持续工作，并说明当前最明显的停滞事实。

记录：首次建设、并行、取消/移动、合并、找到房间详情、指出瓶颈、调整后状态变化。禁止提示正确布局。

关联：E-01、E-06，FH-001至FH-004。

### 9.2 T-02 形成并观察混合虫群

目标：形成工蜂、噬咬体和根脉孢体，并说明三者实际做了什么。

记录：是否尝试个体命令、是否能区分唯一行为、单位数量与画面代表理解。

关联：E-02、E-03，FH-002、FH-005。

### 9.3 T-03 发起风险并处理后果

目标：观察C并决定何时进攻；战况变化后自行决定继续或撤离。

记录：查看的已知事实、进攻理由、撤离/继续理由、战损理解、是否能找到同一节点持续状态。

关联：E-02、E-04，FH-006、FH-007。

### 9.4 T-04 完成普通残骸与样本链

目标：让普通残骸成为可用生物质，并取得第一次基因和候选组。

记录：能否追踪五阶段、消化输出、解析输入、堵塞位置和第一次基因来源。

关联：E-01、E-04，FH-008、FH-009。

### 9.5 T-05 选择并验证突变

目标：比较首次候选，选择一个，并指出后续哪里发生了变化和代价。

记录：选择理由、是否理解组内单选、是否发现收益与压力、修正触发事实。

关联：E-02、E-05，FH-010、FH-011。

### 9.6 T-06 选择自己的出口路线

目标：从C后的已知分支中推进到X；未要求清理全部支路。

记录：支路顺序、回访、单位/房间调整、普通候选来源选择、是否存在唯一被迫路线。

关联：E-05、E-06，FH-012至FH-014。

### 9.7 T-07 读懂第一版远景

目标：在X完成后说明下一地区如何开始，以及飞升将保留和改变什么。

记录：能否区分只读预览与可执行内容、能否复核点数来源、能否比较领袖具体差异。

关联：E-03、E-06，FH-UX-001至FH-UX-004。

## 10. 观察记录

每次任务测试至少记录：

- 参与者类型：首次接触者或熟悉玩家。
- 内容、规则、参数、生成和界面版本。
- 世界种子与候选随机流标识。
- 每个FH里程碑的模拟步骤和现实时间，仅作观察。
- 玩家执行和尝试执行的命令、拒绝原因与页面路径。
- 资源、能源、单位、战损、资产和候选账目摘要。
- 无主持人提示、事实提示、操作提示和错误提示的次数与内容。
- 玩家对瓶颈、战况、数量、突变和远景的原话或编码结论。
- 中断、软锁、恢复、放弃支路和继续游玩的原因。

样本量、任务通过阈值和体验量表在L6测试计划执行前冻结，不能看完结果后修改通过线。

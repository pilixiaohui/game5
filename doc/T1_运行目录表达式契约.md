| 项目 | 内容 |

| 文档层级 | T1，内容编译契约 |

| 权威主题 | 内容数值表达式、上下文绑定、类型与单位、编译结果和错误边界 |
| 当前状态 | 第一版表达式语法已封闭；正式运行目录编译器待按本文实现 |
| 适用范围 | `content.v1.0`运行内容和只读飞升预览内容 |

本文只定义内容字段如何从稳定参数、有限上下文和内容绑定形成确定值。它不确定发布数值，也不允许表达式读取页面、帧率、表现代表、最近失败或任意领域状态。

---

## 1. 唯一输入与编译产物

运行目录编译器只接收：

1. 已通过静态校验的`content.v1.0`定义。
2. 一个明确参数配置：`fixture`、预注册实验配置或`release`。
3. `runtime-expression.v1.0`机器契约。
4. 本契约登记的上下文值和绑定内容ID。

编译结果不是字符串求值器，而是只读的类型化表达式节点。每个结果至少保存：表达式来源内容ID、字段路径、结果类型、单位、参数ID集合、上下文ID集合、绑定槽集合和稳定节点序列。高频模拟不得重新解析JSON或执行动态脚本。

## 2. 根结构

每个计算值统一写为：

```json
{
  "value_type": "float",
  "unit": "seconds",
  "expr": {
    "op": "parameter",
    "parameter_id": "room.digest_pool.cycle_seconds"
  }
}
```

- `value_type`第一版只允许`int`或`float`。
- `unit`必须与参数注册表或上下文登记一致；`scalar`表示无物理单位计数。
- `expr`必须是机器契约登记的AST节点。
- 纯参数字段继续使用`*_ref`并且值必须是单一参数ID；一旦存在绑定或计算，必须使用`*_expr`。
- 同一数值位置不能同时声明有效`*_ref`与`*_expr`；使用表达式时，对应必填引用字段写`none`。

任何包含`+`、`*`、`bound_template_`或`level*`的原始数值字符串都是配置错误。运行时不得调用`Expression`、脚本求值、反射方法或自行拆分字符串。

## 3. 第一版操作符

| 操作符 | 输入 | 结果与约束 |
| --- | --- | --- |
| `constant` | 显式类型、单位和值 | 只允许有限整数或浮点数 |
| `parameter` | 一个活动参数ID | 类型与单位来自参数注册表 |
| `context` | 一个白名单上下文ID | 类型、单位和可用内容类别来自机器契约 |
| `bound_parameter` | 绑定ID和参数槽 | 先解析绑定内容，再由其`binding_parameters`取得参数ID |
| `add` | 至少两个有序参数 | 所有单位相同；全整数得整数，否则得浮点数 |
| `multiply` | 至少两个有序参数 | 只允许一个物理单位，其余必须是`scalar`或`multiplier` |

求值严格按AST数组从左到右、深度优先执行，不交换浮点运算顺序。第一版没有除法、幂、条件、循环、随机、时间读取、集合查询或函数调用。新增操作符必须提升契约版本并提供存档兼容决定。

## 4. 上下文

第一版只有两个数值上下文：

| 上下文ID | 类型/单位 | 可用位置 | 语义 |
| --- | --- | --- | --- |
| `recipe.target_quantity` | `int/scalar` | 配方 | 当前已接受任务的目标批次数量 |
| `permanent.level` | `int/scalar` | 永久收益预览 | 预览快照中的当前等级 |

上下文值必须由调用者在表达式求值前完整提供。缺失上下文使整个目录或该查询失败，不能补零。上下文读取的是同一稳定快照，不允许在一次求值中跨步骤变化。

## 5. 内容绑定

第一版唯一绑定为`room.bound_unit_template`。合法绑定内容必须：

- `kind = unit`、`faction = player`并带有`hatchable`标签。
- 在`spec.binding_parameters`中完整登记`hatch_biomass`、`hatch_larvae`、`hatch_seconds`和`hatch_power`。

配方任务在接受边界把绑定内容ID及四个参数ID写入任务快照。之后改变房间绑定不追溯修改已接受任务，也不能让任务改用新模板参数。

## 6. 类型与单位

- 唯一隐式转换是`int -> float`；禁止`float -> int`隐式截断。
- `add`要求单位完全相同。
- `multiply`忽略`scalar`因子；`multiplier`只能修正另一个单位或形成`multiplier`结果。
- 根声明的类型和单位必须等于AST推导结果。
- 溢出、`NaN`和无穷值均为配置错误，不允许钳制后继续。
- 数量是否必须为整数由消费接口决定；要求整数的接口收到浮点表达式时必须显式拒绝，不能四舍五入。

## 7. 当前结构化公式

第一版使用本语法封闭以下计算：

1. 目标批次数量乘绑定模板幼体成本和生物质成本。
2. 绑定模板的孵化功率与孵化时长。
3. 消化池基础周期乘硬壳时间倍率。
4. 消化池基础生物质输出乘根系回收倍率。
5. 六项永久收益的`基础值 + 等级 * 每级值`及购买后一级预览。

飞升预览使用同一表达式编译结果，不允许页面重复实现公式。

## 8. 编译失败

机器契约登记的错误码是稳定开发接口。出现未知操作符、缺参数、缺上下文、绑定不合格、绑定槽缺失、类型或单位不匹配、原始公式字符串、溢出或非有限值时：

1. `fixture`和实验配置中止该次启动并输出内容ID及字段路径。
2. `release`配置阻止构建或进入配置错误页。
3. 不删除内容、不补默认值、不回退到硬编码目录。

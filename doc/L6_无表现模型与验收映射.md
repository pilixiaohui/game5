## 1. 证据状态

| 状态 | 含义 | 能否作为通过证据 |
| --- | --- | --- |
| `mapped_not_implemented` | 已分配稳定目标套件，尚无对应断言 | 否 |
| `implemented_unverified` | 已有断言，但尚未完成对照、故障注入或重复运行 | 否 |
| `fixture_verified` | 使用机制夹具通过，证明接口可执行、守恒和确定 | 只能证明机制 |
| `release_verified` | 使用已验证参数、发布内容与证据版本通过 | 是 |
| `legacy_probe` | 旧模型测试或探索代码，状态语义与当前L3-1.2不完全一致 | 否 |

任何验收只能由低向高晋升并记录证据版本。一个测试名称、一次成功日志或旧垂直切片行为本身都不是证据。

体验证据另外使用`planned_no_evidence / formative_only / release_collecting / release_passed / release_failed / invalidated`。形成性结果不能作为发布证据，版本变化必须按体验协议使相关证据失效。

## 6. 现有代码迁移判定

旧`V1HeadlessSimulation.gd`和`tests/v1_headless_simulation.gd`继续标记为`legacy_probe`，不能直接晋升。其与L3-1.2的主要差异包括：

- 幼体仍保存在资源字典，而L3-1.2要求幼体只属于单位总账。
- `deploy(unit, count)`允许部分投放，而L3-1.2要求进攻转入全部合法可参战单位。
- `garrisons`、`retained`和返程列表建立了与统一单位位置不一致的数量桶。
- 撤离仍有准备和返程时间参数，而L3-1.2要求第1阶段瞬时原子撤离。
- 工蜂目标可直接增加生物质，尚未实现现场至已回流及房间形成链。
- 候选只覆盖首次受保护组，尚未实现普通槽、来源缓冲和自动迁入。
- 地图校验器只验证给定图，尚未证明完整区域生成、提交、情报和保存。
- 保存只覆盖旧状态字典，尚未实现步骤0、版本头、随机流目录和跨域快照。

旧代码中可复用的是参数CSV解析、批次计算思路、图算法和测试运行方式；旧状态字段与转换不能作为新模型的兼容负担。

独立正式内核位于`res://scripts/simulation/l3/`，权威入口为`L3HeadlessEngine.gd`，状态由`L3StateFactory.gd`从零创建。该目录已经实现稳定ID、命令信封、事件日志、固定步骤、随机子流、快照与迁移，以及资源生产、统一单位战斗、约束地图、候选和飞升预览的第一批机制。证据入口为`res://tests/l3_formal_kernel.gd`；它不读取旧`GameState`字段，也不把旧探针结果计入66项机制夹具。

## 7. 第一批实现状态与后续顺序

1. 已完成独立权威状态根、稳定身份、精确数量、命令信封、事件、十二阶段、随机子流、原子步骤和可迁移快照。
2. 已完成资源与能源、房间事务和生产链、工蜂五阶段、统一单位与战斗、约束地图生成、候选来源缓冲及只读飞升预览的第一批夹具。
3. 已完成单个百万单位规范批次夹具，L6Q-001与L6Q-002达到`fixture_verified`；工程耗时、内存和发布机器预算仍未验证。
4. 下一批按未覆盖验收ID补齐故障注入、非法状态和跨域不变量，直到184项第一版L3与24项INV均有独立证据。
5. 随后建立L4内容静态校验、FH事件覆盖、L5命令查询映射和第一版实际内容包。
6. 最后执行L6P参数实验、冻结参考机器性能测试及E-01至E-06体验协议；这些结果不得由当前夹具外推。

新模型应使用独立状态版本和测试目录；在迁移夹具逐项通过前，不把旧模拟字段静默改名后接入正式存档。

## 8. L6映射完成条件

- 两份L1、两份L2和七份L3的版本、基线绑定与实际SHA-256全部匹配`L3-1.2`机器清单；只匹配标题不算通过。
- 第一版完成定义同时引用184项L3、24项INV、82项L4/L5、12项L6Q、16项L6P和6项体验假设，并保持14项长期契约不阻断发布。
- 198个L3验收ID与24个INV目标在机器登记中无漏号、无重复且范围正确。
- 每项第一版验收拥有可执行测试、故障夹具、断言和证据状态。
- 长期测试不会初始化第一版排除内容。
- 旧探针只能在迁移后逐项替换，不能以文件整体通过作为新机制证据。
- 无表现模型不创建Godot场景节点或读取表现对象。
- 第一版184项机制夹具和24项不变量通过后，才开始参数与首小时批量平衡。
- 82项L4/L5验收均有自动、混合或人工证据入口，不能只停留在文档表格。
- L6Q-001至L6Q-012均有目标套件；候选预算未实测时必须保持`candidate_unverified`，不能据登记表宣称百万级性能达标。
- L6P-001至L6P-016均有目标套件；没有候选范围、模拟、试玩和发布证据时参数只能保持夹具或待验证状态。
- E-01至E-06的样本、任务、通过线和证据槽完整登记；没有发布样本时必须明确保持`planned_no_evidence`。
- 第一版发布还要求同一冻结版本下E-01至E-06全部达到`release_passed`，形成性样本不得合并计算。

## 1. 一页设计简报

| 设计问题 | 第一版回答 |
| --- | --- |
| 游戏是什么 | 单人、生物科幻题材的虫群经营与关卡推进游戏；第一版以Godot桌面版本为验收基线 |
| 玩家是谁 | 塑造虫巢生态和进化方向的虫群领袖，不是单体指挥官或工蜂管理员 |
| 服务谁 | 喜欢观察系统、调整生产关系、积累规模并用构筑验证结果的玩家；同时测试首次接触者和熟悉玩家 |
| 核心幻想 | 统御一座会生产、堵塞、修复和扩张的活体虫巢，让自主虫群从单体成长为宏观洪流 |
| 长期驱动力 | 吞噬更多外部生物质并推进更深区域；能源约束运行强度，基因改变资源到军队和战果的转换方式 |
| 玩家反复做什么 | 观察真实状态，调整房间与生产构成，选择突变，判断是否进攻，并在战斗中决定继续承受损耗或撤离 |
| 核心张力 | 当前虫群规模与构成能否承受已知关卡压力，以及继续推进的边际收益是否值得真实损耗 |
| 第一版承诺 | 在一个初始虫巢和一个受约束随机区域中，完成生产、进攻、损耗、回流、突变和再投资闭环 |
| 第一版终点 | 占领区域出口，看到相邻区域着床条件和按真实账本计算的飞升点、永久收益及领袖差异；不执行正式飞升 |

## 2. 核心体验命题

> 玩家不控制单个单位，仅通过虫巢布局、持续生产、有限候选、进攻与撤离等战略决策，也能理解战果从何而来，并感受到自己正在塑造一支具有生命感、规模感和构筑差异的自主虫群。

命题成立时，玩家能够完成“观察事实 -> 作出取舍 -> 看见系统变化 -> 用结果修正下一次取舍”，不需要系统给出推荐兵种、胜率、战力评分或标准路线。玩家不必理解内部公式，但必须能够从房间、工蜂、战线、伤亡、残骸和账目中识别方向性的因果关系。

命题被反驳的典型表现是：玩家必须操作个体或频繁释放战场技能才能影响结果；只能照着系统推荐行动；战果主要由隐藏倍率决定；改变构筑后看不出差异；或者玩家只需等待数字增长而没有新的取舍。

## 3. 统一核心因果架构

```text
已知外部来源、现场战利品与当前库存
                    |
                    v
工蜂保障和搬运 -> 对应房间处理 -> 生物质、能源能力、基因机会
                    |                         |
                    v                         v
         自由布局、功率与自动生产 <---- 有限突变候选
                    |
                    v
           虫群规模、构成与补充能力
                    |
              玩家选择进攻
                    v
      自主长战场：推进、伤亡、残骸、样本、节点状态
                    |
          +---------+----------+
          |                    |
   玩家选择撤离或继续      清除目标并占领
          |                    |
          +---------+----------+
                    v
       存活单位、留置菌毯和资产按真实状态结算
                    |
                    v
          回流处理、再投资与下一次构筑
                    |
          深度收益下降时形成飞升选择
```

这是一条带反馈的因果链，不是生产、战斗、构筑和飞升四个独立小游戏。生物质是规模目标和主要投入，能源形成吞吐约束，基因提供有限进化机会；节点只提供外部物质来源、抽取容量、环境网络输入、环境容量、房间修正和可追溯战利品，最终核心资源与能源贡献仍由对应房间形成。

玩家只在四类控制点作决定：

1. 虫巢：建设、移动、合并、暂停和重组房间，设置持续生产方向与目标规模。
2. 进化：在有限候选中选择一个会改变既有规则关系的突变。
3. 战役：选择合法节点发起全军进攻，并决定何时瞬时撤离；不配置参战比例、运输、驻军、阵型或个体目标。
4. 长期：第一版只检查准确飞升预览；后续版本由玩家自行决定何时结束本轮并选择永久方向。

## 4. 目标体验与可证伪假设

| ID | 目标体验 | 预期机制原因 | 必须取得的观察证据 | 反驳信号 |
| --- | --- | --- | --- | --- |
| E-01 活体生产生态 | 虫巢像正在运作的生物体，而非菜单列表 | 房间具有空间、输入输出、功率、容量和工蜂交互 | 玩家能从可见事实定位至少一个真实瓶颈，主动调整后观察到对应链路变化；账目与表现一致 | 玩家只看顶部数字、反复领取，或只能靠提示知道哪里停滞 |
| E-02 间接统御 | 不操作个体仍具有控制感和归因感 | 生产构成、唯一单位行为、空间关系和突变共同决定战况 | 同种子同状态的对照中，仅改变一种合法构筑就稳定改变推进、损耗或回流；玩家能指出观察到的差异 | 结果不可重复、差异只能来自隐藏加成，或玩家要求逐单位下令才能影响战局 |
| E-03 吞噬与规模 | 从第一只工蜂成长到宏观虫群具有连续、可信的规模冲击 | 单位形成、死亡、撤离和留置使用同一真实数量账目，画面只作有限代表 | 单位量级跨越可辨认阶段；改变代表实体配额不改变结算；玩家能区分真实数量与画面代表 | 百万级只是文本、账目失真，或数量增大后战线与损耗不可读 |
| E-04 真实代价与恢复 | 进攻和撤离具有紧张感，失败有代价但不摧毁存档 | 实时损耗、原子撤离、残骸回流和不依赖稀有随机的最低恢复链 | 玩家依据真实战况作出继续或撤离选择；极端失败夹具可恢复且守恒；已结算损失不免费返还 | 撤离没有取舍、失败等于回档，或全灭、零能源、堵塞导致永久软锁 |
| E-05 构筑自由与适应 | 玩家形成自己的生产和突变路线，而非执行标准答案 | 自由房间、可选地图支路、有限候选和通用单位互动提供不同代价 | 同一合法种子存在不依赖精确房间顺序或兵种比例的多条完成路径；试玩出现不同且可解释的选择 | 必须使用唯一顺序、唯一候选或唯一单位比例才能完成第一版 |
| E-06 连续首小时 | 新玩家能自然经历完整闭环，熟悉玩家不会被重复等待拖住 | 行为里程碑解锁、并行建设、自动循环与统一时间倍率 | 两类玩家遵守相同账目并触达同一机制集合；熟悉玩家只通过理解、并行和加速缩短现实时间 | 关键机制依赖固定等待，新玩家靠隐藏增益，或熟悉玩家必须跳过因果链 |

上述“玩家证据”的样本量和批次通过比例由L6测试计划在测试前登记；L1不猜测统计阈值。形式守恒、确定性、非法状态和画面配额独立性属于全量通过条件，不能用平均试玩结果抵消。

## 5. 第一版体验契约

| 阶段 | 必须成立的体验事实 |
| --- | --- |
| 起点 | 一个已激活虫巢、少量可读生物质与储能、自由空槽、无现成构筑；第一只工蜂从真实数量`1`建立认知 |
| 建立能力 | 玩家建设并运行对应房间，看见工蜂自动采集、储能与功率供需、持续孵化和真实堵塞 |
| 提交风险 | 玩家比较已知节点压力与当前虫群大军，发起唯一主动战场的全军进攻 |
| 观察结果 | 自主单位在长战场中形成推进、补位、伤亡、菌毯、出兵源和残骸关系，界面只陈述事实 |
| 承担后果 | 玩家至少经历一次撤离、再次进攻或显著战损，存活单位、固定菌毯和五阶段资产按真实状态保留 |
| 改变规则 | 固定样本经回收、分离和解析形成首次基因与候选，玩家选择一个突变并在后续生产或战场中看到差异 |
| 闭合循环 | 回流资产经房间形成核心资源并再投资，玩家占领区域出口，同时仍可返回支路继续积累 |
| 展示远景 | 相邻区域只显示主题、连接、资源倾向和着床条件；飞升页显示准确点数、具体永久收益和领袖差异 |

“首小时”表示新玩家第一次自然游玩的体验容量，不是固定六十分钟的等待脚本。第一版范围以`L1_第一版范围与完成定义.md`为准，具体内容身份由四份L4内容文档维护，体验里程碑与页面路径由两份L5文档维护。

第一版明确不要求正式飞升、第二虫巢、多主动战场、领地失守、塔防、复杂Boss、完整插件、移动端发布、个体命令和战场主动技能。

## 6. 决策与信息契约

每个核心选择必须在行动前显示合法选项、已知事实与未知边界，行动后显示真实资源、状态、时间和损耗变化，并把结果带入后续状态。错误选择应允许转向，但不必撤销已经结算的时间、功率、战损或候选成本。

游戏可以显示房间输入输出、功率、军队构成、已知敌情、战线、伤亡、残骸、样本、节点状态和历史结果；不得生成推荐兵力、胜率、标准构筑、固定路线或“下一步应该做什么”的策略答案。页面访问、动画、画面实体和提示文字不能成为玩法结算条件。

## 7. L1验证门槛

进入L2和风险原型时，所有方案必须通过以下追溯检查：

1. 每项第一版功能能指向因果架构中的输入、玩家决定、状态变化和回流位置。
2. 每项功能至少支持E-01至E-06中的一个假设，并声明可能反驳该假设的观察结果。
3. 无表现模型能够验证资源与单位守恒、确定性、原子交易、失败恢复和画面配额独立性。
4. 对照原型能够只改变一个生产、单位或突变条件并重现不同系统动态。
5. 新玩家任务测试不提供策略答案；熟悉玩家测试不使用隐藏规则或跳过账目。
6. 第一版范围不依赖任何只读预览或明确排除内容才能闭环。

L1通过只表示目标、因果和验证对象已经明确，不表示体验假设已经成立。只有L6登记的模拟与试玩证据通过后，E-01至E-06才能从`设计假设`升级为已验证结论。

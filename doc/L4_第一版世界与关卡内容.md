## 5. H 腐殖母床

| 字段 | 定义 |
| --- | --- |
| 标签 | `initial_hive`、`hive_deployment`、`base_external_source`、`base_thermal_environment`、`safe_node` |
| 初始控制 | 己方控制，虫巢已激活 |
| 战场 | 无主动战斗 |
| 必需对象 | `v1.room.core_chamber`、自由空槽、`v1.source.hive_humus_bed`、`v1.source.base_thermal_gradient` |
| 连接 | 只连接C |
| 经济作用 | 物质来源形成外源有机物现场批次；基础热化环境只提供受容量约束的环境服务，最终生物质与能源能力仍由对应房间形成 |

`v1.source.hive_humus_bed`提供最低完成路线所需的外源物质来源、形成位置和抽取容量。它不直接增加生物质，也不能因工蜂全灭而消失。

`v1.source.base_thermal_gradient`是H在步骤0已经开放的环境网络输入。它提供能够维持低效基础路线的热化环境容量，只允许兼容热化代谢腔通过持续网络贡献形成供给与储能上限；它不是可搬运资产，不直接增加当前储能，也不因玩家跳过E而失效。

## 14. 首轮拓扑硬约束

起始区域生成结果必须同时满足：

1. H、C、X各恰好一个，只有X具有`region_unlock`。
2. H只连接C；C之后至少出现两个分支和一个回环。
3. C到X至少存在两条除端点外内部节点不相交的简单路径。
4. 每条C到X合法路径都经过至少一个P；移除全部P后C与X不连通。
5. 不存在所有路径共同依赖的唯一P；逐个移除任一P后，仍有经过其他P的C到X路径。
6. E和B都存在，但都不是所有出口路径的唯一前置。
7. 至少一个W存在，可跳过且可在出口打开后返回。
8. 所有节点从H可达，额外连接不能产生不经过P的出口捷径。

抽象安全骨架示例只用于校验，不是固定地图：

```text
H -> C
C -> E -> P1 -> X
C -> B -> P2 -> X
E <-> B
P1 <-> P2
W连接上述至少两个非H节点
```

## 15. 内容池与生成顺序

### 15.1 允许随机

- E、B、P、W在地图中的位置和附加兼容标签。
- 满足硬约束后的额外连接、普通节点和回环。
- 空间段排列、长度、支路位置与公开环境。
- 合法敌军模板、批次数量、出兵源位置和有限储备。
- 来源位置、残骸分布、菌毯位置和房间修正。
- 相邻概要原型、连接位置和展示顺序。

### 15.2 禁止随机改变

- H/C/X唯一性、C固定样本、至少两个P和至少一个W。
- 三类不同战场分别存在于C、P、X。
- 基础内容可完成、第一样本可恢复和无P出口捷径。
- E/B非唯一前置、X唯一出口和至少一个相邻概要。
- 当前构筑、军队、最近失败、资源不足和领袖不参与内容答案生成。

### 15.3 原子生成流程

```text
确定腐殖盆地与相邻概要
-> 放置H/C/E/B/P/W/X职责实例
-> 生成并校验节点连接
-> 为每个节点组合空间、单位、来源、环境、残骸和特殊对象
-> 校验基础可达、占领、经济和恢复闭环
-> 原子提交完整区域及随机游标
-> 建立情报遮罩
```

失败尝试只操作事务随机副本。达到参数化上限时使用已验证安全骨架，仍由同一种子决定非关键内容。

## 16. 情报内容映射

| 情报深度 | 可以显示 | 不能显示 |
| --- | --- | --- |
| 区域概要 | 主题、连接、资源倾向、整体压力、虫巢入口 | 完整节点网、敌军和战利品答案 |

玩家对合法相邻节点提交一次`观察节点`命令才能提升到可观察；重复打开详情和进入页面是只读查询。观察不会创建或重抽对象。

## 17. 占领后内容结果

- H初始安全；其他节点只有满足L3阻断与特殊目标集合为空时才占领。
- E、B和W占领只开放已经生成的来源、环境或房间修正，核心资源仍由房间形成。
- 已占节点停止普通敌方恢复和战斗，保留来源、残骸、固定菌毯和安全领地活动。
- X占领只打开世界区域图已声明的相邻连接并固定概要。
- 全部节点控制只形成`完全吞噬`统计与飞升贡献，不重复发放资源。

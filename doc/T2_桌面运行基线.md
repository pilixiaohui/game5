# T2 桌面运行基线

| 项目 | 内容 |
| --- | --- |
| 文档层级 | T2，平台与运行验收基线 |
| 平台基线版本 | `T2-1.0` |
| 权威主题 | 第一版桌面平台、窗口、分辨率、DPI、输入、焦点、文件、导出和运行验收 |
| 当前状态 | 目标基线已确定；当前`540x960`竖屏主场景不符合本基线，不能用于第一版桌面完成验收 |
| 适用范围 | 第一版Windows桌面发布候选、开发调试包和无窗口自动化 |
| 直接依赖 | `T1_Godot技术实现基线.md`、`L5_页面信息架构与交互契约.md`、三份L6验证文档 |

桌面端是第一版交互和性能验收门槛。本文定义可测试的运行环境，不把窗口尺寸、DPI、帧率或输入设备变成玩法条件。

## 1. 平台范围

| 项目 | 第一版决定 |
| --- | --- |
| 发布平台 | Windows 10 22H2及Windows 11，x86-64桌面 |
| 主要输入 | 鼠标与键盘 |
| 显示形态 | 横向可调整窗口、无边框窗口和全屏 |
| 单机能力 | 不依赖网络即可新建、载入、保存和完成首小时切片 |
| 多窗口 | 第一版单主窗口，不弹出独立工具窗口 |
| 控制器 | 可保留映射接口，不作为第一版完成门槛 |
| 触控与移动 | 不属于第一版发布门槛 |
| 其他桌面系统 | 可用于开发兼容检查，不作为第一版完成声明的一部分 |

发布包不得要求玩家安装编辑器、脚本运行时或开发依赖。

## 2. Godot项目设置目标

正式桌面入口完成迁移后，`project.godot`应表达以下目标：

| 设置 | 目标 |
| --- | --- |
| 项目名 | `异种起源：无尽洪流`，不保留“垂直切片”发布后缀 |
| 引擎功能版本 | `4.7` |
| 渲染方法 | `gl_compatibility`作为第一版2D桌面兼容基线 |
| 参考视口 | `1600 x 900` |
| 拉伸模式 | `canvas_items` |
| 纵横策略 | `expand`，页面使用容器和最小约束适配额外空间 |
| 窗口调整 | 允许调整大小 |
| 高DPI | 启用；结合OS缩放和应用UI缩放测试 |
| 初始模式 | 普通窗口；记住玩家上次合法窗口模式和尺寸 |
| 垂直同步 | 默认开启，可在画面设置中切换 |
| 退出 | 主菜单退出与窗口关闭共用安全保存/确认流程 |

选择兼容渲染基线是因为第一版为2D房间、地图和长战场，不依赖高阶3D管线。若后续视觉需求必须更换渲染方法，应先在最低显卡、导出包、字体和全部页面上重跑桌面验收。

当前工程仍使用`540 x 960`竖屏入口和旧主场景。只有新页面达到本文件验收后才能修改项目默认入口；不能仅改两个分辨率数字后宣称桌面迁移完成。

## 3. 窗口与分辨率

### 3.1 验收矩阵

第一版至少测试：

| 模式 | 客户区或桌面分辨率 | 目的 |
| --- | --- | --- |
| 最小窗口 | `1280 x 720` | 最小有效信息密度与所有关键操作可达 |
| 常见笔记本 | `1366 x 768` | 非整数缩放和紧凑高度 |
| 参考窗口 | `1600 x 900` | 设计参考布局 |
| 全高清 | `1920 x 1080` | 默认发布截图和长战场视野 |
| 16:10 | `1440 x 900`、`1920 x 1200` | 额外纵向空间分配 |
| 高分辨率 | `2560 x 1440` | 高DPI、字体和图标清晰度 |
| 超宽 | `2560 x 1080` | 横向扩展不导致控件散开或战场失真 |

小于`1280 x 720`时允许阻止继续缩小，或进入明确的“窗口过小”保护布局；禁止静默裁掉导航、进攻、撤离、保存、关闭弹窗或房间停止操作。

### 3.2 稳定布局原则

- 根UI使用`Control`与容器布局，不使用参考分辨率下手工定位的关键控件。
- 顶部状态、主导航、主动战场带、页面主体和弹层层级按L5固定职责组织。
- 额外宽度优先增加地图、虫巢剖面和长战场可视区域，其次增加详情栏，不把正文无限拉宽。
- 紧凑宽度下详情栏可以收起、覆盖或切换为独立页，但主视图和主要命令不能互相遮挡。
- 科技候选、房间建设、详情、撤离和飞升使用独立弹层或页面，不把所有按钮压入同一屏。
- 长战场保持横向连续空间；窗口比例变化只改变可视范围和投影密度，不改变真实距离、接触容量或命中。
- 固定格式控件使用明确最小尺寸、网格轨道或宽高比，数字变化、悬停和状态徽标不能推动整页跳动。

### 3.3 全屏与多显示器

- 窗口、无边框窗口和全屏切换不得重建游戏会话或推进额外步骤。
- 切换显示器后使用目标显示器有效工作区约束窗口，不能把标题栏或确认按钮留在屏幕外。
- 记住窗口位置前先验证其仍属于当前显示器布局；显示器移除后回到主显示器可见区域。
- `Alt+Enter`可以切换全屏，设置页必须提供等价可见入口。

## 4. DPI、UI缩放与文本

### 4.1 测试组合

至少在Windows显示缩放`100% / 125% / 150% / 200%`下检查参考窗口和全屏。高DPI测试必须使用导出包，不只查看编辑器预览。

### 4.2 应用UI缩放

设置提供离散档位：

```text
100% / 115% / 130%
```

UI缩放是玩家设置，不随视口宽度连续改变字号。每个档位重新计算控件最小尺寸、边距和滚动区域；不能只放大字体后让文本溢出按钮。

### 4.3 文本规则

- 第一版中文正文在100%应用缩放下不小于16逻辑像素。
- 高密度表格、次级标签和时间戳不小于14逻辑像素。
- 主要按钮、资源总量、房间阻断和战场关键数字不小于16逻辑像素。
- 文本允许正常换行；稳定ID、数值和不可断词内容提供省略与悬停完整值。
- 字体后备必须覆盖简体中文、ASCII数字和常用符号；缺字进入发布错误清单。
- 字号不使用视口宽度公式，字距固定为`0`。

这些是可读性下限，不替代真实玩家可用性测试。

## 5. 输入基线

### 5.1 鼠标

| 输入 | 通用行为 |
| --- | --- |
| 左键 | 选择、确认、切换、拖动合法布局对象 |
| 右键 | 返回上层、关闭非阻断详情或取消尚未提交的布局意图 |
| 滚轮 | 滚动当前悬停容器；地图/战场缩放仅在明确画布区域生效 |
| 中键拖动 | 地图和长战场平移，可由设置关闭 |
| 悬停 | 显示工具提示，不作为发现、解锁或命令条件 |

危险操作使用明确确认：撤离确认后是不可取消的原子命令；退出当前未保存会话、删除存档和正式未来飞升同样需要相应确认。普通页面切换、查看详情、改变筛选和预览不需要确认。

### 5.2 键盘

| 按键 | 通用行为 |
| --- | --- |
| `Esc` | 关闭最上层弹窗；无弹窗时返回上层或打开系统菜单 |
| `Enter` / `Space` | 激活当前键盘焦点控件 |
| `Tab` / `Shift+Tab` | 在可交互控件间顺序移动焦点 |
| 方向键 | 在标签、列表、网格或地图邻接项间导航 |
| `Ctrl+S` | 请求手动保存；不能绕过保存状态检查 |
| `F11` | 切换全屏 |

页面级快捷键可以增加，但必须满足：

- 有可见等价控件。
- 输入焦点位于文本输入时不误触发。
- 不与系统和无障碍常用键冲突。
- 只提交现有命令，不创建快捷键专用玩法路径。

### 5.3 键盘焦点

- 弹窗打开时焦点进入弹窗并被限制在其内部。
- 弹窗关闭时焦点回到触发控件或其稳定替代项。
- 页面切换后焦点进入页面标题或第一个主要控件，不留在已释放节点。
- 颜色、缩放、动画和加载状态不能移除焦点轮廓。

## 6. 页面运行约束

页面内容和命令仍以L5为权威；桌面实现额外满足：

### 6.1 虫巢

- 首屏主体是虫巢剖面和实际房间，不是按钮总表。
- 每个房间工作区能显示任务阶段、功率、输入输出、工蜂交互和阻断投影。
- 相邻同类房间合并后仍能看出多个模块工作区，只打开一个群详情。
- 建设选择、房间群详情和布局编辑不会同时永久占据主视图。
- 缩放与平移不改变槽位拓扑或合并判定。

### 6.2 地图

- 世界图和区域图使用独立层级，返回后保留中心、缩放、选中节点和详情上下文。
- 敌我状态、已知连接、情报深度、职责和可进攻性不能只靠颜色区分。
- 节点详情不遮挡地图返回、进攻入口和关键来源事实。
- 相邻生态预览明确不可进入，不产生可点击但必然失败的伪操作。

### 6.3 长战场

- 战场主体保持最大可用区域，战况摘要和进攻/撤离命令不覆盖主要接触区。
- 画面代表、背景流量、残骸和警告区域分别投影，不能让一个代表等同于一个真实单位。
- 玩家能查看进度、敌我真实数量、出生、死亡、残骸、战利品保障和来源状态。
- 代表数量变化、镜头缩放和窗口尺寸变化不影响战斗结果。

### 6.4 进化与飞升预览

- 进化主页面显示持有组、来源、科技族和已同化突变；候选选择在独立弹层完成。
- 候选卡显示对象、收益、压力、来源、形成成本和同化成本，不显示推荐答案。
- 飞升预览显示同一快照的六项贡献、预计点、参数版本、六项永久收益和三位领袖差异。
- 发布参数缺失时显示配置错误并阻止发布验收，不用形容词或静态示例代替数值。

## 7. 动画、闪烁与声音

### 7.1 动画强度

设置至少提供：

```text
完整 / 降低 / 最低
```

- 降低档减少镜头位移、屏幕震动、粒子密度和重复房间动作幅度。
- 最低档保留状态变化所需的淡入、进度和警告，不播放大幅循环运动。
- 调整动画速度或强度不改变任务进度、战斗周期、候选形成或保存时机。

### 7.2 减少闪烁

- 开启后禁止快速全屏亮度跳变和高频颜色反转。
- 伤害、能源不足和关键警告使用边框、图标、文本和受控亮度共同表达。
- 任意必要闪烁都必须受频率与持续限制，并有非闪烁等价反馈。

### 7.3 音频

- 总音量、音乐、环境、效果和界面分别可调并可静音。
- 失焦静音是独立设置，不能暂停模拟。
- 音频播放完成、声道被抢占或设备切换不产生玩法事件。

## 8. 焦点、最小化与系统暂停

| 情况 | 运行规则 |
| --- | --- |
| 窗口失焦 | 默认继续请求统一模拟步骤；输入状态清空，避免按键卡住 |
| 最小化 | 不依赖渲染帧推进；按L3时间和离线边界在恢复时补算 |
| 系统睡眠 | 唤醒后作为离线经过时间处理，不假定进程持续执行 |
| 显示器切换 | 重建UI投影，不重建会话 |
| 音频设备变化 | 重建音频输出，不改变模拟 |
| 系统时间回拨 | 由离线时间校验处理，不能产生负推进或重复收益 |

失焦、最小化和恢复前后保存载入必须得到同一规范状态哈希，允许的差异只来自明确经过并被接受的模拟步骤。

## 9. 本地文件基线

第一版所有可写数据位于`user://`：

```text
user://settings.json
user://saves/index.json
user://saves/slot_01.sav
user://saves/slot_01.bak
user://logs/latest.log
user://diagnostics/                # 仅玩家主动导出或调试构建
```

- 安装目录只读时游戏仍能运行和保存。
- 文件路径、存档名和日志正确处理中文Windows账户路径。
- 自动保存和手动保存使用同一原子仓储。
- 设置损坏回退到明确的安全表现默认值，并保留损坏文件供诊断；玩法存档不能无提示重置。
- 删除存档必须确认，并在成功后更新索引；失败时索引不能声称已经删除。
- 日志轮换有上限，不无限占用磁盘。

## 10. 导出包基线

### 10.1 发布候选

- 使用Godot `4.7.stable`匹配的Windows导出模板。
- x86-64单机导出，项目数据打包，开发控制台默认关闭。
- 调试符号和详细诊断包与玩家发布包分离保存。
- 包内记录构建版本、Git提交、规则、内容、参数、生成和飞升公式版本。
- 发布包首次运行不写安装目录，不访问网络也能完成核心流程。

### 10.2 包外验证

每个发布候选解压到没有工程源码的新目录后执行：

1. 首次启动。
2. 新建存档。
3. 手动保存、退出、重启和继续。
4. 窗口/全屏、DPI、音频和文本设置。
5. 首小时任务脚本或稳定自动化路径。
6. 配置错误和损坏存档的安全拒绝。
7. 性能与内存证据采集。

编辑器中运行成功不能替代导出包验证。

## 11. 性能边界

桌面基线固定体验目标，但不在本文猜定未经测试的硬件结论：

- 前台交互目标为60 Hz显示更新。
- 模拟结果与30、60、120、144 Hz渲染刷新无关。
- 长战场和虫巢画面使用对象池与表现预算，不按真实数量创建节点。
- 页面切换、打开候选和保存不会长时间阻断输入。
- 离线长推进可显示同步状态，但结果必须完整且可取消到上一个已提交存档边界，不能产生半提交。

具体模拟时间、主线程、内存、快照大小和载入时间预算由`L6_百万级状态队列与性能预算.md`登记。发布前必须记录实际CPU、GPU、内存、操作系统、分辨率、DPI、Godot版本、导出模式、内容/参数版本和测试种子；没有这份报告不能声明最低配置或百万级性能达标。

## 12. 桌面验收清单

### 12.1 启动与配置

- `[T2D-001]`正确配置下从导出包进入标题页，没有脚本错误和缺失资源。
- `[T2D-002]`内容、参数或版本错误时进入配置错误页，不创建会话。
- `[T2D-003]`新游戏和继续游戏不会同时建立两个状态根。

### 12.2 布局与DPI

- `[T2D-004]`第3.1节全部尺寸和第4.1节全部DPI组合无关键文本裁切、控件重叠或不可达操作。
- `[T2D-005]`100%、115%、130%应用缩放均可完成标题、虫巢、地图、战场、进化、保存和设置流程。
- `[T2D-006]`超宽只增加合理可视区域，不把信息拉散到无法扫描。
- `[T2D-007]`窗口过小时明确保护，不出现半屏确认弹窗。

### 12.3 输入

- `[T2D-008]`鼠标可以完成全部第一版任务。
- `[T2D-009]`键盘可到达并激活全部关键控件，焦点顺序稳定可见。
- `[T2D-010]`弹窗焦点被正确限制和恢复。
- `[T2D-011]`快捷键不绕过命令资格或确认边界。

### 12.4 模拟独立性

- `[T2D-012]`同一初态和命令在30、60、120、144 Hz渲染条件下产生相同规范状态哈希。
- `[T2D-013]`改变窗口大小、DPI、动画强度、音量、镜头和代表配额不改变资源、单位、战斗或候选结果。
- `[T2D-014]`失焦、最小化、睡眠恢复和读档只按合法统一步骤产生差异。

### 12.5 保存与恢复

- `[T2D-015]`中文用户路径下保存、备份、读档和删除正常。
- `[T2D-016]`写入中断保留上一份有效存档。
- `[T2D-017]`损坏、未来版本或未知内容ID明确拒绝，不静默重置。
- `[T2D-018]`自动保存和手动保存的规范负载均可通过校验和与不变量审计。

### 12.6 表现与可访问性

- `[T2D-019]`房间生产、工蜂交互、战场损耗、样本链和候选结果都能从权威事实重建。
- `[T2D-020]`减少闪烁与最低动画强度下信息仍完整。
- `[T2D-021]`颜色不是敌我、阻断、资格和错误的唯一编码。
- `[T2D-022]`音频完全静音后全部关键状态仍可理解。

### 12.7 证据产物

每次桌面发布验收保存：

- `[T2D-023]`构建与全部规则版本。
- `[T2D-024]`测试机器登记、分辨率、DPI和UI缩放矩阵结果及各主要页面截图。
- `[T2D-025]`输入、焦点、窗口模式和保存日志。
- `[T2D-026]`确定性哈希对照。

## 13. 完成定义

桌面运行基线只有`T2D-001`至`T2D-029`全部达到要求且同时满足以下条件才算完成：

1. 正式项目入口已从竖屏旧主场景迁移为横向桌面游戏外壳。
2. 全部分辨率、DPI和应用缩放组合通过布局与文本检查。
3. 鼠标和键盘都能完成第一版核心流程，且不存在只能靠快捷键完成的操作。
4. 窗口、帧率、DPI、动画、音频和代表数量与玩法结算严格分离。
5. 失焦、最小化、睡眠、退出、保存和恢复遵守统一时间与存档契约。
6. `[T2D-029]`Windows导出包在工程外目录完成启动、保存、重启和首小时流程。
7. 发布构建在已登记参考机器上取得L6性能证据。
8. 所有未通过项有明确阻断状态，不能以“编辑器中基本可用”替代。

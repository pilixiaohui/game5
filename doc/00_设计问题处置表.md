## 1. 状态规则

| 状态 | 含义 | 是否阻塞当前第一版设计 |
| --- | --- | --- |
| `已关闭` | 已有唯一权威规则或内容定义 | 否；按所列权威执行 |
| `待参数验证` | 机制身份已定，只缺候选范围、模拟和试玩 | 不阻塞机制实现；阻塞发布值 |
| `待体验验证` | 页面、信息或可理解性需真人任务证明 | 不阻塞原型；阻塞对应体验发布 |
| `待性能与体验验证` | 同时依赖参考机器和玩家可读性 | 不阻塞语义实现；阻塞表现发布 |
| `长期延期` | 第一版排除或只有长期内容进入时才需决定 | 否；不得扩大第一版 |
| `已废弃` | 问题前提已被新机制明确取消 | 否；不得恢复旧参数或旧流程 |

`待参数验证`统一服从`L6_平衡参数与首小时规模验证.md`，不能在本表写中心值。`长期延期`重新激活时必须建立新决策和适用版本，不能直接引用旧问题作为实现许可。

## 4. 当前阻塞结论

105条旧问题中，没有仍需在L1至L4先行裁定、才能开始第一版机制实现的未登记分歧：

- `已关闭`项直接读取所列权威文档。
- `待参数验证`项进入L6P实验，不允许现在猜正式值。
- `待体验验证`和`待性能与体验验证`项允许制作原型，但阻塞对应发布证据。
- `长期延期`项不进入第一版命令、内容、参数或完成依赖。

这不表示所有问题已经获得答案，而是每个问题已经拥有唯一责任层、当前状态和合法重新打开条件。

## 5. 新问题登记模板

后续新增问题使用：

| 字段 | 要求 |
| --- | --- |
| 问题ID | 使用下一连续`Q-`编号 |
| 所属层级 | L0至L6或治理 |
| 问题 | 可证伪、不可把方案写进问题 |
| 当前阻塞 | 明确阻塞的文档、原型或证据；无则写`none` |
| 方案 | 2至3个可区分方案及主要代价 |
| 所需证据 | 形式、模拟、性能或玩家任务 |
| 责任层 | 唯一决策所有者 |
| 状态 | 使用第1节状态 |
| 重新打开条件 | 哪个版本、内容或证据变化触发 |

旧问题不得在原文件继续追加；原文件只保留历史文本。

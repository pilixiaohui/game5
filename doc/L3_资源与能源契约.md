## 1. 契约职责与所有权

### 1.1 本文负责

- 生物质与基因的可用量、核心资源预留、正式入账、消费、释放和返还。
- 房间结构投入如何由预留形成、如何在重组中保持、如何在拆除或降级后返还。
- 过程资产批次的唯一所有权、五阶段、处理预留、转化、终结和物质来源链。
- 外部来源、残骸、物理输出和网络输出何时才形成可用资源。
- 当前储能、储能上限、功率供给、请求负载、已服务功率、充放能和溢散。
- 固定能源安全层级、同层分配、可缩放与整笔负载、任务进度和堵塞边界。
- 第一能源房间建设、能源冷启动和第一只工蜂应急直连的防软锁规则。
- 资源、过程资产和能源事件的账目投影、保存事实、离线等价与自动验收。

### 1.2 本文不负责

- 房间蓝图、具体配方、单位模板、节点来源内容、候选成员或飞升收益内容。
- 工蜂如何选择目标、移动、受伤和死亡；本文只验证并提交其资产变化草案。
- 房间任务的队列策略、结构事务、模块合并与动画；前3项由`L3_虫巢与房间契约.md`负责，本文只提供输入、功率与输出事务。
- 具体成本、产率、容量、功率、时长、倍率、最小单位和首小时数值曲线。
- 玩家界面布局、音效和表现实体。

### 1.3 唯一写入者

| 权威事实 | 唯一写入接口 | 其他系统只能做什么 |
| --- | --- | --- |
| 生物质与基因账目 | 本文核心资源事务 | 请求预留、消费、释放、返还或入账 |
| 能源状态 | 本文功率服务事务 | 登记供给贡献、储能贡献和负载请求 |
| 结构投入 | 本文结构投入事务 | 请求形成、迁移、返还或明确永久损失 |
| 过程资产批次与阶段 | 本文过程资产事务 | 提交生成、预留、搬运、转化或终结草案 |
| 外部物质来源储量 | 世界契约与本文组成的跨域事务 | 世界提供合法来源，本文同步生成有来源资产 |

账目条目是已提交事件的审计投影，不是第二次资源变化。界面缓存、房间面板、节点摘要和离线摘要都不得成为第二份库存。

## 2. 数量语义与轮次初始状态

### 2.1 精确数量

- 所有可守恒数量必须使用离散最小单位或等价的精确表示。
- 显示小数、缩写和插值动画只是表现投影，不能反向写入账目。
- 批次合并、拆分、时间倍率和离线事件跳跃不能因舍入改变总量。
- 配方结果不能表示为无法在声明精度下复算的无限小尾数；参数登记必须同时声明量纲与精度。
- 任何负库存、负预留、负资产数量或超出储能上限且没有溢散事件的状态都属于实现异常。

### 2.2 轮次初始注入

当前轮次建立时，在`L3_飞升保存与轮次契约.md`协调的步骤`0`轮次建立事务中执行一次原子的经济子事务`轮次初始注入`：

- 建立初始生物质、初始基因、当前储能、基础储能上限和对应来源明细。
- 数量由新存档、永久收益和领袖等已提交跨轮事实共同求值。
- 初始注入不计入本轮外部同化、内部回收、房间产出或节点收益。
- 同一轮次标识只能拥有一个初始注入事件；载入、复制界面或重建派生状态不能再次执行。
- 初始注入完成后，新增核心资源必须遵循本文的房间形成、正式入账或合法返还边界。

必须满足：

```text
0 <= 初始当前储能 <= 初始储能上限
初始可用生物质 >= 0
初始可用基因 >= 0
```

初始值是L6参数，不是节点奖励。第一版必须通过恢复可达性测试证明这些初始事实能够建立第一只工蜂和第一座能源房间。

## 3. 生物质与基因账目

### 3.1 权威字段

对生物质和基因分别保存：

| 字段 | 含义 |
| --- | --- |
| 可用量 | 尚未被任何任务锁定、可以参与下一笔合法交易的数量 |
| 核心资源预留 | 已由唯一任务或交易锁定、尚未消费的数量与预留记录 |
| 累计正式入账 | 本轮通过合法形成边界新增的数量 |
| 累计返还 | 由未消费预留或结构投入合法回到可用量的数量 |
| 累计消费 | 已在声明边界不可逆提交的数量 |
| 初始注入 | 本轮开始时的一次性期初事实 |

任一账期必须满足L2公式：

```text
期初可用 + 期初预留 + 本期正式入账 + 本期返还
= 期末可用 + 期末预留 + 本期已提交消费
```

初始注入只进入第一账期的期初，不重复列入正式入账。

### 3.2 核心资源预留记录

每笔预留至少记录：

- 预留标识、资源类型和精确数量。
- 唯一拥有该预留的命令、过程任务或原子交易标识。
- 建立步骤与阶段、来源事件和参数版本。
- 声明的消费边界、取消边界与最迟释放原因。
- 当前状态：`持有 / 已消费 / 已释放 / 已终止`之一。

同一数量不能被两个预留拥有。预留记录只有一个终结结果，重复消费或重复释放必须由幂等标识拦截。

### 3.3 原子预留

一笔交易需要多种资源、槽位、输入或容量时，必须一次验证并原子建立全部预留：

```text
读取同一稳定事实
-> 校验全部可用量与目标
-> 按L3事件排序键确定冲突顺序
-> 同笔减少各资源可用量
-> 同笔增加各资源预留量
-> 建立唯一预留记录和任务
```

- 任一输入不足时整笔不成立，不允许只锁生物质而未锁基因或容量。
- 同一边界内先提交的合法命令先占用资源；后续命令读取更新后的可用量。
- 玩家命令不足时进入`已拒绝`；自动循环不足时不创建任务并派生`缺料`，不是命令失败。
- 预览不建立预留，也不能暂时阻止其他交易。
- 仅位于队列、尚未形成过程任务的条目不占用资源。

### 3.4 三种消费边界

每个配方必须从以下模式中选择一种，不能使用“过程中大概扣除”：

| 模式 | 规则 | 第一版典型用途 |
| --- | --- | --- |
| 完成提交消费 | 任务建立时完整预留，输出或结构成功提交时才消费 | 建设、幼体形成、单位分化、主动诱导 |
| 开始提交消费 | 进入明确不可逆阶段时消费；之后取消不返还，结果必须拥有安全待提交位置 | 明确声明的设施形成阶段 |
| 即时交易消费 | 在一个边界事务中预留、再次校验、消费并提交结果 | 候选同化、后续一次性能力 |

- 第一版不使用模糊的生物质或基因“每秒扣除”。持续配方必须拆成可审计的离散任务批次。
- 能源不使用本表；持续能耗由第7节功率服务结算。
- 任务暂停保留未消费预留和进度。取消只释放尚未消费的预留。
- 已消费输入不能因输出不受欢迎、页面关闭、任务拆分或读档而返还。

### 3.5 正式入账、返还与消费

| 交易 | 可用量 | 预留量 | 累计事实 |
| --- | --- | --- | --- |
| 正式入账 | 增加 | 不变 | 正式入账增加；按来源更新外部同化或内部回收 |
| 建立预留 | 减少 | 增加 | 不记消费 |
| 释放预留 | 增加 | 减少 | 记释放，不记结构返还或产出 |
| 消费预留 | 不变 | 减少 | 消费增加 |
| 结构投入返还 | 增加 | 不变 | 结构返还增加，不记正式产出 |

正式入账必须带有直接来源对象、上游物质来源事件和入账端引用。不能以“节点完成”“动画结束”“离线奖励”或“战后领取”作为资源来源。

### 3.6 同一步可用性

- 第1阶段释放的预留可以被同阶段中接收序号更晚的玩家命令重新预留。
- 第10阶段形成的生物质和第11阶段形成的基因、结构返还不能追溯满足本步骤第1或第3阶段已经失败的预留。
- 唯一例外是`L3_虫巢与房间契约.md`声明的替换阶段转换：同一第11阶段原子事务先提交旧结构返还，再从提交后的可用量完整建立该替换的新结构预留。返还在事务前不可见，不能服务其他任务或追溯补足更早阶段请求。
- 自动循环最早在下一完整步骤第3阶段读取这些新增可用量。
- 所有任务在开始前锁定输入；第11阶段不能临时抢用本步骤刚形成的资源完成未预留输出。

## 4. 结构投入契约

### 4.1 建设与升级

```text
建设命令合法
-> 第1阶段完整预留结构资源并占用槽位
-> 第3阶段按已服务功率推进建设任务
-> 第11阶段原子消费结构预留并形成房间模块结构投入
-> 新房间、供给、储能上限和运行资格按时间契约从下一步骤生效
```

- 建设、升级和替换不能把预计总能耗作为第二笔能源库存预扣。
- 建设达到完成进度但目标槽位或模块状态失效时保持待提交，不消费结构预留。
- 一个模块的结构投入按形成事件和等级分项记录，不能只保存无法审计的总分。

### 4.2 取消、移动、合并与拆分

- 未完成建设取消时，全部未消费结构预留在第1阶段返回可用量。
- 已经结算的模拟时间、功率服务、一次性储能消费和其他任务结果不回滚。
- 移动、房间群合并与拆分不改变结构投入、累计消费或累计返还。
- 重组任务可以产生时间与临时负载，但不能重新扣除同一结构材料。
- 暂停重组保留原结构投入；取消未完成移动恢复原拓扑。

### 4.3 拆除、替换与降级

- 玩家主动拆除完整模块时，结构投入一直保留到拆解任务完成。
- 第11阶段完成拆解时，模块失效与可回收结构投入返还必须同笔提交。
- 玩家主动降级只返还被移除等级对应的完整可回收结构投入。
- 替换必须按房间契约执行旧结构拆解与新结构建设两段事务；只有旧投入在第11阶段实际提交返还后，同一事务才能完整建立新投入预留，不能提前借用尚未提交的返还。
- 战斗破坏、腐化或其他风险造成的结构损失必须形成明确永久损失事件；不能伪装成布局编辑成本。

### 4.4 容量下降

自愿拆除、替换或降级若会使以下任一事实失去合法去向，最终提交保持堵塞：

- 当前过程资产、房间输入、输出或安全回流缓冲。
- 当前储能高于变更后的全局储能上限。
- 尚未提交的任务输出、候选输出或资源预留。

玩家可以先消耗储能、清空缓冲或建立替代容量。第一版自愿布局操作不静默销毁资产或储能。强制破坏或临时储能修正到期无法等待时，超额储能必须形成可见的`储能溢散`事件。

## 5. 过程资产账本

### 5.1 批次最小字段

每个过程资产批次至少记录：

- 稳定批次标识、资产类型、精确数量和参数版本。
- 唯一来源事件、直接来源对象和物质谱系。
- 当前五阶段之一，以及唯一位置或携带者引用。
- 采集任务、过程输入任务和容量预留引用。
- 腐败、敌方夺回或其他终结条件的进度事实。
- 最早可参与目标选择或处理的步骤。

只有资产类型、来源谱系、阶段、位置、风险状态和预留事实全部兼容的批次才能合并。拆分必须保存总量与来源比例，不能通过拆分刷新腐败、改变优先级或复制外部同化资格。

### 5.2 五阶段与合法变化

```text
现场 -> 已预留 -> 携带中 -> 已保障 -> 已回流
```

并非每份资产必须经过全部阶段。合法捷径只包括：

- `携带中 -> 已回流`：工蜂直接到达兼容房间输入。
- `携带中 -> 终结`：工蜂到达账本接入口，物理资产终结并同笔正式入账。
- `现场 -> 终结`：腐败、敌方夺回、明确摧毁或不产生物质的处理。

任何节点控制、页面操作、战后摘要或动画都不能把`现场`、`携带中`、`已保障`或`已回流`直接改写为核心资源。

### 5.3 两种资产预留

| 预留 | 适用资产 | 阶段变化 | 释放规则 |
| --- | --- | --- | --- |
| 采集资产预留 | 位于`现场`的资产 | `现场 -> 已预留` | 任务失效或撤离时回到`现场` |
| 过程输入预留 | 已位于兼容输入缓冲的`已回流`资产 | 阶段保持`已回流`，增加唯一任务锁 | 取消未消费任务时解除任务锁 |

两种预留都必须同时锁定所需数量和目的容量，但不能互相冒充。`已预留`只表示采集阶段；房间正在处理的输入不能再改名为第二份`已预留`数量。

#### 5.3.1 采集交互预留束中的资源份额

资源契约为每笔采集资产预留返回稳定句柄，至少包含预留束、资产批次、精确数量、原现场位置、行为尝试、建立步骤、当前状态和可逆截止边界。路径、源交互端、投递端和工蜂活动份额只保存所有者句柄引用，不由资源账本复制。

第2阶段进攻只允许对仍处于`已预留`、尚未形成取货草案且属于最终释放集合的句柄准备`已预留 -> 现场`。准备无副作用；提交必须与房间、世界、单位转移和主动战场建立使用同一进攻事务标识。任一子事务失败时资产保持`已预留`且所有容量仍被原作业占用。已携带与过程输入预留不属于该释放接口。

### 5.4 阶段提交位置

| 变化 | 合法阶段 | 关键条件 |
| --- | --- | --- |
| 外部物质来源或房间生成`现场`资产 | 第11阶段 | 来源扣减、输出容量和资产生成同笔提交 |
| 死亡输出生成`现场`残骸 | 第9阶段 | 唯一死亡事件且模板允许输出 |
| `现场 -> 已预留` | 第7阶段 | 源数量、交互端、路径和投递容量同时预留 |
| `已预留 -> 现场` | 第1、2、7或第9阶段 | 撤离、进攻释放可逆工蜂作业、目标失效或执行者死亡，释放全部关联预留 |
| `已预留 -> 携带中` | 第10阶段 | 执行者存活且取货草案仍合法 |
| `携带中 -> 现场` | 第9阶段 | 携带者死亡，资产落在死亡位置 |
| `携带中 -> 已保障/已回流/终结` | 第10阶段 | 执行者存活且交接、投递或账本端仍合法 |
| `已保障 -> 已回流` | 第3或第10阶段 | 安全交接获得功率或工蜂投递，兼容容量存在 |
| `已回流 -> 处理终结与输出` | 第11阶段 | 过程任务完成、输入和全部输出可原子提交 |
| 腐败、夺回或永久损失 | 第3或第9阶段 | 对应非战场计时或战斗效果达到终结条件 |

第9或第11阶段新生成的资产最早从下一步骤参与第7阶段目标选择。第10阶段刚回流的输入不能补进本步骤已经执行完毕的第3阶段过程。

### 5.5 处理与转化

一个房间处理事务必须同时声明：

- 输入资产批次、数量和过程输入预留。
- 处理进度、功率请求和消费模式。
- 每种物理输出、网络输出和明确永久损失。
- 输出缓冲、账本端或网络端的合法提交位置。
- 输入与输出的物质谱系和参数版本。

第11阶段提交时：

```text
再次校验任务、输入与全部输出位置
-> 输入资产以已处理或明确永久损失终结
-> 形成一个或多个有来源物理输出，或提交网络输出
-> 释放过程输入与容量预留
-> 写入同一上游事件下的全部账目条目
```

任一输出没有合法去向时整笔保持`待提交`或进入声明的来源缓冲。输入不能先消失，任务不能重复消费，已经完成的处理进度不再重复请求运行功率；允许存在明确的待机负载。

### 5.6 物理输出与网络输出

| 输出类型 | 提交结果 | 是否立即成为核心资源 |
| --- | --- | --- |
| 物理输出 | 在房间输出缓冲形成新的`现场`过程资产 | 否 |
| 生物质浆账本投递 | 生物质浆终结并增加生物质可用量 | 是 |
| 基因网络输出 | 样本组织终结并增加基因，同时提交样本标签事件 | 是 |
| 能源网络输出 | 建立或改变功率供给与储能上限贡献 | 否；从下一步骤参与能源状态 |
| 条件网络输出 | 提交发现、样本标签或内容池资格 | 否，不是核心资源 |

腐殖滤囊和腐解消化池形成的生物质浆必须由工蜂或合法应急直连投递到账本接入口。突触解析腔和热化代谢腔不生成虚构的“基因物品”或“电池物品”；它们在任务完成后提交对应网络结果。

### 5.7 外部与内部物质谱系

- 外界物质、敌方残骸和敌方已处理储备携带`外部未同化`谱系。
- 己方单位残骸、玩家结构回收物和已经入账后再次形成的物质携带`内部循环`谱系。
- 外部物质第一次形成生物质时，谱系转为`已同化`并增加外部同化统计。
- 内部循环重新形成生物质时只增加内部回收，不增加外部同化。
- 转化、合并和拆分必须守恒分配谱系数量；同一谱系份额不能被两个输出同时继承。
- 同一敌方对象的内容定义必须区分库存、尸体、结构拆解物和样本组织，不能重复形成完整直接缴获与完整残骸。

内部死亡、回收和再孵化闭环不得产生净生物质。具体回收率属于参数，但任何允许无外部输入无限增长的参数组合必须被L6拒绝。

### 5.8 撤离与安全回流

撤离顺序由L3时间契约控制，本文只负责资产事务：

```text
携带中 -> 已保障
已保障中可被当前兼容容量接纳的部分 -> 已回流
超出容量部分继续保持已保障并进入安全回流缓冲
已预留 -> 现场并释放交互、路径和投递容量
其他现场资产保持原位置
```

- 容量不足不能拒绝撤离，也不能把超额资产变成资源或永久损失。
- 第一版安全回流缓冲中的资产不腐败、不被敌方夺回，但仍会堵塞后续交接。
- 撤离只释放当前战场采集预留，不释放建设、孵化、解析或候选核心资源预留。
- 同一撤离幂等标识不能重复转换资产。

## 6. 第一版资源形成链

### 6.1 外部来源

节点与世界只能提供：

- 具有有限储量、持续流量或周期恢复的外部物质来源，以及对应抽取容量。
- 不形成可搬运物质的环境网络输入，以及对应环境容量。
- 兼容处理链和公开房间修正。
- 敌人、结构、环境对象及其物理战利品。

外部物质来源抽取必须在同一事务中减少或占用来源可抽取量，并生成有来源过程资产。抽取容量不足时停止形成，不产生负储量。节点占领只开放来源与修正，从下一合法步骤参与形成，不能提交资源包。

外部物质来源形成的过程资产位于来源节点或初始虫巢母床的明确`现场`位置。腐殖滤囊只能处理已经由工蜂按五阶段带回其物理输入端的外部原料，不能凭一个远程来源引用直接终结节点储量或把物质瞬移进房间。`外部物质来源端`只登记来源兼容、抽取上限和形成位置；实际物质仍由单位契约完成预留、携带、保障与回流。

不形成可搬运物质的环境输入，例如热化梯度，使用第7.2节`持续网络贡献`的环境容量接口。环境输入不会伪装成过程资产，也不能直接增加当前储能或核心资源。

### 6.2 生物质形成

基础链：

```text
外部原料或残骸
-> 采集、保障与回流
-> 腐殖滤囊或腐解消化池处理
-> 房间输出缓冲中的生物质浆
-> 工蜂或应急直连投递到账本接入口
-> 生物质正式入账
```

敌方已经处理好的储备可以直接形成兼容生物质浆过程资产，从采集开始进入链条；它仍不能在敌人死亡或节点占领时直接增加生物质。

### 6.3 样本与基因形成

```text
样本残骸
-> 保障与回流
-> 腐解消化池分离
-> 普通组织形成生物质浆
-> 样本组织形成物理输出并回流突触解析腔
-> 解析任务完成
-> 样本组织终结，基因与样本标签作为网络结果提交
```

- 消化池不能直接形成基因。
- 解析腔不能直接消费样本残骸。
- 没有解析容量时，样本组织保留在合法输入或来源缓冲，不自动丢失。
- 第一次样本资格、图鉴和候选来源使用各自幂等标识，不能因同一解析事件重复触发。

### 6.4 基因消费

- 首次固定样本解析形成基因时不额外支付候选生成基因。
- 首次候选同化在关键选择暂停边界使用即时交易消费。
- 主动诱导开始时预留生成基因，候选组成功提交时消费，形成前取消时释放。
- 已明确由生成成本支付的候选组，成员选择不能再次隐藏扣除同一成本。
- 设施和特殊来源可以声明成员同化成本，但必须在候选形成时保存并在确认前公开。

候选组、槽位和突变生命周期由`L3_候选与进化契约.md`负责；本文只保证基因交易原子、可追溯且不重复。

## 7. 储能与功率服务

### 7.1 权威能源状态

每个稳定边界至少保存：

| 字段 | 含义 |
| --- | --- |
| 当前储能`E` | 下一完整步骤可用于弥补供给缺口或即时消费的累积量 |
| 储能上限`C` | 所有当前有效储能贡献之和 |
| 功率供给贡献 | 来源房间、虫巢、修正、有效步骤和供给值 |
| 请求负载 | 每个消费者的请求值、负载类别、安全层级和服务模式 |
| 已服务功率 | 本步骤实际分配给每个请求的功率 |
| 净功率 | 总供给减去总请求负载 |
| 实际充放能功率 | 总供给减去总已服务功率 |
| 储能溢散 | 因达到上限或强制容量下降无法保存的明确数量 |
| 恢复保障状态 | 当前是否存在第一能源建设或冷启动保障及其唯一目标 |

供给、负载和储能贡献全局共享，但每项保留虫巢、房间、任务或效果来源。第一版没有跨区域传输距离、传输损耗或本地能源库存。

### 7.2 持续网络贡献

`持续网络贡献`用于表达热化代谢腔这类在合法环境条件下持续提供功率供给或储能上限、但不按普通批次消费自身输出的房间能力。它不是过程任务，也不占用模块的过程任务执行位。

每项贡献至少登记：

- 稳定贡献标识、来源模块、虫巢、内容定义和参数版本。
- 所需环境来源、环境容量请求、适用修正与最早生效步骤。
- 贡献类型：功率供给、储能上限或两者。
- 环境服务模式：可缩放或整笔，以及最低环境服务量。
- 暂停、结构失效、冷启动和来源失效时的退出条件。
- 本步骤获得的环境服务量、实际贡献值和来源明细。

第3阶段先为全部合法贡献分配环境容量，再重建正常功率供给与储能上限：

1. 排除尚未生效、玩家暂停、结构不可运行、冷启动未完成或环境来源失效的贡献。
2. 按环境来源分别汇总请求；整笔请求使用`连续未服务步骤降序 -> 稳定贡献标识`，可缩放请求在满足最低值后按未满足比例分配。
3. 同笔提交环境容量占用与本步骤实际贡献；没有取得最低环境服务的贡献本步为零。
4. 贡献只读取环境容量，不消耗其自己产生的功率，也不产生可搬运物品。
5. 第11阶段新形成或重新启动的贡献最早从下一完整步骤参与分配。

多个房间共享一个环境来源时不能各自读取完整容量。玩家暂停房间会释放下一步骤的环境请求，但不会返还历史供给、储能变化或已经发生的溢散。若后续内容需要持续消耗生物质或过程资产，必须改用离散过程任务形成供给贡献，不能把物质消耗藏进持续网络贡献。

### 7.3 功率请求

每个请求至少声明：

- 稳定请求标识和来源对象。
- `基础负载`或`临时负载`类别。
- 能源安全层级。
- 请求功率、最低服务功率和服务模式。
- 得到服务后的进度或效果接口。
- 连续未服务步骤、最近服务步骤和最早生效步骤。

第一版服务模式只有：

| 模式 | 服务规则 | 进度规则 |
| --- | --- | --- |
| 可缩放负载 | 达到最低服务功率后可以获得部分功率 | 按`已服务 / 请求`比例推进 |
| 整笔负载 | 必须获得完整请求功率 | 满足时完整推进本步骤，否则为零 |

最低服务功率、请求功率和基础进度是参数；模式和安全层级是机制或内容身份，不能在运行时因容器顺序变化。

请求功率为零的过程不进入能源分配，也不使用`已服务 / 请求`除法；它只按自身输入、时间与容量条件推进。

### 7.4 第3阶段求值顺序

每个完整步骤第3阶段统一执行：

1. 读取第1阶段已经提交的暂停、取消、即时储能消费和任务变化。
2. 由`L3_虫巢与房间契约.md`按阶段开始快照接纳合法自动任务，原子建立输入、目标份额和输出容量预留；每个空闲模块本步骤至多接纳一个任务。
3. 按第7.2节为持续网络贡献分配环境容量，并从实际贡献、修正和既有有效来源重建本步骤供给与储能上限。
4. 从任务与基础消费者重建请求负载，移除尚未到最早生效步骤、已暂停、已完成待提交或没有合法输入的运行请求。
5. 计算普通能源预算，并按第7.6节安全层级分配已服务功率。
6. 对仍满足第8节条件的唯一能源恢复目标判定是否使用恢复保障功率。
7. 原子提交环境服务、供给贡献、各请求已服务功率、储能变化、溢散和未服务事实。
8. 后续房间与世界过程只按本步骤已服务结果推进。

不能先推进房间再补算能源，也不能让某个房间本步骤新形成的供给反向服务同一步较早任务。

### 7.5 储能公式

设固定步长为`Δt`，第3阶段开始时当前储能为`E0`，储能上限为`C`，正常总供给为`S`：

```text
普通可服务功率预算 B = S + E0 / Δt
总已服务功率 P = 所有普通请求的已服务功率之和
约束：0 <= P <= B

步骤末原始储能 Eraw = E0 + (S - P) * Δt
步骤末当前储能 E1 = clamp(Eraw, 0, C)
储能溢散 = max(0, Eraw - C)

净功率 = S - 总请求负载
实际充放能功率 = S - P
```

- 第一版不设置独立充电或放电功率上限；储能在一个步骤内最多提供`E0 / Δt`的等效功率。
- 恢复保障功率不计入`S`、`P`或`E1`，只推进其唯一合法恢复任务。
- 供给不足且储能耗尽时，未服务请求为零进度，不形成负储能或能源债务。
- 达到储能上限后的多余供给形成溢散统计，不转化为生物质、基因或飞升资源。

### 7.6 固定能源安全层级

第一版从高到低固定为：

| 层级 | 服务对象 |
| --- | --- |
| 0 受保护最低闭环 | 核心维持、第一只工蜂恢复任务及已激活的恢复控制过程 |
| 1 已承诺过程 | 已保障资产交接、已经形成且需要整笔提交的安全过程 |
| 2 基础生产 | 外部物质来源抽取、生物质形成、消化和基础孵化 |
| 3 研究构筑 | 样本解析、候选设施和其他构筑过程 |
| 4 非必要强度 | 加速、过载和其他高耗能可选效果 |

- 玩家不能逐房间设置配电优先级，只能暂停房间群、循环、模式或任务来改变请求集合。
- 基础撤离不请求功率，不因能源不足延迟或失败。
- 科技、插件和领袖可以改变请求值、供给、容量或提供明确保留功率，但第一版内容不能新建隐藏优先级。
- 某个配方的安全层级必须在内容定义中公开登记，不能根据玩家最近失败或当前页面动态改变。

### 7.7 同层分配

当预算不足以满足同一层全部请求时：

1. 整笔负载按`连续未服务步骤降序 -> 最近服务步骤升序 -> 稳定请求标识`扫描。
2. 能完整容纳的整笔请求获得全额服务；不能容纳的请求保持未服务，并继续扫描后续请求。
3. 可缩放请求按相同等待顺序建立活动集合：最低服务功率为零的请求全部进入；最低值大于零的请求只有在剩余预算能够完整覆盖该最低值时才进入。
4. 先为活动集合提交各自最低服务功率，再把剩余预算按各请求尚未满足的功率比例分配；达到请求上限的部分退出后继续重算，直到预算用尽或全部满足。
5. 未进入活动集合的请求保持零服务并继续累积未服务步骤；本层无法使用的剩余供给可以为储能充电，不能保存的部分形成溢散。

连续未服务步骤只在同一请求持续存在且合法时累积；玩家暂停、任务取消或请求身份变化会停止累积。该字段用于消除隐式对象顺序，不保证物理上永远不足的请求能够运行。

### 7.8 功率与任务进度

- 可缩放任务的本步进度乘以服务比例，不能按请求功率假装满速。
- 整笔任务没有达到完整服务时不推进，也不消费本步运行输入。
- 缺能不会取消任务、释放核心资源预留或清除已有进度。
- 已完成并等待第11阶段提交的任务不再请求运行功率；如内容声明，可以保留明确待机负载。
- 取消任务只移除后续请求，已经结算的储能变化和功率服务不返还。
- 时间倍率、帧率和离线事件跳跃不能改变相同模拟时长的总服务量与任务结果。

### 7.9 供给与储能贡献生效

- 第11阶段新完成的能源房间、供给输出和储能贡献从下一完整步骤第3阶段生效。
- 第1阶段暂停一个已运行能源房间时，其正常供给不参与本步骤第3阶段；恢复时必须满足正常启动或冷启动条件。
- 暂停房间不移除其物理储能上限贡献，只有结构失效、拆除、降级或明确修正终止才改变容量。
- 节点控制只开放环境输入、抽取容量或修正，从下一合法步骤影响能源房间；不直接增加供给或当前储能。
- 自愿容量下降遵循第4.4节堵塞规则；强制下降原子截断超额并记录储能溢散。

### 7.10 即时储能消费

后续内容可以声明一次性储能消费，但必须：

- 在第1阶段边界事务中检查并原子扣除当前储能。
- 与效果提交使用同一事件，失败时整笔不发生。
- 不能再为同一效果登记重复持续功率成本。
- 在本步骤第3阶段之前完成，因此会减少本步骤可服务预算。
- 第一版基础进攻、撤离和页面操作都不是即时储能消费。

## 8. 恢复保障契约

### 8.1 共同限制

恢复保障只解决循环依赖，不免费补发资源或倒转损失：

- 每种保障都有唯一目标、唯一任务和明确退出条件。
- 保障功率不进入普通供给、不为储能充电、不转给其他任务。
- 使用保障仍需要合法结构资源、外部输入、容量、模拟时间和配方成本。
- 同一恢复任务不能同时取得普通功率和保障功率。
- 保障过程、目标和累计服务必须保存，载入不能重置或重复形成输出。

### 8.2 第一能源房间建设保障

只有同时满足以下条件才可启用：

1. 全局没有可运行能源房间。
2. 不存在更优先的已完成能源房间冷启动目标。
3. 本步骤正常供给与可用储能无法让任一合法能源恢复任务达到最低服务功率。
4. 至少一个能源房间建设任务已合法预留结构资源与槽位。
5. 任务没有暂停、取消、输出容量问题或非法引用。

多个任务合法时，按命令接收序号和稳定任务标识选择一个。保障只推进该任务；取消后可以重新选择，但已经经过的保障时间不产生储能或可转移功率。房间完成后保障立即关闭，新供给仍从下一步骤生效。

### 8.3 能源房间冷启动

当当前储能为零、正常供给不能启动任何能源房间且存在合法环境输入时：

- 从已恢复运行且具备冷启动能力的完整能源房间中选择一个稳定目标。
- 多个候选按连续合法等待步骤降序、恢复命令接收序号和稳定模块标识选择；玩家可以通过暂停状态改变候选集合。
- 使用专用保障推进其冷启动过程，不向全局供给或储能写入保障值。
- 冷启动完成后只建立下一步骤可用的正常供给贡献。
- 已有普通功率足以启动时不启用保障；玩家暂停的房间不会被系统暗中启动。

### 8.4 第一只工蜂应急直连

只有全局不存在存活采集单位，也不存在已经完成待提交的采质工蜂时，核心巢室才允许建立一个应急任务：

- 只转移形成第一只采质工蜂所需的最小合法物理输出或账本输入。
- 不能绕过外部来源、腐殖滤囊、孵化室、结构资格、真实生物质成本、功率和时间。
- 使用层级0普通功率；它没有独立免费供能通道。
- 不能形成超出第一只工蜂需求的可用库存、过程资产或第二个并行恢复任务。
- 第一只工蜂形成待提交结果或进入存活单位账目后立即停止，后续物流恢复正常工蜂规则。

## 12. 拒绝、取消、暂停与堵塞

| 结果 | 使用条件 | 资源与能源结果 |
| --- | --- | --- |
| 命令拒绝 | 可用资源、资格、槽位、目标或安全去向不合法 | 不建立预留，不推进随机或功率 |
| 自动任务未建立 | 输入、容量或资格不足 | 队列保留，派生缺料或堵塞 |
| 任务暂停 | 玩家暂停或前置暂时失效 | 保留进度与未消费预留，不请求运行功率 |
| 缺能 | 请求未获得所需服务 | 本步零进度或比例进度，不释放输入 |
| 输出堵塞 | 完成结果没有合法输出位置 | 保留待提交结果，不重复消费或运行 |
| 资产堵塞 | 已保障/已回流资产没有兼容容量 | 保持当前阶段和数量，不静默损失 |
| 任务取消 | 契约允许且结果尚未不可逆提交 | 释放未消费预留，历史功率不返还 |
| 实现异常 | 负数量、重复预留、谱系复制、储能越界或守恒失败 | 整步不发布，保留诊断与上一稳定快照 |

拒绝与堵塞只显示当前事实，例如“可用生物质不足”“普通槽已预留”“已服务功率低于最低值”“输出缓冲无容量”。系统不据此推荐玩家停哪座房间或选择哪种构筑。

## 13. 保存、离线与确定性

### 13.1 必须保存

- 生物质与基因可用量、全部预留记录和累计账期事实。
- 每个模块的结构投入、形成来源、可回收部分和重组任务。
- 全部过程资产批次、谱系、阶段、位置、预留、风险进度和累计终结。
- 当前储能、储能上限贡献、供给贡献、请求、已服务结果和同层服务等待事实。
- 持续网络贡献的来源、环境容量请求、实际服务量、启动或冷启动事实、最早生效步骤和退出条件。
- 恢复保障目标、进度、累计服务、退出条件和幂等标识。
- 外部物质来源已抽取量、剩余量或流量进度，环境容量服务明细，以及全部账目事件序号。
- 轮次初始注入标识、规则版本、参数版本和数量精度版本。

### 13.2 离线等价

无主动战场时，离线安全结算必须与没有新玩家命令的在线完整步骤等价：

- 使用相同功率供给、储能积分、安全层级和任务服务模式。
- 继续受输入、工蜂、容量、堵塞、腐败、来源流量和输出投递限制。
- 不自动暂停房间、改变循环、拆除结构、选择候选或使用即时储能能力。
- 恢复保障只在在线状态同样满足条件时推进，不因离线获得额外产率。
- 有主动战场时整个世界冻结，资源、能源、腐败和后台过程都不推进。

事件跳跃必须依次处理储能耗尽/充满、来源耗尽、任务完成、缓冲满、资产腐败、保障退出和容量变化等条件边界。

### 13.3 确定性

- 同一快照、命令、参数和模拟时长必须产生相同预留胜者、功率服务、资产谱系、账目和期末状态。
- 批次合并、场景树、线程、帧率、页面和表现实体不能改变结果。
- 载入不能重新执行初始注入、正式入账、结构返还、样本解析或恢复输出。
- 离线摘要只汇总已经提交的账目事件，不额外计算奖励。

- 生物质与基因的可用量、预留量、本期入账、返还、消费和来源。

## 15. 第一版约束与排除

- 核心资源只有生物质、能源和基因，不增加第四种顶部货币。
- 生物质与基因全局共享；能源全局共享且保留来源明细。
- 幼体是单位，不是资源或过程资产。
- 节点不直接形成核心资源，节点占领不发资源包。
- 生物质物理输出必须投递，基因和能源使用合法网络输出。
- 第一版不提供手动领取、能源债务、逐房间配电、跨区传输损耗或本地核心资源仓库。
- 第一版储能没有独立充放电功率上限。
- 第一版已占节点安全，不因领地活动产生隐藏战损或无来源资源。
- 第一版布局编辑不损失可回收结构投入；真实时间和已服务功率仍然结算。
- 第一版正式飞升不可执行，只读取同一步资源与统计快照。

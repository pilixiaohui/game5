# 《异种起源：无尽洪流》玩法设计文档目录

本目录是后续玩法、界面、数值、内容、实现与验收设计的主要来源，代替单一白皮书。设计方向、工作顺序、证据要求和文档权威关系以`00_设计宪章与方法论.md`为最高层基线；产品简报、目标体验、第一版范围和统一核心因果架构以两份L1为权威来源；共享对象、所有权、状态、不变量和规范术语以两份L2文档为权威来源。七份L3维护系统契约，L4维护第一版具体内容，L5维护首小时进程、页面和反馈，L6维护参数与证据；T文档维护Godot实现和桌面运行基线；`验收游玩测评/`维护真人场次、事件采集、PXI、缺陷与终验标准；旧编号文档只保留迁移素材、场景和历史设计理由。

当前阶段遵循：

- 先确认机制关系与系统边界，再进入敌人、单位和关卡内容量产。
- 解锁围绕实际游玩循环建立可选支路，不用唯一任务链规定玩法。
- 系统显示真实状态与结果，不生成标准策略答案。
- 成本、产量、容量、时长、概率和倍率必须经过模拟与实际游玩测试，未测试值不作为开发定案。
- 新增内容优先复用通用房间、单位、节点、候选、账目和时间规则。

## 当前已确认方向

### 核心循环与资源

- 游戏由虫巢生产、战场验证、基因构筑和飞升四种互相咬合的设计压力组成，不是四个独立菜单玩法。
- 核心资源为生物质、能源和基因。生物质承载规模和吞噬目标，能源承载储能与运行强度，基因承载进化路线。
- 外源物质、残骸和样本统一处于现场、已预留、携带中、已保障或已回流，必须由对应房间形成最终资源。
- 节点占领后的经济效果只提供外部物质来源、抽取容量、环境网络输入、环境容量或房间修正，不能发放资源包或直接增加生物质、能源或基因；路径、着床和区域解锁只改变世界状态。
- 样本先由消化池分离普通组织与样本组织，再由基因解析腔产出基因和样本标签。
- 己方尸体回收属于内部循环，不重复增加本轮外部同化量，也不能形成净生物质套利。

### 解锁与构筑

- 解锁读取真实生产、战场、样本、构筑、扩张与飞升里程碑。
- 重要内容采用必要安全条件加任一可选支路满足，不固定唯一建设顺序、节点顺序或兵种答案。
- 房间蓝图解锁、单位可孵化、科技族可用、候选生成和突变获得是不同状态。
- 突变候选至少来自基因主动诱导、设施周期生成和关卡特殊战利品。
- 第一版在首次样本组之后实际开放主动诱导、诱变培养腔和普通候选槽，不只展示预览。
- 一次生成机会形成一个全局候选组，整个组占一个持有槽；选择一个突变后，同组其他候选消失。
- 科技采用肉鸽候选，不使用可以线性点满的固定长树。
- 候选不会根据最近一次失败自动保证出现克制答案。
- 高强度战斗兴奋、短时孵化加速、过载和菌毯脱根准备由科技、突变、房间或领袖解锁，并通过房间模式、战前策略、持续效果或自动触发生效；第一版不增加战场技能按钮，基础撤离始终是瞬时原子结算。

### 虫巢与房间

- 所有最终资源形成归属具体房间；核心巢室只为零能源房间状态下的一座能源房间建设提供最低保障功率。
- 空槽没有预设功能；房间蓝图解锁后可在任意兼容空槽建设。
- 相邻且兼容的同类房间自动打通，作为一个大房间显示和管理，同时保留模块账目。
- 建设和升级可以并行，并共同形成真实功率负载。
- 取消建设全额返还任务预留资源；已经经过的模拟时间和功率账目不倒转。
- 房间可以移动、拆除、替换与降级。移动和兼容重组保留结构投入、队列、进度、等级和插件；不兼容事实必须安全迁移、完成或合法取消，不设置隐藏损耗。
- 完整结构投入可以全额回收，但房间变形、替换和重新建设承担真实模拟时间与功率。
- 本轮发现和蓝图不会因拆除房间而遗忘；高级能力仍检查当前实际生态条件。
- 房间队列支持自动循环，常规产出不需要手动领取。
- 物理产出和节点外源物质经过五阶段采集状态，再由对应房间形成或处理；缺少单位、端口或容量时形成可观察堵塞。
- 玩家可以暂停部分房间群、停止循环、升级或重组布局，以改变缺料、缺能和堵塞状态。

### 单位与战场

- 幼体、工蜂、战斗个体、菌毯形态、敌方生物、阻断结构和可破坏出兵点使用同一单位模板与批次底层；房间、过程资产、节点和地形不属于单位。
- 每种单位至多拥有一个核心功能行为；移动、受伤、死亡、被动范围和撤离响应是共同规则。
- 单位行为统一产生伤害、搬运资产、自身变形、生成单位、施加状态或生成过程资产六类效果，不为单个物种新增专用状态机。
- 单位无目标时只读取待命或阵营推进锚点；目标失效、行为进度、效果提交、变形继承和死亡输出都使用统一生命周期。
- 工蜂唯一核心行为是采集，环境资源、房间产物和尸体残骸使用统一可采集交互。
- 玩家通过建设、分化、暂停和重组单位生产房间决定虫群构成；关卡中只下达进攻和撤离等战役级命令。
- 新生单位离巢后归属全局单位体系：没有主动战场时进入虫群大军；战斗期间非工蜂单位自动补入当前主动节点左侧后备，新生工蜂先满足虫巢与领地合法采集需求，只有富余部分参战。进攻时全部可参战移动单位进入目标节点。
- 混合战群由不同单位的目标、空间、范围效果和行为自然涌现，不使用隐藏编队协同倍率。
- 战斗行为锁定由目标组、作用空间和兼容状态队列组成的稳定目标池；批次合并或拆分不能改变命中和伤亡。
- 根脉孢体通过扎根等量变形为固定根脉菌毯；菌毯以普通伤害行为参与主动和后台战斗，同时提供明确的被动范围效果。
- 节点由连续长横向场景表示，我方从左侧进入，敌方从右侧进入。
- 中途增援必须来自可观察、可攻击的出兵单位；固定结构形态同样使用单位底层。
- 清除敌军、全部持续出兵源和节点标签要求的可见目标后占领节点，不使用隐藏控制阈值。
- 区域出口与普通节点共用同一底层；每个区域只有一个节点带有`区域解锁`标签，Boss、精英、资源、特殊战利品和虫巢部署由标签组合形成。
- 节点允许多次进入。撤离后保存单位、出兵源、菌毯、残骸和场景战线状态。
- 未占节点的留置菌毯与敌方恢复执行完整后台战斗；我方后台死亡不生成残骸或资源收益。
- 已占节点中工蜂自动采集，战斗单位随机游荡；领地活动仍属于虫群大军，不建立驻军或运输。

### 数量、后备与占领

- 敌我真实数量、战群批次和画面代表三层分离；画面实体数量不参与战斗、采集或回收结算。
- 单位真实位置统一为虫巢内、虫群大军、节点内或死亡；当前战场、节点留置和领地活动均由位置、节点和移动形态派生，不建立第二份库存。
- 宏观兵力差通过接触面占用、补位、缺口、战线突破和后备暴露影响结果，不直接生成隐藏伤亡。
- 单位真实位置、空间位置、核心行为、领地活动和待处理命令分层记录，交叉状态不作为独立数量桶重复相加。
- 常规攻击只伤害可达接触目标；穿透、范围、炮击等纵深伤害必须有明确来源和覆盖范围。
- 撤离确认后不可取消；命令在下一完整步骤开始前原子提交且不再形成攻击，存活可机动单位回归，固定菌毯留置，携带中资产进入安全保障缓冲并按兼容容量继续回流。
- 节点占领时移动单位同样立即回归，已占节点不保留需要召回的移动单位。
- 首版只允许一个主动战场；已占节点保持安全，不得通过后台规则暗中扣减或占用全局虫群大军。

### 条件随机世界与多虫巢

- 首轮使用受强约束随机骨架，关键机制必定存在；新存档提交起始地图时一次生成全部节点敌人、空间段、资源源和修正，情报只控制显示。
- 世界随推进深度逐层生成，具有清晰主方向和区域级支路，理论上可以持续扩张。
- 每个区域只有一个出口节点，区域内部使用大量互相连接的开放网状节点。
- 占领区域出口即可开放后续区域，不要求清理全部支路。
- 新区域通过已打开出口的世界连接变为可达；玩家用全局虫群大军清除虫巢节点后安全建设本地虫巢。
- 地图先选择生态原型，再从兼容内容池随机组合，不读取玩家当前构筑生成答案。
- 所有旧虫巢保持完整模拟，并提供玩家生产方案批量应用和全局异常汇总。
- 所有虫巢生产的新生单位共同归属全局单位体系，并进入虫群大军或当前主动节点，统一用于后续合法关卡。
- 节点占领后的经济效果只提供外部物质来源、抽取容量、环境网络输入、环境容量或房间修正，最终资源与能源贡献由房间形成；路径、着床和区域解锁只改变世界状态，完全吞噬区域只计入统计与飞升成果。

### 飞升、恢复与数值

- 飞升按六类真实贡献和当前参数版本显示准确点数，并预览六项具体永久收益与三位领袖数值差异；首小时不执行正式飞升。
- 飞升后重新生成世界地图，重置区域、节点、虫巢和本轮资产；保留图鉴、领袖、永久收益和明确永久化内容。
- 领袖提供特性、专属科技池与明确弱项，但不能锁住所有玩家需要的通用能力。
- 房间布局、队列和未完成建设属于可逆试错；单位战损、已结算能源、完成的候选诱导和未保障战利品不会免费返还。
- 工蜂全灭、储能归零、生产堵塞和构筑偏差都必须存在不依赖稀有随机内容的恢复路径。
- 第一只工蜂从真实数量`1`开始，首小时末以百万级虫群作为体验目标；具体曲线、阶段时点和验收范围需由平衡测试确定。
- 首小时只展示第二地区虫巢节点、进攻入口和建设条件，不实际建立第二虫巢。
- 除明确体验量级外，当前文档中的成本、产量、容量、秒数、概率和倍率都不视为已定数值。

### 第一版封闭范围

- 第一版是首小时可玩切片，只实际推进腐殖盆地和一个初始虫巢。
- 正式飞升、第二虫巢、完整插件和无限世界只做预览或保留扩展边界；全局虫群大军是长期统一军队规则。
- 区域状态拆为情报、着床、节点控制汇总、出口通路和活动状态；争夺中与通路已打开可以并存。
- 占领出口只固定相邻区域概要和虫巢节点入口，首次确认进攻该节点时才提交完整节点网；第一版不会生成第二地区节点网。
- 第一版六个实际房间、敌我单位、节点职责和六项候选池已经确定，具体数值继续后测。
- 工蜂使用固定服务层级、等待老化和原子预留；战斗使用唯一十二阶段顺序、稳定命中槽分配和同时伤害，第一版不使用随机命中。
- 幼体使用虫巢内单位守恒和两阶段分化；撤离按移动形态划分立即回归与固定留置，并把超出处理容量的资产保存在安全保障缓冲。
- 首次候选固定来自共振样本解析，生成不额外扣基因，同化确认时才支付。
- 共振孵育床能在节点占领前后恢复丢失样本，但恢复载体不产生重复资源、候选或飞升成果。
- 首轮地图使用职责锚点和约束随机，C、P、X三个战场职责不能合并。

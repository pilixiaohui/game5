## 1. 平衡工作的边界

### 1.1 平衡不是猜中心值

任何成本、产量、容量、时长、概率、权重、耐久、伤害、射程、倍率和表现配额，在取得证据前都不能因出现在文档、表格、场景或代码中成为正式设计。当前允许存在机制夹具，但必须明确它只用于让规则可执行。

平衡工作依次回答：

1. 参数是否具有合法身份和明确单位。
2. 改变它是否产生可预测的单调影响。
3. 参数组合是否守恒、可恢复且不存在非法状态。
4. 候选范围是否排除软锁、爆炸增长和事实上的唯一路线。
5. 真人是否能观察、理解并利用产生的差异。
6. 冻结发布候选能否在同一构建、内容和协议下复现。

### 1.2 参数不能改变的内容

下列变化必须回到L1至L4，不能伪装成参数调整：

- 新增或删除核心资源、过程资产阶段、单位位置或世界状态。
- 改变十二阶段、原子撤离、保存、地图提交或候选单选边界。
- 为单位增加第二个核心行为，或增加第七类通用效果。
- 让节点直接发放最终资源，或让表现实体参与玩法结算。
- 开放第二虫巢、多主动战场、正式飞升或其他第一版排除内容。
- 根据玩家落后程度暗中补发单位、资源、候选或克制答案。

### 1.3 体验目标不是硬编码纠偏线

第一只工蜂以真实数量`1`建立认知，首小时末约百万级是真人体验研究目标，不是运行时控制器。系统不得检测玩家低于目标后补发，也不得因高于目标而隐藏削减。若测试分布偏离目标，应调整可追溯生产、吞吐、关卡和构筑参数后重新测试。

## 2. 参数所有权与唯一来源

| 事实 | 唯一所有者 | 其他层的权限 |
| --- | --- | --- |
| 参数是否存在、作用于哪个接口 | L4内容定义与对应L3契约 | L6审计完整性，不能改变接口 |
| 参数值、候选范围、状态和证据 | 本文及机器参数登记 | L3/L4只引用ID，不复制值 |
| 机制测试夹具 | L6无表现测试 | 不能作为发布值 |
| 性能预算 | `L6_百万级状态队列与性能预算.md` | 平衡扫描不能突破语义边界 |
| 体验通过线 | `L6_体验假设验证计划.md` | 参数结果必须服从预注册协议 |

第一版机器参数唯一入口为`data/balance/v1_parameter_registry.csv.txt`。Godot内容、资源、场景和脚本不得维护第二份发布值；测试可以在内存中覆盖候选值，但必须保存基准参数版本、覆盖集和随机种子。

旧无表现模型仍读取的撤离准备和返程参数属于`legacy_probe`依赖，不属于L3-1.2参数身份。`battle.retreat_prepare_seconds`和`battle.return_seconds`不得进入新模型、候选扫描或发布配置；它们已迁入`data/balance/legacy_probe_parameter_registry.csv.txt`，该文件不能作为发布参数入口。

## 3. 参数登记结构

### 3.1 必需字段

参数CSV固定使用以下列：

| 列 | 规则 |
| --- | --- |
| `id` | 稳定、全局唯一；名称表达领域、对象和作用量 |
| `domain` | 资源、能源、房间、单位、战斗、地图、候选、飞升、表现等责任域 |
| `value_type` | `int / float / bool / string`之一 |
| `unit` | 明确量纲；倍率、比例、步骤、秒、数量、功率和资源不能混用 |
| `fixture_value` | 机制夹具值；必须可解析，但不承担好玩或发布含义 |
| `release_value` | 仅`release_verified`状态允许存在 |
| `candidate_min` | 已批准实验包允许的候选下界；空表示尚未建立范围 |
| `candidate_max` | 已批准实验包允许的候选上界；空表示尚未建立范围 |
| `status` | 使用第3.2节状态 |
| `evidence_version` | 支撑当前状态的证据包标识；没有证据写`none` |
| `description` | 作用对象、方向和不允许改变的语义 |

CSV使用结构化解析器。字段缺失、重复ID、未知单位、错误类型、上下界倒置、发布值无证据或L4引用不存在时，发布配置必须失败。

### 3.2 参数状态

| 状态 | 含义 | 允许用途 |
| --- | --- | --- |
| `fixture_only` | 只有机制夹具值，没有候选范围 | 无表现接口和守恒测试 |
| `range_pending` | 参数身份成立，候选范围尚待建立 | 不能进入批量平衡结论 |
| `design_target` | 体验量级或方向性目标，不是运行值 | 研究问题与报告 |
| `scan_ready` | 候选范围、约束和指标已预注册 | 批量模拟 |
| `simulation_candidate` | 通过形式和批量模拟 | 形成性试玩 |
| `playtest_candidate` | 通过形成性试玩并冻结候选包 | 发布验证 |
| `validated` | 同一冻结版本达到发布门槛 | `release_value`读取 |
| `retired` | 机制已删除或参数身份被替代 | 只能存于历史清单，不能加载 |

现有加载器在支持完整状态集前，可以把`scan_ready`之后的状态保留在证据登记中，并只向运行配置暴露`fixture_only / range_pending / design_target / validated`。这种兼容不能把未验证值标成`validated`。

### 3.3 参数身份审计

每次内容或契约版本变化后必须执行：

1. 从全部第一版L4定义收集`parameter_refs`。
2. 验证每个引用在参数表中恰好存在一次。
3. 验证参数表中的活跃ID至少被一项内容、机制或发布预算引用。
4. 验证脚本、场景和资源不存在未登记玩法常量。
5. 验证已废弃机制参数没有进入新模型必需集合。
6. 生成新增、删除、改名和语义变化报告；参数改名必须有存档迁移策略。

## 4. 数值分层

第一版分别控制四层，不能用一个全局增长倍率统一驱动：

| 层 | 对象 | 主要研究问题 |
| --- | --- | --- |
| 核心资源 | 生物质、储能、供给、负载、基因 | 建设、运行和路线取舍是否可读 |
| 过程资产 | 外源物质、残骸、样本、候选组、缓冲 | 处理链和容量瓶颈是否真实 |
| 单位规模 | 幼体、工蜂、战斗单位、菌毯、敌军、结构 | 从单体到宏观规模是否守恒且可观察 |
| 空间与时间 | 槽位、端口、场景段、接触、补位、任务周期 | 玩家是否有观察和改变状态的窗口 |

跨层影响只能通过L3已声明接口发生。增加孵化批次不能自动增加生物质来源，扩大功率不能直接生成单位，增加后备不能提高单体伤害，降低代表实体配额不能改变真实战斗。

## 5. 首小时规模阶段

规模体验按可观察状态而非固定分钟划分：

```text
单体可辨认
-> 小群体可直接观察
-> 自动批次生产形成
-> 真实数量与画面代表分离
-> 多来源持续补充与后备形成
-> 百万级宏观虫群
```

每次跃迁至少暴露一个新瓶颈：物质、功率、幼体、采集、端口、缓冲、空间、出兵源、候选代价或战损。瓶颈不能只允许一个预设解法；测试必须观察房间重组、节点支路、单位构成、候选和继续积累能否形成不同可行路径。

记录阶段时使用事件和账目，不使用“第几分钟强制进入下一阶段”。阶段时间分布是结果指标，不能成为解锁条件。

## 6. 实验包

参数不按单个孤立滑杆调整。每次实验使用版本化实验包，声明固定项、变量、约束、种子集、策略样本、输出指标和停止条件。

| 包 | 先验证什么 | 核心输出 |
| --- | --- | --- |
| B-01 守恒与最低恢复 | 资源、单位、结构、资产和候选不复制 | 守恒差、恢复可达、套利路径 |
| B-02 初始资源与冷启动 | 首工蜂、首能源房间和错误布局仍可恢复 | 最低闭环步骤、停滞率、恢复耗时分布 |
| B-03 房间与物流吞吐 | 端口、缓冲、工蜂、路径和自动循环 | 堵塞占比、利用率、等待老化、空转 |
| B-04 储能与功率 | 并行建设、运行、暂停和冷启动 | 净功率、储能轨迹、未服务请求、停机选择 |
| B-05 幼体与单位增长 | 两阶段孵化、目标规模和批次选择 | 形成率、构成变化、量级跃迁、超产 |
| B-06 战场空间与伤亡 | 接触、纵深、后备、出兵源和范围 | 推进、补位、伤亡来源、结构持续时间 |
| B-07 撤离与持续节点 | 原子边界、留置、资产阶段和恢复 | 回归恒等式、留置战局、再次进入状态 |
| B-08 回收与样本 | 五阶段、腐败、分离、解析和恢复载体 | 外部同化、内部回收、样本损失和堵塞 |
| B-09 候选与构筑 | 三类来源、槽位、支付和六项候选 | 选择分布、可支付性、差异幅度、支配率 |
| B-10 地图种子 | 拓扑、职责、来源和敌方组合 | 完成率、路径多样性、软锁和收益分布 |
| B-11 首小时综合 | 全链、两类玩家和继续游玩 | FH触达、规模阶段、停滞、恢复和完成分布 |
| B-12 表现与性能 | 代表配额、状态队列和参考机器 | 可读性、帧时间、内存和结算等价 |

前一包存在形式失败时，不得用后一包的平均体验掩盖。B-11只有B-01至B-10达到模拟候选后才能形成发布候选，B-12必须与同一候选参数包共同冻结。

## 7. 各领域最低实验设计

### 7.1 生物质与外部同化

每个候选包必须分别记录：初始可用量、外部同化入账、内部回收、结构回收、建设消费、孵化消费和转化损失。己方死亡、回收和再孵化不得形成净生物质。

至少覆盖稳定虫巢来源、战场外源物质、敌我残骸、已占节点来源和结构事务。比较不同来源组合时保持同一账期和同一内容种子。

### 7.2 初始储能与功率

初始续航不先规定固定分钟，而验证一组行为是否可完成：查看空槽、启动并行建设、观察一次生产与入账、识别净功率变化，并通过建设供给或降低负载形成可持续状态。

必须覆盖零储能冷启动、首能源房间保障、多任务争用、整笔与可缩放服务、暂停恢复和取消未来负载。核心保障不能成为正常供给。

### 7.3 房间、工蜂与批次

孵化和处理批次分别扫描输入、幼体、功率、周期、输出容量和目标规模。解锁大批次后必须保留小批次，避免资源不足时只能等待。

工蜂实验同时改变真实工蜂数、携带、服务周期、端口、路径和目标老化，检测局部最优是否导致永久饥饿。不能把画面工蜂数作为变量。

### 7.4 战场容量与宏观兵力差

首小时至少包含三类夹具：小规模开阔接触、受限接触与出兵源、长场景综合推进。逐步改变敌我真实兵力时，应主要改变补位持续性、空间占用和后备暴露；不得通过隐藏倍率直接清空后备。

范围、穿透和纵深攻击分别测试声明覆盖。普通接触伤害的过量伤害不得转移到后备。

### 7.5 原子撤离

撤离没有准备时间、成功率、途中受击和返程时间，因此不扫描这些量。只验证不同数量级下的快照可读性、命令唯一提交、回归/留置恒等式、五阶段资产去向和误确认率。

### 7.6 候选交易

首次解析预算始终满足范围文档的防软锁不等式。普通候选实验同时观察主动诱导、诱变培养腔和特殊战利品来源；不要求三者平均，也不根据当前弱项发放答案。

候选效果比较必须保留明确收益与压力，检测选择率、后续行为变化和结果差异。高选择率本身不等于支配，只有同时压缩可行路线并缺少真实代价时才判为问题。

## 8. 策略样本与随机控制

自动模拟不假装完整替代玩家。每个综合实验至少包含：

- 多种合法房间建设、暂停与重组顺序。
- 多种单位生产构成，不使用固定标准比例。
- 多种节点支路与撤离时机。
- 六项候选的不同合法组合及延后选择。
- 多个受约束地图种子和长战场组合。
- 保守、扩张、高损耗回收和低功率等行为倾向。
- 极端资源耗尽、全灭、满缓冲和满候选槽故障夹具。

随机流、种子集和策略样本必须版本化。发现异常后可以增加反例，但不能删除失败种子来提高通过率。地图和候选随机不得读取模拟策略生成克制答案。

## 9. 新玩家与熟悉玩家

两类玩家使用完全相同的初始状态、地图、内容池、参数、时间和账目。熟悉玩家只可通过理解、并行建设、较少停留和统一时间倍率缩短现实时间。

记录：

- FH-001至FH-014触达与先后关系。
- 每个规模阶段的模拟时间与现实时间。
- 停滞、恢复、撤离、重组和候选选择。
- 资源、功率、单位、战损和回流曲线。
- 页面停留、误操作和策略提示依赖。

固定等待造成熟悉玩家无法压缩，或隐藏新手增益让两组账目不同，都判为机制或节奏失败。

## 10. 参数晋升流程

```text
参数身份完整
-> 机制夹具通过
-> 候选范围预注册
-> 批量模拟排除非法区
-> 形成性试玩调整
-> 冻结发布候选包
-> 发布样本与性能验证
-> release_verified
```

### 10.1 从夹具到扫描范围

必须先证明参数改变方向符合定义，候选范围不会越过机制不变量，并记录关联参数。范围不能只围绕一个作者偏好的中心值窄幅采样。

### 10.2 从模拟候选到试玩候选

必须通过全部形式验收、极端恢复、地图种子和防套利检查，并且没有单一策略在所有合法种子上同时支配推进、损耗、回流和恢复。

### 10.3 发布值

只有同一冻结构建、规则、内容、生成、UX和协议下完成发布试玩与性能验证，才能写入`release_value`并标记`validated`。发布值必须随证据包记录：

- 参数表哈希和内容版本。
- 构建标识、Godot版本和参考机器。
- 模拟种子、策略集、失败清单和汇总制品。
- 新玩家与熟悉玩家有效样本。
- E-01至E-06、L6Q和L6P判定。
- 审核人、时间和失效条件。

## 11. 证据失效

以下变化会使关联参数证据失效：

- L1范围、L2对象或L3事务语义变化。
- L4配方、单位行为、节点组合、候选作用接口变化。
- L5信息、命令路径或表现分配规则影响玩家判断。
- 参数单位、ID、作用对象、候选范围或关联参数变化。
- 地图/候选随机算法、内容池或随机流变化。
- 性能实现改变批次、状态队列或超预算边界。
- 测试协议、构建或参考机器不再匹配。

纯表现资源替换只有在不改变可读性、碰撞、时长认知和任务路径时才不使玩法参数失效；仍需通过对应视觉回归。

## 12. 报告要求

每个实验包报告至少包含：

1. 研究问题和预注册假设。
2. 固定参数、变量、范围和组合方法。
3. 规则、内容、参数、生成和构建版本。
4. 种子集、策略集、样本排除和故障记录。
5. 守恒、恢复、软锁、支配、节奏和性能指标。
6. 分布、极端值与失败案例，不只给平均数。
7. 结论是扩大范围、缩小范围、形成候选、回退还是升级机制。
8. 证据制品路径和下一次失效条件。

没有原始制品、只有截图或只有一次成功流程的结果不能晋升参数。

## 1. 契约职责与写入边界

### 1.1 本文负责

- 发现、内容池资格、当前可执行性、实例化、强化和永久化如何分离。
- 行为里程碑如何从已提交玩法事件形成资格变化。
- 科技族和候选定义如何进入合法候选池。
- 首次样本解析如何形成一次受保护候选组。
- 主动诱导、诱变培养腔和特殊战利品如何形成普通组。
- 候选随机流、成员抽取、多样性、互斥与防动态答案。
- 受保护槽、普通槽、槽位预留和来源缓冲。
- 候选同化、组内其他成员终结、普通组舍弃和本轮突变生效。
- 候选、资格、修正和随机结果的保存、离线和幂等。

### 1.2 本文不负责

- 基因入账、预留、消费和释放；由资源契约负责。
- 样本残骸、样本组织和五阶段过程资产；由资源与单位契约负责。
- 解析、设施周期和房间缓冲过程任务；由房间契约负责。
- 节点特殊战利品和样本恢复来源；由世界契约负责。
- 突变修改后的单位战斗、房间运行和资源结算；由目标契约执行。
- 正式飞升中的永久收益购买和领袖选择；由飞升保存契约负责。
- 具体候选名称、图标、文案、数值和内容数量。

### 1.3 唯一写入者

| 权威事实 | 唯一写入边界 | 其他系统只能做什么 |
| --- | --- | --- |
| 发现与内容池资格 | 本文里程碑和资格事务 | 提供已提交事件、标签与来源 |

| 受保护槽、普通槽和槽位预留 | 本文容量事务 | 永久收益提供容量修正 |
| 本轮突变与修正实例 | 本文同化事务 | 目标系统校验接口与执行修正 |

进化页面、候选卡片、通知和候选动画都不是写入者。

## 2. 进化事实分层

### 2.1 六个维度

| 维度 | 权威含义 | 禁止混同 |
| --- | --- | --- |
| 发现 | 玩家是否接触并记录对象或概念 | 当前能否使用 |
| 内容池资格 | 定义是否可进入建设、孵化或候选池 | 当前资源是否足够 |
| 当前可执行性 | 从资源、空间、槽位、样本和世界事实派生 | 永久解锁 |
| 实例化 | 房间、单位、候选组或突变已经形成 | 看见图标 |
| 强化 | 修正实例改变现有规则 | 新对象身份 |
| 永久化 | 资格或收益明确跨轮保留 | 图鉴自动授权 |

不得保存单一`已解锁`布尔值替代这些维度。

### 2.2 发现记录

发现记录至少包含：对象定义、来源事件、首次步骤、图鉴资格和本轮/跨轮范围。发现不会直接形成候选、基因、蓝图或单位。

### 2.3 科技族资格

科技族是候选分类和来源权重集合。资格可以来自：

- 已解析样本标签。
- 已发现单位、房间、节点或环境关系。
- 已提交生产、战场、回收、扩张和构筑里程碑。
- 当前领袖、永久收益和明确前置突变。

科技族可用只使其成员有机会进入合法池，不直接获得全部效果。

### 2.4 当前可执行性

每次预览和命令验证从当前稳定边界重算：

- 候选来源是否有合法池。
- 普通槽或受保护槽是否可用。
- 生成与同化成本是否可支付。
- 目标规则接口和互斥是否仍合法。

该结果不保存为可单独修改的资格。

## 3. 行为里程碑与可选支路

### 3.1 里程碑来源

里程碑只能由已提交事件形成，例如：

- 房间完成有效循环、能源从短缺恢复或出现真实堵塞。
- 接触敌人、摧毁出兵点、撤离或占领节点。
- 保障过程资产、完成外部同化或解析样本。
- 同化突变并实际触发其规则。
- 打开出口、激活虫巢或完全吞噬区域。

不使用平行教程步骤和页面访问记录。

### 3.2 资格规则

重要内容采用：

```text
全部必要安全条件成立
并且
至少一条可选支路成立
```

- 安全条件只防止无效状态，例如目标接口尚不存在。
- 支路可以来自生产、战场、样本、节点、突变或永久收益。
- 支路不能要求先拥有它自己解锁的内容。
- 内容定义必须登记来源、条件、结果和是否永久。

### 3.3 资格提交

第12阶段按稳定顺序处理里程碑：

1. 收集本步骤已提交事件。
2. 形成无副作用条件结果。
3. 对首次满足的资格建立唯一事件。
4. 更新发现、科技族或内容池资格。
5. 新资格最早影响下一次候选池生成，不追溯改写已形成组。

## 4. 候选定义、合法池与生成算法

### 4.1 候选定义最低字段

每个候选必须登记：

- 稳定定义标识、所属科技族和来源标签。
- 目标对象类型、标签、作用范围和生命周期。
- 规则变化、显式代价或新增压力。
- 必要前置、互斥组、重复与叠加规则。
- 允许的来源类型和基础权重。
- 同化成本模式与目标接口检查。
- 发现、资格和图鉴映射。

候选不能以显示文案或当前页面决定效果目标。

### 4.2 合法池

一个定义进入本次池需要：

- 科技族与来源资格成立。
- 目标内容已发现或是本轮必然可重建的基础内容。
- 前置成立且没有已生效互斥。
- 重复和叠加规则允许再次出现。
- 作用不会创建第二核心行为、非法状态、负容量或守恒捷径。
- 第一版不依赖第二虫巢、正式飞升或未开放插件。

合法池不足以满足来源最低组约束时，来源命令拒绝或自动任务不建立；不能生成空组、无效成员或临时重复答案。

### 4.3 权重与抽取

候选生成统一执行：

```text
建立按稳定定义标识排序的合法池
-> 应用来源、样本、领袖、已有构筑和重复衰减的公开权重修正
-> 使用来源专属候选随机子流无放回抽取
-> 校验组内作用对象、代价结构和互斥多样性
-> 必要时在事务随机副本中确定性重试
-> 原子提交组内容、顺序和随机游标
```

具体组大小、权重和尝试上限是参数。内容配置必须提供一个合法安全组合；达到尝试上限仍无法形成组属于内容配置错误，不向玩家提交非法或空组。

### 4.4 多样性边界

- 同一组不能全部成为同一对象、同一代价结构的无条件强化，除非来源明示为单成员定向组。
- 已有构筑可以提高相关权重，但不能把所有异向候选权重降为零，除非显式互斥使其非法。
- 连续重复可以衰减，但不会保证完整套装。
- 稀有度表达规则幅度、条件或构筑要求，不只是更大数字。

### 4.5 禁止动态答案

候选生成不能读取最近战损、敌方当前防线或兵力差来保证克制项。战场只能通过真实取得的样本、战利品、发现和里程碑改变合法池。

## 5. 候选组、槽位与来源缓冲

### 5.1 候选组最小记录

- 稳定组标识、来源类型、来源对象、来源事件和形成步骤。
- 成员定义引用、顺序、公开生成成本和每成员同化条件。
- 候选随机流标识、提交前后游标和参数版本。
- 槽位类型、槽位或来源缓冲引用。
- 生命周期：`形成中 / 来源缓冲 / 持有 / 已选择 / 已舍弃`。
- 形成、选择、舍弃和终结幂等标识。

组内成员不是独立库存，不能分别占槽或被分别出售。

### 5.2 受保护槽

- 每轮步骤0建立一个只服务首次固定样本组的受保护槽身份。
- 普通组与普通槽预留不能占用。
- 首次组形成后可以长期持有，不倒计时、不腐败、不允许舍弃或重抽。
- 首次同化后受保护槽关闭，不转为普通容量。

### 5.3 普通槽

- 第一版至少存在一个普通槽，具体容量属于参数和永久收益。
- 已持有组与已预留槽之和不得超过容量。
- 主动诱导开始时预留一个槽。
- 普通组可以舍弃，舍弃不返还生成成本。
- 选择一个成员后释放槽位。

### 5.4 来源缓冲

候选组已经形成但无法进入持有槽时：

- 组由进化状态继续唯一拥有，并保存唯一来源缓冲引用。
- 每个来源缓冲最多容纳声明容量；第一版普通来源至少支持一个不可覆盖结果。
- 成员、顺序、成本和随机游标已经提交，释放槽位后不能重抽。
- 来源任务停止形成新组，并派生堵塞。
- 来源缓冲不占普通持有槽，但不是免费无限仓库。

来源缓冲迁入普通槽是自动容量事务，不是玩家命令：

1. 同化、舍弃、容量增加或其他合法事务释放普通槽后，在同一第1阶段事务末尾尝试迁入。
2. 第11阶段处理新候选输出前，再对本步骤开始时已经存在的来源缓冲执行一次迁入。
3. 多个缓冲竞争时按组形成步骤、来源事件排序键、组稳定标识升序选择最早组。
4. 迁移只原子改变`来源缓冲 -> 持有`及缓冲/槽引用，不改变成员、成本、随机游标或来源，也不再次触发生成效果。
5. 槽仍不足或目标槽预留失败时，组保持原缓冲和原堵塞；不得跳过旧组接纳同一步较新的组。

来源房间重组、拆除或来源对象失效不能删除缓冲组。对应结构事务必须等待缓冲迁出，或由未来明确的组终结规则处理；第一版不存在静默丢弃。

## 6. 首次适应与固定样本

### 6.1 一次性资格记录

每轮保存一个`首次适应记录`：

- 固定样本定义与当前有效来源事件。
- 首次解析是否已经提交。
- 受保护组引用或已同化突变引用。
- 样本恢复来源的唯一链标识。

界面所显示的“待保障、处理中、待形成、已持有”等阶段从过程资产、房间任务、候选组和该记录共同派生，不保存第二套可独立推进的教程枚举。

### 6.2 完整形成链

```text
固定样本载体死亡
-> 样本残骸保障与回流
-> 腐解消化池分离普通组织与样本组织
-> 突触解析腔解析
-> 同笔形成基因、样本标签、发现/科技族资格和首次候选组
-> 组进入受保护槽
```

首次组形成不额外支付生成基因。解析输出、基因入账、标签、资格、随机组和受保护槽必须作为一个跨域原子事务；池配置错误属于实现阻断，不能只发基因而丢失首次组。

### 6.3 首次组约束

- 组内至少两个真实可选成员，具体数量待测。
- 成员不能全部作用于同一对象类型或使用同一代价结构。
- 每个成员作用于已发现或必然可重建的基础内容。
- 每个成员同化成本不超过首次解析所得可用基因。
- 第一版首次组成员使用同一成本档。
- 不包含只作用于排除内容的成员。

### 6.4 样本损失与恢复

- 样本保障前永久损失时，世界契约的共振孵育床启动唯一恢复链。
- 未占节点形成带`恢复载体`标签的单位；已占节点形成受容量约束的现场恢复过程资产。
- 恢复来源不生成普通资源、外部同化、额外候选或飞升成果。
- 已保障后堵塞只保留资产和任务，不重新启动恢复。
- 首次解析提交后恢复来源休眠，重复样本不形成第二个受保护组。

### 6.5 首次预算约束

L6参数必须满足：

```text
首次解析基因产量
>= 首次组最高同化成本
 + max(主动诱导生成成本, 第一版普通池最高同化成本)
```

这是防软锁关系，不是发布数值，也不保证两条普通来源同时免费完成。

## 7. 普通候选来源

### 7.1 基因主动诱导

主动诱导是进化状态拥有的过程任务：

1. 验证普通合法池、可用基因和普通槽。
2. 同笔预留生成基因与一个普通槽。
3. 在第3阶段按研究构筑安全层级请求功率并推进时间。
4. 第11阶段抽取并形成组，消费预留基因，组进入预留槽。
5. 形成前取消释放基因与槽；时间和功率不返还。

第一版主动诱导成员选择不再收第二笔基因。组不受欢迎也不退款。

### 7.2 诱变培养腔

- 房间任务使用生物质、已解析样本资格、功率和时间。
- 没有可预留普通槽时不开始新周期。
- 周期完成时若同一步槽竞争使预留失效，组进入该房间来源缓冲。
- 组形成后成员同化时支付各自公开基因成本。
- 房间暂停、取消和堵塞遵循房间与资源契约。

### 7.3 特殊战利品

- 战利品先完成五阶段与声明处理链。
- 默认不收主动诱导的生成基因成本。
- 普通槽满时，已形成组进入处理来源缓冲，不销毁战利品结果、不重抽。
- 第一版成员按组中公开条件支付基因同化成本。
- 单项特殊候选仍是只有一个成员的候选组。

### 7.4 来源统一边界

三种来源都必须提供稳定来源、合法池标签、成本模式、组约束、随机流、槽位或缓冲、取消边界与幂等标识。新增来源不能绕过候选组和组内单选。

## 8. 同化、舍弃与修正生效

### 8.1 同化预检查

玩家确认前检查：

1. 组处于`持有`且未被其他交易锁定。
2. 成员属于该组并仍合法。
3. 公开同化成本可支付；已由来源支付时成本为零而非隐藏省略。
4. 目标接口、作用范围、前置和互斥仍成立。
5. 修正不会破坏所有权、守恒、单一行为、容量和恢复不变量。

失败不消费资源、不终结成员、不推进随机流。

### 8.2 原子同化

在`关键选择暂停`边界提交：

```text
预留并再次校验公开同化成本
-> 消费成本
-> 建立本轮突变与修正实例
-> 标记所选成员
-> 终结同组其他成员
-> 候选组进入已选择
-> 释放普通槽或关闭首次受保护槽
-> 更新后续资格、互斥、统计和自动保存请求
```

整笔要么提交一次，要么完全不发生。未选成员不形成基因、生物质、候选机会或返还。

### 8.3 普通组舍弃

- 只有处于普通槽的`持有`组可以舍弃。
- 舍弃终结整组并释放普通槽。
- 生成成本、处理资产、时间和功率不返还。
- 受保护首次组和来源缓冲组不能通过该命令舍弃。
- 舍弃不重放来源或自动形成替代组。

### 8.4 修正作用

本轮突变保存来源、目标、作用范围、生命周期、叠加、互斥和最早生效步骤：

- 默认持续到本轮结束。
- 作用于内容定义时，同时影响现有和未来合法实例。
- 房间拆除重建不删除定义级突变。
- 修改单位时只能调整已有核心行为的条件、目标、周期或效果；新动词需要新模板或新形态。
- 修改资源或房间时只能调用对应契约的通用条件、配方和修正接口。
- 不能暗中修改地图随机流、自动选择候选或伪造玩家命令。

## 9. 关键选择、通知与页面边界

- 候选形成只产生非强制通知，不自动遮挡主动战场。
- 玩家主动打开组内选择时，在最近稳定边界进入`关键选择暂停`。
- 查看、比较和关闭不消费资源、不推进随机流。
- 同化和普通组舍弃可以在暂停边界原子提交。
- 关闭页面恢复进入前的世界运行状态和倍率。
- 页面显示来源、生成成本、同化成本、规则变化、代价、生命周期、互斥和组内单选。
- 页面不显示推荐候选、最优标记或隐藏敌情克制提示。

## 10. 十二阶段接入

| 阶段 | 本文职责 |
| --- | --- |
| 步骤0 | 建立首次适应记录、受保护槽、普通槽基础容量和初始永久资格投影 |
| 稳定边界/第1阶段 | 主动诱导建立/取消、普通组舍弃、候选同化、释放槽后的缓冲迁入和关键选择状态 |
| 第3阶段 | 推进主动诱导；房间候选任务由房间契约推进 |
| 第4至10阶段 | 只读取已生效修正；不生成或选择候选 |
| 第11阶段 | 先迁入既有来源缓冲，再让样本解析、主动诱导、设施和特殊来源形成候选组或新的来源缓冲 |
| 第12阶段 | 行为里程碑、发现、科技族资格、互斥、统计、守恒和通知 |

第11阶段形成的组在本步骤发布后可查看；同化产生的修正按时间契约最早生效步骤执行。

## 11. 命令契约

| 命令 | 验证 | 成功结果 | 取消/拒绝 |
| --- | --- | --- | --- |
| 启动主动诱导 | 合法池、基因、普通槽和任务上限 | 建立基因/槽预留和形成任务 | 任一不足整笔拒绝 |
| 取消主动诱导 | 任务尚未形成组且可取消 | 释放未消费基因与槽预留 | 历史时间功率不返还 |
| 同化候选成员 | 组、成员、成本、目标接口和互斥 | 突变生效、整组终结、槽释放 | 任一失败无变化 |
| 舍弃普通组 | 普通持有组未被交易锁定 | 整组终结并释放槽 | 不返还生成成本 |

设施循环由房间策略驱动，不在进化页面增加复制按钮。查看和关闭未确认选择不是命令。

## 12. 事件与跨域接口

### 12.1 事件类别

| 事件 | 提交阶段 | 原子结果 |
| --- | --- | --- |
| 发现已记录 | 第12阶段或轮次建立 | 新知识与来源事实 |
| 科技族资格已形成 | 第12阶段或轮次建立 | 定义进入后续合法池 |
| 候选形成任务已建立/取消 | 第1阶段 | 资源、槽与任务一致 |
| 候选组已形成 | 第11阶段 | 随机成员、位置和游标同笔提交 |
| 候选组已移入持有槽 | 第1或11阶段 | 按形成步骤、来源事件和组标识稳定选择，来源缓冲与槽位引用互斥变化 |
| 候选已同化 | 暂停边界 | 成本、修正、组终结和槽释放 |
| 普通组已舍弃 | 暂停边界 | 组终结且无返还 |

### 12.2 资源房间接口

- 请求基因预留、消费与释放，不能直接修改账目。
- 接收解析任务的基因、标签和首次组跨域提交请求。
- 接收房间候选输出和来源缓冲引用，不拥有房间任务。

### 12.3 单位世界接口

- 读取已提交单位死亡、样本恢复、节点控制和战利品来源事件。
- 不改变单位目标、地图内容和控制状态。
- 突变效果通过通用修正接口进入目标契约。

### 12.4 飞升接口

- 提供本轮已同化突变数量、发现和可永久化资格。
- 读取当前领袖与永久科技族作为初始合法池来源。
- 第一版只允许飞升预览读取，不执行永久化。

## 13. 保存、离线与确定性

### 13.1 必须保存

- 发现、内容池资格、行为里程碑和来源事件。
- 首次适应记录、受保护槽和普通槽容量/预留。
- 全部候选组、成员、顺序、成本、位置、状态和来源缓冲。
- 候选形成任务、输入预留、进度、输出承诺和幂等标识。
- 候选随机流种子、标识、游标和已提交结果。
- 本轮突变、修正、互斥、叠加、最早生效步骤和统计。

### 13.2 离线

- 无主动战场时，主动诱导和设施任务按正常功率、输入、槽位和容量推进。
- 槽满或来源缓冲满时正常堵塞，不自动舍弃或选择。
- 候选组默认不随离线时间消失。
- 有主动战场时整个世界冻结。

### 13.3 确定性

- 相同快照、来源事件、合法池、参数和随机流得到相同组成员与顺序。
- 预览、拒绝、堵塞和页面访问不推进随机流。
- 读档不重新形成、重抽或再次同化已经提交的组。
- 容器、帧率、时间倍率和画面顺序不改变候选结果。

## 14. 第一版范围与排除

第一版实际支持：

- 固定样本完整处理和首次受保护组。
- 至少一个普通持有槽。
- 基因主动诱导、诱变培养腔和首轮出现的特殊战利品来源。
- 普通组持有、舍弃和组内单选。
- 首次同化后至少一种普通来源的实际生成与处理。
- 本轮突变对现有房间、单位或战场规则产生可观察变化。

第一版排除：

- 正式飞升永久化、领袖实际生效和领袖专属池。
- 完整插件安装和候选改造。
- 已同化突变撤销、重配或免费替换。
- 候选倒计时、战斗失败保底克制和固定线性科技树。

## 17. 不变量责任与完成条件

### 17.1 不变量责任

本文直接维护：

- INV-001至INV-003：进化对象稳定、唯一所有且页面不成第二真相。
- INV-011：每组占一个槽，选择一个后其余终结且无返还。
- INV-012：发现、资格、可执行、实例化、强化和永久化分离。
- INV-019：候选随机结果提交后不重抽。
- INV-020至INV-023：候选生成和同化确定、表现独立、原子且幂等。

本文协同维护：

- INV-004至INV-009：成本和修正调用资源、房间和单位契约，不绕过守恒与功率。
- INV-017：已占节点特殊来源不能重启敌方战斗。
- INV-024：首次样本恢复、受保护槽和预算不等式防止进化软锁。

### 17.2 完成条件

- 所有进化事实分层明确，行为里程碑不依赖教程或页面。
- 首次样本、受保护组和三类普通来源均有唯一生成、支付、槽位和失败边界。
- 候选池、随机、多样性、来源缓冲和持有在读档与堵塞后保持稳定。
- 同化交易不能半扣资源、半生效或保留同组未选成员。
- 突变只通过目标系统通用修正接口生效，不破坏单一行为和全局不变量。
- CEC-001至CEC-030全部映射到L6自动测试；未通过前本文状态不能升级为已验证机制。

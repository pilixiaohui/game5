## 1. 契约职责

### 1.1 本文负责

- 统一模拟时间、完整步骤与时间倍率的关系。
- 正常、暂停、关键选择、离线和主动战场冻结的运行边界。
- 命令如何接收、排序、验证、拒绝、创建任务或原子提交。
- 同一步撤离、进攻、房间操作和候选选择发生冲突时的唯一结果。
- 事件草案何时形成、何时必须结算、何时因执行者死亡或条件失效而终止。
- 房间、战斗、后台战斗、恢复、候选、地图与账目共用的十二阶段。
- 自动保存、手动保存、载入、离线结算与确定性重放的稳定时点。
- 其他系统必须接入的阶段、命令和事件接口。

### 1.2 本文不负责

- 具体资源成本、任务周期、步长、倍率、离线上限和警报阈值。
- 能源分配层级内部算法、房间配方、工蜂目标选择、伤害公式或地图生成内容。
- 具体页面布局、动画速度和输入设备实现。
- 正式飞升交易；第一版只允许读取飞升预览。

## 2. 统一时间模型

### 2.1 完整步骤

模拟时间只通过固定时长的`完整步骤`推进：

```text
下一模拟时间 = 当前模拟时间 + 固定步长
下一步骤号 = 当前步骤号 + 1
```

- 固定步长是L6参数，运行中不能因帧率、页面或单位数量改变。
- 所有持续时间、任务进度、冷却、腐败和恢复最终换算为步骤边界。
- 现实帧可以执行零个、一个或多个完整步骤，但不能执行半个可保存步骤。
- 百万级单位使用批次和状态队列，不改变时间语义。
- 性能优化可以跨越没有事件的步骤，但必须与逐步执行产生完全相同的事件顺序和结果。

### 2.2 三种时间输入

| 时间 | 用途 | 能否直接结算玩法 |
| --- | --- | --- |
| 模拟时间 | 所有玩法状态变化 | 是，唯一依据 |
| 表现时间 | 动画、镜头、声音、界面过渡 | 否 |
| 离线经过时间 | 计算允许补算的模拟时间输入 | 否；必须先进入离线安全结算 |

动画可以追赶、简化或跳过；表现尚未完成不能阻止下一模拟步骤，也不能补发资源、伤亡或冷却。

### 2.3 时间倍率

时间倍率只决定单位现实时间内调度多少完整步骤：

- 所有在线系统使用相同倍率。
- 不允许只加速己方生产、只减速敌方战斗或让腐败停在原倍率。
- 改变倍率不改变固定步长和步骤内顺序。
- 相同起始状态、命令和模拟时长在任意倍率组合下结果相同。
- 倍率切换记录在界面与会话日志中，但不作为资源或战斗修正。

## 3. 世界运行状态

| 状态 | 是否推进完整步骤 | 进入条件 | 退出条件 |
| --- | --- | --- | --- |
| `正常运行` | 是 | 默认在线游玩或恢复运行 | 玩家暂停、关键选择或关闭游戏 |
| `全局暂停` | 否 | 玩家暂停、系统菜单或设置 | 玩家恢复或关闭游戏 |
| `关键选择暂停` | 否 | 玩家打开需要不可逆确认的候选或后续飞升界面 | 完成、取消查看或关闭界面 |
| `离线安全结算` | 批量推进 | 无主动战场时从关闭状态载入 | 有效离线时长结算完成 |
| `主动战场冻结` | 否，整个世界冻结 | 有主动战场时关闭游戏 | 重新载入并回到在线状态 |

规则：

- 页面切换、详情抽屉、地图观察、日志和图鉴默认不改变运行状态。
- 打开候选选择前，先在最近稳定边界进入`关键选择暂停`，不能冻结在步骤中间。
- 关闭游戏时只读取最近完成的稳定边界；有主动战场则房间、战场和世界一并冻结。
- 已占节点安全领地活动不是主动战场，不阻止离线安全结算。
- 运行状态变化必须形成有序事件，但不推进模拟时间。

## 4. 稳定边界与命令模型

### 4.1 稳定边界

稳定边界是一个完整步骤结束后、下一个步骤尚未开始的状态。只有稳定边界允许：

- 接受和排序新的玩法命令。
- 在暂停状态执行不推进时间的边界事务。
- 写入权威快照。
- 切换世界运行状态和时间倍率。
- 发布上一完整步骤的表现事件与警报。

当前轮次尚不存在时，由`L3_飞升保存与轮次契约.md`协调一次跨域`轮次建立事务`，并首次发布步骤`0`稳定边界。世界地图、初始虫巢、单位、进化状态、随机流与资源初始注入分别由其权威契约提供子事务；`L3_资源与能源契约.md`只写入经济子事务，不能单独代表整轮建立。步骤`0`不是一个省略阶段的完整步骤，只能写入L2声明的初始对象、初始资源、版本和来源事实；发布后的全部玩法变化继续遵循本文边界和十二阶段。

玩家可以在表现帧中的任意时刻操作，但命令只绑定到下一个合法稳定边界。界面预览可以即时刷新，不能提前修改权威事实。

正常运行时，边界先处理世界控制与保存调度；若仍为正常运行，则封存其余命令并由下一个完整步骤读取，若进入暂停，则只提交第5.2节允许的边界事务。暂停期间可以在同一稳定边界连续提交多笔允许的边界事务。所有命令接收序号在当前边界内单调递增，不能因打开页面、关闭弹窗或经过新的表现帧而重新编号。

### 4.2 玩法命令信封

每条玩法命令至少记录：

| 字段 | 含义 |
| --- | --- |
| 命令标识 | 全局唯一且用于防重复提交 |
| 命令类型 | 建设、暂停、进攻、撤离、同化等登记类型 |
| 发起者 | 玩家、明确的系统控制器或恢复流程 |
| 目标引用 | 稳定实例标识和需要的定义标识 |
| 公开参数 | 玩家实际选择的槽位、模板、候选成员等 |
| 接收边界 | 命令被系统接收的步骤边界 |
| 接收序号 | 同一边界内的稳定先后顺序 |
| 最早提交步骤 | 允许该命令首次验证和提交的步骤 |
| 幂等标识 | 网络重试、输入重复或载入后用于识别同一意图 |

命令不能携带由客户端自行计算的资源结果、伤亡结果、胜率或随机结果。

### 4.3 命令生命周期

```text
已接收 -> 待验证
待验证 -> 已接受 | 已拒绝
已接受 -> 已提交 | 已取消
```

- `已接受`只表示命令拥有合法提交位置；不表示长时任务已经完成。
- 建设、重组、诱导等命令提交后创建过程任务，命令本身结束，后续由任务生命周期负责。
- `已拒绝`不产生预留、消费、随机抽取或部分状态。
- 只有契约明确允许取消的待处理命令可以进入`已取消`。
- 关闭撤离确认不创建命令；撤离一经确认并`已接收`便不可取消。

### 4.4 预览与验证

- 预览和详情属于`只读查询`：只读取最近稳定边界和公开参数，不进入命令队列、不预留资源、不推进随机流、不创建实例或玩法事件。
- 命令必须在实际提交边界重新验证，不能依赖预览时的旧状态。
- 验证只回答是否合法以及缺少什么事实，不生成策略建议。
- 等待中的任务若后来缺料、缺能或堵塞，进入任务阻断状态，不把原命令重新解释为失败。

### 4.5 第一版边界输入登记

| 命令族 | 第一版命令 | 提交位置 | 提交后的直接结果 |
| --- | --- | --- | --- |
| 世界控制 | 暂停、恢复、设置时间倍率、打开/关闭关键选择 | 稳定边界 | 改变运行状态或现实时间调度，不推进模拟时间 |
| 地图情报 | 观察节点 | 第1阶段边界事务 | 把合法相邻节点从轮廓已知提升到可观察；不生成或重抽内容 |
| 房间结构 | 建设、取消建设、移动、功能重组、拆除、替换、降级、升级、提交批量结构事务 | 第1阶段边界事务 | 拒绝，或原子创建/取消一笔或一组有预留的普通结构过程任务 |
| 房间运行 | 暂停/恢复房间群、设置循环模式、目标规模和孵化模板、提交单次批次/移除其未接纳余量、取消可取消当前批次 | 第1阶段边界事务 | 改变后续任务资格、队列条目或合法任务预留；不追溯改变已消费输入和完成输出 |
| 候选来源 | 启动/取消主动诱导、舍弃普通候选组 | 第1阶段边界事务 | 创建/取消任务，或终结一个普通组 |
| 进化选择 | 同化候选成员 | 关键选择暂停的第1阶段边界事务 | 原子消费公开成本、激活一个突变并终结整组 |
| 战役 | 进攻 | 第2阶段 | 占用主动战场槽并转移全部可参战单位 |
| 战役 | 撤离 | 第1阶段最高优先级 | 原子回归、留置、资产换阶段并释放战场槽 |
| 保存服务 | 请求手动保存 | 稳定边界 | 排队写入当前或下一完整稳定快照，不改变玩法 |

表中除`保存服务`外均为当前轮次玩法命令并使用第4.3节生命周期。`请求手动保存`是边界服务请求，只由保存调度记录请求、目标稳定边界、写入结果和失败，不进入玩法命令批次，也不生成玩法事件。新建、载入、继续、覆盖和删除存档属于飞升保存契约定义的存档生命周期请求，不进入本表。

打开页面、读取飞升预览、读取已揭示详情、选择列表项、关闭未确认弹窗、镜头移动和表现跳过属于只读查询或表现操作，不是玩法命令。`观察节点`因会提升权威情报而是明确登记的地图命令；普通详情访问不能替代它。第一版不存在战场技能、部分单位投放、运输、驻军分配和正式飞升命令。

表中“第1阶段边界事务”在正常运行时由下一完整步骤第1阶段提交；在暂停状态下则使用第5.2节的同序边界事务提交，不伪造一个只执行部分阶段的完整步骤。

## 5. 命令排序与冲突

### 5.1 边界命令批次

同一稳定边界的命令形成一个批次，并按以下规则处理：

1. 当前主动战场的已确认撤离在全部玩法命令中拥有最高优先级，最多接受一条；它在正常运行的第1阶段或暂停边界事务中都先于其他玩法交易验证和提交。
2. 世界控制命令随后按接收序号处理，确定本边界结束后的运行状态与时间倍率；同批多次设置以最后一条合法命令的结果为准。
3. 其他第1阶段命令按接收序号稳定处理，每条都读取之前已提交命令产生的新状态。若结果状态为暂停，只提交第5.2节允许的边界事务；否则保留给下一完整步骤第1阶段。
4. 进攻命令只在第2阶段提交，不在边界事务中立即移动单位。
5. 当前边界成功撤离后，立即相邻的下一完整步骤内所有进攻命令均以“不能回归后立即再次进入战场”拒绝；玩家必须在该步骤发布后的新稳定边界重新发起。
6. 没有撤离时，多个进攻命令按接收序号验证；第一条合法命令占用唯一主动战场槽，其余拒绝。
7. 同一槽位、任务、候选组或资源预留发生冲突时，先提交者生效，后续命令基于新事实验证并拒绝或采用仍合法的结果。

不得使用对象容器顺序、页面层级、帧到达先后或内容稀有度改变上述顺序。

### 5.2 暂停中的边界事务

处于`全局暂停`或`关键选择暂停`时，可以在当前稳定边界原子提交：

- 候选同化、普通组舍弃等必须由玩家确认的进化交易。
- 合法相邻节点的观察命令；重复观察已达到当前上限的节点幂等返回且不形成新事件。
- 房间暂停/恢复、队列模式、目标规模、单次条目提交/未接纳余量移除、可取消当前批次终结，以及建设、升级、移动、功能重组、降级、拆除、替换和对应取消等合法结构事务的建立。
- 已确认撤离。
- 保存请求和运行状态切换。

这些事务不推进模拟时间。结构事务可以建立、取消或改变排他声明，但建设、重组和拆解进度只在恢复运行后的第3阶段推进。新任务、突变、房间状态和回归单位最早影响下一个完整步骤；不得追溯修改已经完成的步骤。进攻可以排队，但只在恢复运行后的第2阶段提交。

### 5.3 内容不能增加隐式命令

单位自主移动、目标选择、房间自动循环、工蜂采集、敌方恢复和后台战斗是模拟规则，不是玩家命令。科技、领袖和突变可以修改既有条件、配方或参数，但不能伪造玩家点击、自动选择候选、自动进攻、自动撤离或自动飞升。

## 6. 事件模型

### 6.1 事件草案与已提交事件

| 对象 | 含义 | 是否进入稳定权威快照 |
| --- | --- | --- |
| 效果配方 | 内容定义声明可能产生的变化 | 否 |
| 事件草案 | 当前步骤工作状态中已经形成、等待本步骤指定阶段提交的确定结果 | 否；只存在于未发布步骤的事务工作区 |
| 事件 | 已在合法阶段原子提交的状态变化 | 是 |
| 账目条目 | 事件对资源、单位和资产变化的审计投影 | 是，不再次改变状态 |

其他文档所称“已形成攻击、出兵或状态事件”，在本文中统一解释为`事件草案`；只有本步骤完成提交并通过第12阶段校验后才随稳定边界发布为事件。第一版事件草案不得跨完整步骤存活。需要等待未来条件的意图或结果必须归入一个已经登记所有者和生命周期的对象：有序待处理命令、队列条目、过程任务、待提交输出、单位行为进度，或带最早生效/失效步骤的单位、状态和修正实例；不能建立通用`计划事实`，也不能把旧快照草案留到未来重新提交。

### 6.2 事件最小字段

- 唯一事件标识和事件类型。
- 完整步骤事件记录步骤号、阶段号和阶段内顺序；稳定边界事件记录已完成步骤号和边界事务顺序。
- 上游命令、任务或事件标识。
- 来源、目标、所有者和位置引用。
- 变化前后状态、数量和参数版本。
- 是否要求来源在提交阶段仍存活。
- 最早生效步骤与一次性幂等标识。

### 6.3 事件排序键

非同时事件使用以下规范排序字段：

```text
时点坐标：完整步骤与阶段，或已完成步骤后的边界事务
-> 事件类别固定顺序
-> 上游命令接收序号或来源事件序号
-> 来源稳定标识
-> 目标稳定标识
-> 定义内效果槽序或规范批次分片序
```

- 稳定边界事件排在刚完成步骤的全部事件之后、下一完整步骤的全部事件之前；同一边界按边界事务顺序排列。
- 定义内效果槽序必须来自内容定义；规范批次分片序必须由稳定数量区间推导，不能使用创建或遍历先后。
- 全局事件序号在全部排序字段确定后依次分配，不能反过来作为打破平局的输入。
- 若两个待提交事件的全部排序字段仍相同，系统必须按契约合并它们或将其判为重复身份错误，不能任意选择先后。
- 同时效果集合的排序只用于幂等、账目和表现发布，不能把同时伤害重新解释为逐条抢先生效。
- 任何系统不得依赖字典、数组、场景树、线程完成或画面实体顺序。

### 6.4 两类行为草案

| 类别 | 第一版效果 | 来源同一步死亡后的处理 |
| --- | --- | --- |
| 形成后必结算 | 伤害、已完成的状态脉冲、已完成的生成单位 | 按第9或第11阶段继续提交 |
| 存活后提交 | 搬运资产、自身变形 | 第9阶段死亡后失效并释放预留或掉落携带物 |

任务完成输出不依赖表现对象存活，但仍需在第11阶段检查输出容量和目标容器。容量不足时任务保持`待提交`或进入来源缓冲，不能重复消费输入或覆盖旧输出。

### 6.5 原子性与幂等

- 一个事件的全部写入要么提交一次，要么完全不提交。
- 事件提交前失败不得留下数量扣减、状态改变或随机游标推进。
- 事件提交后即使表现失败、自动保存延迟或页面关闭，也不能再次提交。
- 死亡输出、节点占领、出口打开、候选组形成、候选同化和首次样本资格必须具有稳定一次性标识。
- 读档后已经提交的事件只用于重建和审计，不重新进入待提交队列。

一个完整步骤整体是一个原子发布单元：各阶段提交的事件先写入本步骤工作状态，可供后续阶段读取，但只有第12阶段校验通过后才一起发布为新的稳定边界。校验失败时丢弃本步骤工作状态，不允许界面、保存服务或其他线程观察到半个步骤。暂停期间的单笔边界事务同样必须在写入后先通过受影响不变量检查，再原子发布。

事件草案随未发布工作状态一同提交或丢弃。实现可以为异常诊断保留草案副本，但诊断副本不属于可继续游玩的存档，不能在载入后恢复提交。

### 6.6 随机事务

- 只有声明需要随机的命令、任务输出或世界事件可以读取指定确定性随机流。
- 随机请求先按事件排序键串行化，再使用当前游标的事务副本形成结果；结果、游标推进和对应事件必须在声明阶段原子提交。
- 预览、合法性验证、命令拒绝、任务尚未达到提交条件或输出无合法去向时，不能推进游标。
- 已提交地图、候选组或其他随机结果保存实际内容与提交后的游标，载入和重复观察不得重新抽取。
- 第一版战斗命中、伤害、死亡和目标平局不读取随机流。

## 7. 唯一十二阶段

每个完整步骤按以下顺序执行。阶段号和含义属于固定机制，不是参数。

| 阶段 | 名称 | 主要职责 |
| --- | --- | --- |
| 1 | 边界命令与撤离 | 读取并处理有序命令；优先原子提交撤离；提交其他合法边界事务 |
| 2 | 进攻与延迟位置变化 | 提交合法进攻、上一步形成的进入/回归以外位置变化 |
| 3 | 能源与非战场过程 | 结算能源服务，推进房间、建设、解析、候选设施、来源抽取和敌方恢复计时 |
| 4 | 状态与生效资格 | 激活/失效状态，确定本步骤可行动单位、任务与修正 |
| 5 | 范围、路径与容量 | 按存活固定单位重建被动范围、路径、空间和交互容量 |
| 6 | 移动与补位 | 推进移动、进入空间层、补位和战线位置 |
| 7 | 目标与预留 | 过滤并稳定选择目标，建立行为需要的资产、容量和目标预留 |
| 8 | 形成事件草案 | 从统一快照形成伤害、搬运、变形、出兵、状态和过程资产草案 |
| 9 | 同时效果、死亡与输出 | 同时提交伤害和状态脉冲，随后统一结算死亡、携带物掉落和死亡输出 |
| 10 | 存活后提交效果 | 提交仍合法且要求执行者存活的搬运资产和自身变形 |
| 11 | 任务输出与新实例 | 提交出兵、房间离巢、恢复输出、房间物理/网络输出、候选组和其他新实例 |
| 12 | 世界判定、校验与发布 | 检查占领、战场槽、解锁、统计、飞升预览、随机游标、守恒、日志、警报和表现事件 |

### 7.1 第1阶段：边界命令与撤离

- 读取本边界命令批次，先处理当前主动战场撤离。
- 撤离成功后，该节点本步骤不形成移动、攻击、出兵、采集、扎根或环境伤害。
- 撤离只跳过该节点的战斗阶段；能源、房间、其他节点后台战斗和世界过程继续本步骤。
- 提交房间、队列和候选等边界事务；新修正记录`最早生效步骤=当前步骤`的第4阶段，若命令在暂停边界提交则为恢复后的下一步骤。
- 进攻只验证并保留到第2阶段，不在本阶段移动单位。

### 7.2 第2阶段：进攻与延迟位置变化

- 选择第一条仍合法的进攻命令，但在跨域准备完成前不占用主动战场槽、不释放预留也不移动单位。
- 转移工蜂前，单位契约按稳定顺序分类工蜂，并调用资源、房间和世界接口无副作用准备释放其仍可无损中断的采集资产、路径、源交互端和投递端预留；正常不可释放作业不进入最终集合。
- 最终集合确定后，以同一进攻事务标识原子提交全部准备释放、工蜂行为/活动清空、单位转入、战役期初账目和唯一主动战场引用。任一已准备子事务不匹配时本次进攻完全不发布。
- 全部可参战单位从虫群大军转入目标节点左侧后备，不建立行军或运输状态。
- 已成功撤离的单位本步骤不能重新进入任何节点。
- 进入单位原本已经生效，可以在本步骤第4至第8阶段行动；第11阶段新形成单位不能。
- 新区域虫巢节点进攻若需要提交完整区域内容，必须先原子生成、校验并提交地图；失败则进攻整笔拒绝且不留下半张地图。

### 7.3 第3阶段：能源与非战场过程

- 阶段开始先由`L3_虫巢与房间契约.md`按稳定快照接纳至多一个适配各空闲模块的新任务，原子锁定其输入、目标份额和输出容量，并登记本步骤功率请求。
- 随后根据本步骤开始时已存在的供给、新接纳任务负载和安全层级确定已服务功率及储能变化。
- 再推进得到服务的建设、重组、生产、消化、解析、候选形成、着床建设和其他过程任务。
- 推进已占来源抽取、残骸腐败、敌方恢复计时和允许的离线过程。
- 本阶段完成的房间、供给、输出和恢复只形成待提交结果，在第11阶段提交并从下一步骤参与供需或行为，避免同阶段递归。
- 取消或暂停命令已在第1阶段生效的任务，本阶段不再消耗功率或推进。

### 7.4 第4至第8阶段：资格、空间与草案

- 第4阶段只激活已到生效步骤的单位、状态和修正；过期状态在声明边界失效。
- 第5阶段只从当前存活固定单位和权威地形重建范围、路径与容量。
- 第6阶段移动只改变位置，不直接造成伤亡、资源或控制权。
- 第7阶段目标与预留使用稳定标识和平局规则，不读取玩家最近失败或画面对象。
- 第8阶段使用第7阶段完成后的统一快照形成草案；同类来源批次合并或拆分不能改变草案总量和目标池。

### 7.5 第9阶段：同时效果与死亡

- 同一步敌我伤害从同一快照形成后同时提交。
- 本阶段死亡的单位仍完成已经形成的`形成后必结算`草案。
- 伤害与状态提交后统一结算死亡、携带物掉落、死亡输出、范围移除和占用释放。
- 同一死亡只能形成一次死亡输出；后台我方固定单位死亡按已确认规则跳过可回收输出，但仍登记死亡。
- 本阶段新施加且声明下一完整步骤生效的状态，最早在下一步骤第4阶段生效。

### 7.6 第10阶段：存活后提交

- 搬运资产要求执行者、来源、目标、路径和容量仍合法。
- 自身变形要求执行者仍存活，按统一继承规则等量提交。
- 不合法草案原子失效并按来源契约释放预留；不能部分搬运或同时保留变形前后数量。

### 7.7 第11阶段：任务输出与新实例

- 提交第3和第8阶段已经完成的单位、过程资产、物理输出、网络输出和候选组结果。
- 新形成或刚变形单位记录最早生效步骤为下一完整步骤，不能在本步骤再次移动、选目标或执行行为。
- 新完成能源房间和新供给从下一完整步骤第3阶段参与供需。
- 输出容量冲突保留为`待提交`、`堵塞`或`来源缓冲`，不重复消费输入、不覆盖旧结果。
- 同一步多个来源争用候选槽或容器时，按事件排序键提交，后续结果进入各自合法缓冲。

### 7.8 第12阶段：世界判定、校验与发布

按以下顺序收尾：

1. 检查主动与后台节点的占领条件。
2. 对满足条件的节点提交控制、移动单位回归、出口通路和来源开放。
3. 对已经撤离、已经占领或不再存在玩家可机动在场单位的节点释放主动战场槽；固定单位保持节点位置，未占节点从下一步骤转入后台规则。
4. 更新行为里程碑、内容池资格、候选资格和飞升贡献统计。
5. 检查L2守恒、所有权、位置、槽位、随机游标和幂等不变量。
6. 生成日志、账目、离线摘要增量、警报和表现事件。
7. 响应等待中的自动保存或手动保存请求。
8. 发布新的稳定边界并递增步骤号。

若不变量检查失败，本步骤不能发布为可继续模拟的状态；处理方式见第11节。

### 7.9 主动与后台战斗

- 主动战场执行第1至12阶段，并允许撤离命令。
- 未占节点后台战斗执行第3至12阶段，不创建画面代表、不占主动战场槽、不接受战役命令。
- 已占安全领地只执行来源、采集、房间修正和表现投影，不执行敌方战斗阶段。
- 所有节点共享同一状态、行为、伤害、死亡和占领顺序，不能维护“离线比例结算”。

## 8. 生效边界

为防止同一步递归和顺序套利，统一采用：

| 本步骤形成的事实 | 最早影响时点 |
| --- | --- |
| 第1阶段暂停/取消任务 | 本步骤第3阶段 |
| 第1阶段同化的突变 | 本步骤第4阶段；若在暂停边界提交则为恢复后下一步骤 |
| 第2阶段进入的既有单位 | 本步骤第4阶段 |
| 第3阶段完成的房间或能源供给 | 下一步骤第3阶段 |
| 第9阶段施加的持续状态 | 下一步骤第4阶段，除非定义明确声明即时一次性效果 |
| 第10阶段完成的变形单位 | 下一步骤第4阶段 |
| 第11阶段形成或离巢的单位 | 下一步骤第4阶段 |
| 第12阶段开放的节点来源与房间修正 | 下一步骤对应阶段 |

任何内容要突破上述边界，必须先新增通用规则并证明不会产生同一步无限触发，不能由单个突变或Boss自行覆盖。

## 9. 暂停、警报与页面

### 9.1 全局暂停

- 暂停请求在当前步骤完成后的稳定边界生效。
- 暂停期间不推进能源、任务、战斗、恢复、腐败、状态或冷却。
- 玩家可以检查和创建边界事务；新任务在恢复运行后才推进。
- 恢复请求发布下一个可运行边界，原时间倍率保留。

### 9.2 关键选择暂停

- 候选组形成只发送非强制通知，不自动暂停或遮挡战场。
- 玩家主动进入候选选择时，在稳定边界进入关键选择暂停。
- 查看、比较和关闭不消费资源或推进随机流。
- 同化命令在当前暂停边界原子提交；突变不得改写已完成步骤或已形成草案。
- 关闭后恢复进入前的运行状态与时间倍率。

### 9.3 页面与警报

- 切换虫巢、地图、战场、进化、图鉴或统计不暂停世界。
- 主动战场在其他页面运行时必须保持入口、真实损耗和撤离入口可见。
- 警报由第12阶段事件和派生阈值形成，只描述状态变化。
- 设置为自动暂停或降速的警报在当前步骤发布后生效，不能插入步骤中间改变结果。
- 警报不生成推荐布局、兵种、兵力或撤离时机。

## 10. 离线安全结算

### 10.1 进入条件

- 关闭时不存在主动战场：保存稳定边界和现实时间戳，载入后进入离线安全结算。
- 关闭时存在主动战场：保存稳定边界并进入主动战场冻结，离线经过时间不转换为模拟时间。
- 第一版相邻区域概要、只读飞升和未生成世界内容不参与离线推进。

### 10.2 等价要求

离线结算等价于在没有新玩家命令的情况下执行相同有效模拟时长的完整步骤：

- 继续遵守能源、输入、工蜂、队列、容量、腐败和敌方恢复。
- 未占节点固定单位与恢复敌军运行同一后台战斗。
- 后台满足占领条件时可以提交节点控制，但不发放资源包。
- 已占节点只推进安全来源、领地采集、回流与房间形成，不生成战损。
- 不自动建设、改变队列、发起进攻、选择候选、购买永久收益或执行飞升。
- 候选槽、输出或房间容量满时正常堵塞，不能为提高离线收益绕过容量。

实现可以使用确定性事件跳跃，但必须按时间顺序处理每个会改变条件的边界，并通过逐步执行对照测试。

### 10.3 有效时长与摘要

- 有效离线时长使用一个全局上限，具体值属于L6参数。
- 超出上限的现实时间不继续推进任何玩法系统。
- 离线摘要只汇总离线期间实际提交的事件，不再次计算奖励。
- 摘要至少区分资源入账、任务完成、堵塞、单位形成、过程资产变化、敌方恢复、后台伤亡和节点占领。

## 11. 保存、载入与异常

### 11.1 保存请求

- 手动保存、自动保存和退出保存都排队到最近稳定边界。
- 步骤中途发起保存时，当前完整步骤先全部完成或整体不发布。
- 撤离、节点占领、候选组形成、候选同化、地图提交和后续正式飞升属于不可逆边界，提交后请求自动保存。
- 保存延迟不能阻止已提交事件，也不能让同一交易再次执行。

### 11.2 必须保存的时序事实

- 世界运行状态、模拟时间、步骤号、边界事务顺序、事件序号和固定步长版本。
- 有序命令队列、已确认撤离和最早提交步骤。
- 进行中任务、预留、进度、待提交输出与最早生效步骤。
- 状态和修正实例的生效/失效步骤、单位最早生效步骤、行为进度、队列条目、过程任务与待提交输出；稳定快照不保存步骤内事件草案或未提交界面草案。
- 主动战场引用、后台战斗、随机流游标和一次性事件标识。
- L2规定的全部权威事实与累计守恒账目。

### 11.3 载入与重放

- 载入先按L2顺序完成版本迁移、引用与不变量校验，再发布状态。
- 已提交事件不重放为写操作；待处理命令、队列条目、进行中任务、待提交输出、行为进度及单位/状态/修正实例按各自保存的生命周期继续，步骤内事件草案和未提交界面草案不会出现在可载入快照中。
- 待处理命令保持原接收序号和幂等标识。
- 载入不能重新选目标、重建攻击结果、刷新候选或重抽地图。
- 同一快照及后续命令在容器顺序变化后仍必须产生相同事件序列。

### 11.4 不变量失败

若第12阶段发现守恒、重复所有权、负数量、无效引用或重复一次性事件：

1. 不发布本步骤为新的稳定边界。
2. 保留最近已验证快照和导致失败的命令、事件草案与诊断。
3. 禁止通过静默补数、删除资产、改变随机游标或继续模拟掩盖问题。
4. 可以确定性回退到最近已验证快照，并明确报告未提交步骤。

这属于实现阻断错误，不是玩家需要通过玩法恢复的失败状态。

## 12. 拒绝、取消与阻断

| 结果 | 使用条件 | 对权威事实的影响 |
| --- | --- | --- |
| 命令拒绝 | 提交时目标、资源、槽位、战场或资格不合法 | 无变化，不推进随机流 |
| 命令取消 | 已接受但尚未提交，且契约明确允许取消 | 释放命令级预留，不回滚已完成事件 |
| 任务未接纳或零推进 | 缺料使自动任务无法建立，或缺能使已建立任务本步骤无法推进 | 不建立任务，或保留已有任务、进度和合法预留 |
| 任务暂停 | 玩家暂停、结构事务或系统契约声明的显式暂停原因 | 保留任务、进度和合法预留，不请求普通运行功率 |
| 任务堵塞 | 结果已形成但输出容量或槽位无去向 | 保留待提交结果，不重复消费 |
| 任务失败 | 目标永久失效且L3契约声明无法恢复 | 按契约释放或终结预留并形成一次失败事件 |
| 实现异常 | 违反L2不变量或事件顺序 | 不发布步骤，进入诊断与确定性恢复 |

拒绝信息必须指出当前事实，例如“已有主动战场”“目标槽已占用”“候选组已被选择”。不得把拒绝包装成推荐路线或自动替玩家选择其他目标。

## 14. 第一版约束

- 全局主动战场上限为`1`，不存在提高上限的第一版内容。
- 节点页面的战役命令只有`进攻`和`撤离`；科技不增加主动战场技能命令。
- 撤离没有持续时间、成功率、途中受击、容量失败或取消阶段。
- 有主动战场时关闭游戏冻结整个世界，不产生离线收益或损失。
- 第一版可以同化候选，但不能提交正式飞升、新轮次重建或领袖生效命令。
- 时间倍率从开局可用，新玩家与熟悉玩家使用相同模拟规则。
- 已占节点保持安全，不执行敌方战斗阶段。
- 表现实体配额、画质、帧率和页面停留不参与任何阶段。

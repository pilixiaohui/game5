1. 两份L1首次进入精确冻结组成，第一版体验目标、实际制作范围和L3机制不再能够分开解释。

4. 百万级有限代表只读取稳定玩法快照；关键对象、关键行动、接触面、空间覆盖和未分配数量的表现优先级被显式封闭。
5. 旧撤离准备与返程时间参数不属于当前接口，活动参数登记与旧探针夹具分离。
6. 冻结时Q-001至Q-093获得唯一处置状态；D-062随后补充Q-094至Q-105和387个逐节覆盖单元，不改变L3-1.2机制语义。

## 3. 冻结规则

- L4内容只能通过已登记条件、效果、标签、参数和修正接口扩展，不能新增隐式命令、阶段、位置或资源捷径。
- L5只能调用L3命令或只读查询并显示权威事实，不能用页面访问、代表实体和动画形成结算。
- L6以机器清单中的文档、版本和SHA-256为输入；旧编号文档、标题字符串或过期哈希不能覆盖接口。
- 任何冻结文件字节变化都会使当前基线测试失败。文案澄清也必须评估语义、提升受影响版本、刷新人读表和机器清单后再建立新基线。
- 任何机制语义变化必须先登记新决策，分析L1/L2/L4/L5/L6影响，更新受影响验收，再提升基线和对应文档版本。
- 已冻结不等于不可修改；它表示修改必须显式留下版本、原因和证据影响。

## 4. 验证状态

当前完成的是设计输入冻结、机器映射和哈希校验入口。`mapped_not_implemented`仍表示尚无玩法断言；`fixture_only`仍表示没有发布参数；候选性能预算、体验协议和问题处置登记也不能单独证明发布完成。

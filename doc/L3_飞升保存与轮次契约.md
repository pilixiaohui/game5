## 1. 契约职责与所有权

### 1.1 本文负责

- 本地存档槽、跨轮进度和零或一个当前轮次的生命周期。
- 新游戏与后续新轮次的步骤0建立事务。
- 权威快照包含哪些事实、何时写入和如何原子发布。
- 状态、规则、参数、内容和生成版本如何登记与迁移。
- 载入校验、确定性恢复、离线安全结算和主动战场冻结入口。
- 飞升六类贡献、准确只读预览、永久收益和领袖对照快照。
- 后续正式飞升的选择、点数、跨轮保留、当前轮次终结和新世界建立。
- 保存失败、迁移失败和不变量失败时的非破坏恢复。

### 1.2 本文不负责

- 完整步骤内部顺序、暂停和事件草案；由统一时间契约负责。
- 各聚合根内部字段转换与守恒；由对应L3负责。
- 地图、候选和战斗随机结果；本文只保存其随机流与提交事实。
- 永久收益和领袖的具体数值、成本、图标和内容文案。
- 云存档、联机、成就和多设备冲突合并；第一版排除。

### 1.3 唯一写入者

| 权威事实 | 唯一写入边界 | 其他系统允许提供 |
| --- | --- | --- |
| 存档槽与当前轮次引用 | 本文存档和轮次事务 | 各L3提供聚合根快照 |
| 跨轮进度 | 本文正式飞升或明确永久事件 | 图鉴、领袖和长期目标来源事件 |
| 步骤0轮次根对象 | 本文轮次建立协调事务 | 各L3提供原子子事务 |
| 权威快照与版本头 | 本文保存事务 | 时间契约提供稳定边界 |
| 飞升预览 | 本文派生快照 | 资源、世界、单位、进化提供同一步统计 |
| 正式飞升点结算 | 本文正式飞升事务 | L4提供成本与效果定义 |
| 各域运行事实 | 对应L3契约 | 本文只协调、校验和持久化 |

存档列表卡片、继续游戏按钮、预览缓存和离线摘要不拥有第二份存档事实。

## 2. 存档与轮次生命周期

### 2.1 存档槽

每个本地存档槽至少拥有：

- 稳定存档标识、状态版本和创建/最后保存信息。
- 跨轮进度。
- 零或一个当前轮次引用。
- 最近已验证权威快照和可选上一份已验证回退快照。
- 保存事务序号、校验事实和最近载入结果。

第一版至少提供一个可独立新建、保存、退出和载入的槽。

### 2.2 跨轮进度

跨轮进度只包含：

- 图鉴和已发现内容知识。
- 已解锁领袖与当前选择资格。
- 已购买永久收益及等级。
- 明确永久化的科技族、插件或蓝图资格。
- 未消费飞升点余额。
- 历史统计、飞升记录和长期目标。

它不包含本轮地图、资源、单位、房间、候选组和本轮突变。

图鉴和已发现内容知识是跨轮知识记录，不自动等于科技族、蓝图或候选池资格。新轮步骤0只把明确永久化资格投影为初始资格；其余本轮发现记录必须由新轮事件重新形成。

### 2.3 当前轮次生命周期

| 生命周期 | 含义 |
| --- | --- |
| 无当前轮次 | 新存档尚未建立、上一轮已结束或存档明确终止 |
| 建立事务中 | 只存在于未发布工作区，尚不能保存或游玩 |
| 运行中 | 已发布步骤0及后续稳定边界 |
| 飞升选择中 | 后续版本在关键选择暂停中编辑未提交选择草案 |
| 飞升提交中 | 只存在于原子工作区，旧轮与新轮均未被部分发布 |
| 已结束 | 旧轮结果已写入跨轮进度且不再可继续模拟 |

存档稳定快照只保存`无当前轮次`、`运行中`或已经完整提交的`已结束`结果，不保存半个建立或飞升事务。

### 2.4 第一版生命周期限制

第一版只允许：

- 无当前轮次 -> 步骤0建立 -> 运行中。
- 运行中保存、载入、离线安全结算或主动战场冻结。
- 读取飞升预览。

第一版不能进入飞升选择、提交和已结束于正式飞升的状态。

## 3. 步骤0轮次建立事务

### 3.1 输入

轮次建立必须冻结并登记：

- 存档与新轮次标识。
- 状态、规则、参数、内容、生成和飞升公式版本。
- 当前跨轮进度快照。
- 本轮根种子与各确定性子流派生规则。
- 当前领袖或基础中性意志；第一版不激活三位预览领袖效果。
- 新游戏、正式飞升后新轮或明确测试夹具的来源类型。

### 3.2 跨域子事务

按以下依赖构建同一工作状态：

1. 建立当前轮次聚合根、模拟时钟、事件账本和随机流目录。
2. 由世界契约生成世界区域图、完整起始区域和初始情报遮罩。
3. 由房间契约建立初始虫巢、核心巢室、自由空槽、端口和最低容器。
4. 由资源契约执行一次轮次初始注入，建立生物质、基因、储能和基础容量。
5. 由单位契约建立地图初始单位与零或声明的初始玩家单位；第一只工蜂仍按第一版孵化链形成。
6. 由候选契约建立首次适应记录、受保护槽、普通槽容量和永久资格投影。
7. 建立第一版完成里程碑记录、飞升贡献统计和只读目录引用。
8. 运行身份、引用、所有权、位置、守恒、地图、恢复和参数完整性校验。
9. 原子发布步骤`0`稳定边界并请求初始自动保存。

步骤0不执行普通十二阶段，不计入房间产出、外部同化、单位形成、候选或飞升贡献。

### 3.3 失败边界

任一子事务、地图校验或不变量失败时：

- 不发布当前轮次引用。
- 不执行部分初始注入，不消耗正式随机游标。
- 不覆盖原存档的跨轮进度和上一轮快照。
- 保存诊断和可复现输入。

地图安全骨架属于世界子事务的合法结果，不是静默补写非法轮次。

### 3.4 初始事实幂等

轮次标识、初始注入、初始地图、虫巢、首次槽与初始事件各有稳定一次性标识。载入、迁移、重建派生状态和重复新游戏请求不能在同一轮再次建立。

## 4. 权威快照

### 4.1 保存边界

只有统一时间契约定义的稳定边界可以保存：

- 手动、自动和退出保存使用同一事实范围。
- 步骤中请求保存时，当前步骤完整发布或整体丢弃后再保存。
- 暂停边界事务提交后可以立即保存。
- 地图提交、撤离、节点占领、候选形成、候选同化和后续正式飞升请求自动保存。
- 事件草案只存在于步骤工作区，不进入可载入快照。

### 4.2 快照版本头

每份快照至少登记：

- 状态版本、规则版本、参数版本、内容版本和生成版本。
- 飞升公式与永久收益目录版本。
- 存档、轮次、步骤、模拟时间、事件序号和边界事务序号。
- 世界运行状态、关闭现实时间戳和固定步长版本。
- 完整性校验事实和上一个已验证快照引用。

### 4.3 快照事实范围

必须包含：

- 跨轮进度和当前轮次全部聚合根。
- 生物质、基因、能源、结构投入、过程资产和累计账目。
- 世界、区域、节点、虫巢、房间、任务、单位、战役、进化和统计。
- 有序待处理命令、队列条目、过程任务待提交输出、单位行为进度，以及单位、状态和修正实例各自的最早生效/失效步骤。
- 所有随机流、游标、已提交随机结果和一次性事件标识。
- 离线或冻结判定需要的主动战场引用和时间事实。

派生状态与表现缓存不属于必要真相。保存缓存时必须可丢弃重算。

### 4.4 原子保存

保存事务必须满足：

```text
读取单一稳定边界
-> 序列化完整权威事实
-> 校验身份、引用、版本和不变量
-> 写入候选快照
-> 验证候选快照可完整读取
-> 原子提升为当前已验证快照
```

失败不能破坏上一份已验证快照，也不能把半写入结果显示为可继续游戏。

### 4.5 账本压缩

历史事件可以按版本化规则压缩，但必须保留：

- 当前状态所需累计守恒。
- 未终结对象的来源链。
- 一次性与幂等标识。
- 飞升六类贡献和历史审计所需事实。
- 可定位异常区间的摘要。

压缩不能改变飞升预览、恢复路径和确定性重放边界。

## 5. 载入、迁移与恢复

### 5.1 载入顺序

```text
读取候选权威快照
-> 校验快照完整性和版本头
-> 执行必要状态迁移
-> 校验身份、引用、所有权、位置、状态组合和随机流
-> 校验各L3守恒与恢复不变量
-> 重建派生状态
-> 判定在线恢复、离线安全结算或主动战场冻结
-> 原子发布可游玩状态
```

### 5.2 状态迁移

- 每个状态版本必须有明确、确定且可测试的迁移路径。
- 迁移只改变数据表达，不暗中改变规则、参数或随机结果。
- 迁移输入保持不变，只有全部成功后才形成新快照。
- 无法迁移的版本明确拒绝，不静默删除对象、补资源或重抽地图候选。
- 规则和参数版本不因状态迁移自动升级；是否兼容由发布规格明确决定。

### 5.3 不合法快照

遇到损坏、负数量、重复所有权、无效引用、随机游标冲突或守恒失败：

1. 不发布为可游玩状态。
2. 保留诊断和原始快照。
3. 可以尝试上一份已验证快照，并明确报告丢失的已提交区间。
4. 不通过静默修复伪装成功载入。

### 5.4 载入后时间状态

- 关闭时有主动战场：进入`主动战场冻结`，离线经过时间不推进任何系统。
- 关闭时无主动战场：按全局上限进入`离线安全结算`。
- 暂停或关键选择状态关闭：载入保持其稳定状态，不将关闭时间转为模拟进度。
- 离线摘要只汇总实际提交事件。

## 6. 确定性重放

重放输入至少包含：

- 同一权威快照。
- 同一规则、参数、内容和生成版本。
- 同一随机流与游标。
- 同一有序命令和有效模拟时长。

结果必须具有相同事件序列、账目和最终权威状态。页面、帧率、线程、容器顺序、表现配额和保存次数不能改变结果。

重放用于诊断和测试，不把历史事件重新作为写操作提交到当前存档。

## 7. 飞升贡献与只读预览

### 7.1 六类指标

同一步骤统计快照读取：

| 指标 | 来源 |
| --- | --- |
| `M_external` | 本轮累计外部同化物质，不含内部重复回收 |
| `M_nodes` | 已控制节点唯一数量 |
| `M_power` | 满足参数化稳定窗口的最高稳定供能 |
| `M_samples` | 合法已处理样本事件，恢复复制不重复计入 |
| `M_mutations` | 已同化本轮突变，每组最多一次 |
| `M_regions` | 完全吞噬区域唯一数量 |

不存在隐藏第七项。虫巢、Boss或特殊路线可以影响发现和领袖资格，但不暗中改变第一版点数公式。

### 7.2 公式

```text
C_kind = floor(weight_kind * pow(M_kind, exponent_kind))

飞升原始贡献
= C_external + C_nodes + C_power + C_samples + C_mutations + C_regions

预计飞升点
= max(0, floor(飞升原始贡献 / point_scale))
```

权重、指数、换算尺度和精确数值语义由L6参数登记。缺少任一发布参数时，第一版发布配置不能把预览标记为可验收。

### 7.3 预览快照

每次读取必须冻结：

- 账本步骤、六个原始指标与六个分项贡献。
- 原始贡献、预计点数、公式版本与参数版本。
- 六项永久收益的当前等级、下一级成本和具体效果。
- 三位领袖的基准、修正、结果、独有机制、专属池和弱项。
- 保留、清除和第一版不可执行边界。

预览是可丢弃派生快照，不消费点数、不建立选择、不改变当前轮次。

### 7.4 已知增量边界

“继续推进可能增加”只能列出已知来源和公式关系，不预测未揭示地图、不推荐刷取路线，也不伪造下一档保证值。

## 8. 第一版永久收益与领袖目录

### 8.1 永久收益预览身份

第一版固定读取以下内容定义，数值由参数表提供：

| 永久收益 | 作用边界 |
| --- | --- |
| 胚床记忆 | 下一轮首座胚流孵化室建设时间 |
| 代谢火种 | 下一轮初始储能 |
| 回流本能 | 工蜂单次携带容量 |
| 突触余响 | 突触解析时间 |
| 群识扩容 | 普通候选组持有槽整数容量 |
| 韧性外膜 | 玩家可机动基础单位最大耐久 |

预览必须显示当前、购买后、成本、作用对象和叠加边界，不能只显示方向。

### 8.2 领袖预览身份

| 领袖 | 主要强化 | 明确弱项 | 地图生成影响 |
| --- | --- | --- | --- |
| 裂殖母巢 | 建设与孵化 | 基础战斗伤害 | 无 |
| 噬脑主宰 | 解析与候选空间 | 腐殖滤囊形成速度 | 无 |
| 噬界暴君 | 战斗残骸与前线采集 | 稳定功率供给 | 无 |

第一版只显示基准值、修正值和修正后值。三位领袖不实际选择或生效。

### 8.3 目录与参数分离

目录定义作用关系，参数定义具体量。缺失目标对象、参数或作用范围属于发布配置错误，不能显示静态示例冒充当前存档预览。

## 9. 正式飞升选择事务

本节属于长期契约，第一版不得调用。

### 9.1 进入选择

后续版本满足已公开飞升资格且没有主动战场、未提交命令和冲突不可逆事务时，可以进入`关键选择暂停`并建立一个只读飞升来源快照。

玩家编辑未提交选择草案：

- 分配本次获得点数与已有未消费余额。
- 购买或升级合法永久收益。
- 保留当前领袖、选择已解锁领袖，或支付公开成本解锁并选择新领袖。
- 查看完整保留与清除范围。

选择草案不是权威事实，不进入存档和确定性重放输入。取消选择恢复原运行状态，不消费点数、不结束轮次；在该界面关闭程序时只保存最近稳定边界和`关键选择暂停`状态，载入后从同一只读来源快照重新建立空白界面草案。

### 9.2 点数预算

```text
可用飞升点
= 跨轮未消费余额 + 本次结算点数

期末未消费余额
= 可用飞升点 - 永久收益购买 - 领袖解锁或重选公开成本
```

成本不能隐藏，预算不能为负。领袖重选是否收费由L4内容定义和参数声明，但交易结构不变。

### 9.3 确认前验证

- 飞升来源快照仍对应当前冻结轮次和参数版本。
- 所有购买、升级、前置、领袖和点数合法。
- 当前存档可以按当前生成版本建立下一轮安全世界。
- 保留和清除清单覆盖每个当前轮次对象类型。
- 选择不会删除通用基础恢复能力。

任一失败时不修改旧轮、跨轮进度或点数。

## 10. 正式飞升原子提交

本节属于长期契约，第一版不得调用。

### 10.1 提交顺序

```text
冻结并再次校验旧轮最终统计
-> 结算本次飞升点
-> 原子更新永久收益、领袖解锁/选择、余额、图鉴和历史记录
-> 将旧轮标记为已结束
-> 派生全新轮次标识和世界根种子
-> 执行第3节完整步骤0轮次建立事务
-> 校验新轮全部不变量
-> 同笔发布跨轮进度与新轮步骤0稳定边界
-> 写入不可逆飞升记录与自动保存
```

### 10.2 跨旧轮与新轮的原子性

新地图生成、初始注入或任一子事务失败时，旧轮保持运行中，点数、永久收益和领袖选择均不提交。不存在“旧轮已删除但新轮未建立”的合法存档。

### 10.3 新世界独立性

- 新轮使用新根种子与独立随机流。
- 不保留旧区域连接、节点内容、控制、情报和具体来源位置。
- 图鉴只提供内容知识，不泄露新世界组合。
- 领袖和永久收益只改变玩家能力、初始事实和候选池，不暗中修改地图生成权重。

## 11. 飞升保留与重置

### 11.1 保留

- 图鉴和内容知识。
- 已解锁领袖与确认选择。
- 已购买永久收益。
- 明确永久化科技族、插件和蓝图资格。
- 未消费飞升点余额。
- 历史统计、飞升记录和长期目标。

### 11.2 重置

- 本轮世界种子、区域图、节点、情报、控制和持续战局。
- 虫巢、槽位拓扑、房间、任务、结构投入和本地容器。
- 生物质、基因、储能、过程资产和本轮统计账期。
- 幼体、虫群大军、敌我单位、固定菌毯、残骸和样本。
- 候选组、本轮突变、本轮发现记录及其来源事件、未永久化资格。
- 主动战场、命令、离线状态和本轮随机流。

任何未列明对象类型在进入正式飞升实现前必须被加入保留或重置清单，不能使用默认猜测。

### 11.3 永久收益初始求值

永久收益只在声明关系和步骤生效：

- 初始注入型收益在步骤0经济子事务求值。
- 初始资格型收益在步骤0进化子事务求值。
- 模板或房间修正在目标实例生效边界求值。
- 不追溯增加旧轮成果，也不重复执行一次性初始事件。

## 12. 十二阶段与稳定边界接入

| 时点 | 本文职责 |
| --- | --- |
| 步骤0 | 协调所有聚合根建立并原子发布初始稳定边界 |
| 稳定边界 | 接收保存、载入入口、关键选择和后续飞升命令 |
| 第1阶段 | 第一版只处理保存调度；后续处理飞升选择边界，不在运行步骤中半重置 |

正式飞升在关键选择暂停的独立边界事务中提交，不能塞入普通步骤中间。

## 13. 生命周期请求、玩法命令、服务请求与查询

### 13.1 存档生命周期请求

| 请求 | 第一版 | 验证与结果 |

| 创建存档并新建游戏 | 支持 | 目标槽合法且无当前轮次时建立步骤0；失败不产生半槽或覆盖旧槽 |

| 删除存档 | 支持产品确认 | 明确确认后删除槽文件与产品索引；不提交当前轮次玩法事件 |

这些请求由存档槽服务在当前轮次之外排序和审计，不进入玩法命令队列，不使用玩法事件重放，也不占用模拟步骤号。创建请求只有在步骤0全部子事务通过后才发布一个新当前轮次。

### 13.2 当前轮次玩法命令与边界服务请求

| 输入 | 分类 | 第一版 | 验证与结果 |
| --- | --- | --- | --- |
| 请求手动保存 | 边界服务请求 | 支持 | 排队到当前或下一稳定边界，不改变玩法事实 |
| 进入/取消飞升选择 | 玩法命令 | 拒绝 | 后续版本进入关键选择暂停或无损取消 |
| 确认正式飞升 | 玩法命令 | 拒绝 | 后续版本执行第10节原子事务 |

第一版拒绝正式飞升必须返回`尚未开放`，不能消费资源、创建新种子或改变当前轮次。保存服务请求只产生保存调度与存储审计，不产生玩法事件。

### 13.3 只读查询

| 只读查询 | 输入边界 | 结果 | 禁止变化 |
| --- | --- | --- | --- |
| 读取飞升预览 | 最近已发布稳定边界及指定公式/参数版本 | 形成或复用可丢弃预览快照 | 不创建命令、玩法事件、选择草案，不消费点数或改变轮次 |

打开、关闭和重复读取飞升预览均不进入命令队列。为诊断形成的查询审计记录位于独立非权威日志，可以删除，不进入权威快照、玩法事件流或确定性重放输入。

## 14. 玩法事件与非玩法审计

### 14.1 当前轮次玩法事件

| 事件 | 提交边界 | 关键事实 |
| --- | --- | --- |
| 轮次已建立 | 步骤0 | 全部聚合根和初始来源一次发布 |
| 正式飞升已提交 | 后续暂停边界 | 旧轮、新轮、点数、选择和保留清单 |

正式飞升事件必须有跨旧轮和新轮的唯一幂等标识。离线结算期间产生的资源、任务、单位和世界变化仍使用各自普通玩法事件，不压缩成一个可重复执行的“离线收益事件”。

### 14.2 存档生命周期与存储审计

| 审计记录 | 边界 | 关键事实 |
| --- | --- | --- |
| 存档槽创建/覆盖/删除结果 | 生命周期请求 | 请求身份、目标槽、确认、旧新索引和结果 |
| 权威快照写入并验证 | 稳定边界与存储完成 | 快照身份、步骤、版本、校验和写入结果 |
| 存档迁移完成/失败 | 载入事务 | 旧新版本、迁移路径、原快照和结果 |
| 离线结算会话完成 | 载入后 | 现实经过、有效时长和已提交普通事件摘要 |

这些审计记录不能作为玩法资源、进度或重放写操作。清理审计日志不改变已验证快照中的当前轮次事实；执行已确认的`删除存档`生命周期请求则会按请求删除目标槽及其快照。

### 14.3 只读查询审计

飞升预览查询审计只记录查询身份、来源稳定步骤、公式/参数版本、结果摘要和诊断时间。它不是事件，没有事件排序键，不触发自动保存、点数结算、里程碑或后续效果，也不进入确定性重放。

## 15. 发布配置与参数身份

### 15.1 发布完整性

第一版发布配置至少需要：

- 所有必需状态、规则、内容、生成和参数版本。
- 六类飞升贡献权重、指数和换算尺度。
- 六项永久收益的预览等级、成本和具体效果参数。
- 三位领袖全部基准、修正、结果和适用标签参数。
- 固定步长、离线上限、保存间隔和数量精度版本。

任一必需发布值为空或状态不是已验证时，发布配置不能通过第一版完成验收。

### 15.2 待平衡参数族

| 参数族 | 需要登记 | 必须验证 |
| --- | --- | --- |
| 保存 | 自动保存间隔、回退保留和账本压缩 | 性能、最大回退和诊断能力 |
| 离线 | 有效上限与摘要分组 | 容量、堵塞、恢复与公平性 |
| 飞升公式 | 六类权重、指数和点数尺度 | 首次意义、深度收益与无单项支配 |
| 永久收益 | 等级、成本、效果和递增 | 压缩旧流程但保留核心体验 |
| 领袖 | 特性、弱项、专属池与选择成本 | 差异、非唯一最优和可比较性 |
| 世界重建 | 新种子和生成版本兼容 | 新世界差异、合法性与原子建立 |

## 16. 第一版范围与排除

第一版必须实际支持：

- 新游戏和步骤0完整建立。
- 至少一个本地存档槽、自动保存、手动保存、退出和继续游戏。
- 状态迁移框架、非法快照拒绝和上一验证快照恢复边界。
- 无主动战场离线安全结算和有主动战场全世界冻结。
- 准确飞升点、六项永久收益与三位领袖只读预览。
- 明确显示正式飞升、领袖选择和新世界重建尚未开放。

第一版明确排除：

- 正式飞升确认。
- 飞升点消费和永久收益购买。
- 领袖解锁、选择和实际生效。
- 结束当前轮次和建立飞升后新世界。
- 云存档、联机同步和存档合并。

## 18. 不变量责任与完成条件

### 18.1 不变量责任

本文直接维护：

- INV-001至INV-003：存档、轮次、快照和预览身份稳定，所有权唯一且缓存不成第二真相。
- INV-012：跨轮发现、资格、实例、强化和永久化分离。
- INV-019：地图、候选和随机结果保存后不重抽。
- INV-020、INV-021：保存、载入、重放、离线和表现不改变相同模拟结果。
- INV-022、INV-023：步骤0、保存、迁移和正式飞升原子且幂等。
- INV-024：非法快照不发布，当前轮失败仍保留合法恢复或上一验证边界。

步骤0、保存和飞升边界协调检查INV-004至INV-018，但各守恒和状态算法仍由其直接责任L3维护。

### 18.2 完成条件

- 存档槽、跨轮进度、当前轮次、步骤0和权威快照各有唯一职责。

- 飞升预览使用同一步真实账本和发布参数，不能改变玩法。
- 第一版不能执行任何正式飞升、点数消费或领袖生效。
- 后续正式飞升若启用，旧轮结算、跨轮选择和新轮建立必须跨域原子提交。

## 1. 契约职责与写入边界

### 1.1 本文负责

- 虫巢、自由空槽、房间模块和固定基础设施分别保存哪些事实。
- 槽位相邻、模块兼容与房间群派生的唯一规则。
- 建设、升级、移动、功能重组、降级、拆除和替换如何建立并完成结构事务。
- 房间运行策略、队列条目、过程任务和自动循环之间的区别。
- 模块输入缓冲、输出缓冲、端口、容量与工蜂交互的接入边界。
- 采集交互预留中属于虫巢的路径、取货端、投递端和安全交接容量如何预留、提交与释放。
- 暂停、缺料、缺能、堵塞、运行和待机如何从真实事实派生。
- 第一版六个可建设房间如何接入通用模型。
- 多虫巢完整模拟、批量命令和生产方案不能绕过的校验边界。

### 1.2 本文不负责

- 生物质、基因、结构投入、过程资产和能源的守恒及功率分配算法；它们由`L3_资源与能源契约.md`负责。
- 固定步骤、命令总排序、事件顺序、离线和保存时点；它们由`L3_统一时间命令与事件契约.md`负责。
- 工蜂目标选择、搬运、单位形成后的真实位置、伤害或战斗行为。
- 房间蓝图的正式配方、成本、周期、产率、容量、升级分支和插件内容。
- 页面布局、动画频率、画面工蜂数量、声音和镜头表现。
- 候选组内部生成与同化、地图来源开放和正式飞升交易。

### 1.3 唯一写入者

| 事实 | 唯一写入边界 | 其他系统允许请求的内容 |

## 2. 虫巢与房间权威模型

### 2.1 虫巢实例

每个虫巢至少保存：

- 稳定虫巢实例标识、所属区域和虫巢节点引用。
- 激活来源事件、创建步骤和当前生命周期事实。
- 槽位定义、有效连接边和单调递增的拓扑版本。
- 房间模块、结构事务、过程任务和运行策略记录。
- 本地输入、输出、幼体、特殊产物和安全交接容器引用。
- 账本接入口、网络接入口、路径锚点和固定基础设施引用。
- 当前生效的区域、领袖、突变、插件和房间修正引用。

虫巢不拥有第二份核心资源、过程资产或单位库存。幼体和过程资产仍分别由单位总账与过程资产总账拥有，只把唯一位置或容器引用指向虫巢。

### 2.2 核心巢室

第一版核心巢室是虫巢固定基础设施，不是可建设房间模块：

- 不占自由空槽，不参加房间群，不使用普通生产队列。
- 提供账本接入口、最低本地容器和虫巢拓扑锚点。
- 只通过资源契约调用第一能源房间建设保障和第一只工蜂应急直连。
- 不形成常规生物质、基因、功率供给或免费单位。
- 第一版不能移动、替换、降级、拆除或安装普通房间插件。

后续内容若要改变核心巢室，必须建立独立的虫巢基础设施事务，不能把恢复保障伪装成普通房间产出。

### 2.3 槽位实例与排他声明

每个槽位至少记录：

- 稳定槽位标识、所属虫巢和拓扑位置。
- 尺寸、形状、接入类型及其他结构兼容标签。
- 与其他槽位或基础设施之间的有效连接边。
- 当前房间模块引用，或当前结构事务的排他声明引用。

第一版一个房间模块恰好占用一个槽位。较大的可见房间由多个模块组成房间群，不使用一个隐藏多槽实例。以后若加入多槽模块，必须先扩展占用原子性与拓扑验收，不能仅由表现跨越多个槽位。

`自由空槽`表示当前既没有模块，也没有任何结构事务排他声明。建设或移动目标在命令提交时就被声明，不能等完成时再与其他任务争抢。

### 2.4 房间模块实例

每个房间模块至少保存：

- 稳定模块实例标识、蓝图定义标识、所属虫巢和当前槽位引用。
- 结构状态、形成事件、结构投入分项、等级、功能分支和运行绑定。
- 插件引用、显式修正引用和最早生效步骤。
- 玩家暂停事实、运行策略、队列条目和当前任务引用。
- 输入缓冲、输出缓冲、单位容器和网络端口引用。
- 持续网络贡献定义引用、启动或冷启动事实、环境请求与最早生效步骤。
- 当前结构事务、阻断事实、等待事实和最近有效运行步骤。

房间模块是最小运行实例。等级不同、插件不同或临时状态不同的模块仍然保持各自账目，不能因画面合并而丢失或平均化。

## 3. 槽位拓扑、兼容键与房间群

### 3.1 唯一相邻规则

两个槽位只有在同一虫巢的槽位拓扑中存在有效连接边时才相邻：

- 屏幕距离、斜角接触、动画边界和场景树父子关系不构成相邻。
- 拓扑边只能由已提交的虫巢建立、扩建或结构事件改变。
- 每次拓扑变化递增拓扑版本，并在第12阶段重新构建派生关系。
- 基础相邻只影响房间群、路径、端口和显式声明的修正，不提供隐藏产率加成。

科技、插件或领袖可以在既有边上建立公开邻接修正，但必须声明来源、目标、作用边、生效步骤、叠加规则和断开结果。

### 3.2 群兼容键

每个已形成模块按内容定义生成一个`群兼容键`：

```text
群兼容键
= 房间族定义标识
 + 改变主要输入输出的功能分支
 + 内容明确声明影响共享任务资格的运行绑定
```

- 蓝图显示名称不能作为兼容键。
- 等级、插件、玩家暂停、缺料、缺能、堵塞、任务进度和临时效果不进入兼容键。
- 运行绑定只有在它改变主要输入、主要输出或任务资格时才进入兼容键。第一版孵化模块的最终单位模板属于此类绑定。
- 两个内容若不能安全共享待执行任务，就必须产生不同兼容键，不能依靠领取任务时临时猜测。

### 3.3 房间群派生

同一虫巢中，同时满足以下条件的模块按相邻连通分量形成一个房间群：

1. 结构已经形成；`建设中`模块尚未加入。
2. 当前占据槽位之间通过有效边相邻连通。
3. 群兼容键完全相同。

`可运行`、`重组中`和`拆解中`模块在结构终结前仍参与空间连通，但只有`可运行`模块具有运行资格。房间群没有两格或三格的机制上限。

房间群标识由`虫巢标识 + 拓扑版本 + 兼容键 + 排序后的成员模块标识`确定性派生。它只用于当前稳定边界的显示、查询和命令快照，不能跨拓扑变化被当作永久实例引用。

### 3.4 房间群不能拥有第二份事实

房间群只允许派生：

- 成员模块集合、连通形状和有效端口集合。
- 输入、输出、容量、供给、负载、吞吐和状态的汇总。
- 成员队列条目与过程任务组成的共享管理视图。
- 同一运行字段是统一值还是`混合配置`的摘要。

房间群不能拥有第二份模块、资源、过程资产、结构投入、任务、冷却、插件、缓冲或产物。保存文件可以缓存房间群用于加速载入，但必须从权威模块重算并校验；缓存不一致时不得反向覆盖模块。

## 4. 模块结构状态与运行资格

### 4.1 正交维度

房间不能用一个总枚举覆盖全部情况。至少分开记录或派生：

| 维度 | 权威或派生事实 |
| --- | --- |
| 结构状态 | `建设中 / 可运行 / 重组中 / 拆解中`之一 |
| 玩家控制 | 玩家暂停事实、循环是否开启、运行策略版本 |
| 过程任务 | `已预留 / 推进中 / 暂停 / 待提交 / 已完成 / 已取消 / 已失败` |
| 输入条件 | 各兼容输入是否足够、是否已经唯一预留 |
| 能源条件 | 请求功率、最低服务功率和本步骤已服务功率 |
| 输出条件 | 输出位置、容量、网络端和目标承诺是否仍合法 |
| 结构事务 | 无，或一个当前排他的结构事务引用 |
| 派生运行摘要 | 运行、待机、缺料、缺能、堵塞及其全部原因集合 |

`全局暂停`冻结世界时间，但不把每个任务改写成`暂停`。`玩家暂停`或结构事务导致的任务暂停才保存任务暂停原因。

### 4.2 结构状态转换

```text
新建：建设中 -> 可运行
升级、移动、功能重组、降级：可运行 -> 重组中 -> 可运行
拆除：可运行 -> 重组中（需要清空时） -> 拆解中 -> 终结
替换：旧模块可运行 -> 重组中 -> 拆解中 -> 旧模块终结 + 新模块建设中 -> 新模块可运行
```

- 取消尚未提交的建设会终结临时模块，不形成可运行模块。
- 取消尚未完成的重组或拆解会恢复原结构；已经合法转移出去的资产保持现位置，不会瞬移回房间。
- 结构状态改变不自动清除玩家暂停、队列策略、任务进度或插件；不兼容事实必须通过明确迁移或拒绝处理。
- 结构完成事件在第11阶段提交，第12阶段把新结构纳入下一稳定边界的房间群投影；供给、容量和运行资格最早从下一完整步骤生效。

### 4.3 运行资格

一个模块只有同时满足以下条件时才能领取新过程任务：

- 结构状态为`可运行`且已经到达最早生效步骤。
- 没有排他的结构事务处于清空或结构推进阶段。
- 未被玩家暂停，循环或单次请求仍允许建立任务。
- 当前没有占用该模块执行位的非终结过程任务。
- 运行绑定、配方、输入、输出承诺和当前生态资格合法。

供能不足不影响任务建立所需的资源与容量校验，但会按资源契约使任务本步骤零进度或比例推进。第一版每个模块只有一个过程任务执行位；百万级产出使用批次数量，不靠增加同一模块的隐藏并发槽。

## 5. 通用结构事务

### 5.1 结构事务最小记录

建设、升级、移动、功能重组、降级、拆除和替换都必须建立结构事务。每笔至少记录：

- 事务、来源命令、目标虫巢、源模块、源槽位和目标槽位标识。
- 操作类型、目标蓝图、等级、分支、绑定和预期群兼容键。
- 结构资源预留、当前结构投入、预计返还及各自独立事件引用。
- 临时功率请求、进度、最早生效步骤和完成幂等标识。
- 保留清单：运行策略、队列条目、过程任务、插件、缓冲、端口和修正。
- 清空清单：必须完成、取消、解除预留或转移的本地事实。
- 目标容量、端口、储能上限及其他安全提交检查。
- 允许取消的最后边界和取消后的唯一去向。

预计返还只是预览，不能作为新结构预留的可用资源。结构事务没有隐藏全局队列；多个模块事务可以并行，并共同参与真实功率竞争。

### 5.2 通用阶段

```text
命令验证
-> 原子建立资源、槽位和目标容量预留
-> 必要时停止新任务并等待清空
-> 模块进入重组中或拆解中
-> 按已服务功率推进结构任务
-> 达到完成条件后进入待提交
-> 第11阶段再次校验并原子提交
-> 第12阶段重建拓扑派生与房间群
```

`等待清空`不是第五种结构状态。原模块可以暂时保持`可运行`，但禁止领取新任务，只允许完成兼容任务和通过普通工蜂交互移出物理资产。没有安全去向时事务保持可观察堵塞，玩家可以建立容量、等待处理或取消事务；系统不能静默删除资产。

### 5.3 建设

建设命令必须原子完成：

1. 检查虫巢已激活、蓝图具有当前内容池资格、槽位兼容且为自由空槽。
2. 检查初始运行绑定合法；只有玩家明确选择的高级分支或绑定才检查其公开的当前生态条件。
3. 完整预留结构资源并声明目标槽位。
4. 建立`建设中`临时模块和建设任务；临时模块没有有效结构投入和运行资格。
5. 第3阶段按已服务功率推进。
6. 第11阶段消费结构预留、形成模块结构投入并转为`可运行`。

取消未完成建设会完整释放未消费结构预留、释放槽位并终结临时模块。已经结算的时间、功率和一次性能源效果不返还。建设完成前不形成正式输出、不提供供给或容量，也不加入房间群。

第一版房间建设与结构升级使用生物质形成结构投入，并在推进期间请求能源功率；能源不是第二笔预扣库存，基因不作为普通结构材料。具体生物质成本、请求功率和周期均由L6测试确定。

基础蓝图不能把槽位功能或当前环境输入作为隐藏建设限制。缺少外部来源或运行输入时，房间仍可建成并派生缺料或待机；当前生态只限制已公开的高级能力执行资格。

### 5.4 升级

- 升级预留新增等级所需的完整结构资源，不重扣已有结构投入。
- 模块进入`重组中`后停止正式运行；当前任务、进度、输入预留、缓冲、插件和策略保持。
- 完成时新增预留转为分项结构投入，等级和容量从下一步骤生效。
- 取消时释放未消费的升级预留并恢复原等级与运行资格。
- 若升级同时改变主要输入输出，应改用功能重组或替换，不能借“升级”绕过清空检查。

### 5.5 移动

- 移动不预留或返还结构材料，但必须排他声明目标槽位。
- 结构推进期间源槽位继续被原模块占用，目标槽位同时被事务声明。
- 模块标识、结构投入、等级、插件、运行策略、队列、任务进度和本地缓冲保持。
- 完成时同笔释放源槽位、占用目标槽位并更新模块位置；取消则释放目标槽位并恢复源位置。
- 移动只限同一虫巢。模块及其缓冲随同一实例改变位置，不等于把过程资产转入另一虫巢。

### 5.6 功能重组与运行绑定

改变功能分支、孵化模板或其他会改变群兼容键的绑定属于功能重组：

- 事务开始后停止领取使用旧绑定的新任务。
- 已开始任务只有在新旧绑定都明确兼容时才保留；否则必须先完成、合法取消或保持事务等待清空。
- 不兼容插件、输入、输出、候选缓冲和单位容器必须存在合法去向。
- 完成时同一模块切换绑定并重算群兼容键，不刷新冷却、一次性效果或历史产出。
- 第一版功能重组只消耗真实时间和功率，不额外扣除结构材料。若目标能力需要新增结构投入，必须使用升级或替换事务，不能把材料收费藏在绑定切换中。

### 5.7 降级

- 降级在完成前保留全部原结构投入和原容量。
- 被移除等级对应的可回收结构投入只在第11阶段完成事件中返还一次。
- 低等级仍兼容的插件、运行策略、队列和任务可以保留。
- 超出新容量或失去端口资格的资产、待提交输出和预留必须先获得安全去向。
- 当前储能高于降级后的全局上限时，最终提交保持堵塞，不允许自愿操作产生静默溢散。

### 5.8 拆除

- 拆除先停止新任务并等待非结构任务、插件和本地资产完成清空。
- 模块进入`拆解中`后不再运行普通配方，但结构投入仍然有效存在。
- 完成时模块终结、槽位释放和全部可回收结构投入返还必须同笔提交。
- 历史运行输入、功率、产物和已消费配方成本不返还。
- 战斗破坏、腐化或事件损毁不属于玩家拆除，必须由风险系统形成明确永久损失。

### 5.9 替换

替换不是即时修改蓝图，而是一笔包含拆解与新建两个提交边界的组合结构事务：

1. 校验旧模块清空清单和新蓝图全部资格。
2. 排他占用原槽位并推进旧结构拆解；此时旧结构投入仍然存在，预计返还不能被任何任务使用。
3. 拆解达到待提交后，按事件排序重新检查：`当前可用量 + 本次将提交的旧结构返还`能否完整覆盖新结构成本。
4. 条件不足时旧模块保持`拆解中 / 待提交`，不终结、不返还，也不建立部分新预留。
5. 条件满足时，第11阶段同笔先终结旧模块并提交旧结构返还，再从提交后的可用量完整建立新结构预留，创建`建设中`新模块并把槽位声明转给新建阶段。
6. 新建阶段继续请求真实功率和时间，最终按普通建设规则消费新预留并使新模块转为`可运行`。

新模块使用新的实例标识，并通过父替换事务引用旧模块。只有明确兼容的运行策略、插件和未开始队列条目可以迁移；已开始且配方不兼容的过程任务不能改名后继续。

- 旧结构终结前取消替换会恢复旧模块，不产生结构材料变化。
- 旧结构已返还且新结构仍在建设时取消，会按普通建设取消规则释放新预留并留下自由空槽；旧模块不会无来源复原，但全部可回收结构材料仍在账目中。
- 两个阶段已经结算的功率、时间和物理资产转移都不回滚。

### 5.10 结构事务冲突

- 一个模块任意时刻最多拥有一个非终结结构事务。
- 一个槽位任意时刻最多被一个模块或结构事务排他声明。
- 同一边界冲突按L3命令接收顺序处理，先提交者生效，后续命令基于新事实拒绝。
- 结构任务已经进入`待提交`但最终安全条件失效时保持待提交，不重复推进或消耗功率。
- 只有终结前且仍在声明取消边界内的事务可以取消；完成事件提交后，反向操作必须建立新的结构事务。

## 6. 房间群管理与拓扑变化

### 6.1 群命令快照

房间群命令必须携带：

- 虫巢标识、拓扑版本、群兼容键和排序后的成员模块标识。
- 要改变的字段及旧策略版本。
- 是否为全体原子批量操作。

提交时重新派生房间群。若拓扑版本、成员集合或兼容键已经变化，命令以目标快照过期拒绝，不能把操作悄悄应用到新出现或已经离开的模块。

### 6.2 合并与拆分

房间群合并和拆分是拓扑变化后的派生结果，不存在玩家直接支付的“合成命令”：

- 合并不复制模块、任务、缓冲、冷却、插件、结构投入或一次性效果。
- 拆分时已开始任务留在当前执行模块，不能重新分配进度。
- 未开始队列条目保留原始模块引用，只能由拆分后同一连通分量中的兼容模块领取。
- 合并后两个原群的未开始条目进入同一派生队列视图，并按稳定顺序竞争空闲模块。
- 端口和缓冲仍属于具体模块；群总容量只是求和摘要，不能让资产无事件地换到另一模块。

### 6.3 混合配置

运行策略保存在模块上。两个配置不同的房间群因移动而合并时：

- 不自动选择任一侧为胜者，也不改写已有模块。
- 群摘要把不同字段显示为`混合配置`，仍按各模块现有策略运行。
- 玩家下一次对全群设置某字段时，命令原子把该字段写入当前全部成员。
- 拆分后每个模块继续保留自己最后提交的策略。

建设新模块若将加入配置一致的相邻房间群，可以在建设命令中复制该群策略；若相邻目标是混合配置，必须显式选择初始策略，不能使用容器中的第一个模块。

### 6.4 批量结构操作

第一版登记一个玩法命令`提交批量结构事务`，只包含建设、取消建设、移动、功能重组、拆除、替换、降级和升级等普通结构子命令。群级暂停、恢复、循环、目标和绑定继续使用房间运行命令，不混入结构批次。

批量命令至少保存批次命令标识、玩家确认的有序子命令、共同目标拓扑版本、公开参数、接收边界和幂等标识。它在隔离事务工作区中展开并验证：

- 按子命令登记顺序和稳定子命令标识求值联合槽位声明、最终拓扑、全部资源预留和冲突。
- 玩家确认的批次必须先验证全部目标与全部预留；任一目标不合法则整批拒绝，正式状态保持不变。
- 成功后可以创建多个独立过程任务，它们照常竞争功率和容量。
- 不提供隐式“尽可能执行一部分”模式；若后续需要，必须作为不同且明确的命令类型设计。
- 批量操作不能改变单模块结构投入、任务身份、取消边界或完成顺序。
- 相同批次幂等重试只能返回同一接受、拒绝或已提交结果。

后续生产方案只能先编译成可丢弃预览，再由玩家确认生成普通结构与策略命令。第一版尝试`应用生产方案`必须明确拒绝，不能借用批量结构命令复制模块、投入、任务或资产。

## 7. 运行策略、队列条目、过程任务与持续网络贡献

### 7.1 四类对象不能混同

| 对象 | 含义 | 是否锁定输入 |
| --- | --- | --- |
| 运行策略 | 模块希望持续采用的配方、模式、目标与开关 | 否 |
| 队列条目 | 尚未分配执行模块的一次生产意图或显式批次 | 否 |
| 过程任务 | 已分配模块并原子预留输入、目标和容量的运行实例 | 是 |
| 持续网络贡献 | 在合法环境容量下持续形成供给或储能上限的网络能力 | 只预留本步骤环境容量，不锁定物质输入 |

队列条目不足以改变核心资源或过程资产。只有任务建立成功后才形成预留、功率请求和输出承诺。常规生产完成后自动提交，不产生“等待玩家领取”的任务状态。

持续网络贡献不是隐藏循环任务。它不具有生产批次、完成输出或领取状态，也不占过程任务执行位；每个步骤由资源契约根据模块资格、环境容量和暂停事实确定实际贡献。需要持续消耗物质的能力必须使用离散过程任务，不能借持续贡献绕过输入守恒。

一个显式单次条目至少记录条目标识、原始模块、配方、精确提交总量、未接纳余量、各任务承诺量、已完成量和已终结未完成量，并始终满足：

```text
提交总量
= 未接纳余量
 + 当前任务承诺量
 + 已完成量
 + 已终结未完成量
```

队列条目只保存数量归因，不拥有任务输入或输出。

### 7.2 第一版循环模式

| 模式 | 建立任务的条件 | 停止条件 |
| --- | --- | --- |
| 持续处理 | 配方、输入、输出承诺和执行模块均合法 | 循环关闭、玩家暂停或任一条件暂不成立 |
| 维持目标 | 声明目标指标低于目标值且存在正缺口 | 目标覆盖量达到目标值 |
| 处理全部 | 存在尚未被过程任务锁定的兼容输入 | 当前没有兼容未锁定输入 |
| 单次批次 | 玩家已提交一个明确配方和数量的条目 | 该条目完成、合法取消或失败后不再自动建立 |

`停止循环`只阻止建立新自动任务，已经建立的任务继续。`玩家暂停`同时阻止新任务并暂停可中断的当前过程任务。改变模式不追溯改变已经消费的输入或已经形成的输出。

玩家可以提交单次批次队列条目，也可以移除其中尚未被任务接纳的余量。未接纳余量没有锁定输入，因此不触发资源返还；已经形成的任务分片必须改按任务消费与取消边界处理。

### 7.3 任务接纳与分配

每个完整步骤第3阶段开始时，按阶段开始快照执行一次接纳：

1. 排除结构不可运行、玩家暂停、执行位被占或绑定无效的模块。
2. 从成员模块的队列条目和运行策略形成本步骤候选请求；显式条目可按配方批次上限和兼容空闲模块确定性拆成多个正数量分片，分片总量不得超过未接纳余量。
3. 按`显式单次条目优先 -> 条目形成步骤 -> 命令接收序号 -> 原始模块标识 -> 条目标识`稳定排序。
4. 对每个候选，选择`连续空闲步骤最长 -> 最近领取步骤最早 -> 模块稳定标识`最靠前的兼容模块。
5. 原子预留全部核心资源、过程输入、单位输入、目标份额和输出容量；任一项不足则不创建任务。
6. 建立过程任务、分配执行模块并登记本步骤功率请求，随后参与资源契约的能源服务。

每个空闲模块本步骤至多接纳一个分片；接纳时同笔减少条目未接纳余量并增加任务承诺量。一个模块在本步骤完成任务后不能在同一第3阶段再次接纳新任务。第11阶段新输出最早在下一完整步骤接纳中使用，避免零周期递归和同一步无限生产。

### 7.4 过程任务最小记录

每个过程任务至少保存：

- 稳定任务标识、任务类型、配方、参数版本和来源队列条目。
- 所属虫巢、执行模块和需要的运行绑定。
- 核心资源预留、过程输入锁、单位输入锁和目标覆盖承诺。
- 输出配方、物理缓冲、网络端、单位容器或候选来源缓冲承诺。
- 消费模式、功率服务模式、安全层级、请求功率和最低服务功率。
- 当前进度、任务状态、暂停原因、阻断原因和连续等待事实。
- 完成草案、提交阶段、最早生效步骤和幂等标识。

任务进度必须能在批次合并、时间倍率和保存载入后精确重算。表现动画帧和可见器官动作不进入任务记录。

### 7.5 任务生命周期

```text
已预留 -> 推进中 <-> 暂停 -> 待提交 -> 已完成
             \-> 已取消
             \-> 已失败
```

- `已预留`已经锁定全部开始输入，但尚未获得第一次合法推进。
- `推进中`可以因本步骤缺能而得到零进度；缺能本身不把任务改成暂停或失败。
- `暂停`只用于玩家暂停、结构事务或配方声明的显式暂停原因，并保留进度和未消费预留。
- 达到完成条件后任务进入`待提交`，停止请求运行功率并保存唯一输出承诺。
- 输出位置暂不可用时保持`待提交`，不重复消耗、抽取随机、推进或形成结果。
- `已失败`只用于目标永久失效且对应契约声明无法等待或恢复的情况；临时缺料、缺能、路径中断和容量不足不是失败。
- 完成、取消和失败都是终结状态，同一任务不能再次形成输出。

单次条目的任务完成时，同笔把对应任务承诺量转入已完成量；取消或失败时转入已终结未完成量，不自动回到未接纳余量。重试必须形成新的玩家条目，不能由载入或自动循环暗中补回。

已开始任务是否允许取消由配方的消费边界决定。取消只释放未消费预留；已经消费的输入、经过的时间和已服务功率不返还。第一版必须允许玩家取消尚未越过不可逆消费边界的当前批次，以解除能源停摆时的资源锁定，但不提供逐项拆分输入或回滚部分进度的微操；结构清空调用同一取消边界。取消任务不自动关闭持续循环或维持目标，条件仍成立时下一步骤可以重新建立任务，玩家需要停止对应循环才能持续释放该负载。

### 7.6 维持目标与防超产

每个`维持目标`策略必须声明一个规范目标指标和作用范围，例如某模板的全局可机动存活量。不能只写模糊“兵力”。

```text
目标覆盖量
= 当前符合指标的真实数量
 + 已预留任务承诺形成且尚未提交的数量

可建立数量
= max(0, 目标值 - 目标覆盖量)
```

- 同一步多个模块按任务接纳顺序原子建立目标覆盖承诺，后续模块读取更新后的缺口。
- 尚未建立任务的无限循环意图不计入覆盖量。
- 目标值降低不会删除现有单位、资产或已完成输出，也不追溯取消已经消费输入的任务。
- 战损或其他合法终结使真实数量下降后，下一步骤可以重新出现缺口。
- 目标指标、范围和是否计入特定位置由L4内容登记，不由页面当前筛选决定。

### 7.7 批次语义

- 一个任务可以处理任意正数量的同质批次，不为每个单位或每份物质建立独立任务。
- 一个显式条目可以被多个兼容模块并行接纳，但每个数量份额只能进入一个任务分片。
- 条目拆成多个过程任务属于第7.3节的可观察调度，会按真实模块分别请求功率和推进；单个过程任务内部的批次表示拆分或合并才是执行优化，且不得改变输入、进度、功率、输出、事件顺序和账目总量。
- 只有配方、来源谱系、任务状态、修正集合、输出去向和消费边界完全相同时才能规范合并。
- 不同来源需要分别追溯外部同化、内部回收或样本资格时，不能为了性能丢失来源份额。
- 单次可处理数量、规范分片大小和数量精度都是L6参数，不改变本节语义。

## 8. 缓冲、端口、容量与路径

### 8.1 端口种类

房间蓝图按需要声明以下通用端口：

| 端口 | 接收或形成的事实 | 是否需要工蜂 |
| --- | --- | --- |
| 物理输入端 | 已回流过程资产进入具体模块输入缓冲 | 是，或明确的安全交接规则 |
| 物理输出端 | 模块输出缓冲中的`现场`过程资产被取走 | 是 |
| 账本接入口 | 兼容物理资产终结并正式入账 | 是，或资源契约允许的应急直连 |
| 单位容纳端 | 幼体等`容纳`单位进入本地容器或任务预留 | 否，遵循单位事务 |
| 单位离巢端 | 新生可机动单位进入后续合法位置 | 否，遵循单位事务 |
| 能源网络端 | 功率供给、请求和储能上限贡献 | 否 |
| 突触网络端 | 基因、样本标签、资格或其他条件网络结果 | 否 |
| 外部物质来源端 | 登记已开放物质来源、抽取容量、形成位置和兼容处理链 | 形成现场资产后需要工蜂回流 |
| 环境网络输入端 | 读取热化梯度等不形成可搬运物质的环境容量或房间修正 | 否；按持续网络贡献分配 |

端口类型是接入资格，不是产量。外部物质来源端不能把节点物质远程传入房间；来源先形成现场过程资产，再由工蜂送入物理输入端。网络输出仍需合法任务或持续网络贡献边界，不能因为不搬运就绕过输入、环境容量、功率或事件来源。

### 8.2 本地缓冲

- 输入和输出缓冲属于具体模块或固定基础设施，不属于房间群。
- 缓冲保存过程资产或单位总账的容器引用，不复制其数量。
- 过程输入预留锁定已回流资产，阶段仍保持`已回流`。
- 物理输出提交后形成位于输出缓冲的`现场`资产，最早从下一步骤参与工蜂目标选择。
- 特殊不可堆叠输出必须进入声明的唯一来源缓冲，不能覆盖旧内容。

### 8.3 容量承诺

任务建立时必须同时承诺精确的输入与输出容量：

- 承诺不等于已经形成产物，但会阻止其他任务超额占用同一容量。
- 正常结构操作不能撤销已承诺容量；若强制规则使容量失效，任务保持待提交或资产保持原合法位置。
- 自愿降级、拆除和替换在容量安全前不能完成。
- 房间群显示的总容量只是成员容量与预留的派生汇总，不允许跨模块无事件借用。

### 8.4 路径与交互容量

- 第5阶段根据当前槽位拓扑、模块结构状态、有效端口和显式通路重建路径与交互容量。
- 第7阶段由单位契约同时请求资源资产预留、路径、取货端和投递端容量；本文只写入虫巢内路径与房间/基础设施端口份额，并返回稳定预留句柄。
- 第10阶段只有执行者、路径和端口仍合法时才提交搬运。
- 房间群显示成一个大房间不会把多个物理端口压成一个，也不会生成无限端口。
- 工蜂目标选择和服务层级由单位契约负责；本文只暴露合法目标、容量和阻断事实。

画面工蜂、房间动画和端口特效只代表真实交互批次，不决定吞吐。

### 8.5 采集交互预留与第2阶段释放

一次第7阶段采集接纳由单位契约建立`采集交互预留束`。本文为每个房间域份额返回：

- 预留束、行为尝试、工蜂批次与真实工蜂数量。
- 所有者为虫巢、模块或固定基础设施的稳定路径句柄。
- 取货端、投递端或安全交接端的稳定容量句柄与精确份额。
- 建立步骤、端口/拓扑版本、当前状态和可逆截止边界。

预留束只是跨域引用集合，不复制路径、端口容量或过程资产。节点路径和节点来源交互份额由世界契约返回自己的句柄；资产阶段由资源契约拥有；工蜂活动和行为进度由单位契约拥有。

第2阶段进攻准备对候选工蜂调用无副作用`准备释放交互预留`：

1. 校验所有房间句柄仍属于同一预留束、同一行为尝试和精确工蜂份额。
2. 校验资产尚未取货、没有第10阶段待提交搬运草案，并且端口份额尚未消费或转移。
3. 返回`可释放`及拟释放句柄；准备调用不改变容量和状态。
4. 已携带、不可中断交接或句柄已消费的工蜂返回`不可释放`，保持原作业且不进入最终可参战集合。
5. 所有者、数量、版本或状态不一致属于不变量失败，不得通过只排除该工蜂掩盖。

单位契约确定最终释放集合和最终可参战集合后，进攻跨域事务同笔提交：

- 本文把集合内全部虫巢路径、取货端、投递端和安全交接容量句柄从`已预留`改为`已释放`。
- 资源契约把对应资产`已预留 -> 现场`。
- 世界契约释放节点域路径与来源交互句柄。
- 单位契约清空行为目标、路径、未提交进度和活动份额，并转移最终可参战单位。
- 世界与时间契约建立目标节点活动和唯一主动战场引用。

任一已准备子事务在提交时不再匹配，整笔进攻不发布，全部句柄、资产、工蜂作业和战场引用保持阶段开始状态。相同进攻事务标识重试只能返回同一结果；一个句柄最多从`已预留`提交到`已释放`一次。

## 9. 房间输出与跨域提交

### 9.1 四类输出

| 输出 | 第11阶段结果 | 后续所有者 |
| --- | --- | --- |
| 物理过程资产 | 在具体输出缓冲形成`现场`资产 | 过程资产总账 |
| 网络结果 | 提交能源、基因、条件或资格事件 | 对应全局聚合根 |
| 单位形成或变形 | 形成幼体、等量分化或离巢事件 | 单位总账 |
| 候选来源结果 | 形成候选组请求或进入来源缓冲 | 进化状态 |

房间模块只保存上游任务和端口引用，不拥有输出的第二份副本。

### 9.2 提交约束

- 任务的全部输入终结、输出形成、预留释放和账目必须原子提交。
- 任一输出没有合法去向时，整个任务保持`待提交`；不能先扣输入再等待剩余输出。
- 一个配方有多个物理输出或网络输出时，共享同一上游完成事件并保持各自来源份额。
- 第11阶段形成的房间、供给、单位和物理输出不追溯参与本步骤早期任务。
- 自动入库只表示不需要玩家领取；物理输出仍受工蜂、路径、端口和容量约束。

### 9.3 两阶段孵化的通用表达

第一版孵化不建立物种专用状态机，而使用一个有向无环配方链：

```text
最终单位队列条目
-> 检查并承诺现有未预留幼体
-> 对缺口建立幼体形成子任务
-> 幼体形成并直接归入该条目的单位输入承诺
-> 建立分化任务
-> 等量变形为目标模板
-> 提交离巢请求
```

- 幼体始终是`容纳`单位，不是房间物品或核心资源。
- 幼体形成与分化分别使用普通过程任务、输入预留、功率和完成边界。
- 子任务在第11阶段形成的幼体最早从下一步骤参与分化。
- 更改绑定模板不能删除已经形成的通用幼体；已为旧任务预留的幼体必须先释放或完成合法任务。
- 离巢后单位的虫群大军、主动节点或虫巢作业归属由单位契约决定。

同一配方图机制以后可以表达多阶段解析或组织培育，但配方图必须无环，不能在同一步消费自身输出。

## 10. 暂停、循环与运行阻断

### 10.1 玩家可以控制的基础事实

- 暂停或恢复当前房间群的普通运行。
- 开启或停止自动循环。
- 设置兼容循环模式、配方、孵化模板和目标规模。
- 取消尚未越过不可逆消费边界的当前批次。
- 发起建设、升级、移动、功能重组、降级、拆除或替换。

第一版玩家不设置逐任务能源优先级、不手动指派工蜂目标，也不点击领取常规产出。

### 10.2 暂停语义

- 房间群暂停命令原子写入命令快照中的全部成员模块。
- 暂停使普通运行请求从本步骤后续能源汇总退出，保留任务进度、未消费输入和输出承诺。
- 暂停能源生产模块会移除其正常供给资格，但不会仅因暂停移除物理储能上限贡献。
- 结构任务不受普通房间群暂停控制；玩家通过取消对应结构事务停止未来结构负载。
- 恢复只重新取得运行资格，不保证本步骤一定获得功率、输入或端口。

### 10.3 派生阻断事实

| 事实 | 判定 |
| --- | --- |
| 缺料 | 合法自动请求因核心资源、过程输入、单位输入或外部来源不足而未建立；已建立任务的消费输入在开始前已经锁定 |
| 缺能 | 合法请求没有达到配方所需已服务功率 |
| 堵塞 | 输出、目标、端口、路径、槽位、候选来源缓冲或结构安全条件没有合法去向 |
| 运行 | 至少一个模块的合法任务在本步骤实际推进，或合法持续网络贡献实际生效 |
| 待机 | 模块可运行且未暂停，但当前没有需要推进的任务，也没有阻断其应当建立的任务 |

同一房间群可以同时存在多个模块和多个阻断原因。系统必须保留完整原因集合；单一主状态只允许作为可重算摘要，不能覆盖任务事实。

阻断反馈只陈述事实、位置和受影响数量，例如“输出缓冲已承诺满”“两个任务未获得最低服务功率”。它不推荐暂停哪座房间、生产哪种单位或选择哪项突变。

## 11. 第一版房间接入

### 11.1 固定基础设施

| 内容 | 通用接入 | 禁止的捷径 |
| --- | --- | --- |
| 核心巢室 | 拓扑锚点、账本接入口、最低容器、两项恢复保障接口 | 常规产出、普通供给、免费工蜂、加入房间群 |

### 11.2 六个可建设房间

| 房间 | 功能类别 | 通用任务与端口要求 | 输出边界 |
| --- | --- | --- | --- |
| 腐殖滤囊 | 资源形成 | 外部物质来源兼容声明、物理输入端、形成任务、物理输出缓冲与取货端 | 只处理工蜂已回流原料并形成生物质浆；投递账本接入口后才增加生物质 |
| 热化代谢腔 | 能源生产与储能 | 环境网络输入端、持续网络贡献、能源网络端与冷启动资格 | 按实际环境服务建立供给和储能上限贡献 |
| 胚流孵化室 | 孵化 | 最终模板绑定、两阶段配方链、幼体容器与单位离巢端 | 先形成幼体，再等量分化并离巢 |
| 腐解消化池 | 回收与转化 | 物理输入端、过程输入预留、多输出承诺与物理输出端 | 普通组织形成生物质浆；样本残骸分离样本组织 |
| 突触解析腔 | 研究与解析 | 样本组织输入、解析任务、突触网络端 | 形成基因、样本标签和首次候选来源事件 |
| 诱变培养腔 | 候选形成 | 生物质预留、样本资格、功率、普通候选来源缓冲 | 向进化契约请求形成一个普通候选组 |

上述内容只指定接口，不固定配方数值。多功能房间可以组合多个通用端口和任务标签，但不能拥有内容专用任务生命周期。

### 11.3 内容定义最低要求

每个可运行房间蓝图必须登记：

- 房间族、结构兼容、群兼容键构成和允许的运行绑定。
- 至少一个明确功能类别及其配方图。
- 输入类型、输出类型、消费模式和来源谱系规则。
- 持续网络贡献的环境来源、服务模式、最低环境服务量、贡献类型和退出条件；不使用持续贡献时明确为空。
- 结构与运行功率请求的服务模式及安全层级。
- 模块缓冲、端口、容量、目标指标和阻断条件。
- 建设、升级、功能重组、降级、拆除和替换时的保留与清空规则。
- 所有修正的作用范围、生命周期、叠加与最早生效步骤。

缺少任一必要字段的蓝图不能用脚本默认值补齐后进入发布内容。

## 12. 多虫巢与生产方案

### 12.1 多虫巢扩展规则

长期版本中的每个已激活虫巢都保持完整模拟：

- 独立保存槽位拓扑、模块、运行策略、任务、端口、路径和本地过程资产容器。
- 生物质、基因和能源按资源契约全局共享，并保留来源虫巢与模块明细。
- 一个虫巢缺料、堵塞或重组不会直接冻结其他虫巢。
- 本地幼体、过程资产、输入缓冲和输出缓冲不能因核心资源全局共享而瞬移。
- 基础模块移动只限同一虫巢；跨虫巢移动或资产交接不是第一版结构操作。

第一版只实际激活一个虫巢，但对象身份、事件和参数不得假设“虫巢标识永远等于初始虫巢”。

### 12.2 生产方案

生产方案只保存玩家希望复用的声明：

- 相对槽位关系与房间蓝图。
- 等级、分支、运行绑定、循环模式和目标指标。
- 哪些字段必须一致，哪些字段允许在应用时重新选择。

生产方案不保存或复制模块实例、结构投入、任务、进度、插件实例、缓冲资产、功率或核心资源。应用方案时必须：

1. 先把相对槽位映射到目标虫巢的兼容槽位。
2. 生成普通建设、移动、功能重组和策略命令预览。
3. 对全部目标、资格、资源、容量和冲突进行普通校验。
4. 玩家确认后按批量结构操作规则提交。

第一版不要求实际提供跨虫巢方案应用，但后续实现不能把方案做成免费复制生产生态的捷径。

## 13. 统一十二阶段接入

本文不复制十二阶段，只登记接入职责：

| 时点 | 本文职责 |
| --- | --- |
| 稳定边界/第1阶段 | 验证房间命令；建立或取消结构事务；设置暂停、循环、绑定和目标；处理群快照冲突 |
| 第2阶段 | 为进攻事务准备并原子释放参战工蜂在虫巢域持有的路径、取货端、投递端和安全交接容量句柄；不改变资产或单位 |
| 第3阶段开始 | 按稳定快照接纳自动任务、原子预留输入与容量；登记持续网络贡献环境请求与任务功率请求 |
| 第3阶段服务后 | 按已服务功率推进结构与过程任务，形成完成草案和阻断事实 |
| 第5阶段 | 从有效拓扑、模块和端口重建路径、作用范围与交互容量 |
| 第7阶段 | 向单位契约提供可采集目标、端口与容量预留接口；本文不选择工蜂 |
| 第10阶段 | 接收存活工蜂的取货、投递和回流结果，更新唯一容器引用 |
| 第11阶段 | 原子提交模块形成、移动、升级、重组、降级、拆除、替换和任务物理/网络/单位/候选输出 |
| 第12阶段 | 重算房间群与摘要，检查槽位、任务、容量、幂等和守恒接口，发布日志与警报 |

阶段内约束：

- 第3阶段任务接纳先于本步骤功率请求重建，接纳完成后由资源契约统一分配功率。
- 第3阶段完成只进入`待提交`，不能直接形成模块、资源、单位或候选。
- 第11阶段新结构与输出最早从下一步骤参与对应规则。
- 第12阶段房间群重算失败属于不变量错误，整个步骤不能发布。

## 14. 命令契约

| 命令 | 主要验证 | 成功的直接结果 | 不能发生 |
| --- | --- | --- | --- |
| 建设房间 | 蓝图资格、槽位、结构资源、初始绑定 | 建立结构预留、槽位声明、临时模块与建设任务 | 即时完成或预扣总能源 |
| 取消建设 | 任务未终结且仍可取消 | 释放未消费预留、槽位和临时模块 | 返还历史功率与时间 |
| 升级模块 | 等级资格、资源、任务与容量 | 建立升级事务和新增结构预留 | 重扣已有结构投入 |
| 移动模块 | 同虫巢目标槽、排他声明和结构状态 | 建立移动事务并保留模块全部合法事实 | 复制模块或跨虫巢瞬移 |
| 功能重组 | 新绑定、任务、插件、输入输出去向 | 建立重组事务 | 删除幼体、刷新冷却或即时换产线 |
| 降级模块 | 目标等级、容量与储能安全 | 建立降级事务，完成后返还移除投入 | 提前返还或静默溢散 |
| 拆除模块 | 清空条件、结构状态 | 建立清空与拆解事务 | 即时删除缓冲或任务 |
| 替换模块 | 旧模块清空、新蓝图资格和拆解后完整预留新结构的可达性 | 建立两段组合事务 | 提前使用未提交旧返还或瞬时切换蓝图 |
| 暂停/恢复房间群 | 群快照、成员和策略版本 | 原子更新当前成员暂停事实 | 自动调整其他群或结构任务 |
| 设置循环模式 | 群快照、配方支持 | 更新后续任务建立策略 | 追溯改写已开始任务 |
| 设置目标规模 | 规范目标指标和合法值 | 更新后续目标缺口 | 删除超额单位或资产 |
| 设置运行绑定 | 模板或分支资格 | 无需结构变化则更新策略，否则建立功能重组 | 绕过兼容键与清空检查 |
| 提交单次批次/移除未接纳余量 | 配方、数量、群快照和条目剩余事实 | 建立条目或终结不锁输入的剩余数量 | 把已开始任务分片伪装成未接纳余量取消 |
| 取消当前批次 | 任务仍可取消且消费边界未越过 | 释放未消费预留、终结任务并释放后续负载 | 返还已消费输入、历史功率或时间 |
| 提交批量结构事务 | 批次身份、有序子命令、共同拓扑版本、联合目标、全部预留和冲突 | 原子建立全部普通结构事务 | 隐式只执行可成功部分、混入未登记策略命令或应用生产方案 |

查看预览、切换房间、展开模块明细和播放动画不是玩法命令。命令拒绝不建立部分预留、任务、槽位声明或随机结果。

## 15. 事件与跨域接口

### 15.1 本文事件类别

| 事件类别 | 提交时点 | 原子结果 |
| --- | --- | --- |
| 槽位已声明/已释放 | 第1或第11阶段 | 排他关系改变，不形成模块副本 |
| 结构事务已建立/已取消 | 第1阶段 | 任务、预留、目标和取消结果同笔变化 |
| 模块已形成 | 第11阶段 | 结构预留消费、结构投入形成和结构状态改变 |
| 模块已移动/已重组/已升级 | 第11阶段 | 同一模块位置或结构事实改变 |
| 模块已降级/已拆除 | 第11阶段 | 结构变化与可回收投入返还同笔提交 |
| 替换阶段已转换 | 第11阶段 | 旧实例终结与返还提交后，完整建立新预留和`建设中`新实例 |
| 模块替换已完成 | 第11阶段 | 新预留消费、新结构投入形成和新模块转为`可运行` |
| 运行策略已改变 | 第1阶段 | 当前目标模块策略版本递增 |
| 过程任务已建立/暂停/恢复/取消/失败 | 第1或第3阶段 | 输入、执行位、预留和任务状态一致变化 |
| 过程任务输出已提交 | 第11阶段 | 输入终结、输出形成、预留释放和任务完成同笔提交 |

房间群重建是派生投影，不生成可重复领取的“合并完成事件”。可以记录拓扑投影日志，但日志不能触发资源、冷却或一次性效果。

### 15.2 调用资源能源接口

本文每次建立或完成任务必须向资源契约提供：

- 稳定任务、配方、模块、虫巢、输入、输出和参数版本。
- 核心资源预留、过程输入锁、结构投入或返还请求。
- 功率服务模式、安全层级、请求值、最低服务值和本步骤服务结果。
- 输出容量、物质谱系、消费边界、取消边界和幂等标识。

资源契约拒绝或未服务时，本文只能保持未建立、零进度、既有显式暂停或堵塞，不能改用房间本地库存补数。

### 15.3 调用单位、地图与候选接口

- 单位接口：请求形成幼体、预留幼体、等量分化、离巢和释放单位输入；本文不直接写单位数量。
- 进攻释放接口：按预留束和进攻事务标识准备/提交虫巢域路径与端口释放；本文不改变资产阶段、工蜂活动或战场位置。
- 地图接口：读取已经开放的外部物质来源、抽取容量、环境网络输入、环境容量和房间修正；本文不提交节点控制或资源包。
- 候选接口：提交候选来源输出和来源缓冲引用；本文不抽取成员、不选择突变也不拥有候选组。
- 修正接口：读取作用于模块、房间群、虫巢或全局的明确修正；修正不能替换本契约状态机。

## 16. 保存、离线与确定性

### 16.1 必须保存

- 虫巢身份、所属世界引用、槽位拓扑、拓扑版本和全部有效边。
- 每个槽位的模块或结构事务排他声明。
- 每个模块的蓝图、结构状态、结构投入、等级、分支、绑定、插件、修正和最早生效步骤。
- 玩家暂停、运行策略、策略版本、队列条目和目标覆盖承诺。
- 全部结构事务、过程任务、输入锁、容量承诺、进度、功率请求、阻断、取消边界和幂等标识。
- 输入输出容器、端口、路径所需权威引用和待提交输出。
- 未终结采集交互预留束的房间域句柄、精确份额、状态、版本和最近释放事务标识。
- 连续空闲、最近领取、连续未服务等确定性排序事实。

房间群、运行摘要、总产率和画面实体不需要作为权威事实保存；若缓存，载入后必须重算校验。

### 16.2 离线等价

无主动战场时，离线安全结算按在线完整步骤推进：

- 自动循环只在相同输入、目标、容量和运行策略下建立任务。
- 房间、结构任务和输出继续受同一功率、工蜂、路径、端口与堵塞限制。
- 不自动建设、移动、暂停、恢复、重组、拆除、替换、选择模板或调整目标。
- 任务完成、缓冲满、输入耗尽、目标达成、储能边界和结构完成都必须成为事件跳跃边界。
- 有主动战场时整个世界冻结，房间不独自获得离线推进。

### 16.3 确定性

- 同一快照、命令、参数和模拟时长必须得到相同任务接纳、模块分配、功率请求、输出、拓扑和账目。
- 任务与模块排序不能读取字典、场景树、线程、画面位置或动画完成先后。
- 房间群重算结果只依赖拓扑版本、兼容键和模块事实。
- 载入不能再次形成结构投入、返还、模块、任务输出或一次性网络结果。
- 批次压缩、离线跳跃和多倍率必须通过与逐步骤执行等价的HRC验收。

## 17. 第一版范围与排除

第一版实际支持：

- 一个已激活初始虫巢、固定核心巢室和自由空槽拓扑。
- 六个可建设房间及其通用建设、运行、暂停、循环、缓冲和堵塞规则。
- 相邻兼容模块自动形成房间群，并允许群级暂停、循环、目标与批量结构命令。
- 并行建设与重组、两阶段孵化、物理输出搬运和网络输出。
- 保存、载入、无主动战场离线推进和完整可观察异常。

第一版不支持：

- 第二虫巢实际激活、跨虫巢模块移动或玩家配置的运输系统。
- 多槽房间模块、房间受伤、维修、塔防、敌方破坏虫巢或已占虫巢失守。
- 工蜂参与建设、维修、研究或玩家指定优先级。
- 手动领取、逐房间配电、隐藏全局建设队列或任意不同房间相邻加成。
- 完整插件装卸流程、正式飞升房间交易和生产方案跨虫巢实际应用。
- 内容专用任务状态机、动画回调结算或按画面单位数量计算吞吐。

结构和数据可以保留上述扩展接口，但第一版运行状态不能意外执行排除内容。

## 20. 不变量责任与完成条件

### 20.1 不变量责任

本文直接维护：

- INV-001：虫巢、槽位、模块、队列条目、任务和结构事务具有稳定身份。
- INV-002：虫巢拥有结构与任务，单位和过程资产只通过唯一位置或容器引用进入。
- INV-003：房间群、运行摘要、产率预测和表现不成为第二份真相。
- INV-007：房间群只由相邻兼容模块派生，结构操作不复制任务、投入、冷却或产物。
- INV-022：结构、任务建立、取消和输出提交不存在半状态。

本文协同维护：

- INV-004：调用资源契约保持核心资源、结构投入和过程资产守恒。
- INV-005：孵化链只请求幼体单位形成、预留和等量分化。
- INV-008：房间只处理地图已经开放的来源，核心资源仍按正式形成边界入账。
- INV-009：未服务房间任务不推进，结构与运行能源不双重收费。
- INV-020、INV-021：稳定排序、固定步骤和派生重算保持重放、倍率与表现等价。
- INV-023：模块形成、结构返还、任务输出和网络结果全部使用唯一幂等标识。
- INV-024：可逆布局、暂停、清空和资源契约恢复保障共同维持第一版恢复可达。

### 20.2 完成条件

- 虫巢、槽位、模块、房间群、队列条目和过程任务各只有一种规范含义。
- 每个结构操作都有验证、预留、推进、清空、提交、取消、返还和冲突边界。
- 房间群合并与拆分在任意拓扑中都能从模块事实唯一重算。
- 自动循环不能在无输入时建任务、在同一步递归、超出目标或绕过容量。
- 每种第一版房间都只通过通用配方、端口、任务和跨域输出接口实现。
- 第2阶段进攻能够无半状态地释放参战工蜂的虫巢路径与端口份额，未参战工蜂的作业不被误释放。
- 暂停、缺料、缺能、堵塞、运行和待机均能从真实任务与容量事实解释。
- 保存、离线、多倍率、批次压缩和页面表现不改变结算结果。
- HRC-001至HRC-032全部映射到L6自动测试；第一版发布必须通过其中标记为`第一版发布`的项目，`长期契约`项目在对应能力进入实现前通过。未达到相应范围前，本文只能维持`已确认机制`状态。

单位、世界、候选和轮次调用规则已经由对应L3契约封闭；本文后续只随虫巢房间接口变更同步更新。

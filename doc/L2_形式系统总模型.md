本文定义“世界中有哪些事实，以及这些事实怎样才能保持自洽”。它不定义具体房间、单位或节点内容，不确定参数，不规定页面布局，也不直接规定Godot代码结构。

发生冲突时，L2决定对象是什么、归谁所有、位于哪里、哪些状态正交、什么必须守恒；L3只能在这些边界内定义转换条件和执行顺序。L4至L6不得通过内容、界面、参数或实现建立第二份玩法真相。

## 1. 事实层级

所有设计信息只能属于以下四层之一：

| 层 | 含义 | 是否能直接改变玩法 |
| --- | --- | --- |
| 内容定义 | 房间蓝图、单位模板、生态原型、候选定义等稳定规则和标签 | 不能；必须被实例或事件引用 |
| 权威事实 | 当前存档中已经提交的资源、实例、位置、状态、任务、随机游标和事件 | 能；只能通过合法交易改变 |
| 派生状态 | 可从权威事实唯一重算的汇总、分组、资格或摘要 | 不能单独写入；底层事实变化后重算 |
| 表现投影 | 场景、动画、有限代表实体、页面文本、声音和镜头 | 不能；只读取事实和派生状态 |

核心规则：

- 内容定义与运行实例分离；修改显示名称不能改变已保存对象语义。
- 权威事实只能由已验证并原子提交的事件改变。
- 派生状态不得成为另一份可分别修改的存档真相；允许缓存时也必须能够丢弃重算并校验一致。
- 表现时间、帧率、代表实体数量、页面访问和动画完成度不参与结算。
- 参数只改变已经登记的量和关系，不能暗中增加对象类型、位置或状态维度。

## 2. 存档根与所有权树

```text
存档
├─ 跨轮进度
│  ├─ 图鉴
│  ├─ 已解锁领袖
│  ├─ 永久收益
│  └─ 历史统计
└─ 当前轮次
   ├─ 模拟时钟、规则版本、参数版本与随机流
   ├─ 全局经济：生物质、能源、基因与统计账目
   ├─ 过程资产总账：全部过程资产批次、唯一阶段与位置
   ├─ 世界：区域图 -> 区域 -> 节点 -> 空间段、来源与容器
   ├─ 虫巢：槽位 -> 房间模块、任务与本地容器
   ├─ 单位总账：虫巢内 / 虫群大军 / 节点内 / 死亡，以及采集交互预留束引用
   ├─ 进化：发现、科技族资格、候选组与本轮突变
   ├─ 当前主动战场引用与有序命令
   └─ 事件账本与飞升贡献统计
```

第一版不执行正式飞升，但仍保存可复算飞升预览所需的本轮事实和跨轮定义。不存在当前轮次时，存档只能处于新建、已结束或待创建轮次状态。

### 2.1 聚合根

| 聚合根 | 唯一所有者 | 直接拥有 | 生命周期 |
| --- | --- | --- | --- |
| 存档 | 本地存档槽 | 跨轮进度、零或一个当前轮次 | 跨多轮 |
| 当前轮次 | 存档 | 世界、经济、单位、进化、时钟、随机流和账本 | 飞升或明确终止前 |
| 世界 | 当前轮次 | 世界区域图和已提交区域 | 本轮 |
| 区域 | 世界 | 节点网、生态事实、着床事实和区域虫巢引用 | 本轮 |
| 节点 | 区域 | 空间段、来源、修正、容器和持续战局；单位与资产只通过位置引用进入 | 本轮 |
| 虫巢 | 当前轮次，并引用所属区域与节点 | 槽位、房间模块、任务和本地容器；幼体与资产只通过位置引用进入 | 本轮 |
| 全局经济 | 当前轮次 | 核心资源、能源状态和统计账目 | 本轮 |
| 过程资产总账 | 当前轮次 | 全部过程资产批次、阶段、位置和累计终结事实 | 本轮 |
| 单位总账 | 当前轮次 | 所有阵营的单位批次和累计形成、变形、死亡事实 | 本轮 |
| 进化状态 | 当前轮次 | 科技族资格、候选组、槽位预留和已同化突变 | 本轮 |
| 跨轮进度 | 存档 | 图鉴、领袖、永久收益与历史记录 | 跨轮 |

对象只能有一个直接所有者。引用其他对象时保存稳定引用，不把目标对象复制进自己的数量或状态。

## 3. 定义、实例与身份

### 3.1 两类身份

| 身份 | 作用 | 示例 |
| --- | --- | --- |
| 定义标识 | 指向不会因存档重载改变的内容规则 | 房间蓝图、单位模板、过程资产类型、候选定义 |
| 实例标识 | 指向当前存档中的一个真实对象或事件 | 房间模块、单位批次、节点、候选组、建设任务 |

每个运行实例至少拥有：

- 全局唯一且本轮稳定的实例标识。
- 所引用的定义标识。
- 唯一所有者或唯一真实位置。
- 形成它的来源事件标识。
- 创建步骤和最后变更步骤。
- 与其类型兼容的数量、状态和引用。

不得使用显示名称、数组位置、场景节点路径或画面实体作为稳定玩法身份。

### 3.2 标签与参数

- 标签只表达资格，例如`可采集`、`占领阻断`、`区域解锁`；标签本身不是生命周期状态。
- 状态表达已经发生或正在发生的事实，不能用标签代替互斥状态。
- 参数表达成本、速度、容量、概率或倍率，不能决定对象属于哪个聚合根。
- 内容定义可以组合标签、条件和通用效果，但不能覆盖本文件的不变量。

### 3.3 数量与来源

- 单位真实数量使用非负整数。
- 资源和过程资产数量必须使用可确定重算的数值语义；具体数值表示由实现决定，但不得因浮点误差破坏守恒。
- 每次数量增加都必须具有来源事件；每次减少都必须具有目标事件或终结原因。
- 数量为`0`的批次可以被清理，但累计账目和来源事件不能因此消失。

| 对象 | 类型 | 权威事实 | 不属于该对象的内容 |
| --- | --- | --- | --- |
| 世界区域图 | 运行实例 | 区域实例、区域连接、主深度方向和已公开连接 | 区域内部节点连接 |
| 区域 | 运行实例 | 生态原型、深度、节点网提交状态、着床和出口引用 | 单一“区域颜色状态” |
| 节点 | 运行实例 | 节点连接、内容、控制、持续战局、来源和修正 | 核心资源奖励包 |
| 空间段 | 运行实例 | 节点内位置、连接、容量、地表和推进锚点 | 独立关卡结算 |
| 外部物质来源 | 运行实例 | 资产类型、储量或流量、抽取容量、现场形成位置、物质谱系和兼容处理链 | 已入账核心资源或房间内远程输入 |
| 环境网络输入 | 运行实例 | 环境类型、来源位置、可分配容量、兼容贡献和公开修正 | 可搬运过程资产或直接核心资源 |

| 槽位 | 运行实例 | 位置、连接、尺寸兼容和当前占用 | 预设房间功能 |
| 房间模块 | 运行实例 | 蓝图、结构投入、等级、分支、运行策略、队列条目、任务引用、端口、缓冲、持续网络贡献和运行条件 | 房间群汇总 |
| 房间群 | 派生状态 | 同虫巢中相邻且兼容模块的连通集合 | 第二份模块、队列或资源 |
| 过程任务 | 运行实例 | 执行模块、输入预留、目标与容量承诺、进度、功率需求、输出配方和提交边界 | 队列意图或已经完成的结果 |
| 生物质与基因账目 | 运行实例 | 可用量、资源预留和顺序账目事件 | 能源状态、残骸、幼体、样本组织 |
| 能源状态 | 运行实例 | 储能、上限、供给、需求、已服务负载、环境容量分配、持续贡献和来源明细 | 可搬运能源物品的默认假设 |
| 过程资产总账 | 运行实例 | 全部过程资产批次与累计生成、转化和终结事实 | 节点或虫巢中的第二份资产库存 |
| 过程资产批次 | 运行实例 | 类型、来源、数量、阶段、唯一位置或携带者 | 核心资源 |
| 单位模板 | 内容定义 | 阵营资格、标签、移动形态、耐久、行为与死亡输出 | 存活数量 |
| 单位批次 | 运行实例 | 模板、唯一位置、规范状态队列、真实数量和统计归因份额 | 画面代表数量或运行容器分片 |
| 规范状态单元 | 单位批次内规范值 | 完整玩法键、真实数量和对应统计归因份额 | 独立运行对象、平均状态或表现实体 |
| 采集交互预留束 | 单位行为拥有的跨域协调记录 | 工蜂活动份额、行为尝试、资产预留、路径、源交互端和投递端的稳定所有者引用及可逆边界 | 第二份资产、路径、容量或独立运输任务 |
| 候选组 | 运行实例 | 来源、成员、槽位类型、成本事实和生命周期 | 已生效突变 |
| 突变 | 内容定义与本轮实例 | 规则变化、代价、作用范围及本轮获得来源 | 科技族全部内容 |
| 解锁记录 | 运行事实集合 | 发现、内容池资格、永久化来源与行为里程碑 | 当前资源是否足够 |
| 当前主动战场引用 | 全局单例引用 | 零或一个节点标识、开始步骤和待处理战役命令 | 第二份节点单位 |
| 命令 | 玩家意图 | 类型、目标、参数、提交顺序和请求步骤 | 已经发生的事实 |
| 事件 | 已提交事实 | 唯一标识、步骤、原因、来源、目标和变化 | 可以再次提交的指令 |
| 账目条目 | 事件投影 | 资产类型、增减、前后状态和来源去向 | 第二次资源变化 |
| 飞升预览 | 派生快照 | 同一步骤的贡献、参数版本、点数和只读对照 | 第一版可执行飞升 |

房间、节点、过程资产和地形不能为了复用战斗代码而被称为单位。可受伤、死亡、占据战场空间的敌方结构和出兵点属于单位。

### 4.1 条件、效果与修正

内容差异使用同一组合语法：

| 构件 | 定义 | 限制 |
| --- | --- | --- |
| 条件 | 对权威事实和合法派生状态进行确定性判断的无副作用谓词 | 不能创建事件、消费资源或推进随机流 |
| 效果配方 | 声明满足条件时允许形成哪些事件及其输入输出 | 不是已经提交的事件 |
| 修正实例 | 由突变、插件、领袖、永久收益、区域或房间产生的运行规则变化 | 必须有来源、目标、范围、生命周期和叠加规则 |
| 作用范围 | 通过对象类型、所有者、位置、标签和空间关系筛选目标 | 不能读取显示名称或画面实体 |
| 生命周期 | 本步骤、持续步骤、本轮、领袖有效期或跨轮永久 | 必须具有明确开始与终止边界 |
| 叠加规则 | 刷新持续、取最强、独立实例、固定叠层、互斥或规则替换 | 未声明时不得进入运行状态 |

多个修正作用于同一关系时统一按以下阶段求值：

```text
目标资格与作用范围
-> 规则替换、前置与互斥
-> 固定数值修正
-> 比例修正
-> 上下限、容量与安全约束
-> 形成结果事件与有序后续触发
```

- 规则替换必须声明互斥组或覆盖关系，不能依赖文档、载入或容器排列顺序。
- 动态资格变化只影响规则声明的后续求值边界，不能追溯改写已形成事件。
- 后续触发必须保留来源事件和深度边界，不能无限递归生成资源、单位或伤害。
- 内容定义不能通过专用效果绕过所有权、守恒、原子性、确定性和玩家控制边界。

## 5. 唯一位置与正交状态

### 5.1 核心资源与过程资产

- 已正式入账的生物质和基因归属全局经济，不再拥有节点或虫巢位置。
- 能源归属全局能源状态，但每项供给、负载和储能贡献保留来源虫巢与房间引用。
- 未处理的物理物质属于过程资产，必须保留节点、虫巢、房间缓冲或携带单位等唯一位置。
- 同一过程资产不能因为“已预留”而同时计入现场可用量。

### 5.2 单位唯一位置

每个存活单位批次只能处于以下一个位置：

| 位置 | 所需引用 | 合法对象 |
| --- | --- | --- |
| `虫巢内` | 虫巢标识和容器 | 幼体等`容纳`单位 |
| `虫群大军` | 当前轮次 | 玩家可机动单位 |
| `节点内` | 节点与空间段标识 | 敌我可机动单位、固定菌毯和可破坏结构 |
| `死亡` | 死亡事件标识 | 已终结单位累计账目 |

`当前战场`、`节点留置`、`虫巢作业`、`领地采集`、`自由游荡`、`待命`、`接触`、`纵深`和`后备`都是位置之上的筛选或活动，不是额外数量桶。

### 5.3 世界运行状态

| 状态 | 模拟时间 | 说明 |
| --- | --- | --- |
| `正常运行` | 推进 | 默认在线游玩 |
| `全局暂停` | 停止 | 玩家暂停或系统菜单 |
| `关键选择暂停` | 停止 | 候选同化、正式飞升等不可逆选择 |
| `离线安全结算` | 按规则批量推进 | 无主动战场时关闭游戏 |
| `主动战场冻结` | 整个世界停止 | 有主动战场时关闭游戏 |

页面切换不是运行状态。时间倍率改变模拟推进速度，不改变相同模拟时长下的结果。

### 5.4 区域状态维度

区域不存在单一总状态，必须分别记录或派生：

| 维度 | 权威状态 |
| --- | --- |
| 节点网提交 | `未提交 / 已提交`；已提交后本轮不可重抽 |
| 情报 | `未发现 / 概要已知 / 节点网已知 / 深入侦察 / 完全解析` |
| 着床 | `不可着床 / 待着床 / 虫巢节点争夺中 / 着床安全区 / 虫巢建设中 / 虫巢已激活` |
| 节点控制汇总 | `敌方占据 / 区域争夺中 / 完全吞噬`，由节点和特殊目标派生 |
| 出口通路 | `出口未知 / 出口已知 / 出口可达 / 出口争夺中 / 通路已打开` |
| 活动摘要 | 主动战场、敌方恢复、后台战斗、领地采集、建设和处理等事实集合的派生摘要 |

`虫巢已激活 + 区域争夺中 + 通路已打开`是合法组合。`完全吞噬`是统计事实，不生成第二次节点收益。

### 5.5 节点状态维度

| 维度 | 权威状态或派生规则 |
| --- | --- |
| 情报 | `未揭示 / 轮廓已知 / 可观察 / 已解析` |
| 连通 | `不可达 / 可达`，由图连接、已占路径和着床规则派生 |
| 控制 | `敌方控制 / 争夺中 / 己方控制` |
| 活动 | `主动战场 / 敌方恢复 / 后台战斗 / 安全领地活动`等可组合事实；互斥关系由L3声明 |
| 完成要求 | 存活占领阻断单位和未完成可见特殊目标的实际集合 |

`争夺中`可以没有当前主动战场；它表示节点保存了未完成战局。节点不保存隐藏控制百分比，敌我推进由单位、结构、空间和事件表达。

### 5.6 虫巢、槽位与房间状态

- 虫巢生命周期与所属区域着床事实一致；同一地区最多有一个当前虫巢实例，长期扩展若改变该约束须另行决策。
- 槽位只记录拓扑、尺寸兼容和占用，不记录预设功能。
- 房间模块的结构状态是`建设中 / 可运行 / 重组中 / 拆解中`之一；空槽不是房间状态。
- `玩家暂停`是明确指令事实。
- `缺料`、`缺能`、`堵塞`、`运行`和`待机`由任务、输入、已服务功率、容量及暂停事实派生，可以同时存在多个阻断原因；界面主状态只是优先摘要。
- 房间群由兼容模块的相邻连通分量重算，不保存可独立改变的第二份群状态。
- 运行策略和队列条目保存在模块上；房间群只派生共享管理视图与混合配置摘要。

### 5.7 过程任务状态

所有建设、重组、生产、解析和候选形成任务复用以下生命周期：

```text
验证通过 -> 已预留 -> 推进中 <-> 暂停 -> 待提交 -> 已完成
                    \-> 已取消
                    \-> 已失败
```

- `已预留`表示输入或容量已经唯一锁定，但尚未形成结果。
- `待提交`表示完成条件已满足，结果只能在统一事件边界提交一次。
- 取消、失败和完成都是终结状态，不能从同一任务再次产出。
- 不同任务的可取消投入和不可逆消耗由L3契约声明，不由任务名称猜测。
- 队列条目只表达尚未接纳的意图，不属于过程任务，也不建立输入或容量预留。

### 5.8 单位状态维度

每个单位批次至少分开记录：

- 单位模板、阵营和真实数量。
- 移动形态：`容纳 / 可机动 / 固定`。
- 唯一真实位置。
- 节点内空间段和互斥空间层：`接触 / 纵深 / 后备`。
- 虫群大军活动：`虫巢作业 / 领地采集 / 自由游荡 / 待命`。
- 剩余耐久与状态效果组成的规范状态队列。
- 至多一个核心行为的目标、进度、预留和最早生效步骤。

规范状态队列以“完整玩法键 -> 真实数量”保存无损状态单元。纯统计生产归因只保存在并列份额表中；只有明确改变资格、目标或效果的来源标签才进入玩法键。移动、受伤、死亡、被动范围和响应进攻/撤离不是额外核心行为。单位批次可以合并或拆分，但容器分片、插入顺序和纯统计归因不得改变逻辑个体状态、目标或结果。

### 5.9 发现、资格与构筑状态

“解锁”必须拆成不同事实：

| 维度 | 含义 |
| --- | --- |
| 发现 | 玩家是否见过对象或概念，是否进入图鉴 |
| 内容池资格 | 房间蓝图、单位模板或科技族是否可以进入对应选择池 |
| 当前可执行性 | 当前资源、空间、功率、样本、槽位和世界条件是否允许行动；始终派生 |
| 实例化 | 是否已经建造房间、形成单位、生成候选或同化突变 |
| 强化 | 等级、插件、突变、领袖和永久收益对实例或定义的显式修正 |
| 永久化 | 飞升后是否保留发现、资格或收益 |

发现不等于获得，内容池资格不等于当前可执行，候选出现不等于突变生效。

### 5.10 候选组状态

| 状态 | 含义 |
| --- | --- |
| `形成中` | 来源任务已经预留合法输入和槽位 |
| `来源缓冲` | 组已形成但合法持有槽暂不可用，保留在唯一来源缓冲 |
| `持有` | 整组占用一个受保护或普通槽，可查看和选择 |
| `已选择` | 一个成员已同化，同组其余成员已终结 |
| `已舍弃` | 普通组被主动删除且来源成本不返还 |

受保护首次组不能舍弃。候选成员不是独立持有资产，不能分别占槽。

所有候选组始终由进化状态唯一拥有；`来源缓冲`和`持有`只表示互斥的当前容器引用，不把候选组复制给房间、节点或槽位。

## 6. 资产与账目模型

### 6.1 三类核心资源

| 核心资源 | 权威含义 | 正式形成边界 |
| --- | --- | --- |
| 生物质 | 可用于结构和孵化的已同化物质 | 对应房间形成物理输出并由工蜂投递到账本接入口 |
| 能源 | 储能、储能上限、功率供给、负载和已服务功率组成的运行能力 | 能源房间完成合法形成事件；不是节点直接奖励 |
| 基因 | 可用于候选生成、同化和进化条件的已解析信息资源 | 样本组织由解析房间完成解析事件 |

核心资源全局共享，但每笔形成、预留、消费和返还必须追溯到来源对象与事件。

`轮次初始注入`在步骤0稳定边界首次发布前建立期初事实，不属于本轮正式入账，也不计入房间产出或外部同化。初始边界发布后，任何新增核心资源都必须遵循上表形成边界或合法结构返还。

### 6.2 过程资产五阶段

每个可采集过程资产批次任意时刻只处于一个阶段：

```text
现场 -> 已预留 -> 携带中 -> 已保障 -> 已回流 -> 房间处理
```

`已处理`、`腐败`、`敌方夺回`和`永久损失`是离开五阶段的终结事件。房间处理可以终结一种过程资产并形成另一种位于房间输出缓冲的`现场`过程资产。

`采集资产预留`锁定仍在现场的物质；`核心资源预留`锁定已经入账的库存。两者必须使用不同字段和不同释放规则。

### 6.3 统计账目

外部同化量、内部回收量、最高稳定供能、已处理样本、已同化突变、控制节点和完全吞噬区域属于统计事实，不是可消费资源。

- 己方尸体回收只增加内部回收，不增加外部同化。
- 节点占领只改变控制、来源、容量、修正和统计，不直接增加核心资源。
- 飞升预览只能读取同一步骤、同一参数版本的统计快照。

## 7. 守恒公式

### 7.1 核心资源账期

对生物质或基因的任一账期：

```text
期初可用 + 期初预留 + 本期入账与返还
= 期末可用 + 期末预留 + 本期已提交消费
```

房间结构完成时，结构预留成为已提交消费并同步增加模块的结构投入；拆除或降级形成新的返还事件。历史功率、运行输入和战损不因结构返还而倒转。

### 7.2 结构投入

```text
累计形成结构投入
= 当前有效结构投入
 + 累计已返还结构投入
 + 累计明确永久损失
```

移动、合并、拆分和未完成建设取消不能改变累计形成结构投入。

### 7.3 过程资产

```text
累计生成某类过程资产
= 现场 + 已预留 + 携带中 + 已保障 + 已回流
 + 累计已处理 + 累计腐败/夺回/永久损失
```

同一数量不能同时出现在两个阶段；转化事件必须同时终结输入并形成有来源的新输出。

### 7.4 单位

对每个单位模板：

```text
期初数量 + 累计形成 + 累计变形进入
= 虫巢内存活 + 虫群大军存活 + 全部节点内存活
 + 累计变形离开 + 累计死亡
```

等量变形对全部模板总和不增加单位。画面代表、当前战场、节点留置、活动和空间层不进入公式。

### 7.5 单次战役

```text
战役期初节点单位 + 本期进入 + 本期形成 + 本期变形进入
= 本期死亡 + 本期回归 + 战役结束节点单位 + 本期变形离开
```

战役结束后主动战场引用必须释放；固定单位仍可合法留在节点。

### 7.6 虫群大军活动

对每种可机动玩家单位：

```text
虫群大军
= 虫巢作业 + 领地采集 + 自由游荡 + 待命
```

所有节点的活动投影之和不得超过对应真实活动数量。

### 7.7 能源

```text
请求负载 = 基础负载 + 临时负载
请求净功率 = 总供给 - 请求负载
储能变化 = 在容量上下限内对实际充放能的模拟时间积分
```

供给不足时，L3能源契约按固定安全层级确定已服务功率；未服务负载不能继续推进任务，也不能形成隐藏能源债务。建设不能同时完整预扣能源并再次按时间结算同一消耗。

### 7.8 候选槽

```text
已持有组数 + 已预留槽数 <= 对应槽位容量
```

每组恰好占一个槽，组内成员数不改变槽位占用。来源缓冲中的组不占持有槽，但必须占用唯一来源缓冲并阻止覆盖。

## 8. 时间、命令与随机

### 8.1 模拟时间

- 所有玩法系统读取同一模拟时间和完整步骤序号。
- L3统一时间契约负责声明一个完整步骤内的事件顺序；其他系统只能引用。
- 新形成、刚变形或刚进入的单位何时首次生效，必须由步骤号而不是动画帧决定。
- 表现时间只负责追赶状态，离线经过时间只用于计算允许推进的系统。
- 存在主动战场时关闭游戏，整个世界冻结；不存在主动战场时才允许离线安全结算。

### 8.2 命令与事件

```text
玩家或系统形成命令
-> 按稳定顺序验证
-> 必要时原子预留
-> 立即提交，或建立过程任务
-> 任务达到提交边界
-> 形成一个或多个有因果顺序的事件
-> 更新事实并写入账目
```

命令是意图，事件是已经发生的事实。验证失败不能产生部分预留、部分消费、半个实例或重复事件。

`只读查询`只读取一个已经发布的稳定边界并形成可丢弃结果，不进入命令生命周期、不预留或消费、不推进随机流，也不形成玩法事件。打开飞升预览、读取房间详情和重复查看已揭示情报属于只读查询；会改变情报事实的`观察节点`仍是显式命令，不能用页面访问替代。

每个事件至少记录：

- 唯一事件标识和事件类型。
- 模拟时间、步骤号和步骤内顺序。
- 原因命令或上游事件。
- 来源、目标、所有者和位置引用。
- 变化前后状态与数量。
- 使用的规则版本和必要参数版本。

重复载入、动画回调和页面访问不能再次提交同一事件。

### 8.3 随机流

- 当前轮次保存根种子；地图、候选和其他需要随机的系统使用可区分的确定性子流。
- 每个随机流保存稳定标识和已消费位置，不能依赖容器遍历顺序。
- 已向玩家提交的区域概要、节点内容和候选组不能因读档、撤离、失败或构筑变化重抽。
- 情报只揭示已存在事实，不推进内容生成随机流。
- 地图和候选随机不能读取最近失败生成策略答案；领袖不能暗中修改地图流。
- 第一版战斗命中、伤害和伤亡不使用随机流。

### 8.4 保存、载入与重放

保存只能发生在完整步骤边界。权威快照至少包含：

- 状态版本、规则版本、参数版本、模拟时间、步骤号和事件序号。
- 当前轮次全部聚合根、跨轮进度及其稳定身份与引用。
- 有序待处理命令、队列条目、进行中的任务、资源/资产/容量预留、待提交输出、行为进度，以及单位、状态和修正实例的最早生效/失效步骤；步骤内事件草案和未提交界面草案不进入稳定快照。
- 有序命令队列、当前主动战场引用、世界运行状态和关闭时的现实时间戳。
- 全部确定性随机流标识、种子和游标。
- 地图与候选内容提交事实、一次性事件标识和累计守恒账目。

房间群、区域控制汇总、当前战场数量、节点留置数量、预计续航和画面代表等派生状态不作为必要真相保存。为性能保存缓存时，载入后必须与重算结果一致，否则丢弃缓存。

载入按以下逻辑边界执行：

```text
读取权威快照
-> 按状态版本执行确定性迁移
-> 校验身份、引用、所有权、位置、状态组合和随机游标
-> 校验资源、结构、过程资产、单位、战役和候选槽不变量
-> 重建全部派生状态
-> 确定在线恢复、离线安全结算或主动战场冻结
-> 原子发布为可游玩状态
```

- 迁移必须全部成功或保持原存档不变，不能留下半迁移状态。
- 非法引用、重复所有权、负数量、重复一次性事件和守恒失败不能通过静默补数、删除资产或重抽内容修复。
- 可以回退到已验证的上一份权威快照，但必须报告丢失的模拟区间；不能伪装成成功载入。
- 手动保存、自动保存和退出保存使用同一事实范围。
- 相同快照、规则、参数、随机流和后续命令必须能够确定性重放到相同状态。
- 账本可以按明确版本压缩历史明细，但必须保留当前守恒所需累计、未终结来源引用和幂等事件标识。

## 9. 跨域交易边界

下表只定义读写边界，具体检查和步骤由L3系统契约唯一维护。

| 交易 | 主要读取 | 原子提交结果 | 禁止的捷径 |
| --- | --- | --- | --- |
| 建设房间 | 蓝图资格、槽位、核心资源、功率条件 | 结构资源预留、槽位占用和建设任务 | 直接生成完成房间或双重扣能 |
| 取消未完成建设 | 建设任务和结构预留 | 返还全部结构预留、释放槽位和未来负载 | 返还已结算功率或时间 |
| 房间形成核心资源 | 输入资产、任务、功率、输出容量 | 终结输入、形成物理输出或网络结果；满足入账边界后记账 | 节点或动画直接加顶部资源 |
| 单位形成与分化 | 孵化任务、本地幼体、资源和功率 | 幼体形成或等量变形，单位进入唯一位置 | 把幼体当资源或复制前后形态 |
| 发起进攻 | 节点合法性、主动战场槽、可参战单位及工蜂可逆预留 | 释放参战工蜂的可逆采集预留，全部可参战单位转入目标节点并建立唯一主动战场引用 | 保留已转移工蜂的作业预留、选择投放比例、建立运输或复制军队 |
| 撤离 | 当前主动节点、单位移动形态和采集资产阶段 | 可机动单位回归、固定单位留置、资产原子换阶段、释放战场槽 | 撤离途中二次伤亡、容量不足拒绝或重复结算 |
| 节点占领 | 存活阻断单位和可见目标 | 控制权、来源、容量、修正、统计和移动单位回归 | 隐藏控制阈值或占领奖励包 |
| 样本解析 | 已回流样本组织、解析任务和功率 | 基因、样本标签、图鉴事件和候选资格事件 | 消化池直接产基因或绕过样本阶段 |
| 候选同化 | 持有组、成员条件、公开成本 | 一个突变生效、成本提交、整组终结并释放槽 | 只删除部分成员、隐藏收费或读档重抽 |
| 飞升 | 本轮账本快照、永久收益选择和领袖选择 | 后续版本原子结束轮次并建立新轮次 | 第一版执行、自动选择收益或保留旧地图细节 |

## 10. 派生状态与表现投影

以下对象必须由权威事实重算：

| 派生对象 | 来源事实 |
| --- | --- |
| 房间群 | 同虫巢槽位拓扑、已形成模块与群兼容键 |
| 当前战场数量 | 当前主动节点中的玩家单位 |
| 节点留置数量 | 各节点中的玩家固定单位 |
| 虫群大军活动摘要 | 虫群大军单位的活动字段 |
| 节点连通 | 图连接、已占路径、着床和特殊虫巢节点规则 |
| 区域控制汇总 | 已提交节点控制与区域级可见特殊目标 |
| 完全吞噬 | 全部节点控制和区域完成目标 |
| 能源净变化与预计续航 | 当前供给、请求负载、已服务功率、储能和容量 |
| 当前可执行性 | 内容资格、资源、空间、功率、槽位和目标状态 |
| 飞升预览 | 同一步骤统计账目与同一参数版本 |
| 画面代表实体 | 单位批次、空间、表现预算和可见性 |

表现投影可以减少、合并或跳过对象，但不得反向改变真实数量、目标选择、伤害、采集、资源、候选或时间。

## 11. 全局不变量

| ID | 不变量 |
| --- | --- |
| INV-001 | 每个定义、实例、命令和事件拥有稳定唯一身份，显示名称和容器顺序不作为身份 |
| INV-002 | 每个运行实例恰有一个直接所有者；单位批次只由单位总账拥有，过程资产批次只由过程资产总账拥有，节点和虫巢只作为其唯一位置或容器引用 |
| INV-003 | 派生状态和表现投影不能成为可独立修改的第二份玩法真相 |

## 12. 恢复可达性

第一版合法状态不能只满足静态守恒，还必须满足动态可恢复：

```text
可行动状态
= 当前存在至少一个合法玩家命令
或存在一个按统一时间继续推进后会形成合法命令的最低自治过程
```

无工蜂、零储能、全军覆没、房间堵塞、候选槽满和不理想突变可以叠加，但不得让存档永久离开可行动状态。最低恢复过程仍消耗已有来源、模拟时间、功率或结构机会，不等于免费返还损失。

恢复证明至少需要：

- 给出失败入口、保留资产、永久损失和最低恢复路径。
- 证明路径不要求已经全部损失的同类资产。
- 证明路径不依赖随机候选、第二虫巢或正式飞升。
- 证明取消、拆除、回收和应急直连不能形成净资源套利。
- 用状态搜索或极端夹具证明能够回到至少一个可持续生产并可进攻的状态。

## 13. 第一版状态边界

第一版实际实例化：一个当前轮次、一个完整提交全部节点内容的起始区域、一个已激活虫巢、一个全局经济、一个过程资产总账、一个单位总账、零或一个主动战场，以及首次与普通候选流程。玩家至少推进到区域出口，未选择支路仍保留其已提交事实。

第一版只读派生：相邻区域概要与着床入口、准确飞升点、永久收益目录和三位领袖对照。

第一版不得提交：正式飞升、新轮次重建、第二地区完整节点网、第二虫巢、多主动战场、已占节点失守、塔防和个体单位命令。

数据结构可以保留长期扩展身份，但第一版运行状态不能因预留字段而执行排除内容。

## 14. L3交接契约

后续每份L3系统契约必须声明：

1. 读取和写入哪些L2聚合根与对象。
2. 接受哪些命令，产生哪些事件。
3. 转换前置、原子提交点、取消、拒绝和恢复边界。
4. 与其他系统交换哪些稳定引用和账目条目。
5. 维护哪些INV编号。
6. 哪些值属于L6参数，不能写入机制正文。

建议按依赖顺序提取：统一时间与事件、资源能源、虫巢房间、单位行为与战斗、世界地图与节点、候选进化、飞升保存。L3不得分别维护不同的时间步骤、单位位置、过程资产阶段、候选生命周期或区域状态枚举。

## 15. L2完成条件

- 第一版每个运行对象均能归入唯一聚合根、唯一位置和明确生命周期。
- 同一术语在`L2_统一术语表.md`中只有一个规范含义。
- 区域、节点、房间、单位、过程资产、候选和解锁均使用正交状态，不再用单一模糊枚举覆盖多个事实。
- INV-001至INV-024都能映射到至少一个L3责任方和一个L6自动验收。
- 旧文档中的共享对象定义已降为迁移来源，不再覆盖L2。
- 未确定的数值、内容数量和表现方案没有被写成形式系统事实。

达到这些条件只表示模型边界封闭；模型是否可执行、可恢复和产生L1目标体验，仍需L3契约、无表现模拟和玩家测试证明。

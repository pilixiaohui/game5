## 1. 契约职责与写入边界

### 1.1 本文负责

- 世界区域图、区域、区域节点网、节点、空间段和连接的运行实例。
- 世界、区域和节点随机流如何派生、校验、提交与保存。
- 区域节点网提交、区域五个正交状态维度和节点四个正交维度。
- 节点连通、观察、进攻资格、持续战局、敌方恢复活动和安全领地。
- 节点初始单位、结构、空间、来源、环境和修正的生成请求。
- 节点占领、单位回归、来源开放、区域出口和完全吞噬的跨域事务。
- 新区域虫巢节点、着床安全区、虫巢建设与普通节点开放。
- 第一版受约束随机骨架和相邻区域只读概要。

### 1.2 本文不负责

- 单位移动、目标、伤害、死亡、变形、进攻转移与撤离；由`L3_单位与战斗契约.md`负责。
- 外部物质来源形成的过程资产数量、五阶段和核心资源入账；由资源契约负责。
- 虫巢槽位、房间和本地建设任务；由房间契约负责。
- 候选战利品组的抽取、持有与同化；由候选进化契约负责。
- 飞升确认、新轮次建立和跨轮保留；由飞升保存契约协调。
- 节点、敌人、生态和空间段的具体内容数量与表现布局。

### 1.3 唯一写入者

## 2. 三层世界模型

### 2.1 世界区域图

世界区域图至少保存：

- 本轮稳定身份、根种子、规则版本与随机流引用。
- 区域实例、世界深度、主推进关系和区域级支路。
- 有向世界连接、来源出口和目标区域引用。
- 已公开连接、尚未提交区域概要和下一层生成边界。

世界分支只发生在区域图。一个区域内部只有一个出口节点，但该出口可以公开一个或多个后续区域。

### 2.2 区域

每个区域至少保存：

- 区域身份、生态原型、深度、概要事实和生成版本。
- 节点网提交状态及提交事件。
- 一个虫巢节点引用和一个区域出口节点引用。
- 区域节点网、区域级可见特殊目标和虫巢引用。
- 情报、着床、出口与统计所需底层事实。
- 独立区域随机流及已消费游标。

区域没有单一总颜色状态。

### 2.3 节点

每个节点至少保存：

- 节点身份、所属区域、定义组合、标签与生成来源。
- 节点连接、情报、控制与活动事实。
- 连续空间段、左右入口、路径、容量、地表和推进锚点。
- 初始和当前敌我单位位置引用、可见特殊目标和持续战役引用。
- 外部物质来源、环境网络输入、抽取容量、房间修正与过程资产容器引用。
- 敌方恢复来源、有限储备、恢复进度与后台战斗事实。
- 占领、出口、首次样本和其他一次性事件标识。

节点不拥有第二份单位或过程资产库存。

### 2.4 空间段

空间段是节点内位置实例，保存连接、方向、容量、地表、视线、环境和推进锚点。它不是独立关卡，不拥有控制权、奖励或结算页。

## 3. 条件随机与原子提交

### 3.1 随机流分离

至少分离：

- 世界区域图流。
- 每个区域的节点网流。
- 每个节点的内容和空间组合子流。
- 纯表现随机流。

候选随机由进化契约独立维护。随机键由轮次、区域、节点和用途稳定身份派生，不能依赖容器遍历顺序。

### 3.2 生成顺序

```text
选择或读取已固定区域概要
-> 选择兼容生态原型
-> 放置结构职责与节点身份
-> 生成节点连接
-> 为每个节点生成空间、单位、结构、来源、环境、修正和特殊对象
-> 运行结构、内容、可达、经济和恢复校验
-> 原子提交全部区域内容与随机游标
-> 建立情报遮罩
```

任一校验失败时，候选结果与游标只存在于事务副本。系统使用确定性下一尝试；达到参数化尝试上限后使用已验证安全骨架，并继续由当前种子决定非关键内容。

### 3.3 提交边界

- 起始区域在步骤0轮次建立事务中完整提交。
- 后续区域的概要可以先于节点网提交，并在公开时固定。
- 后续区域首次确认进攻虫巢节点时，先完整生成并提交该区域，再执行进攻。
- 取消进攻、命令拒绝或生成失败不留下半张地图，也不推进正式随机游标。
- 区域提交后，情报、撤离、失败、构筑和读档不能重抽任何内容。

### 3.4 禁止自适应答案

地图生成不得读取：

- 当前军队数量或单位比例。
- 最近一次战损或撤离。
- 当前突变构筑。
- 当前库存不足。
- 领袖身份的隐藏地图权重。

深度压力必须落实为内容、空间、来源和公开环境规则，不使用追赶式隐藏倍率。

## 4. 区域正交状态

### 4.1 节点网提交

| 状态 | 含义 |
| --- | --- |
| `未提交` | 只存在稳定概要和虫巢节点入口，不存在可玩的内部节点网 |
| `已提交` | 全部节点、连接和内容已原子建立，本轮不可重抽 |

第一版起始区域为`已提交`，相邻区域保持`未提交`。

### 4.2 区域情报

| 状态 | 最低已知事实 |
| --- | --- |
| `未发现` | 不显示位置和主题 |
| `概要已知` | 生态主题、世界连接、资源倾向、压力摘要和虫巢入口 |
| `节点网已知` | 已提交节点网的已揭示连接和轮廓 |
| `深入侦察` | 部分节点更精确的单位、空间、来源和样本信息 |
| `完全解析` | 当前规则允许揭示的区域事实达到最高精度 |

情报不创建对象、不改变随机流，也不生成推荐构筑。

### 4.3 区域着床

| 状态 | 转换条件 |
| --- | --- |
| `不可着床` | 没有来自已打开出口的合法世界连接 |
| `待着床` | 至少一个合法世界连接已公开 |
| `虫巢节点争夺中` | 目标虫巢节点是主动战场 |
| `着床安全区` | 虫巢节点已控制，普通反扑停止 |
| `虫巢建设中` | 虫巢结构任务已合法建立 |
| `虫巢已激活` | 虫巢实例完成，普通节点和房间建设开放 |

初始区域在步骤0直接建立已激活虫巢，并保存其初始来源事件。非初始区域不能跳过安全区和建设。

### 4.4 节点控制汇总

| 派生值 | 条件 |
| --- | --- |
| `敌方占据` | 本地虫巢未激活，或尚无普通己方控制节点 |
| `区域争夺中` | 虫巢已激活且至少一个已提交节点未控制 |
| `完全吞噬` | 全部节点控制且区域级可见特殊目标完成 |

完全吞噬只更新统计和飞升贡献，不形成第二次节点收益。

### 4.5 出口通路

| 状态 | 含义 |
| --- | --- |
| `出口未知` | 出口身份尚未揭示 |
| `出口已知` | 出口身份已知但不可进入 |
| `出口可达` | 至少一条己方控制路径连接出口 |
| `出口争夺中` | 出口存在主动或保存的未完成战局 |
| `通路已打开` | 出口已经控制，世界连接已公开 |

通路打开后，本轮不会因支路恢复关闭。

## 5. 节点正交状态

### 5.1 节点情报

`未揭示 / 轮廓已知 / 可观察 / 已解析`严格只描述可见深度。相邻关系在第12阶段自动形成`轮廓已知`；玩家对合法相邻节点提交一次`观察节点`命令后，在第1阶段提升到`可观察`。重复查看详情是只读查询，不能代替命令或继续提升情报；侦察效果只提高允许精度。

### 5.2 节点连通

`不可达 / 可达`是派生事实：

- 普通节点要求本区域虫巢已激活，并存在从虫巢节点经己方控制节点到达目标的合法连接。
- 待着床虫巢节点只要求存在来自已打开出口的世界连接。
- 节点标签、敌军数量和当前构筑不参与拓扑可达性。

### 5.3 节点控制

| 状态 | 含义 |
| --- | --- |
| `敌方控制` | 尚无保存的己方争夺事实，或从未进入 |
| `争夺中` | 存在主动战场，或保存了未完成敌军、菌毯、残骸与推进事实 |
| `己方控制` | 阻断单位和可见特殊目标全部完成 |

撤离后可以处于`争夺中`但没有主动战场。连通与控制不能合并为一个状态。

### 5.4 节点活动

以下活动按规则组合：

- 主动战场。
- 敌方恢复。
- 后台战斗。
- 安全领地活动。
- 外部物质来源形成与工蜂采集，以及环境网络输入服务。
- 观察与只读表现。

第一版`己方控制`节点只允许安全领地活动、来源和观察，不允许敌方恢复或战斗。

## 6. 节点进攻资格与持续战局

### 6.1 进攻资格

普通节点必须同时满足：

- 区域节点网已提交。
- 区域虫巢已激活。
- 节点至少`可观察`且派生为`可达`。
- 节点不是己方控制的安全节点。
- 主动战场槽空闲。
- 单位契约报告至少一个可参战单位。

后续区域虫巢节点使用着床例外。第一版相邻区域入口明确禁用进攻命令。

### 6.2 持续节点

进攻、撤离、全灭和再次进入都读取同一节点实例：

- 敌军、出兵源、储备、固定结构和当前耐久保持。
- 玩家固定菌毯、残骸、样本和过程资产保持。
- 空间、路径、最远位置和已摧毁对象保持。
- 敌方恢复只通过已生成来源形成实际变化。

退出节点不会重置关卡，也不形成抽象失败奖励或惩罚。

### 6.3 主动战场建立

第2阶段原子提交：

1. 再次验证节点、连通、情报和主动槽。
2. 必要时先提交完整区域地图。
3. 调用单位契约转入全部合法可参战单位。
4. 节点控制进入或保持`争夺中`。
5. 保存战役开始步骤与幂等标识。

## 7. 节点内容、来源与环境

### 7.1 长战场最低事实

每个可战节点必须有：

- 玩家左侧入口、敌方右侧入口和连续空间段。
- 至少一条基础单位可通行路径。
- 接触、纵深、后备与交互容量。
- 所有完成必需单位和特殊目标的可见或可揭示方式。
- 撤离无需返回路径。

### 7.2 外部物质来源

外部物质来源保存类型、储量或流量、抽取容量、形成位置、物质谱系、恢复规则和兼容处理链。

- 节点控制只开放来源资格，不增加核心资源。
- 来源在合法阶段与资源契约同笔减少可抽取量并在来源位置形成`现场`过程资产。
- 工蜂必须按单位契约预留、携带、保障并回流到房间物理输入端。
- 源容量不足时停止形成，不允许房间远程终结储量。
- 来源可以有限、持续流量或周期恢复，但每种机制都必须有可追溯事件。

### 7.3 环境网络输入与房间修正

- 热化梯度等环境输入不形成物理资产，向资源契约提供环境容量与来源身份。
- 热化代谢腔等持续网络贡献必须竞争实际环境容量。
- 房间修正只修改明确房间关系、参数或资格，从下一合法步骤生效。
- 环境输入和房间修正不能直接增加当前储能、生物质或基因。

### 7.4 特殊战利品

特殊战利品是具体单位死亡输出、结构对象或过程资产。它必须经过保障与声明处理链；候选进化契约只在来源结果合法提交后生成候选组。

## 8. 敌方恢复、后台战斗与安全领地

### 8.1 敌方恢复来源

节点生成时明确每个恢复来源：

- 存活出兵点。
- 右侧有限储备与入口。
- 明确环境恢复规则。
- 修复或生成单位的可见单位行为。

恢复来源死亡、储备耗尽或资格终止后不能因退出或读档恢复。

### 8.2 无玩家固定单位

未占节点没有玩家固定单位时，第3阶段推进恢复来源和敌方位置。恢复不是直接修改“敌占百分比”，而是形成单位、修复合法对象或推动实际战线。

### 8.3 有玩家固定单位

留置菌毯与恢复敌军并存时，节点进入后台战斗：

- 世界契约提供节点、恢复和空间事实。
- 单位契约执行完整战斗。
- 后台满足占领条件时可以直接请求占领。
- 后台不创建画面实体、不占主动战场槽。

### 8.4 安全领地

节点己方控制后：

- 停止普通敌方恢复、出兵与后台战斗。
- 不发生控制反转和隐藏战损。
- 保留未回收残骸、固定菌毯、外部来源、环境输入和房间修正。
- 工蜂采集和战斗单位游荡使用虫群大军活动投影。
- 特殊样本恢复源可以按自身软锁规则继续形成过程资产，但不能生成敌军或撤销控制。

## 9. 节点占领原子事务

### 9.1 占领条件

第12阶段只读取：

```text
存活敌方占领阻断单位集合为空
并且
未完成可见特殊目标集合为空
```

不读取隐藏控制值、战斗时长、推荐兵力或是否打开结算页面。

### 9.2 原子提交

占领事务必须同时：

1. 再次验证阻断和特殊目标事实。
2. 把节点控制设为`己方控制`。
3. 调用单位契约让玩家可机动单位回归，固定单位留置。
4. 结束主动或后台战役活动并释放主动槽。
5. 停止普通恢复来源。
6. 开放外部来源、环境容量和房间修正，从下一步骤生效。
7. 更新控制节点统计和一次性里程碑。
8. 若为虫巢节点，进入着床安全区。
9. 若为区域出口，提交打开通路子事务。

任一子结果失败则整笔不发布。节点单位死亡输出和资产仍按各自阶段保留，不转为奖励包。

### 9.3 幂等

每个节点只有一个占领事件标识。主动与后台同时观察到条件时只能有一笔获胜；重复检查、读档和再次进入不重复开放来源、统计或出口。

## 10. 区域出口与完全吞噬

### 10.1 唯一出口

每个区域提交时恰有一个节点带`区域解锁`标签。它仍使用普通节点战斗和控制规则。

### 10.2 打开世界通路

出口首次占领时：

- 将出口通路设为`通路已打开`。
- 公开世界区域图已经声明的后续连接。
- 固定相邻区域生态主题、深度、资源倾向、压力摘要和虫巢入口。
- 相邻区域进入`概要已知 + 待着床`。
- 不自动生成其完整节点网。

一个出口可以公开多个区域支路，但区域内部不增加第二出口。

### 10.3 完全吞噬

区域节点网已提交、全部节点控制且区域级可见目标完成时，派生完全吞噬并提交一次统计事件。它不关闭来源、不删除支路、不发放核心资源，也不要求玩家停止继续游玩。

## 11. 新区域着床与多虫巢边界

### 11.1 长期着床流程

```text
前一区域出口已打开
-> 相邻区域待着床
-> 首次确认进攻时提交完整节点网
-> 全军进攻虫巢节点
-> 节点占领并进入着床安全区
-> 建立虫巢建设任务
-> 虫巢完成激活
-> 区域普通节点和本地房间建设开放
```

### 11.2 着床安全区

- 虫巢节点清除后普通反扑停止。
- 建设期间不发生塔防或隐藏伤亡。
- 玩家可以延后、取消或重新建设。
- 取消只返还未消费结构预留，不返还功率与时间。
- 未激活虫巢前，普通节点不可进攻，其他房间不可建设。

### 11.3 第一版边界

第一版只显示相邻区域概要、虫巢入口与建巢条件。进攻入口为明确`尚未开放`，不提交第二地区节点网、单位、来源或虫巢任务。

## 12. 首轮受约束地图

### 12.1 职责锚点

起始区域必须含：

| 锚点 | 固定职责 |
| --- | --- |
| H 腐殖母床 | 已激活虫巢、自由空槽、初始外部来源和恢复闭环 |
| C 共振前庭 | 首次小规模接触、固定样本与可恢复样本源 |
| E 燃脉裂隙 | 非唯一前置的环境能源支路 |
| B 腐殖渗床 | 非唯一前置的可再生外部物质支路 |
| P 出兵源压力 | 受限接触和可见可破坏出兵源 |
| W 蜕壳遗场 | 可选回收与特殊战利品支路 |
| X 壳门长廊 | 唯一区域出口和综合长战场 |

H、C、X唯一；C、P、X必须为不同节点。第一版必须至少生成一个W职责节点，但W是可跳过的支路，不能承担首次样本、出口或唯一恢复职责。

### 12.2 拓扑硬约束

- H只通过C进入网状部分。
- C后至少两个分支和一个回环。
- C到X至少两条内部节点不相交合法路径。
- 每条合法出口路径经过至少一个P职责。
- 移除全部P后X不可达；移除任一单个P后仍有另一条路径。
- E和B均不是所有出口路径的唯一前置。
- 所有节点从H可达，只有X有`区域解锁`标签。

### 12.3 内容与软锁约束

- C包含固定样本体和独立恢复孵育床。
- 基础噬咬体可以低效处理所有必要阻断结构。
- 不占领E和B仍存在低效完成路径。
- 第一样本损失后不依赖节点敌方控制即可恢复。
- 恢复载体不形成普通资源、重复候选或飞升成果。
- 所有完成必需目标可见、可达且有有限来源或失效条件。

### 12.4 允许随机

可以改变非硬约束连接、职责附加标签、敌军组合、空间段排列、来源位置、残骸、房间修正和相邻概要。不得改变硬约束、基础可完成性和固定样本存在。

## 13. 情报揭示

### 13.1 基础揭示

- 相邻区域：主题、世界连接、资源倾向、压力摘要和虫巢入口。
- 节点轮廓：位置、已知连接和模糊标签。
- 相邻观察：地形倾向、已发现敌人类型、出兵迹象、来源和残骸类别。
- 进入节点：当前可见单位、结构、空间、来源和持续状态。
- 已解析：更准确数量、抗性、样本和规则事实。

基础观察不需要侦察单位，必须提供承担风险所需事实。

### 13.2 情报修正

侦察单位、房间、突变和领袖只能：

- 提高揭示范围。
- 提高数量或属性精度。
- 更早识别样本、来源和环境标签。

它们不能新增地图对象、重抽内容或生成策略推荐。

### 13.3 已知与未知边界

界面必须区分已知精确事实、已知区间和未知字段。隐藏字段不能以胜率、战力或推荐兵种替代。

## 16. 事件与跨域接口

### 16.1 世界事件

| 事件 | 提交边界 | 原子结果 |
| --- | --- | --- |
| 世界区域图已建立 | 步骤0或后续飞升 | 区域概要、连接与世界随机流提交 |
| 区域节点网已提交 | 步骤0或第2阶段 | 全部节点、内容和游标一次建立 |
| 情报已揭示 | 第1或第12阶段 | 观察命令或相邻/世界事件只改变可见事实集合 |
| 节点战役已开始 | 第2阶段 | 节点活动与单位转入一致 |
| 节点已占领 | 第12阶段 | 控制、回归、恢复停止和来源开放 |
| 区域出口已打开 | 第12阶段 | 通路和相邻概要提交一次 |
| 着床状态已改变 | 第2、第11或12阶段 | 节点、虫巢任务和区域资格一致 |
| 区域已完全吞噬 | 第12阶段 | 只更新统计与飞升贡献 |

### 16.2 资源房间接口

- 提供来源、储量/流量、抽取容量、现场形成位置与物质谱系。
- 提供环境网络输入容量和房间修正。
- 接收过程资产生成或来源扣减子事务。
- 为采集交互预留束返回节点路径与来源交互容量句柄；第2阶段按进攻事务标识无副作用准备并原子释放最终参战工蜂的句柄。
- 不直接写生物质、基因、储能和房间任务。

### 16.3 单位接口

- 地图提交请求形成初始敌我与中立单位。
- 进攻提供节点、空间和入口；单位契约转移数量。
- 占领只读取单位契约提供的阻断与特殊目标集合。
- 敌方恢复提供合法形成来源，不直接计算伤亡。

### 16.4 候选接口

节点只提供特殊战利品、样本标签或候选来源资产。候选组由进化契约生成，地图不预抽组内成员。

## 17. 保存、离线与确定性

### 17.1 必须保存

- 世界、区域、节点、连接、生态、深度和生成版本。
- 所有随机流种子、标识、游标、尝试或安全骨架身份。
- 区域节点网提交、情报、着床、出口与一次性事件事实。
- 节点情报、控制、空间、来源、环境、修正、恢复与持续战局引用。
- 未终结采集交互预留束的节点路径/来源交互句柄、精确份额、状态和最近释放事务标识。
- 已提交全部隐藏内容身份；情报遮罩单独保存。
- 相邻区域已公开概要和第一版禁用入口。

派生连通、区域汇总和地图主标签可以重算，不保存第二份真相。

### 17.2 离线

- 无主动战场时推进敌方恢复、后台战斗、来源流量、领地采集和样本恢复。
- 有主动战场时整个世界冻结。
- 已占节点不会离线生成战斗损失。
- 后台占领可以在离线第12阶段提交，并进入摘要。

### 17.3 确定性

- 相同轮次种子、规则和生成版本得到相同世界与区域结果。
- 生成失败尝试和安全骨架选择可重放。
- 读档不重新运行已提交区域生成器。
- 情报、页面、画面位置和代表实体不改变隐藏内容。

## 18. 第一版范围与排除

第一版实际支持：

- 一个完整提交并可推进的腐殖盆地。
- 一个已激活初始虫巢。
- H/C/E/B/P/W/X职责和受约束随机拓扑。
- 持续节点、敌方恢复、后台菌毯战斗、占领、安全来源与出口。
- 一个或多个相邻区域稳定概要、待着床入口和建巢条件。

第一版排除：

- 进攻第二区域虫巢节点。
- 第二地区节点网提交和第二虫巢。
- 无限深度实际生成。
- 已占节点失守、塔防和虫巢破坏。
- 领袖修改地图生成。

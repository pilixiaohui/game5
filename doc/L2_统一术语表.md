## 1. 使用规则

1. 描述状态变化时必须使用最具体的规范术语，不能只写“资源变化”“已经回收”或“区域完成”。
2. 同一对象的内容定义、运行实例、派生摘要和表现实体必须使用不同术语。
3. 术语表定义“是什么”，`L2_形式系统总模型.md`定义“归谁所有并如何保持自洽”，L3定义“怎样变化”。
4. 状态词使用反引号标识；内容名称和玩家可见短名不承担状态语义。
5. 新术语必须先证明现有术语无法表达一类可复用事实，再同时更新本文、形式模型和受影响的验收。
6. 英文或稳定数据标识可以存在，但中文设计文档必须能追溯到本文的规范中文术语。

## 2. 文档与数据层术语

| 规范术语 | 定义 | 边界 |
| --- | --- | --- |
| 存档 | 一个本地存档槽中的跨轮进度和零或一个当前轮次 | 不等同于当前地图或单次战斗 |
| 跨轮进度 | 飞升后仍保留的图鉴、领袖、永久收益和历史记录 | 不包含本轮普通资源、地图和突变 |
| 当前轮次 | 从本轮世界建立到正式飞升或终止之间的全部运行事实 | 第一版创建但不会正式结束于飞升 |
| 当前轮次生命周期 | 当前轮次从不存在、建立、运行、飞升选择与提交到结束的正交状态维度 | 世界运行状态或战役状态 |
| 轮次建立事务 | 由飞升保存契约协调、在步骤0原子建立世界、经济、虫巢、单位、进化和版本事实的跨域事务 | 资源初始注入或普通完整步骤 |
| 内容定义 | 描述某类房间、单位、节点、候选等稳定规则的模板 | 没有运行数量、位置和当前状态 |
| 运行实例 | 当前存档中具有唯一身份、所有者和生命周期的真实对象 | 不是场景节点或显示卡片 |
| 权威事实 | 已由合法事件提交、必须保存或可确定重放的玩法事实 | 只能通过交易改变 |
| 派生状态 | 从权威事实唯一重算的分组、汇总、资格或摘要 | 不能成为第二份可独立修改的真相 |
| 表现投影 | 场景、动画、代表实体、声音和界面文本 | 不参与结算 |
| 聚合根 | 拥有一组实例并负责其内部一致性的最高运行对象 | 跨聚合变化必须通过明确交易 |
| 定义标识 | 稳定引用一个内容定义的身份 | 不使用显示名称代替 |
| 实例标识 | 稳定引用当前存档中一个实例的身份 | 不使用数组位置或场景路径代替 |
| 标签 | 表达对象资格、筛选或组合关系的稳定标记 | 不是生命周期状态或隐藏数值 |
| 参数 | 成本、速度、容量、概率、倍率等可调量 | 不能改变对象所属类型和所有权 |
| 内容实例 | 内容定义在当前轮次中的运行实例 | 与“已解锁内容”不同 |
| 规则版本 | 决定状态转换语义的版本 | 与平衡参数版本分开 |
| 参数版本 | 决定本次计算所用数值集合的版本 | 不得改变规则结构 |
| 全局经济 | 当前轮次中拥有生物质、能源、基因和统计账目的聚合根 | 虫巢本地过程资产或单位总账 |
| 生物质与基因账目 | 记录两种离散核心资源可用量、预留、入账和消费的权威账目 | 储能与功率组成的能源状态 |
| 能源状态 | 记录储能、上限、供给、请求负载和已服务功率的权威运行事实 | 单一库存数字 |
| 过程资产总账 | 当前轮次中唯一拥有全部过程资产批次、阶段、位置和终结累计的聚合根 | 节点、虫巢或房间中的第二份资产库存 |
| 单位总账 | 当前轮次中唯一拥有全部阵营单位批次及形成、变形和死亡累计的聚合根 | 节点、虫巢或画面中的第二份单位库存 |
| 进化状态 | 当前轮次中拥有科技族资格、候选组、槽位预留和本轮突变的聚合根 | 跨轮图鉴和永久收益 |
| 事件账本 | 按稳定顺序保存已提交事件及其因果、来源、目标和变化的审计事实 | 再次执行事件的命令队列 |
| 条件 | 对权威事实和合法派生状态进行确定性判断的无副作用谓词 | 事件、消耗或随机抽取 |
| 效果配方 | 声明条件成立时可以形成哪些事件及其输入输出的内容规则 | 已经提交的效果事件 |
| 修正实例 | 由突变、插件、领袖、永久收益、区域或房间形成的运行规则变化 | 无来源的隐藏倍率 |
| 作用范围 | 通过对象类型、所有者、位置、标签和空间关系筛选修正目标的规则 | 显示名称或画面实体筛选 |
| 生命周期 | 一个实例或修正从生效到终止的明确时间边界 | 模糊的“长期有效” |
| 叠加规则 | 刷新持续、取最强、独立实例、固定叠层、互斥或规则替换等组合语义 | 依赖载入顺序的覆盖 |

## 3. 世界与地图术语

| 规范术语 | 定义 | 不应混同 |
| --- | --- | --- |
| 世界 | 当前轮次的世界区域图及其已提交区域 | 跨轮进度 |
| 世界区域图 | 区域实例、区域连接、主深度方向和区域级支路组成的图 | 区域内部节点网 |
| 世界连接 | 两个区域之间由区域出口公开的有向可达关系 | 节点连接或单位运输路线 |
| 世界深度 | 区域在本轮主推进关系中的层级事实 | 隐藏敌方伤害倍率 |
| 区域 | 具有一个生态原型、一个节点网和一个出口的世界层容器 | 单个关卡 |
| 区域生态原型 | 约束区域兼容节点、敌人、环境、空间和来源池的内容定义 | 已生成区域实例 |
| 区域节点网 | 一个区域中节点与连接构成的已提交图 | 世界区域图 |
| 节点网提交 | 校验通过后一次建立区域全部节点、连接和内容的原子世界事件 | 情报揭示或逐节点随机生成 |
| 节点 | 可持续进入并可被占领的关卡实例；单位和过程资产以唯一位置引用指向它 | 静态菜单按钮或独立单位库存 |
| 关卡 | 面向玩家的节点挑战称呼；设计规则涉及状态时一律使用“节点” | 新的对象类型 |
| 长战场 | 节点内由连续空间段组成、可保存的战斗与采集空间 | 一次性战斗结算页 |
| 空间段 | 长战场中的位置、连接、容量、地表和推进锚点单元 | 世界或区域节点 |
| 节点连接 | 区域节点网中两个节点之间的拓扑边 | 世界连接 |
| 节点连通 | 根据节点连接、已占路径和着床条件派生的`不可达 / 可达`事实 | 节点控制 |
| 节点情报 | 玩家对一个已存在节点事实的揭示深度 | 节点内容生成 |
| 区域情报 | 区域级已知主题、节点网和解析深度 | 区域控制 |
| 节点控制 | `敌方控制 / 争夺中 / 己方控制`之一 | 连通、活动或隐藏百分比 |
| 区域控制汇总 | 由节点控制与区域特殊目标派生的`敌方占据 / 区域争夺中 / 完全吞噬` | 虫巢是否激活 |
| 区域出口 | 每个区域唯一带`区域解锁`标签的普通节点 | 独立“区域核心”类型 |
| 通路已打开 | 区域出口已占领并已公开后续世界连接的本轮事实 | 完全吞噬 |
| 完全吞噬 | 区域全部已提交节点和区域可见特殊目标均完成的派生统计 | 打开出口或获得资源包 |
| 虫巢节点 | 新区域中用于争夺着床安全区和建立虫巢的节点身份 | 自动拥有的虫巢 |
| 着床 | 从世界可达、争夺虫巢节点到激活本地虫巢的区域建立过程 | 单位运输或驻军 |
| 着床安全区 | 虫巢节点已占领但本地虫巢尚未激活的安全状态 | 已激活虫巢或塔防阶段 |
| 主动战场 | 当前有玩家可机动单位在场、允许撤离命令且占用全局战场槽的节点 | 只剩固定菌毯的节点 |
| 当前主动战场引用 | 全局指向零或一个主动战场节点的单例引用 | 第二份战场单位库存 |
| 后台战斗 | 未占节点中留置固定单位与敌方恢复单位按完整战斗规则推进 | 比例扣兵或表现动画 |
| 敌方恢复 | 由存活出兵源、入口、修复行为或环境规则形成实际敌方状态变化 | 退出后关卡重置 |
| 安全领地 | 第一版中已经己方控制、不会生成敌军和战斗损耗的节点 | 独立驻军容器 |
| 领地活动 | 虫群大军在已占节点的采集或游荡活动投影 | 单位位置转移或驻军 |
| 节点标签 | 资源、样本、精英、Boss、区域解锁等可组合资格 | 节点生命周期状态 |
| 外部来源 | 外部物质来源与环境网络输入的上位概念 | 核心资源奖励或单一运行对象类型 |
| 外部物质来源 | 在明确位置形成可搬运过程资产、仍需工蜂回流和房间处理的外界来源 | 环境网络输入或远程房间输入 |
| 环境网络输入 | 不形成可搬运过程资产、按环境容量服务兼容持续网络贡献的外界条件 | 当前储能、功率供给或房间修正本身 |
| 抽取容量 | 单位时间内可以从外部物质来源形成现场过程资产的能力上限 | 环境容量或已入账产量 |
| 环境容量 | 一个环境网络输入在同一步可分配给兼容持续网络贡献的服务上限 | 抽取容量、房间缓冲或功率供给 |
| 房间修正 | 节点或其他对象对明确房间规则施加的显式修正 | 节点直接产出 |

## 4. 虫巢、房间与任务术语

| 规范术语 | 定义 | 不应混同 |
| --- | --- | --- |
| 虫巢 | 区域中的生产聚合根，拥有槽位、房间、任务和本地容器；幼体与资产以唯一位置引用指向容器 | 全局虫群大军、单位总账或全局经济 |
| 虫巢基础设施 | 由虫巢直接拥有、不使用普通房间建设与生产队列的固定设施 | 可建设房间模块 |
| 核心巢室 | 第一版预置的虫巢基础设施，提供拓扑锚点、账本接入和最低恢复接口 | 免费资源生产房间或普通房间模块 |
| 自由空槽 | 没有预设功能、没有房间模块且没有结构事务排他声明的槽位 | 指定功能建筑位 |
| 槽位拓扑 | 槽位及有效相邻边构成的虫巢空间图 | 屏幕像素相邻 |
| 有效连接边 | 槽位拓扑中已经提交、可用于相邻、路径与端口求值的连接关系 | 画面接触或场景树关系 |
| 房间蓝图 | 允许建立某类房间模块的内容定义资格 | 已建房间实例 |
| 房间模块 | 占据槽位并保留结构投入、等级、运行策略、任务和缓冲的最小房间实例 | 整体显示的大房间 |
| 群兼容键 | 由房间族、改变主要输入输出的分支和必要运行绑定组成的共享任务兼容身份 | 显示名称、等级或临时状态 |
| 房间群 | 相邻且兼容的同类房间模块连通集合形成的派生管理整体 | 合成后新增的房间实例 |
| 结构投入 | 已从核心资源提交并固化在房间模块中的可追溯材料价值 | 历史功率或运行输入 |
| 结构事务 | 建设、升级、移动、功能重组、降级、拆除或替换的预留、推进与原子提交过程 | 即时布局编辑或隐藏建设队列 |
| 过程任务 | 建设、重组、生产、处理、解析或候选形成的可推进运行实例 | 玩家任务或教程目标 |
| 建设任务 | 预留结构资源并在功率支持下形成房间模块的过程任务 | 即时购买 |
| 重组任务 | 升级、移动、功能重组、降级、拆除或替换中改变模块结构事实的耗时耗能过程任务 | 材料惩罚或房间群自动拆分 |
| 运行策略 | 模块保存的配方、循环模式、目标指标、运行绑定和开关声明 | 已锁定输入的过程任务 |
| 队列条目 | 尚未分配执行模块、尚未锁定输入的一次生产意图或显式批次 | 过程任务或已完成产物 |
| 队列 | 从房间群成员模块的队列条目与过程任务派生的共享管理和接纳顺序 | 拥有第二份任务的房间群库存 |
| 自动循环 | 在合法输入、功率和容量满足时自动建立后续同类任务的模式 | 无条件离线产出 |
| 目标规模 | 单位生产房间希望维持的指定模板可机动存活量 | 本次进攻投放数量 |
| 目标指标 | `维持目标`策略明确登记的数量定义与作用范围；第一版单位生产使用目标规模 | 页面当前筛选或模糊“兵力” |
| 目标覆盖承诺 | 已建立过程任务对尚未提交目标产量的唯一占用事实 | 已形成单位或无限循环意图 |
| 输入缓冲 | 已位于兼容房间、等待任务处理的本地过程资产容器 | 全局资源库存 |
| 输出缓冲 | 房间已形成但尚未完成投递或转移的物理结果容器 | 已入账资源 |
| 容量承诺 | 任务建立时为精确输出或交接结果唯一锁定的目标容量 | 已经形成的产物 |
| 账本接入口 | 接收兼容物理输出并原子终结资产、形成核心资源入账的虫巢端口 | 手动领取按钮 |
| 物理输出 | 任务完成后在输出缓冲形成、仍需搬运或继续处理的过程资产 | 已入账核心资源 |
| 网络输出 | 任务完成后直接提交到能源、突触或条件网络的非搬运结果 | 无来源的即时奖励 |
| 持续网络贡献 | 房间在合法环境容量下逐步骤形成供给或储能上限等网络能力的运行事实 | 生产批次、过程任务或免费常量加成 |
| 外部物质来源端 | 登记外部物质来源、抽取上限、现场形成位置和兼容处理链的接入端 | 把节点物质直接传入房间的远程管道 |
| 环境网络输入端 | 读取不形成可搬运物质的环境容量或房间修正、供持续网络贡献竞争的接入端 | 过程资产输入或节点直接供能 |
| 交互端口 | 工蜂取货、投递或通行所使用并受容量约束的位置 | 画面装饰 |
| 采集交互预留束 | 一次工蜂取货尝试对资产、路径、源交互端、投递端和活动份额的同源跨域预留引用集合；各事实仍由原聚合根拥有 | 新资产库存、独立运输任务或可以部分释放的松散引用 |
| 玩家暂停 | 玩家明确停止房间群或循环继续建立与推进可中断任务的指令事实 | 缺料、缺能或全局暂停 |
| 堵塞 | 合法输出、输入、槽位或容量没有去向而使任务不能继续的事实 | 资源被删除 |
| 缺料 | 当前任务没有足够兼容输入 | 输入永久不存在 |
| 缺能 | 当前任务没有获得其所需的已服务功率或启动储能 | 能源资源为零的唯一情况 |
| 运行 | 当前模块至少有合法任务正在得到输入和已服务功率 | 动画正在播放 |
| 待机 | 当前没有可推进任务且不是玩家暂停或故障 | 房间被删除 |

## 5. 资源、资产与账目术语

| 规范术语 | 定义 | 不应混同 |
| --- | --- | --- |
| 核心资源 | 正式入账后可被全局规则消费的生物质、能源和基因三类资源 | 过程资产、单位或统计量 |
| 正式入账 | 在合法形成边界终结输入并增加全局生物质或基因可用量的原子事件；能源使用网络贡献进入能源状态 | 拾取、保障、回流或动画完成 |
| 轮次初始注入 | 当前轮次步骤0稳定边界发布前唯一一次建立期初资源与能源的事务 | 房间产出或节点奖励 |
| 生物质 | 已完成同化并可用于结构、幼体和单位形成的物质库存 | 残骸或生物质浆 |
| 能源 | 储能、储能上限、功率供给、负载和已服务功率组成的系统 | 只有一个可消费整数的货币 |
| 当前储能 | 能源系统当前可释放的累积量 | 功率供给速率 |
| 储能上限 | 当前储能可以达到的容量上限 | 当前储能 |
| 功率供给 | 每单位模拟时间可服务负载或充能的能力 | 已经消耗的能源总量 |
| 基础负载 | 房间维持和持续运行请求的功率 | 建设等临时任务负载 |
| 临时负载 | 建设、重组、过载等有明确来源和期限的功率请求 | 永久基础负载 |
| 请求负载 | 基础负载与临时负载之和 | 实际已服务功率 |
| 已服务功率 | 按能源安全层级实际分配给对象的功率 | 对象请求值 |
| 净功率 | 当前供给减去请求负载的方向性差值 | 供能不足时所有任务仍能获得的功率 |
| 实际充放能功率 | 当前供给减去总已服务功率、实际改变储能的方向性差值 | 按全部请求计算的净功率 |
| 功率请求 | 一个房间、任务或效果声明的来源、数值、安全层级和服务模式 | 已服务功率 |
| 功率服务模式 | 规定请求获得部分或完整功率时如何推进的身份 | 玩家自定义配电优先级 |
| 可缩放负载 | 达到最低服务功率后按服务比例推进的功率请求 | 获得任意微小功率都满速运行 |
| 整笔负载 | 只有获得完整请求功率才推进本步骤的功率请求 | 可以部分推进的负载 |
| 能源安全层级 | 供给与储能不足时决定不同类别请求先后关系的固定机制层级 | 玩家逐房间排序 |
| 恢复保障功率 | 仅服务唯一能源恢复目标、不进入普通供给或储能的受限功率 | 免费全局供给 |
| 储能溢散 | 供给超过可服务负载与储能容量，或强制容量下降造成的可审计能源损失 | 可返还资源或隐藏删除 |
| 基因 | 已由样本组织解析形成、可用于候选和进化交易的核心资源 | 样本标签或图鉴记录 |
| 过程资产 | 尚未正式入账、仍具有位置、阶段、容量和风险的物理资产 | 第四种顶部资源 |
| 过程资产批次 | 类型、来源、数量、阶段和位置相同的一组过程资产实例 | 单位批次 |
| 物质谱系 | 记录过程资产来自外部未同化物、已同化物或内部循环及其来源份额的守恒事实 | 显示来源文案 |
| `现场` | 位于节点或房间输出位置、尚未被任务锁定的过程资产阶段 | 已安全回流 |
| `已预留` | 仍在现场但被一个合法采集任务唯一锁定的阶段 | 核心资源预留 |
| `携带中` | 由存活工蜂携带、尚未完成安全交接的阶段 | 已保障 |
| `已保障` | 已脱离敌方争夺并进入安全交接缓冲、尚未进入兼容房间的阶段 | 已回流或已入账 |
| `已回流` | 已进入兼容虫巢输入或处理缓冲的阶段 | 已处理或核心资源 |
| 终结事件 | 已处理、腐败、敌方夺回或永久损失等使资产离开五阶段的事件 | 可继续转移的阶段 |
| 核心资源预留 | 为建设、孵化、诱导等任务锁定已入账资源 | 采集资产的`已预留` |
| 核心资源消费模式 | 规定预留在任务完成、不可逆开始或即时交易中何时转为消费的配方身份 | 持续功率服务模式 |
| 采集资产预留 | 把`现场`资产转为`已预留`并唯一锁定源数量、交互和投递容量的事实 | 核心资源预留或过程输入预留 |
| 过程输入预留 | 在不改变`已回流`阶段的前提下，为一个处理任务唯一锁定输入资产的事实 | 采集资产的`已预留`阶段 |
| 安全回流缓冲 | 撤离时容纳已保障但暂时无法进入兼容容量的资产 | 免费仓库或直接入账通道 |
| 残骸 | 单位死亡输出形成的过程资产类别 | 生物质收益 |
| 样本残骸 | 含可分离样本组织标签的残骸 | 基因或候选组 |
| 样本组织 | 消化分离后等待解析的虫巢本地过程资产 | 样本标签 |
| 样本标签 | 解析后记录在本轮进化状态中的资格信息 | 可消费基因或图鉴 |
| 图鉴 | 跨轮保留的已发现内容知识 | 本轮样本资格或地图情报 |
| 生物质浆 | 房间已形成、仍需工蜂投递到账本接入口的物理过程资产 | 已入账生物质 |
| 外部同化 | 外界物质首次完成核心资源入账并记入本轮贡献的事件 | 己方尸体内部回收 |
| 内部回收 | 已属于玩家物质循环的己方残骸重新形成生物质 | 新的外部吞噬贡献 |
| 永久损失 | 腐败、夺回、转化损失或不可逆结算造成的终结数量 | 暂时堵塞或预留 |
| 统计账目 | 记录同化、回收、消耗、损失、控制和峰值等成果的不可消费事实 | 核心资源库存 |
| 账目条目 | 一个已提交事件对资源、单位或资产变化的可审计记录 | 再次改变状态的事件 |

## 6. 单位与战斗术语

| 规范术语 | 定义 | 不应混同 |
| --- | --- | --- |
| 单位 | 具有真实数量，并可按模板受伤、死亡、变形、占据空间或执行至多一个核心行为的对象 | 房间、地形、资源和过程资产 |
| 单位模板 | 定义阵营、标签、移动形态、耐久、核心行为和死亡输出的内容定义 | 存活单位实例 |
| 单位批次 | 玩法事实相同、可以共同结算的一组真实单位 | 生产批次或画面队列 |
| 逻辑个体 | 批次内部仍需保持独立耐久与状态语义的一个真实单位 | 一个Godot节点或画面精灵 |
| 规范状态单元 | 完整玩法键相同、以一个真实整数数量无损聚合的逻辑个体集合 | 运行容器分片、平均单位或画面代表 |
| 规范状态队列 | 单位批次内按完整玩法键唯一排序的规范状态单元序列 | 可以裁剪、取平均或只保留前N项的缓存 |
| 真实数量 | 单位总账中参与形成、伤亡、撤离和守恒的整数数量 | 画面显示数量 |
| 画面代表 | 为表达单位批次而创建的有限表现实体 | 真实单位或伤害来源 |
| 阵营 | 玩家、敌方或中立环境等目标关系资格 | 单位模板名称 |
| 移动形态 | `容纳 / 可机动 / 固定`之一，决定单位可进入的位置类型 | 核心行为 |
| `容纳` | 只能存在于声明虫巢或容器中的移动形态 | 可搬运过程资产 |
| `可机动` | 可以在虫群大军与合法节点间转移的移动形态 | 玩家可逐个控制 |
| `固定` | 保持节点空间、基础撤离时不回归的移动形态 | 无敌或无行为 |
| 单位真实位置 | `虫巢内 / 虫群大军 / 节点内 / 死亡`之一 | 当前战场、活动或空间层 |
| 虫群大军 | 所有当前位于全局军队位置的玩家可机动单位 | 全部存活单位总和或驻军 |
| 当前战场数量 | 当前主动节点内玩家单位的派生统计 | 独立战场库存 |
| 节点留置数量 | 节点内玩家固定单位的派生统计 | 驻军或虫群大军扣留量 |
| 虫群大军活动 | 虫巢作业、领地采集、自由游荡或待命之一 | 单位真实位置 |
| 活动锚点 | 虫群大军活动引用的目标虫巢、节点或工作位置，只用于资格、容量、时间和表现投影 | 单位真实位置、运输路线或驻军所有权 |
| 空间层 | 节点内单位的`接触 / 纵深 / 后备`位置关系 | 第二份军队位置 |
| 接触 | 当前能够占用前线交互容量的空间层 | 所有可攻击单位 |
| 纵深 | 位于接触之后、只可被明确覆盖规则作用的空间层 | 隐藏伤亡区 |
| 后备 | 等待补位或尚未暴露的节点内空间层 | 虫群大军库存或预备队菜单 |
| 核心行为 | 单位模板至多一个主动功能动词 | 移动、受伤、死亡和被动范围 |
| 行为管线 | 过滤目标、选择、接近、检查条件、积累进度并形成效果的通用过程 | 物种专用状态机 |
| 目标池 | 一个行为按模板规则锁定的稳定逻辑目标集合 | 画面上被点中的一个精灵 |
| 效果事件 | 行为完成后形成的伤害、搬运、变形、生成单位、施加状态或生成过程资产事件 | 动画特效 |
| 状态效果 | 具有明确生效步骤、失效步骤和叠加规则的临时规则变化 | 任意“Buff”文案 |
| 被动范围修正 | 由存活单位、位置和范围持续派生，不主动选择目标的效果 | 第二个核心行为 |
| 形成 | 从声明来源新增某模板真实数量的事件 | 变形或死亡后返还 |
| 变形 | 等量单位从一个模板转为另一个模板的事件 | 先死亡再免费重生 |
| 死亡 | 单位离开存活位置并产生一次死亡输出的终结事件 | 画面尸体动画完成 |
| 死亡输出 | 死亡事件按模板形成残骸、样本或特殊事件的结果 | 直接核心资源奖励 |
| 幼体 | 位于虫巢内、移动形态为`容纳`、无核心行为并可等量分化的单位 | 幼虫货币、过程资产或房间物品 |
| 工蜂 | 唯一核心行为为采集的玩家单位角色；第一版内容名为采质工蜂 | 建设单位、维修单位或玩家物流优先级工具 |
| 生产归因 | 记录单位由哪个虫巢和房间形成的统计来源 | 单位可用区域限制 |
| 可破坏结构 | 使用单位耐久、位置和死亡规则的固定单位 | 房间或不可破坏地形 |
| 出兵点 | 核心行为为生成单位、具有真实储备或输入的固定单位 | 隐藏关卡计时器 |
| 占领阻断单位 | 具有`占领阻断`标签、存活时阻止节点控制提交的单位 | 隐藏控制值 |
| 战役 | 从进攻提交到撤离、占领或全灭释放主动战场槽的一段账期 | 整个当前轮次 |
| 进攻 | 把全部合法可参战单位原子转入目标节点并建立主动战场的战役命令 | 分批投放或行军运输 |
| 撤离 | 在统一步骤边界让可机动单位回归、固定单位留置并重分配资产阶段的原子命令 | 有持续时间的逃跑技能 |
| 战损 | 由真实死亡和不可回收资产终结形成的损失事实 | 结算页随机惩罚 |

## 8. 时间、随机与交易术语

| 规范术语 | 定义 | 不应混同 |
| --- | --- | --- |
| 模拟时间 | 所有玩法状态推进使用的唯一时间轴 | 动画播放时间或现实等待 |
| 完整步骤 | 具有稳定步骤号和统一事件顺序的最小确定性模拟边界 | 一个画面帧 |
| 稳定边界 | 一个完整步骤已结束、下一个步骤尚未开始，可接收命令和保存快照的状态 | 步骤中间或动画完成点 |
| 步骤阶段 | 完整步骤内具有固定编号、读写职责和顺序的执行区段 | 可由系统任意重排的更新函数 |
| 表现时间 | 动画、镜头、代表实体和界面过渡使用的时间 | 玩法冷却和产出时间 |
| 离线经过时间 | 关闭存档到再次载入之间的现实时间输入 | 所有系统无条件推进时长 |
| 离线安全结算 | 无主动战场时按在线规则、容量和统一上限批量推进的状态 | 固定“离线收益”奖励 |
| 主动战场冻结 | 有主动战场时关闭游戏导致整个世界停止的运行状态 | 只冻结战斗、房间继续生产 |
| 时间倍率 | 改变模拟时间推进速度但不改变相同模拟时长结果的玩家设置 | 只加速己方生产的增益 |
| 只读查询 | 读取一个已发布稳定边界并形成可丢弃结果，不进入命令生命周期且不改变权威事实 | 命令、事件或会改变情报的观察节点 |
| 命令 | 玩家或系统提出、尚未提交为当前轮次玩法事实的有序意图 | 存档生命周期请求、只读查询或已经完成的事件 |
| 命令信封 | 保存命令身份、类型、目标、公开参数、接收边界、序号和幂等信息的完整记录 | 客户端计算的结果 |
| 命令批次 | 同一稳定边界接收并按契约统一排序的一组命令 | 同一画面帧的无序点击集合 |
| 边界服务请求 | 请求保存服务在稳定边界读取或写入权威快照、但不改变当前轮次玩法事实的输入 | 玩法命令或存档槽生命周期操作 |
| 存档生命周期请求 | 在当前轮次之外创建、载入、继续、覆盖或删除存档槽的产品请求 | 当前轮次玩法命令或玩法事件 |
| 查询审计记录 | 为诊断记录只读查询来源、版本和结果的可丢弃非玩法日志 | 权威快照、玩法事件或确定性重放输入 |
| 边界事务 | 在稳定边界原子提交且不推进模拟时间的命令交易 | 步骤中间插入的即时结算 |
| 事件 | 已验证并提交、具有唯一身份和因果来源的状态变化事实 | 可以重复调用的回调 |
| 事件草案 | 已形成并具有稳定顺序、等待指定阶段提交的过程事实 | 已改变目标状态的事件 |
| 最早生效步骤 | 新单位、状态、修正或输出最早可以参与规则求值的完整步骤号 | 动画开始播放的帧 |
| 事件排序键 | 由步骤或边界事务时点、阶段、类别、上游序号、稳定标识和定义内效果槽序组成的确定性顺序 | 容器、场景树或预分配事件序号 |
| 交易 | 从验证、预留、推进到原子提交或终结的一组有边界变化 | 任意界面点击 |
| 预留 | 为一个合法任务唯一锁定资源、资产或容量的事实 | 已消费或已处理 |
| 提交 | 把满足条件的变化一次性写入权威事实的边界 | 播放完成动画 |
| 取消 | 按契约终止未完成交易并释放可返还预留 | 回滚全部历史时间和功率 |
| 原子 | 一组变化要么完整提交一次，要么完全不改变权威事实 | 分多帧表现 |
| 幂等 | 同一事件因读档、重试或重复请求被再次处理时不产生第二次结果 | 可以重复领取 |
| 确定性随机流 | 由稳定种子、流标识和游标决定的可重放随机序列 | 容器遍历顺序或系统时间 |
| 随机游标 | 一个随机流已经消费到的位置事实 | 玩家情报深度 |
| 来源事件 | 形成某实例或数量的直接上游事件标识 | 显示页面或动画来源 |
| 状态版本 | 决定权威快照字段结构和迁移路径的版本 | 规则版本或参数版本 |
| 权威快照 | 在完整步骤边界保存的全部权威事实、版本、引用、随机游标和待处理事务 | 页面截图或派生缓存 |
| 存档迁移 | 把旧状态版本确定性转换为新状态版本的原子过程 | 静默删除非法数据 |
| 确定性重放 | 从同一快照、规则、参数、随机流和命令序列复现同一结果 | 动画回放 |

## 9. 状态词规范

以下状态只能用于指定维度：

| 维度 | 规范状态 |
| --- | --- |
| 当前轮次生命周期 | `无当前轮次 / 建立事务中 / 运行中 / 飞升选择中 / 飞升提交中 / 已结束` |
| 世界运行 | `正常运行 / 全局暂停 / 关键选择暂停 / 离线安全结算 / 主动战场冻结` |
| 命令生命周期 | `已接收 / 待验证 / 已接受 / 已提交 / 已拒绝 / 已取消` |
| 核心资源预留 | `持有 / 已消费 / 已释放 / 已终止` |
| 区域节点网提交 | `未提交 / 已提交` |
| 单位位置 | `虫巢内 / 虫群大军 / 节点内 / 死亡` |
| 单位移动形态 | `容纳 / 可机动 / 固定` |
| 节点空间层 | `接触 / 纵深 / 后备` |
| 过程资产 | `现场 / 已预留 / 携带中 / 已保障 / 已回流` |
| 过程资产物质谱系 | `外部未同化 / 已同化 / 内部循环` |
| 区域情报 | `未发现 / 概要已知 / 节点网已知 / 深入侦察 / 完全解析` |
| 区域着床 | `不可着床 / 待着床 / 虫巢节点争夺中 / 着床安全区 / 虫巢建设中 / 虫巢已激活` |
| 区域控制汇总 | `敌方占据 / 区域争夺中 / 完全吞噬` |
| 出口通路 | `出口未知 / 出口已知 / 出口可达 / 出口争夺中 / 通路已打开` |
| 节点情报 | `未揭示 / 轮廓已知 / 可观察 / 已解析` |
| 节点连通 | `不可达 / 可达` |
| 节点控制 | `敌方控制 / 争夺中 / 己方控制` |
| 房间结构 | `建设中 / 可运行 / 重组中 / 拆解中` |
| 过程任务 | `已预留 / 推进中 / 暂停 / 待提交 / 已完成 / 已取消 / 已失败` |
| 候选组 | `形成中 / 来源缓冲 / 持有 / 已选择 / 已舍弃` |

同一个词出现在不同维度时必须带对象前缀，例如“区域争夺中”和“节点争夺中”。不得仅保存一个不说明维度的`状态=完成`。

## 10. 歧义与禁用表达

| 避免使用 | 原因 | 应改为 |
| --- | --- | --- |
| 资源 | 无法判断是核心资源、过程资产、外部来源还是统计 | 写明生物质、能源、基因、过程资产、外部来源或统计账目 |
| 能量 | 与规范资源名不一致 | 能源；并进一步写明储能、供给、负载或已服务功率 |
| 节点产出资源 | 容易绕过房间形成链 | 节点提供外部来源、抽取容量或房间修正 |
| 关卡奖励、占领奖励包 | 容易制造无来源资源 | 具体敌人/结构战利品、外部来源开放、世界状态事件或统计 |
| 资源节点、奖励节点 | 把标签误写成独立生命周期 | 带`资源`或`奖励`标签的节点，并列出实际对象 |
| 已回收 | 无法判断风险和处理阶段 | 已预留、携带中、已保障、已回流、已处理或已入账 |
| 样本 | 同时可能指残骸、组织、标签或图鉴 | 样本残骸、样本组织、样本标签或图鉴记录 |
| 产出 | 无法判断形成何物和是否已入账 | 形成单位、生成过程资产、形成物理输出、形成网络结果或正式入账 |
| 解锁 | 混合发现、资格、可执行和获得 | 发现、内容池资格、当前可执行、实例化、强化或永久化 |
| 科技 | 混合科技族、候选和已生效效果 | 科技族、候选成员、本轮突变或插件 |
| 科技树 | 暗示固定线性点亮 | 科技族与肉鸽候选系统 |
| 建筑 | 可能指房间、可破坏结构或地形 | 房间模块、可破坏结构单位或地形 |
| 基地 | 所有权和状态含糊 | 虫巢；必要时写明所属区域 |
| 大房间合成 | 暗示销毁模块并新建实例 | 相邻兼容模块派生为房间群 |
| 幼虫资源 | 幼体不是资源 | 幼体单位 |
| 军队库存、兵力储备 | 可能重复单位位置 | 虫群大军、节点内单位或具体派生统计 |
| 后备军 | 与节点空间层混淆 | 虫群大军；“后备”只用于节点内空间层 |
| 战场单位库存 | 容易建立第二份数量 | 当前主动节点内单位 |
| 驻军 | 当前设计没有独立驻军位置 | 节点留置固定单位或领地活动投影 |
| 节点运输、运输路线 | 当前设计没有玩家配置运输系统 | 过程资产五阶段、工蜂采集任务或单位原子转移 |
| 投放、投放强度、分批投放 | 当前进攻不选择参战比例 | 全军进攻、战时新生自动补充 |
| 战力、胜率、推荐兵力 | 与真实模拟无关且会生成策略答案 | 已知敌情、真实数量、构成、历史推进和实际战损 |
| 关卡失败 | 把多种不利状态压成一个弹窗 | 全灭、撤离、节点未占领、储能归零、堵塞等具体事实 |
| 区域核心 | 暗示独立节点底层 | 区域出口，或带Boss/精英标签的普通节点 |
| 区域完成 | 无法区分打开出口和清理全部节点 | 通路已打开或完全吞噬 |
| 吞噬度、控制百分比 | 当前没有隐藏连续控制量 | 节点单位、阻断对象、节点控制与区域控制汇总 |
| 敌人刷新 | 容易暗示退出后重置 | 出兵点生成、入口增援或敌方恢复 |
| Buff、Debuff | 无法判断生命周期和结算方式 | 状态效果、被动范围修正或明确参数修正 |
| 菌毯覆盖度 | 容易成为无来源百分比 | 节点内固定菌毯单位及其被动范围；确需汇总时声明为派生面积 |
| 离线收益 | 容易暗示固定奖励 | 离线安全结算产生的逐项资源、任务和世界变化 |
| 撤离时间、撤离成功率 | 与基础原子撤离冲突 | 撤离确认快照与下一完整步骤原子提交 |
| 战斗结果奖励 | 容易二次生成收益 | 战役账期中的单位、资产、节点和统计变化 |
| 计划事实 | 没有所有者和生命周期，容易成为绕过命令、任务与状态契约的兜底存档对象 | 待处理命令、队列条目、过程任务、待提交输出、行为进度、状态/修正实例或单位最早生效步骤 |
| 进度 | 可能指任务、战线、情报或完成 | 建设进度、行为进度、最远空间位置、情报状态或完成要求 |

引用历史文案时可以保留原词，但必须标记为历史方案，不能继续用于新规则。

## 11. 命名规则

### 11.1 规范名、内容名与短名

```text
规范术语：工蜂
内容正式名：采质工蜂
界面短名：工蜂
稳定定义标识：由L4内容数据登记
```

- 规范术语描述系统角色，内容正式名描述具体实例，界面短名只服务空间可读性。
- 一个内容可以更换正式名，但稳定定义标识和规范角色不能静默改变。
- 同名内容若使用不同生命周期，必须拆成不同定义标识和明确形态。

### 11.2 状态与事件命名

- 状态使用“对象 + 维度 + 当前值”，例如`节点控制=争夺中`。
- 命令使用玩家意图动词，例如“发起进攻”“确认撤离”“开始建设”。
- 事件使用已经发生的过去事实，例如“进攻已提交”“单位已死亡”“节点已占领”。
- 账目条目使用“资产 + 变化 + 来源/去向”，不能使用“获得奖励”概括。
- 参数名必须指向一个已定义关系，例如“孵化任务基础周期”，不能使用“节奏系数”这类无边界名称。

## 12. 术语表验收

- L2与后续L3权威文档中的每个对象、状态、命令和事件均能映射到一个规范术语。
- 自动扫描不再把歧义词用于新规则；历史文件中的歧义词在迁移时逐项替换或标记废弃。
- 界面短名、内容正式名、稳定定义标识和规范术语具有可追溯映射。
- 状态日志能显示对象、维度、前值、后值和来源事件，不只显示“状态改变”。
- 测试名称使用规范状态和不变量编号，不通过显示文案判断玩法结果。
- 新内容不能通过改名绕过单位、房间、节点、候选和资产的通用模型。

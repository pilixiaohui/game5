## 1. 第一版的正式定义

第一版采用`首小时可玩切片`范围：

- 新玩家以约一小时的自然游玩容量接触完整机制集合，但不承诺固定现实分钟数。
- 熟悉玩家可以通过并行建设、快速决策和统一时间倍率更快完成同一组行为里程碑。
- 玩家实际经营一个已激活的初始虫巢，推进一个受强约束随机生成的起始区域。
- 玩家至少完成一次房间生产、虫群大军形成、关卡进攻、战场损耗、残骸回收、样本解析和突变同化的闭环。
- 玩家占领起始区域唯一的区域出口，并看到下一地区的主题、连接、资源倾向、着床节点和建巢条件。
- 玩家看到基于本轮真实账本计算的飞升预览，但不能在第一版执行正式飞升。

第一版的体验终点不是固定时间、固定资源数量或删除存档，而是满足本文定义的完成里程碑。完成后玩家仍可返回起始区域支路、观察已占节点的领地采集、留置资产与房间来源修正，调整虫巢并继续积累资源。

## 2. 第一版核心承诺

第一版必须证明以下设计承诺能够在同一存档中共同成立：

1. 房间生产、战场推进、突变构筑和飞升预览使用同一资源与事实账本。
2. 生物质、能源和基因具有不同作用，残骸与样本不能绕过物理处理链直接入账。
3. 房间可以自由建设、相邻合并、并行施工、自动循环和无损重组。
4. 工蜂、战斗单位和菌毯单位只执行各自的单一核心行为，不依赖玩家设置任务优先级。
5. 长战场使用真实数量、战群批次和有限画面代表，百万级数量不要求生成百万个实体。
6. 战损、原子撤离、残骸、敌方恢复、留置菌毯后台战斗和再次进入属于同一持续节点，而不是退出后重置关卡。
7. 首轮地图在随机变化的同时，始终提供最低资源闭环、三类战场关系、固定样本和唯一出口，并在地图提交时生成全部节点内容。
8. 一次突变选择能够改变同一套房间、单位或战场规则，并带来可观察代价。
9. 极端失败可以恢复，但不会免费返还已经结算的战损、功率或候选成本。
10. 页面职责分离，虫巢主视图不承载地图、进化、飞升和系统设置的全部操作。

## 3. 第一版必须实际可玩的内容

### 3.1 世界范围

- 一个实际生成并可推进的起始区域。
- 一个已经激活的初始虫巢节点。
- 起始区域唯一的`区域出口`节点。
- 至少三种机制职责不同的实际战场：小规模接触、受限接触与出兵源、长场景综合推进。
- 至少一个占领后提供明确外部资源源与抽取容量的资源节点；最终资源必须由对应房间形成。
- 一个固定首次样本载体及其可恢复来源。
- 一个只生成概要和待着床入口的相邻区域。
- 起始区域内部具有分支、回环和至少两条可抵达出口的合法路线，不退化成单线关卡列表。

共享生成、情报、连通、控制和出口规则见`L3_世界地图与节点契约.md`；第一版生态、节点职责、空间段和首轮组合见`L4_第一版世界与关卡内容.md`。`26_首轮地图骨架与生成约束.md`只保留迁移理由，不是发布校验来源。

### 3.2 虫巢与房间

第一版必须实际建设和运行：

- 生物质来源房间。
- 储能与功率供给房间。
- 基础孵化房间。
- 普通残骸与样本分离房间。
- 基因解析房间。
- 普通候选形成房间。

必须实际支持：

- 自由空槽。
- 同类相邻自动合并。
- 并行建设和功率负载。
- 自动循环与目标数量。
- 工蜂自动采集物理输出。
- 工蜂在已占节点自动抽取外部来源并回流过程资产。
- 缺料、缺能、堵塞、暂停和恢复。
- 移动及其引发的群拆分、替换、降级、拆除与取消建设的账目守恒。

共享槽位、模块、房间群、结构事务、队列、任务和缓冲规则见`L3_虫巢与房间契约.md`；首批房间身份、配方、端口和解锁见`L4_第一版虫巢房间与配方.md`。

### 3.3 单位与战场

第一版必须实际运行同一单位底层中的以下形态链：

- 虫巢内无主动行为、供孵化任务等量分化的基础幼体。
- 只执行采集的工蜂。
- 只执行猎杀的基础战斗单位。
- 只执行扎根并等量变形为固定根脉菌毯的根脉孢体。
- 只执行截杀并提供被动范围修正的固定根脉菌毯。
- 与生物共用耐久、批次和死亡规则的可破坏出兵点与阻断结构。

第一版可以通过可选支路开放只执行破城的单位，但基础战斗单位必须能够低效处理首轮必要阻断结构，避免随机解锁形成硬锁。

战场必须实际支持：

- 左侧我方入口、右侧敌方入口和可见中途出兵源。
- 接触、纵深、后备、通行和交互容量。
- 战时新生自动补充、伤亡、残骸和工蜂回收。
- 菌毯扎根、范围效果、被击杀和效果消失。
- 确认后不可取消的瞬时原子撤离；命令步骤不再形成新的攻击或移动事件。
- 撤离时可机动存活单位回归，固定菌毯留在节点；携带中资产转为已保障，兼容容量内继续回流，超额保持安全堵塞，预留释放回现场。
- 撤离或全灭后保存节点状态并发生有来源的敌方恢复；留置菌毯与恢复敌军运行完整后台战斗，我方后台死亡不形成回收收益。
- 后台战斗满足同一占领条件时正常提交控制，不要求玩家重新进入已经清空的节点。
- 清除敌军、持续出兵源和可见特殊目标后占领节点。
- 占领后移动单位回归、节点安全完成、领地活动、外部来源、房间修正与留置菌毯。
- 节点观察页提供`进攻`，主动战场只提供`撤离`；第一版科技效果不增加战场技能按钮。

单位底层、工蜂采集、形态变化、伤害、进攻、撤离与后台执行统一见`L3_单位与战斗契约.md`；十二阶段顺序见`L3_统一时间命令与事件契约.md`；第一版模板、目标组、效果和死亡输出见`L4_第一版单位与战斗内容.md`。`24_工蜂与战斗执行规则.md`与`29_统一单位模型与战斗内核.md`只保留历史场景素材。

### 3.4 资源、样本与突变

第一版必须实际完成：

```text
外部生物质来源
-> 现场 / 已预留 / 携带中 / 已保障 / 已回流
-> 腐殖滤囊或腐解消化池完成形成
-> 生物质浆进入房间输出缓冲
-> 工蜂自动送到账本接入口
-> 生物质正式入账
-> 建设或孵化再投资

首次样本载体死亡
-> 样本残骸保障与回流
-> 腐解消化池分离普通组织与样本组织
-> 突触解析腔完成解析
-> 基因、样本标签和首次候选机会形成
-> 玩家从候选组选择一个突变并支付同化成本
-> 同组其他候选消失
-> 规则变化在现有房间或战场中可观察

完成首次突变
-> 建设诱变培养腔，或执行基因主动诱导
-> 普通候选组进入普通持有槽
-> 选择一个成员并结算公开成本
-> 同组其他候选消失
```

首次与普通候选的来源、随机、槽位、支付、同化和失败边界见`L3_候选与进化契约.md`；六项候选、里程碑、普通来源和飞升预览目录见`L4_第一版候选解锁与飞升预览内容.md`。`25_首次候选与基因交易规则.md`只保留历史内容素材。

### 3.5 页面与基础产品功能

第一版必须包含：

- 标题页的新游戏、继续游戏和设置入口。
- 至少一个可独立载入的本地存档槽。
- 游戏内自动保存与手动保存入口。
- 虫巢主视图、房间详情和布局编辑。
- 世界地图、区域地图和长战场节点画面。
- 进化页面、候选组选择和已同化突变摘要。
- 诱变培养腔房间管理、普通候选槽和基因主动诱导入口。
- 虫群大军、单位构成、资产位置与各虫巢生产贡献摘要。
- 显示准确飞升点、具体永久收益数值与三位领袖差异的飞升只读预览。
- 基础日志、资源账目和离线摘要。
- 音频、画面、文本大小、减少闪烁和动画强度设置。

桌面端是第一版交互和性能验收基线。移动端信息架构继续保留，但不作为第一版完成门槛；后续版本必须重新进行触控密度和性能验收。

第一版页面职责、命令入口、只读查询和返回路径以`L5_页面信息架构与交互契约.md`为准；首小时因果触达、行为里程碑和任务测试以`L5_前1小时进程与任务测试.md`为准。

## 4. 第一版只展示、不实际开放的内容

以下内容可以显示锁定入口、规则概要或真实预览，但不得让玩家误以为已经可执行：

- 正式飞升确认。
- 飞升后的新世界生成。
- 虫群领袖选择和领袖专属科技实际生效。
- 第二地区虫巢节点进攻。
- 第二虫巢建设和多虫巢管理。
- 完整基因插件管理。
- 后续区域生态、Boss阶段、正式入侵和塔防。

预览内容必须使用明确的`尚未开放`状态，不参与第一版资源消费、解锁判定或完成统计。

## 5. 第一版明确排除的内容

- 正式执行飞升和跨轮永久成长。
- 实际建立第二虫巢。
- 多个主动战场同时运行。
- 无限深度世界的实际持续生成。
- 正式领地失守、塔防和敌方大规模入侵。
- 复杂Boss多阶段战斗。
- 完整插件、成就、云存档和联机功能。
- 玩家配置单位目标权重、任务优先级、阵型或个体路径。
- 移动端发布验收。

排除不等于删除长期设计。相关入口和数据结构可以保留扩展边界，但不能扩大第一版制作清单。

## 6. 起始状态

新存档必须从同一类规则状态开始，具体数值由首小时平衡测试确定：

- 起始区域和首轮地图种子已经生成。
- 初始虫巢处于已激活状态。
- 核心巢室不凭空生产资源，只提供基础账本接入口、最低容量、第一只工蜂应急闭环，以及只服务一座能源房间建设的最低保障功率。
- 玩家拥有少量可读的初始生物质与储能。
- 最小房间蓝图集合可用，并存在自由空槽。
- 全局没有已同化突变，基因为零或处于首轮测试规定的最低状态。
- 第一只工蜂尚未形成，或由首个孵化任务明确形成；无工蜂时只允许受限应急直连服务第一只工蜂。
- 起始区域地图提交时已经生成全部节点敌人、结构、空间段、资源源和修正，但开局只揭示基础连接和已知事实。

起始资源必须通过模拟证明能够形成最低闭环，但不得以未经测试的固定值写入机制文档。

## 7. 第一版完成状态

一个存档满足以下全部行为里程碑后，记为`第一版体验已完成`：

1. 至少一个玩家建设的房间完成过有效运行循环。
2. 第一只工蜂完成过一次房间物理输出采集与合法投递。
3. 玩家实际观察过储能、功率供给、基础负载、临时负载和净变化。
4. 至少两个同类房间模块完成过相邻合并，或玩家主动验证过兼容合并预览。
5. 噬咬体和根脉孢体通过持续生产加入虫群大军，并在进攻后表现出不同核心行为。
6. 玩家经历过一次撤离或同一节点再次进攻，使持续战场状态与单位回归得到验证。
7. 普通残骸经过工蜂回收和腐解消化池处理形成生物质浆，并由工蜂投递成为可用生物质。
8. 固定样本载体的样本残骸完成分离和解析。
9. 玩家从首次候选组中同化一个突变，并在之后的生产或战场中触发其规则变化。
10. 诱变培养腔与基因主动诱导均已解锁；玩家至少实际生成并处理过一个普通候选组，来源由玩家自行选择。
11. 至少一个节点被完全占领，移动单位回归且节点进入安全完成状态；工蜂领地采集和战斗单位自由游荡可观察。
12. 起始区域唯一出口被占领。
13. 世界地图显示相邻区域概要、待着床虫巢节点和建巢条件。
14. 飞升页面根据当前存档真实账本显示准确预计飞升点、具体永久收益数值、三位领袖差异和保留/重置范围。

完成状态只用于版本验收、统计和后续内容边界，不弹出策略评价，也不阻止玩家继续游玩。

## 8. 数值完成条件

机制文档不猜测正式数值，但第一版发布前必须存在一份带版本号的参数表，并满足：

参数机器登记为`data/balance/v1_parameter_registry.csv.txt`，参数身份、实验包和晋升规则以`L6_平衡参数与首小时规模验证.md`为权威，无表现证据状态见`L6_无表现模型与验收映射.md`，百万级工程预算见`L6_百万级状态队列与性能预算.md`。当前参数仍为`fixture_only`且发布值为空，只能用于机制测试，不能视为本节发布条件已经完成；`14_首小时规模曲线.md`和`27_无表现模拟模型与参数登记表.md`只保留迁移素材。

- 每个成本、产量、容量、时长、概率和倍率都有唯一参数标识。
- 每个参数记录测试范围、当前值、关联机制、证据版本和状态。
- 资源、单位、残骸、建设和候选交易能够守恒对账。
- 首次样本解析与候选参数必须满足以下防软锁约束：

```text
首次解析基因产量
>= 首次组最高同化成本
 + max(主动诱导生成成本, 第一版普通池最高同化成本)
```

- 上式只保证玩家在首次同化后仍可选择启动一种普通候选来源，不表示主动诱导与设施/特殊来源可以同时免费完成。
- 第一只工蜂从真实数量`1`开始。
- 新玩家测试能够从单体过渡到宏观虫群；首小时末以约百万级真实数量作为目标体验量级，最终验收区间由测试证据确定。
- 熟悉玩家可以更快完成行为里程碑，但不能跳过资源、战损和样本处理链。
- 不存在依靠取消建设、己方尸体循环、候选舍弃或读档重抽获得净资源的路径。

未经自动模拟和实际游玩验证的参数不得成为第一版完成基线。

### 9.1 功能证据

- 所有第一版必须内容均可从新存档自然触达。
- 所有只读预览内容不能被实际执行或产生资源账目；诱变培养腔和主动诱导不属于预览，必须形成真实候选账目。
- 所有排除内容不成为首轮地图、解锁或软锁恢复的必要条件。
- 关键状态能够保存、退出并恢复，不因读档重新抽取地图、候选或已经结算的战果。
- 幼体、工蜂、战斗生物、固定菌毯、敌方、阻断结构和出兵点均通过同一单位位置、行为管线、效果事件和十二阶段顺序运行；新增首版单位不增加专用战斗流程。

### 9.2 守恒与恢复证据

- 建设取消、房间重组、战斗伤亡、原子撤离、五阶段采集资产和残骸处理通过自动账目测试。
- 幼体形成与分化、孢体变形、状态有效步骤、死亡输出、固定结构阻断资格和单位真实位置分别通过守恒测试。
- 全部工蜂死亡、储能归零、房间堵塞、候选槽满和战场全灭均有可执行恢复路径。
- 应急直连只能在无采集单位时形成第一只工蜂，不能成为常规高效物流。
- 第一份样本即使未能保障，也能通过有代价的节点恢复重新获得，不会永久软锁基因系统。
- 样本恢复在节点已占领后仍然可用，并在节点输出缓冲形成由领地工蜂按五阶段送入虫巢处理链的恢复输出；不会刷新敌军或生成样本以外的资源、候选与飞升成果。

### 9.3 随机地图证据

- 批量生成的每个首轮种子都通过WMC与L4W登记的结构、职责、依赖和内容引用校验。
- 每个起始区域在地图提交时已经生成全部节点内容；改变情报深度不会追加或重抽隐藏内容。
- 同一存档、同一种子和同一已揭示状态在读档后保持一致。
- 不存在唯一出口无法到达、必要资源位于循环依赖之后或必需目标不可发现的种子。
- 第一版相邻区域只保存概要和着床入口，不因占领出口暗中生成未开放的完整节点网。

### 9.4 表现与性能证据

- 不同画面代表配额、画质和帧率下，相同模拟状态与指令得到相同玩法账目。
- 单位批次采用不同合法合并与拆分方式时，稳定目标池、命中分配、伤亡、状态和死亡输出不变。
- L6Q-001至L6Q-007、L6Q-010至L6Q-012证明规范状态、同时效果、超预算和诊断失败不改变玩法语义。
- `million_normal`与`million_stress`在已登记桌面参考机器、发布构建和冻结参数下达到候选时间、内存、快照与确定性预算；正式报告产生前不能声称百万级性能达标。
- 百万级真实单位使用有限代表实体时，战线、后备、损耗、战时新生、领地活动和残骸趋势仍然可读。
- 虫巢房间、长战场、地图和弹窗在桌面验收分辨率范围内不遮挡关键操作和状态。
- 房间建设、运行、缺料、缺能、堵塞、暂停、合并和重组具有可辨认表现。

### 9.5 试玩证据

- 新玩家无需阅读设计文档即可完成基础建设、单位持续生产、进攻、撤离、样本处理和候选选择。
- 游戏可以说明操作、对象和已经发生的事实，但不会给出推荐兵种、推荐布局或标准构筑答案。
- 熟悉玩家能够使用并行建设和时间倍率压缩重复等待。
- 测试记录触达、停滞、恢复、资源曲线、战损和候选选择，而不只记录平均游玩时长。
- E-01至E-06必须按`L6_体验假设验证计划.md`在同一冻结版本上达到`release_passed`；形成性样本、不同版本样本和主持人策略提示不能拼接为发布证据。

## 10. 第一版发布门槛

只有同时满足以下条件，第一版才可标记为完成：

1. 本文所有必须内容已经实现，所有预览和排除边界与实际版本一致。
2. `L3-1.2`十一份冻结输入通过实际SHA-256校验；184项第一版L3验收与24项INV达到对应`release_verified`证据，14项长期契约不阻断发布。
3. 82项L4/L5验收按登记的自动或混合方式通过，旧迁移文档不存在仍被实现、测试或内容数据读取的并列规则。
4. L6P-001至L6P-016达到`release_verified`；参数表已经过身份审计、自动模拟、极端失败和实际试玩，发布值、证据版本和适用构建完整，不再使用`fixture_only`或无证据占位值验收。
5. 新存档、保存、载入、离线结算和主动战场冻结能够保持账目一致。
6. 首轮随机种子批量验证不存在阻断完成里程碑的非法地图。
7. L6Q-001至L6Q-012达到`release_verified`，参考机器报告证明百万级精确模拟与候选性能预算通过。
8. 第一版完成里程碑能够被事件账目准确记录，但不会被表现动画或页面访问伪造。
9. E-01至E-06在同一冻结版本上全部达到`release_passed`，零容忍失败项为零。
10. `00_设计问题处置表.md`不存在仍由旧编号文档独立拥有、且阻塞第一版实现的未处置问题。
11. 没有会导致存档永久软锁、资源复制、候选重抽、单位数量失真、存档损坏或表现改变结算的已知阻断问题。

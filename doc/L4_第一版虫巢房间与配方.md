## 2. 共同结构定义

### 2.1 槽位与群兼容

六个房间共同使用：

- `room_family`等于各自内容ID，不通过本地化名称判断同类。
- `structure_compatibility = organic_standard_slot`。
- `group_compatibility_key = room_family + major_io_branch + required_running_binding`。
- 一个模块占一个自由空槽；第一版没有多槽模块。
- 正交相邻且兼容的模块自动派生为一个房间群。
- 只有会改变主要输入、主要输出或共享任务资格的分支和运行绑定才进入兼容键；纯数值升级分支不进入。
- 等级、插件、玩家暂停、缺料、缺能、堵塞、任务进度、临时效果和结构状态不进入兼容键。已经形成的不同等级模块仍可成群；建设中模块尚未加入，重组中或拆解中的既有模块在结构终结前仍参与空间连通，但只有可运行模块能够执行任务。

模块移动、合并和拆分只改变拓扑投影；结构投入、任务、缓冲和绑定仍归属模块。房间群没有第二份产出库存。

### 2.2 共同结构事务

所有蓝图使用L3通用建设、升级、移动、功能重组、降级、拆除和替换事务：

| 事实 | 第一版内容规则 |
| --- | --- |
| 结构投入 | 预留生物质，并在任务推进期间请求功率；能源只通过功率服务与储能变化结算 |
| 并行建设 | 允许，所有任务共同竞争功率，不存在隐藏全局建设队列 |
| 取消建设 | 返还未消费结构预留；不返还已经过去的模拟时间和已经结算的能源 |
| 移动和兼容重组 | 保留模块、结构投入、队列、进度、缓冲、等级与绑定；承担时间和功率 |
| 拆除和降级 | 清空或保障不兼容事实后返还对应完整结构投入 |
| 替换 | 先合法终结旧结构并形成返还，再完整预留新结构；不能提前花费未提交返还 |

参数族统一使用`room.<slug>.build_biomass`、`build_seconds`、`build_power`、`move_seconds`、`reconfigure_seconds`及相应容量参数。发布前全部ID必须在L6参数表登记。

### 2.3 共同运行状态

所有房间均从任务、功率、端口、缓冲和策略派生完整原因集合：

- `缺料`：合法配方缺少可预留输入或来源服务。
- `缺能`：任务或持续贡献没有取得最低服务功率。
- `堵塞`：输出、端口、容量、目标或候选去向无法承诺。
- `运行`：至少一个任务实际推进或持续网络贡献实际生效。
- `待机`：可运行且未暂停，但当前目标已经满足或暂时没有任务。
- `暂停`：玩家暂停房间群或停止对应自动循环。

同一房间可以同时缺料、缺能和堵塞。L5可以显示一个摘要徽标，但详情必须保留全部原因、影响量和位置。

## 3. 核心巢室

### 3.1 定义

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.core_chamber` |
| 分类 | 固定基础设施，不是可建设房间，不参加房间群 |
| 结构位置 | 初始虫巢拓扑锚点，不占自由空槽 |
| 普通配方 | `none` |
| 普通功率供给 | `none` |
| 物理端口 | 全局生物质账本接入口、最低安全交接容器 |
| 单位接口 | 无采集单位时允许第一只工蜂恢复链申请受限应急交接；单位仍由胚流孵化室形成 |
| 恢复接口 | 第一能源房间建设保障、第一工蜂应急闭环 |

### 3.2 两项受限保障

`v1.recipe.core_first_energy_guarantee`只在全局没有可运行能源房间且正常预算不能服务恢复任务时，为一座`v1.room.thermal_metabolism`建设任务提供最低保障功率。该功率：

- 不进入当前供给或储能。
- 不服务其他房间、孵化、解析或候选任务。
- 目标能源房间提交后立即失去资格。
- 同一时刻最多服务一笔通过稳定排序选出的任务。

`v1.recipe.core_emergency_worker`是恢复协调接口，不是单位生产配方。它只在全局没有存活采集单位、没有已经完成待提交的采质工蜂且没有同类恢复任务时，代替缺失工蜂完成形成第一只`v1.unit.worker`所必需的最小合法物理交接：

- 必须绑定已经形成且合法的腐殖滤囊处理链、胚流孵化室及其工蜂目标任务；核心巢室不能直接形成幼体或工蜂。
- 只替代链条中原本必须由工蜂完成的最小搬运与账本投递，外部来源、腐殖滤囊处理、生物质正式入账、幼体形成、等量分化和离巢仍按各自房间与L3事务执行。
- 胚流孵化室仍需预留真实生物质、取得层级0普通功率并经过正常两阶段时间；核心巢室不提供第二条免费供能通道。
- 直连不能形成超出一只工蜂所需的可用库存、过程资产、幼体或第二个并行恢复任务。
- 第一只工蜂形成待提交结果或进入单位总账后，直连立即终止，后续全部物流恢复普通工蜂规则。

## 4. 腐殖滤囊

### 4.1 蓝图

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.biomass_filter` |
| 功能类别 | 资源形成 |
| 运行绑定 | 外部物质谱系与兼容来源族 |
| 物理输入 | `v1.asset.external_organic_matter`的已回流批次 |
| 物理输出 | `v1.asset.biomass_slurry` |
| 网络输出 | `none` |
| 单位输出 | `none` |
| 候选输出 | `none` |
| 功率层级 | 基础生产安全层级 |

### 4.2 配方图

```text
节点或母床形成外源物质现场批次
-> 工蜂完成五阶段并投递物理输入端
-> v1.recipe.filter_external_organic
-> 生物质浆进入模块输出缓冲
-> 工蜂送到账本接入口
-> 生物质正式入账
```

配方只锁定已回流输入，按完成边界形成物理输出。输入谱系保持外部来源身份，最终入账同笔增加`M_external`可追溯统计；房间不能读取节点储量远程生成生物质。

### 4.3 端口与堵塞

- 输入端：外源有机物，按模块和房间群声明容量。
- 输出端：生物质浆取货端。
- 主要堵塞：输入端满、输出承诺满、账本接入口无可用投递容量、工蜂路径无效。
- 主要缺料：没有已回流外源物质或来源尚未开放。
- 参数引用：建设、运行功率、配方周期、输入量、输出量、输入/输出容量和端口并发。

## 5. 热化代谢腔

### 5.1 蓝图

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.thermal_metabolism` |
| 功能类别 | 能源生产与储能 |
| 运行绑定 | 当前虫巢可连接的热化环境输入 |
| 物理输入/输出 | `none` |
| 网络输入 | 基础热化梯度或已占节点环境容量 |
| 网络输出 | 持续功率供给、储能上限贡献 |
| 功率层级 | 能源恢复与稳定供给层级 |

### 5.2 持续贡献

`v1.recipe.thermal_network_contribution`不是周期发放能源包，而是在第3阶段按实际环境服务建立：

```text
环境输入请求与容量竞争
-> 取得实际环境服务量
-> 求值得到本步骤供给和储能上限贡献
-> 资源能源契约统一分配功率并结算储能
```

新增能源节点只能提供环境输入、容量或明确房间修正，不能直接写当前储能。失去环境服务、房间暂停、重组或未达到最低服务时，贡献按L3边界退出。

### 5.3 冷启动与阻断

- 储能归零不删除房间或环境输入。
- 满足冷启动资格时，使用资源契约的受保护最低服务恢复本房间贡献。
- 主要缺能事实是建设任务或冷启动未获得最低服务，不显示为“缺燃料”。
- 主要堵塞是环境容量已被承诺或储能上限下降需先满足安全处理；储能已满属于待机或供给未被使用，不伪装成物理堵塞。
- 参数引用：建设成本/时间/功率、最低环境服务、供给曲线、储能容量贡献、启动过程与可选生物质转化分支。可选生物质转化不是第一版基础完成依赖。

## 6. 胚流孵化室

### 6.1 蓝图

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.embryo_hatchery` |
| 功能类别 | 孵化 |
| 运行绑定 | 一个已具当前可执行资格的最终单位模板 |
| 单位输入 | 所属虫巢内基础幼体预留 |
| 单位输出 | 等量分化后的目标模板离巢请求 |
| 物理输出 | `none` |
| 功率层级 | 基础生产安全层级；候选模式可声明更高临时负载 |

### 6.2 两阶段配方

```text
v1.recipe.hatch_larva
  生物质预留 + 功率 + 时间
  -> 在所属虫巢幼体容器形成基础幼体

v1.recipe.differentiate_unit
  预留现有幼体 + 生物质预留 + 功率 + 时间
  -> 等量变形为当前绑定模板
  -> 通过单位离巢端提交到虫群大军或当前主动节点
```

一个目标批次先预留可用幼体，只为缺口建立幼体形成子任务。幼体是单位总账中的`容纳`单位，不是房间库存数字。阶段二取消时释放未消费生物质和幼体预留；已形成幼体保持在所属虫巢，不因模板切换删除。

### 6.3 运行策略与堵塞

玩家可以设置绑定模板、循环模式、单次批次、目标规模和暂停。模板变化若改变群兼容键或存在不兼容任务，必须通过功能重组事务。

主要阻断：

- 缺料：生物质或可用幼体不足。
- 缺能：当前阶段没有获得最低服务功率。
- 堵塞：幼体容器、任务执行位、单位离巢端或目标位置容量无法承诺；候选模式产生的输出压力同样使用这些事实。
- 待机：目标规模已满足或绑定模板当前不可执行。

参数引用：两阶段输入、时间、功率、幼体容量、任务批次、目标规模、离巢容量、模板重组时间与功率。

### 7.1 蓝图

| --- | --- |

| 功能类别 | 回收与转化 |
| 物理输入 | 已回流普通组织、硬壳残骸、共振样本残骸、异变腺结或菌毯残骸 |
| 物理输出 | 生物质浆；样本来源额外形成共振样本组织或异变样本组织 |

| 功率层级 | 基础生产安全层级 |

### 7.2 配方分支

| 配方ID | 输入 | 输出 | 谱系规则 |
| --- | --- | --- | --- |
| `v1.recipe.digest_soft_tissue` | 普通软组织 | 生物质浆 | 敌方来源计外部同化；己方来源只计内部回收 |
| `v1.recipe.digest_hard_shell` | 硬壳结构残骸 | 生物质浆及发现事件 | 第一版允许低效处理，不要求专用房间 |
| `v1.recipe.separate_resonance_sample` | 共振样本残骸 | 普通组织输出 + 共振样本组织 | 样本组织保持唯一首次链标识 |
| `v1.recipe.separate_mutagenic_gland` | 异变腺结 | 普通组织输出 + 异变样本组织 | 保持W特殊来源事件和普通候选来源谱系 |
| `v1.recipe.digest_root_remains` | 菌毯残骸 | 生物质浆 | 只计内部回收，不形成外部同化套利 |

多输出配方必须在任务开始时原子承诺全部输出去向。任一必需输出无容量时任务不开始，不能只生成生物质而丢失样本。

### 7.3 堵塞与恢复

输入、普通浆体和样本组织分别使用独立兼容缓冲。样本输出满时保留输入或已承诺任务，不触发第二条样本恢复链。拆除、替换和降级前必须保障全部输入、任务和输出。

参数引用：各配方输入、输出、周期、功率、硬壳效率、输入/输出容量和端口并发。

## 8. 突触解析腔

### 8.1 蓝图与配方

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.synapse_analyzer` |
| 功能类别 | 研究与解析 |
| 物理输入 | 共振样本组织和异变样本组织 |
| 网络输出 | 基因入账、样本标签和发现/资格事件 |
| 候选输出 | 首次或特殊战利品候选来源事件，不直接拥有候选组 |
| 功率层级 | 研究构筑安全层级 |

`v1.recipe.analyze_resonance_sample`原子完成：

```text
终结唯一共振样本组织
-> 基因正式入账
-> 登记本轮样本标签和发现/科技族资格
-> 请求形成首次受保护候选组
```

任何候选池配置、受保护槽或基因输出子事务失败时整笔任务不提交。不能只发基因后丢失首次候选，也不能由页面打开补发候选。

`v1.recipe.analyze_mutagenic_tissue`终结一批异变样本组织，登记对应样本标签，并向候选契约提交`v1.source.mutagenic_gland_analysis`普通候选来源。第一版不要求该配方额外形成基因；组形成失败时样本、标签和候选来源事务全部不提交。普通槽满时，已形成组进入该处理来源缓冲。

### 8.2 阻断

- 缺料：没有合法样本组织。
- 缺能：未取得研究任务最低服务。
- 堵塞：输入或首次候选跨域输出无法原子承诺。
- 解析完成后样本标签保存在本轮进化事实中，拆除房间不会遗忘发现。

参数引用：建设、解析输入、周期、功率、基因输出、输入容量和突触网络并发。

## 9. 诱变培养腔

### 9.1 蓝图与配方

| 字段 | 内容 |
| --- | --- |
| `content_id` | `v1.room.mutation_culture` |
| 功能类别 | 候选形成 |
| 核心输入 | 生物质、已解析样本资格、功率和时间 |
| 候选输出 | 一个普通候选组来源请求 |
| 来源缓冲 | 每个来源至少容纳一个已形成且不可覆盖的组 |
| 功率层级 | 研究构筑安全层级 |

`v1.recipe.culture_candidate_group`在任务建立前预留生物质、执行位和普通候选槽。任务完成时由候选契约抽取并提交组；同一步槽竞争导致预留无效时，组进入该房间的来源缓冲，不能重抽或覆盖。

### 9.2 运行策略与阻断

- 第一版支持自动循环、单次批次、暂停和恢复。
- 普通槽无可预留容量时不开始新周期。
- 来源缓冲已有组且无法迁入时停止形成新组并显示堵塞。
- 房间拆除、替换或不兼容重组必须等待缓冲迁出；第一版不能静默舍弃来源缓冲组。
- 参数引用：建设、输入生物质、周期、功率、来源缓冲容量、合法池标签、组大小和权重配置引用。

## 10. 解锁与可执行性

| 内容 | 初始发现/资格 | 当前可执行性 |

## 11. 表现接口

每个实际房间至少提供以下表现状态键：

```text
construction / forming / running / starving_input / starving_power
/ blocked_input / blocked_output / blocked_port / paused / idle
/ reconfiguring / merged_group / split_group
```

房间动画只读取任务阶段、进度、已服务功率、缓冲和工蜂交互。建议动作：

| 房间 | 关键动作 |
| --- | --- |
| 腐殖滤囊 | 抽取、鼓胀、浆体形成、工蜂取货 |
| 热化代谢腔 | 热雾吸收、腔体压缩、网络脉冲、冷启动衰弱 |
| 胚流孵化室 | 养分注入、幼体形成、分化、破膜离巢 |
| 腐解消化池 | 残骸投入、酸液分解、浆体形成、样本隔离 |
| 突触解析腔 | 样本固定、神经丝接入、信息脉冲、完成输出 |
| 诱变培养腔 | 基质注入、组织分叉、候选成熟、堵塞回缩 |

相邻同类房间群必须显示多个工作区和真实端口活动，并只打开一个群详情；不能只把单房间贴图等比放大。
